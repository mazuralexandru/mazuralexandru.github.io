384. Header Scope: Row Headers (scope="row") Example

Let's modify our table to include row headers as well, demonstrating the use of scope="row".

Scenario: Imagine a table comparing features of different products, where the product names act as headers for their respective rows.

Structure:

  • The first row still contains column headers (<th> with scope="col").
  • Subsequent data rows will now start with a <th> element (containing the row's label, e.g., the product name) using scope="row", followed by <td> elements for the data in that row corresponding to the columns.

Code Change:

We've restructured the table in index.html:

  1. Added an empty top-left cell (<th></th>) in the header row (common practice when both row and column headers exist).
  2. Changed the column headers.
  3. Modified the data rows:
    • The first cell in each data row is now a <th> with scope="row".
    • The subsequent cells are <td>.
<!-- index.html snippet -->
<body>
  <h1>Structuring Tabular Data</h1>
  <p>Adding scope="row" to row headers (first column th):</p> <!-- Adjusted P -->

  <table>
    <tr> <!-- Header Row -->
      <th></th> <!-- Empty top-left corner -->
      <th scope="col">Feature X</th> <!-- Column Header -->
      <th scope="col">Feature Y</th> <!-- Column Header -->
    </tr>
    <tr> <!-- Data Row 1 -->
      <th scope="row">Product A</th> <!-- Row Header -->
      <td>Yes</td> <!-- Data -->
      <td>No</td> <!-- Data -->
    </tr>
    <tr> <!-- Data Row 2 -->
      <th scope="row">Product B</th> <!-- Row Header -->
      <td>No</td> <!-- Data -->
      <td>Yes</td> <!-- Data -->
    </tr>
  </table>

  <p>Scope clarifies header direction for complex tables.</p> <!-- Adjusted P -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

✅ Observation:

  1. Examine the new table structure carefully in index.html. Note the <th> elements in the first column of the data rows now have scope="row". The column headers in the first row still have scope="col".
  2. Look at the browser preview:
    • The column headers "Feature X" and "Feature Y" appear (bold, centered).
    • The row headers "Product A" and "Product B" appear in the first column (also likely bold, but maybe left-aligned depending on browser defaults for row-scoped headers).
    • The data cells ("Yes", "No") appear in their respective intersections.
  3. Accessibility Impact: A screen reader encountering the cell containing "Yes" in the first data row would now be able to announce something like: "Column: Feature X, Row: Product A, Cell: Yes". Both the row and column context are clearly associated thanks to the scope attributes.

✨ Key Takeaway: Use <th> with scope="row" for cells that act as headers for the data in their respective rows (typically the first cell in each data row). Combining scope="col" and scope="row" provides robust accessibility for tables with both column and row headers.
