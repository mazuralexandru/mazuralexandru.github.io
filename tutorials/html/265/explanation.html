265. Global Attributes: Definition and Scope

Now let's formally define Global Attributes.

  • Definition: Global attributes are attributes that can be specified on all standard HTML elements. While they are valid on any element, they might not always have a practical or semantic effect on every single one (e.g., adding a title attribute to the <head> element itself usually has no effect).
  • Purpose: They provide universal mechanisms common across different elements, allowing for consistent ways to:
    • Uniquely identify elements (id).
    • Classify elements for styling or scripting (class).
    • Apply direct inline styles (style).
    • Provide advisory information (title).
    • Specify language and text direction (lang, dir).
    • Influence accessibility and interactivity (tabindex, hidden, ARIA attributes).
    • Store custom data for scripts (data-*).

Other Notable Global Attributes (Brief Mention):

There are other global attributes you might encounter, which often relate to more advanced topics covered in later modules (like Accessibility or JavaScript):

  • tabindex: Controls keyboard focusability and tab order.
  • hidden: Hides elements visually and from assistive technology.
  • draggable, dropzone: Part of the HTML Drag and Drop API.
  • contenteditable: Allows users to edit the element's content.
  • spellcheck: Hints if content should be spell-checked.
  • data-*: Custom data attributes for use by JavaScript.
  • ARIA Attributes (role, aria-*): Attributes specifically for enhancing accessibility.
