200. CSS Colors Revisited

Files Modified: None

We've covered the main ways to specify colors in CSS. Let's quickly recap and introduce a couple more advanced concepts.

Primary Methods:

  1. Keywords: red, blue, transparent, tomato, etc. (Easy for common colors).
  2. Hexadecimal: #RRGGBB or #RGB (Compact, widely used, good for exact RGB values).
  3. RGB/RGBA: rgb(r, g, b) or rgba(r, g, b, alpha) (Explicit RGB values, standard way to handle alpha transparency).

Other Methods/Keywords:

  1. HSL/HSLA: hsl(hue, saturation, lightness) or hsla(h, s, l, alpha).
    • hue: Angle on the color wheel (0-360). 0=red, 120=green, 240=blue.
    • saturation: Intensity of the color (0% = grayscale, 100% = full color).
    • lightness: Brightness (0% = black, 50% = normal color, 100% = white).
    • Often more intuitive for creating color variations (e.g., making a color lighter/darker or less/more saturated).
    .button { background-color: hsl(210, 80%, 50%); } /* A nice blue */
.button:hover { background-color: hsl(210, 80%, 60%); } /* Lighter blue on hover */
  2. currentColor Keyword: A special keyword that represents the computed value of the element's own color property. Useful for making other properties (like border-color or SVG fill) automatically match the text color.
    button {
  color: blue;
  border: 1px solid currentColor; /* Border will also be blue */
}

Choosing the right color format often depends on the specific need (transparency needed? intuitive modification desired? matching text color?) and team conventions. Hex and RGBA are extremely common.
