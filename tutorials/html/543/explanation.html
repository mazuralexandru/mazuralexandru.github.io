543. Multimedia: Video (<video>) - src Attribute

To specify the video file to play when you only need to provide a single format, you use the src attribute directly on the <video> element.

The src Attribute on <video>:

  • Purpose: Specifies the URL (absolute or relative) of the single video file to embed.
  • Syntax: <video src="path/to/video.ext" ...>
  • Use Case: Suitable when you are only providing one video format (e.g., MP4) and don't need to offer alternatives for different browsers using <source> elements.
  • Limitation: If you need to provide multiple formats (like MP4 and WebM) for broader compatibility, you should omit the src attribute on the <video> tag and use multiple <source> elements inside instead.

Code Change:

We've added the src attribute to our <video> tag in index.html, pointing to a hypothetical video file.

<!-- index.html snippet -->
  <!-- Added src attribute -->
  <video src="/videos/example.mp4"> <!-- Hypothetical path -->
    <!-- Fallback content... -->
    <p>Sorry, your browser doesn't support embedded videos.</p>
    <p><a href="/videos/example.mp4">Download the video</a> instead.</p>
  </video>

✅ Observation:

  1. Locate the src="/videos/example.mp4" attribute added to the opening <video> tag.
  2. Look at the browser preview. The appearance might change slightly. The browser now knows which video file to attempt to load.
    • Since the file doesn't exist in our environment, you might see a slightly larger empty box (perhaps reserving default dimensions if none are specified), potentially with a "cannot play" icon or message overlayed.
    • Crucially, the browser still doesn't show playback controls because we haven't added the controls attribute.

✨ Key Takeaway: Use the src attribute on the <video> tag to specify the URL when linking only a single video file format. For multiple formats, omit src and use child <source> elements instead.
