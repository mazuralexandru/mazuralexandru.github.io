306. Margin Collapsing: Scenarios - Empty Blocks

Files Modified: None

A less common but possible scenario involves the margins of an empty block element.

  • Scenario: An element has no border, no padding, no height (or height: auto), no min-height, and no inline content (like text or images) inside it. Essentially, it's an "empty" box.

  • Result: In this specific case, the element's own top margin and bottom margin will collapse with each other. The resulting space taken up by the element vertically will be equal to the larger of its top or bottom margin.

  • Example HTML:

    <div class="empty-block"></div>

  • Example CSS:

    .empty-block {
  margin-top: 20px;
  margin-bottom: 30px;
  /* No height, padding, border, or content */
}
/* The actual vertical space created by this div will be 30px */

This scenario is less frequently encountered in typical layouts but demonstrates another facet of the collapsing rules.
