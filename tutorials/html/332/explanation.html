332. Description Lists (<dl>)

Files Modified: None

The third type of list is the description list (formerly definition list), created using the <dl> element.

  • Purpose: Represents a list of associated names (terms) and values (descriptions). It's ideal for glossaries, metadata lists (like key-value pairs), or any list where each item has a distinct term and an associated explanation.
  • Structure: The <dl> element acts as the container. Inside it, you group terms and descriptions:
    • <dt> (Description Term): Contains the term/name being defined or described.
    • <dd> (Description Details): Contains the description, definition, or value associated with the preceding term(s).
  • Grouping Rules: One or more <dt> elements must be followed by one or more <dd> elements. You can have multiple terms associated with one description, or one term associated with multiple descriptions.
    <dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language - for structure.</dd>

  <dt>CSS</dt>
  <dd>Cascading Style Sheets - for presentation.</dd>
  <dd>Makes things look pretty.</dd>

  <dt>Term A</dt>
  <dt>Term B</dt>
  <dd>Description for both A and B.</dd>
</dl>
  • <div> Wrapper: While not strictly required by older specs, wrapping each term/description pair (or group) within a <div> inside the <dl> is now permitted and often recommended, especially for easier CSS styling (e.g., using Flexbox or Grid on the <div> pairs).
     <dl>
   <div>
     <dt>Term</dt>
     <dd>Description</dd>
   </div>
   <div>
     <dt>Another Term</dt>
     <dd>Another Description</dd>
   </div>
 </dl>
  • Default Styling: Browsers typically render <dd> elements with some indentation (usually margin-left).
