116. Filename Conventions

Files Modified: None

While operating systems allow flexibility, following standard conventions for naming your web files makes your project cleaner, easier to manage, and less prone to issues, especially when deploying to web servers.

Best Practices:

  • Lowercase: Use all lowercase letters (e.g., about-us.html, not About-Us.HTML). Some web servers are case-sensitive, so consistency prevents broken links.
  • No Spaces: Avoid spaces in filenames. Browsers and command lines handle spaces poorly (often requiring encoding like %20).
  • Use Hyphens: Use hyphens (-) to separate words (e.g., contact-form.css). Underscores (_) are sometimes used, but hyphens are generally preferred and more common in URLs.
  • Be Descriptive: Choose names that clearly indicate the file's purpose (e.g., product-gallery.js, main-styles.css). Avoid generic names like file1.html.
  • index.html Significance: The filename index.html has special meaning. Most web servers are configured to automatically serve index.html if a user requests a directory URL (e.g., visiting www.example.com/ or www.example.com/products/ might serve index.html from the root or products directory, respectively). It's the conventional name for the default or home page of a directory.
