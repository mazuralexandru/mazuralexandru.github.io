16. Minimal HTML Document: The <head> Element

Inside the <html> element, the first required element is the <head> element.

Purpose:

  • Acts as a container for metadata. Metadata is information about the HTML document itself, rather than the content that is displayed visibly on the page.
  • It provides instructions and information for the browser, search engines, and other web services.

Syntax:

  • Uses an opening tag <head> and a closing tag </head>.

Placement:

  • The <head> element must be the first element nested directly inside the <html> element.

Content:

The <head> section typically contains elements like:

  • <title>: Defines the title shown in the browser tab or window bar (we'll add this soon).
  • <meta>: Provides various kinds of metadata (e.g., character encoding, viewport settings, page description for search engines).
  • <link>: Used to link external resources, most commonly CSS stylesheets.
  • <style>: Used to embed CSS rules directly within the HTML (less common for larger projects).
  • <script>: Used to embed JavaScript code or link external JavaScript files.
  • <base>: Specifies a base URL for all relative URLs in the document.

Behavior:

With the notable exception of the <title> element, content within the <head> is not displayed directly on the visible web page.

Code Change:

We've added the <head> opening and closing tags inside the <html> element, before the comment.

<!DOCTYPE html>
<html>
<head> <!-- Added opening tag -->

</head> <!-- Added closing tag -->
<!-- We will build the standard document structure soon -->

</html>

✨ Key Takeaway: The <head> element comes first inside <html> and holds metadata (information about the page), not the visible content.
