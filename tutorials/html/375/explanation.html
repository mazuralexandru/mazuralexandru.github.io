375. Basic Structure: Table Rows (<tr>) - Fundamentals

Tables are structured into horizontal rows. The element used to define these rows is <tr>.

The Table Row Element (<tr>)

  • Definition: Represents a single row of cells within an HTML table.
  • Purpose: To group together all the cells (<td> or <th>) that belong on the same horizontal line within the table's structure.
  • Core Concept: Every horizontal line of data or headers you see in a rendered table corresponds to a <tr> element in the HTML source.

Placement & Structure:

  • A <tr> element must be placed inside a table structure element: typically <thead>, <tbody>, or <tfoot>. (If these grouping elements are omitted, the browser implicitly wraps <tr> elements inside a <tbody>, so effectively, <tr> can also appear as a direct child of <table> in simple cases, though explicit <tbody> is recommended).
  • A <tr> element's direct children must be only <td> (data cells) or <th> (header cells). It cannot directly contain text or other elements.

Code Change:

We've added a single, empty <tr> element inside our <table> tag.

<!-- index.html -->
  <table>
    <tr>
      <!-- Table cells (td, th) go here -->
    </tr> <!-- Added empty table row -->
  </table>

✅ Observation:

  1. Note the <tr></tr> tags nested inside the <table> tags in index.html.
  2. Look at the browser preview. There is still no visible change. The <tr> element defines a row structure but has no visual appearance itself until it contains cells (<td> or <th>) with content.

✨ Key Takeaway: The <tr> element defines a horizontal row within a table. It must contain <td> or <th> elements as its children to hold the actual cell data or headers for that row.
