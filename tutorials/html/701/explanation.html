701. Accessibility: ARIA Basics - Purpose & Need

What is ARIA, and why do we need it?

What is ARIA?

  • ARIA stands for Accessible Rich Internet Applications.
  • It's a W3C specification that defines a set of special HTML attributes (like role, aria-label, aria-expanded).
  • These attributes do not change the element's visual appearance or default browser behavior directly.
  • Their sole purpose is to provide additional semantic information specifically for assistive technologies (AT) like screen readers.

Why is ARIA Needed? The Semantic Gap

  • While native HTML5 offers many semantic elements (<nav>, <button>, <input type="checkbox">, etc.), modern web applications often create complex user interface components using generic elements (<div>, <span>) styled with CSS and made interactive with JavaScript.
  • Examples: Custom dropdown menus, sliders, tab panels, tree views, modal dialogs, auto-suggest widgets.
  • The Problem: When these custom components are built using non-semantic elements, assistive technologies often have no way of knowing what the component is (its role) or what its current state is (e.g., is the menu expanded? is the checkbox checked?). A screen reader might just encounter a clickable <div> and announce "clickable," offering no clue that it's supposed to function like a button or a menu.

ARIA Bridges the Gap:

  • ARIA attributes allow developers to add the missing semantic information to these non-semantic elements or to provide more context for native elements in complex situations.
  • It helps you tell assistive technologies:
    • "This <div> is acting as a button." (role="button")
    • "This list represents a tablist." (role="tablist")
    • "This button controls that expandable section, and it's currently expanded." (aria-controls="section-id" aria-expanded="true")
    • "The accessible name for this icon-only button is Close." (aria-label="Close")

✅ Observation: No code changes. Understand the core problem ARIA solves: native HTML doesn't have elements for every conceivable UI pattern (like a tab panel). When developers build these custom widgets using generic elements, ARIA provides the attributes needed to communicate the widget's role, state, and properties accessibly.

✨ Key Takeaway: ARIA (Accessible Rich Internet Applications) provides HTML attributes (role, aria-*) to enhance the accessibility of web content, especially dynamic widgets built with JavaScript, by adding semantic information (role, state, properties) that assistive technologies can understand when native HTML semantics are insufficient.
