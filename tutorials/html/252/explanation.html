252. Figures (<figure>) - Content Model & Use Cases

Let's look more closely at what can go inside a <figure> and its typical uses.

Content Model:

  • A <figure> element typically contains:
    • One or more main content elements: This could be <img>, <svg>, <video>, <audio>, <canvas>, <pre><code>, <table>, <blockquote>, or even just paragraphs (<p>) if the text itself constitutes the self-contained figure.
    • Optionally, ONE <figcaption> element: This provides the caption for the figure (covered next). The <figcaption> can appear as the first or last child within the <figure>.
    • Other flow content as needed to structure the figure.

Use Cases Recap:

  • Image(s) with a caption.
  • Illustration or diagram with a title/description.
  • Code snippet with an explanatory caption.
  • Blockquote with attribution provided in a caption.
  • A video or audio embed with a title.
  • A table presenting data, with a caption explaining it.

Distinction from <div>:

  • While you could use a <div> to group an image and some text, <figure> provides specific semantic meaning: "This is a self-contained unit of content, likely referenced elsewhere, that could potentially be moved without disrupting the main flow." A <div> is a generic container with no inherent semantic meaning. Use <figure> when this specific semantic is appropriate.

✅ Observation: No code changes. Our current <figure> contains only an <img>, which is a valid and common use case. Understand that other elements like code blocks or multiple images could also reside within it, and that a <figcaption> is often included.

✨ Key Takeaway: The <figure> element is a versatile semantic container for various types of self-contained content (images, code, quotes, etc.), optionally paired with a single <figcaption>. Use it when the content functions as a distinct, referencable unit.
