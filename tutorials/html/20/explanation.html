20. Multiple Rules in Internal Stylesheets

You can add as many CSS rules as you need within a single <style> block. Let's add another rule to style the paragraph (<p>) element on the page.

What's New?

  • We added a second CSS ruleset, targeting the <p> element, inside the <style> block in index.html.
<!-- index.html snippet -->
<head>
  <meta charset="UTF-8">
  <title>My First Web Page</title>
  <link rel="stylesheet" href="style.css">
  <style>
    /* Internal CSS rules will go here */
    h1 {
      color: lightseagreen;
      text-decoration: underline;
    }
    /* Added P rule */
    p {
      font-style: italic;
      color: #bbb; /* Slightly lighter gray than default body text */
    }
  </style>
</head>
<body>

Applying Multiple Rules

  • Just list the rulesets one after another within the <style> tags.
  • The new rule targets all <p> elements on this page.
  • font-style: italic; makes the paragraph text italic.
  • color: #bbb; sets the paragraph text color to a specific light gray (#bbbbbb), overriding the slightly darker gray (#E0E0E0) inherited from the body rule in the external style.css.

Internal Stylesheets: Recap

  • Pros: Keeps styles for a single page together, allows use of selectors, better than inline styles for organization within a page.
  • Cons: Styles are not reusable across different HTML pages, still mixes presentation concerns within the HTML file structure.

For multi-page websites or complex applications, external stylesheets (which we linked earlier but haven't used much yet beyond the base styles) are the preferred and standard approach. We'll revisit them next.
