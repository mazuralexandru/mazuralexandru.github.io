161. Attribute Selectors: Purpose Explained

Files Modified: None

Why use attribute selectors when we have classes and IDs? They offer distinct advantages for specific scenarios:

  1. Styling Based on State or Type (without extra classes): You can style elements based on inherent attributes, like form input types ([type="text"], [type="email"]), required fields ([required]), or disabled elements ([disabled]), without needing to add extra classes like class="input-text" or class="is-disabled".
  2. Targeting Links Based on Destination: Style links differently depending on where they point, such as external links (a[target="_blank"]), links to PDF files (a[href$=".pdf"]), or secure links (a[href^="https://"]).
  3. Styling Based on Accessibility Attributes: Target elements with specific ARIA roles or states ([role="button"], [aria-expanded="true"]).
  4. Working with Custom Data Attributes: Style elements based on data-* attributes, which are often used to store custom data for JavaScript ([data-tooltip]).
  5. Avoiding Class/ID Proliferation: In some cases, using an attribute selector can be cleaner than adding numerous specific classes or IDs, especially when the attribute itself already provides the necessary semantic hook.

They provide a way to hook into the existing attributes of your HTML elements for styling purposes.
