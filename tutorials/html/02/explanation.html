02. URLs (Uniform Resource Locators) Breakdown

Files Modified: None

How does your browser know which server to ask and what specific page to request? It uses a URL (Uniform Resource Locator), often called a web address.

Let's break down a typical URL like https://www.example.com/path/to/page.html:

  1. Scheme (https://): Tells the browser how to connect. https means a secure connection, while http is standard (less common now). Other schemes exist (ftp://, mailto:).
  2. Domain Name (www.example.com): Identifies the specific server (or group of servers) hosting the website. It's like the server's street address.
  3. Path (/path/to/page.html): Specifies the exact file or resource being requested on that server. It's like the apartment number within the building. If omitted, the server usually sends a default file (often index.html).

Understanding URLs helps you navigate the web and structure your own sites later.
