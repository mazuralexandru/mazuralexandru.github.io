545. Multimedia: Video (<video>) - width, height, poster Attributes

Similar to images, providing dimensions and a placeholder image improves the loading experience for videos.

1. width and height Attributes:

  • Purpose: Specify the intended display dimensions of the video player in CSS pixels (unitless integer values).
  • Syntax: width="<number>" height="<number>"
  • Behavior:
    • Layout Stability (Prevents CLS): Just like with <img>, these attributes allow the browser to reserve the correct amount of space in the layout before the video metadata (which contains the intrinsic dimensions) loads. This prevents the page content from jumping around when the video player initializes.
    • Aspect Ratio: Use values that match the video's aspect ratio (e.g., width="640" height="360" for 16:9) to avoid distortion if the browser uses these directly or to ensure correct space reservation when CSS makes the video responsive.
    • CSS Interaction: CSS width/height can override these for responsive sizing, but the HTML attributes are still needed for the initial space reservation based on aspect ratio.

2. poster Attribute:

  • Purpose: Specifies the URL of an image to display in the video player area before the video starts playing or while it's downloading.
  • Syntax: poster="path/to/image.jpg"
  • Behavior: Shows a static image placeholder (like a thumbnail or custom splash screen) until the user hits play or the first video frame is ready. Improves visual appeal and perceived performance, giving users something to see immediately.

Code Change:

We've added width, height, and poster attributes to our <video> tag in index.html. Paths are hypothetical.

<!-- index.html snippet -->
  <video src="/videos/example.mp4"
         controls
         width="480" height="270" <!-- Added width/height (16:9 ratio) -->
         poster="/images/video-poster.jpg"> <!-- Added poster -->
    <!-- Fallback content... -->
    <p>Sorry, your browser doesn't support embedded videos.</p>
    <p><a href="/videos/example.mp4">Download the video</a> instead.</p>
  </video>

✅ Observation:

  1. Locate the width, height, and poster attributes added to the <video> tag.
  2. Look at the browser preview.
    • The video player area should now have a defined size (approximately 480x270 pixels).
    • Instead of just an error or black box, you should see a broken image icon representing the poster image (since /images/video-poster.jpg doesn't exist). If the poster did exist, that image would be displayed initially within the player area.
    • The controls are still visible.

✨ Key Takeaway: Add width and height attributes (matching the video's aspect ratio) to <video> to prevent layout shift. Use the poster attribute to specify an image placeholder displayed before playback begins.
