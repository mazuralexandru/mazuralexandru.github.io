54. Text Styling: font-family and Font Stacks

Let's dive deeper into styling text. The font-family property is fundamental, allowing you to specify which font(s) the browser should attempt to use for an element.

font-family Explained

  • Purpose: Specifies a prioritized list (a "font stack") of font family names and/or generic family names for an element.
  • Font Stack: You list multiple font names separated by commas (,). The browser starts with the first font in the list. If that font is available on the user's system (or loaded via @font-face), it uses it. If not, it tries the second font, then the third, and so on.
  • Quoting Names: If a font name contains spaces (like "Times New Roman") or special characters, you must enclose it in quotes (single '' or double ""). Single-word names (like Arial, Georgia) don't strictly require quotes, but it's often good practice to quote them anyway for consistency.
  • Generic Families (Fallback): It is essential to end your font stack with a generic font family keyword. This is the final fallback if none of your specified fonts are available. The browser will use its default font for that category. Common generic families include:
    • serif: Fonts with decorative strokes (serifs) on the ends of characters (e.g., Times New Roman, Georgia).
    • sans-serif: Fonts without serifs (e.g., Arial, Helvetica, Verdana). Often used for body text on the web.
    • monospace: Fonts where all characters have the same fixed width (e.g., Courier New, Consolas). Used for code.
    • cursive: Fonts resembling handwriting (e.g., Comic Sans MS - use with caution!, Brush Script MT).
    • fantasy: Decorative or display fonts (e.g., Impact).
  • Web Safe Fonts: Historically, developers relied on a small set of "web safe fonts" likely to be installed on most computers (Arial, Helvetica, Times New Roman, Georgia, Verdana, Courier New, etc.). While still useful as fallbacks, modern techniques like @font-face (or using font services) allow for much wider font choices.

In Our Example:

  • .serif-text: Tries "Times New Roman", then Georgia, then falls back to the browser's default serif font.
  • .sans-serif-text: Tries Arial, then Helvetica, then falls back to the browser's default sans-serif font (which is likely what our body is already using).
