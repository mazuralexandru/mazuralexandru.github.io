203. Link Content: Good vs. Bad Examples

Let's illustrate the difference between poor, generic link text and good, descriptive link text.

Code Change:

We've added two unordered lists (<ul>) to index.html. The first list demonstrates examples of bad link text, and the second demonstrates good alternatives for similar scenarios.

<body>
  <h1>Link Content & Accessibility</h1>
  <p>Compare descriptive vs. generic link text:</p>

  <h2>Bad Examples (Avoid):</h2>
  <ul>
    <li>To learn more about accessibility, <a href="accessibility-info.html">click here</a>.</li>
    <li>Our latest blog post is available. <a href="blog/latest.html">Read more</a>.</li>
    <li>Download the file: <a href="report.pdf">report.pdf</a></li> <!-- Just the filename -->
  </ul>

  <h2>Good Examples (Prefer):</h2>
  <ul>
    <li>Learn more about <a href="accessibility-info.html">web accessibility guidelines</a>.</li>
    <li>Read our <a href="blog/latest.html">latest blog post on responsive design</a>.</li>
    <li>Download the <a href="report.pdf">Quarterly Sales Report (PDF)</a>.</li> <!-- Descriptive + File Type -->
  </ul>

  <p>Good examples are clear even out of context.</p>

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

✅ Observation:

  1. Examine both lists in index.html.
  2. Look at the browser preview. Both lists contain links.
  3. Critically Compare:
    • Read only the link text from the "Bad Examples" list: "click here", "Read more", "report.pdf". How much information do these give you on their own? Very little.
    • Now read only the link text from the "Good Examples" list: "web accessibility guidelines", "latest blog post on responsive design", "Quarterly Sales Report (PDF)". Notice how much clearer these are about the link's destination, even without reading the surrounding sentence. The last example also helpfully includes the file type.
    • Imagine a screen reader reading out just the links from each list. The second list provides a vastly better experience.

✨ Key Takeaway: Good link text is descriptive and makes sense out of context. Avoid generic phrases; instead, use text that clearly indicates the content or purpose of the link destination.
