294. The dir Attribute - Practical Application & Example

Let's see dir in action.

Document-Level Direction:

  • For documents primarily written in an RTL language, you should set dir="rtl" on the <html> element, usually alongside the appropriate lang attribute.
    • Example: <html lang="ar" dir="rtl">

Element-Level Direction:

  • You apply dir to specific elements when their content's direction differs from the inherited direction.

Code Change:

We've added a new paragraph containing an Arabic phrase. Since our document's base direction is LTR (implied default, or explicitly set on <html>), we need to wrap the Arabic phrase in an element (like <span>) and apply dir="rtl" to ensure it renders correctly. We also add lang="ar" for good measure.

<!-- index.html -->
<body>
  <h1>Global Attributes - Lang & Dir</h1>
  <p>The document language is English (<span lang="fr">C'est la vie</span>). Default direction is LTR.</p> <!-- Adjusted P -->

  <p>Use 'dir' for content in a different direction, like this Arabic phrase: <span lang="ar" dir="rtl">مرحبا بالعالم</span>.</p> <!-- Added RTL example -->
  <!-- The Arabic text reads "Marhaban bil-'alam" (Hello World) -->

  <p>Setting 'dir' ensures correct text flow and alignment for RTL scripts.</p> <!-- Adjusted P -->


  <!-- Removed previous Spanish example for focus -->
  <!-- <p>... Another example in Spanish: <span lang="es">Hola Mundo</span>.</p> -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

✅ Observation:

  1. Locate the <span> containing the Arabic text. Note both lang="ar" and dir="rtl".
  2. Look at the browser preview. Observe the Arabic phrase "مرحبا بالعالم".
    • The characters themselves should flow right-to-left.
    • The punctuation (period .) should appear on the left end of the phrase, which is correct for RTL context adjacent to LTR text.
    • Without dir="rtl", the browser might struggle to render the isolated RTL phrase correctly within the LTR paragraph, potentially misplacing the punctuation or even rendering characters in the wrong order depending on the surrounding LTR characters and browser implementation of the Bidi algorithm. Explicitly setting dir provides the necessary context.

✨ Key Takeaway: Apply dir="rtl" to elements containing right-to-left text when the surrounding context is left-to-right (or vice-versa with dir="ltr"). This ensures correct rendering according to the script's directionality, especially important for punctuation and mixing scripts. Set dir on the <html> tag for documents primarily in RTL languages.
