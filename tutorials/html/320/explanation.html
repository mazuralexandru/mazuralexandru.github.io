320. Deprecated <meta>: Keywords (name="keywords") - Core Mechanics

Let's look at the syntax for the <meta name="keywords"> tag, even though its practical value is minimal today.

Syntax:

  • Placed within the <head> section. <meta name="keywords" content="keyword1, keyword2, phrase keyword, etc">

content Value:

  • Typically, the value is a comma-separated list of words or short phrases considered relevant to the page.
  • Whitespace around the commas is generally ignored.
  • Historically, there were debates about optimal number of keywords, case sensitivity, etc., but these are moot points now given its lack of impact on major search engines.

Code Change:

We've added an example <meta name="keywords"> tag to the <head> of index.html.

<!-- index.html -->
<head>
  <meta charset="UTF-8">
  <meta name="description" content="Learn about HTML metadata elements like meta, link, title, base, style, and script found in the document head section. Essential for SEO and page setup.">
  <meta name="keywords" content="html, metadata, meta tags, head section, seo, web development"> <!-- Added keywords meta tag -->
  <title>HTML Metadata</title>
  <link rel="stylesheet" href="style.css">
  <!-- Other meta tags will be added here -->
</head>
<body>
  <h1>Metadata: Keywords (Historical)</h1>
  <p>Added a keywords meta tag to the head (check the HTML).</p> <!-- Adjusted P -->

  <p>This tag was historically used by search engines but is now largely ignored.</p>

  1. Locate the new <meta name="keywords" content="..."> tag within the <head> in index.html. Examine the comma-separated list in the content attribute.

✨ Key Takeaway: The syntax involves name="keywords" and a content attribute containing a comma-separated list of terms. While syntactically valid, its functional impact on modern SEO is negligible.
