356. Styling Description List Terms (<dt>)

Files Modified: style.css

Let's apply some basic styling to the description terms (<dt>). A common requirement is to make them stand out, often using bold text.

✅ Observe style.css:

  • We've added a new rule targeting dt elements.
  • We're making the text bold and adding a small bottom margin for spacing.
/* Styling Description Lists */
dt { /* Target Description Terms */
    font-weight: bold;
    margin-bottom: 0.25em; /* Space below term, before description */
}

dd { /* Target Description Details */
    /* Default browser margin-left usually provides indentation */
    margin-bottom: 1em; /* Space below description, before next term */
    margin-left: 2em; /* Explicitly set indentation */
}

(Note: We also added some basic margin rules for dd here for better default spacing and indentation).

✨ Observe the Preview:

  • Look at the "Web Technologies" description list.
  • The terms (HTML, CSS, JavaScript) should now appear in bold.
  • There should be a small amount of space between each term and its description below it.
  • The descriptions (<dd>) are indented due to the margin-left and have space below them.
