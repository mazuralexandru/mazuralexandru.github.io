270. The id Attribute - Behavior & Interactions: CSS Styling

One of the primary uses of the id attribute is to provide a unique hook for targeting a specific element with CSS styles.

CSS ID Selector:

  • In CSS, you target an element by its id using the ID selector, which consists of a hash symbol (#) followed immediately by the id value.
    • Syntax: #id-value { ...css properties... }
    • Example: #main-page-title { ... } targets the element with id="main-page-title".

Specificity:

  • ID selectors have a very high specificity in the CSS cascade. Specificity determines which CSS rule applies when multiple rules target the same element and define the same property.
  • Hierarchy (Simplified): Inline Styles (style attribute) > ID Selectors (#id) > Class Selectors (.class), Attribute Selectors ([attr]), Pseudo-classes (:hover) > Type Selectors (h1, p), Pseudo-elements (:before).
  • Implication: Styles applied using an ID selector are difficult to override with other types of selectors (like class or type selectors). You often need another ID selector or the !important flag (which should generally be avoided) to override an ID-based style.

When to Use IDs for Styling:

  • Because of their high specificity and uniqueness constraint, IDs are best reserved for styling unique, major page landmarks or components that are unlikely to be repeated and where overriding styles is generally not desired.
  • Examples:
    • Styling the main site header (id="site-header").
    • Styling the primary content area (id="main-content").
    • Styling a unique sidebar (id="primary-sidebar").
  • Avoid Overuse: Do not use IDs for general styling of elements that might appear multiple times or for creating reusable component styles. Classes (class) are far more flexible and maintainable for these purposes due to their lower specificity and reusability. Overusing IDs for styling often leads to "specificity wars" where developers struggle to override styles and resort to overly complex selectors or !important.

Code Change:

We will add a CSS rule to style.css targeting the <h1> using its ID.

/* style.css snippet - Add AFTER base body style */

/* Style the main heading using its ID */
#main-page-title {
  color: #90EE90; /* LightGreen */
  border-bottom: 1px solid #90EE90;
  padding-bottom: 5px;
}

✅ Observation:

  1. Verify the new CSS rule targeting #main-page-title has been added to style.css.
  2. Look at the browser preview. The main heading ("Global Attributes") should now be:
    • Light green (#90EE90).
    • Have a light green line underneath it.
    • Have a little extra space below the text but before the line.
  3. This confirms that the ID selector in the CSS successfully targeted and styled the specific HTML element with the matching id.

✨ Key Takeaway: Use the CSS ID selector (#id-value) to style unique elements. Be mindful of its high specificity; prefer classes for reusable and general styling.
