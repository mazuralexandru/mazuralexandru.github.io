165. Linking to Fragments: Same-Page Links (href="#id")

Now that we have target elements with id attributes, we can create links that jump to those locations within the same page.

Syntax (Same Page):

To link to an element with a specific id on the current page, set the href attribute value to # followed immediately by the id value.

  • href="#elementID"

Code Change:

We have added two anchor elements inside the navigation (<nav>) section of index.html. Their href attributes point to the ids (#section1 and #section2) we created in the previous step. We wrapped them in a <ul> for structure.

  <nav>
      <p>Navigation:</p>
      <ul> <!-- Added ul for structure -->
        <li><a href="#section1">Go to Section 1</a></li> <!-- Added same-page fragment link -->
        <li><a href="#section2">Go to Section 2</a></li> <!-- Added same-page fragment link -->
      </ul>
  </nav>

  <div class="spacer">Scroll down...</div>

  <h2 id="section1">Section One</h2>
  ...

  <div class="spacer">Scroll down further...</div>

  <h2 id="section2">Section Two</h2>
  ...

✅ Observation:

  1. Locate the new <a> tags within the <nav> section in index.html. Note their href values (#section1, #section2).
  2. Look at the browser preview. You should see the two links ("Go to Section 1", "Go to Section 2") near the top of the page.
  3. Try clicking the links!
    • Clicking "Go to Section 1" should instantly scroll the page down so that the "Section One" heading is at or near the top of the viewport.
    • Clicking "Go to Section 2" should scroll down further to bring the "Section Two" heading into view.
    • Notice the URL in your browser's address bar might now include #section1 or #section2 at the end.
    • The page does not reload; the browser just navigates within the current document.

✨ Key Takeaway: Use href="#elementID" to create links that smoothly scroll the browser to the element with the corresponding id on the same page.
