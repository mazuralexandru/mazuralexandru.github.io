240. width & height: Interaction with Image Dimensions

It's crucial that the values you provide for the width and height attributes accurately reflect the actual dimensions of the image file they refer to.

Best Practice: Match Intrinsic Dimensions

  • Find the actual pixel width and height of your image file (using image editing software, file properties, browser developer tools after loading it, etc.).
  • Use these exact numbers as the values for the width and height attributes.
    • Example: If photo.jpg is truly 800 pixels wide and 600 pixels tall, use width="800" height="600".

What Happens if They Don't Match?

  • If you provide width and height values that differ from the image file's actual dimensions, the browser will stretch or squash the image to fit the dimensions specified in the HTML attributes (unless overridden by CSS).
  • Result: This typically leads to image distortion and poor visual quality (pixelation if scaling up, loss of detail or unnatural appearance if scaling down or changing the aspect ratio).

Example of Distortion (Conceptual):

<!-- Image file is actually 200x200 pixels -->
<img src="square.png" alt="Square" width="400" height="100">
<!-- Browser will stretch the 200x200 image into a 400x100 rectangle -->

Getting Dimensions:

  • Most operating systems let you view image dimensions via file properties (right-click -> Properties/Get Info).
  • Image editing tools (Photoshop, GIMP, etc.) clearly show dimensions.
  • You can open the image directly in a browser and use Developer Tools (F12 -> Inspect) to see its intrinsic size.

✅ Observation: No code changes. Look at our example: width="150" height="50". We are assuming our non-existent logo.png file is actually 150x50 pixels. If the real file were, say, 300x100, the browser (if displaying it) would render it at 150x50, scaling it down. If the real file were 150x150 (square), the browser would squash it into a 150x50 rectangle based on the attributes.

✨ Key Takeaway: Ensure the width and height attribute values match the actual pixel dimensions of the image file to prevent distortion and ensure accurate space reservation for layout stability.
