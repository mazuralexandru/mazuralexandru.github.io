341. list-style-type Affecting Nested Lists

Files Modified: style.css

How does list-style-type affect nested lists?

  • Inheritance: The list-style-type property is inherited. If you set it on a parent list (ul), child lists (ul nested inside an li) will inherit that same style unless overridden.
  • Browser Defaults: Browsers often have default styles that apply different markers to different nesting levels (e.g., ul { list-style-type: disc; }, ul ul { list-style-type: circle; }, ul ul ul { list-style-type: square; }). These browser defaults might override simple inheritance if your own rules aren't specific enough.
  • Explicit Styling: To ensure consistent or specific styling for nested lists, you need to write CSS rules that target them directly, often using descendant combinators.

✅ Observe style.css:

  • We've added a new rule targeting nested unordered lists: ul ul.
  • This rule sets their list-style-type to circle.
/* Style Nested Unordered Lists */
ul ul {
    list-style-type: circle; /* Use circles for nested items */
    margin-top: 0.25em; /* Add some space above nested list */
    margin-bottom: 0.5em; /* Add space below nested list */
}

  • The nested items under "Apples" ("Granny Smith", "Fuji") should now have circle markers, because the more specific ul ul rule overrides the inherited square style.
  • We also added some vertical margin to space out the nested list slightly.
