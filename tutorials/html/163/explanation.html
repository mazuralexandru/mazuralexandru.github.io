163. Linking to Fragments: Definition & Mechanism

Beyond linking to different pages or resources, HTML allows you to create links that navigate the user to a specific section within an HTML document (either the current page or another page). This is achieved using fragment identifiers.

Definition:

  • A fragment identifier (or simply "fragment") is the part of a URL that appears after a hash (#) symbol.
  • It identifies a specific location or element within the target HTML document.

Mechanism: The id Attribute

  • To create a targetable location within a page, you give an HTML element a unique id attribute.
    <h2 id="section-intro">Introduction</h2>
<p id="para-important">This is an important paragraph.</p>
  • The id attribute's value acts as the anchor point for the fragment link.
  • Crucial Rule: The value assigned to an id attribute must be unique within the entire HTML document. No two elements can have the same id. Valid id values should typically start with a letter and contain letters, numbers, hyphens (-), or underscores (_).

Linking to the Fragment:

You then create an anchor (<a>) element whose href attribute value starts with # followed by the exact id value of the target element.

  • Syntax (Same Page): href="#targetElementID"
  • Syntax (Different Page): href="page.html#targetElementID"

When a user clicks such a link, the browser attempts to find the element with the matching id in the target document and scroll the page to bring that element into view.

✨ Key Takeaway: Fragment identifiers (#elementID) in URLs link to specific locations within HTML documents, identified by unique id attributes on target elements.
