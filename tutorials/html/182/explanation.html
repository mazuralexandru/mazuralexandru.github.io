182. Specificity Calculation Method

Files Modified: None

Specificity is typically calculated and compared using a four-component system, often represented as (A, B, C, D):

  1. A (Inline Styles): Counts 1 if the declaration comes from an inline style="..." attribute. Otherwise 0. (This is the highest level).
  2. B (IDs): Counts the number of ID selectors (#idname) in the overall selector.
  3. C (Classes, Attributes, Pseudo-classes): Counts the number of class selectors (.classname), attribute selectors ([attribute]), and pseudo-classes (:hover, :focus, :nth-child(), etc.) in the selector.
  4. D (Types, Pseudo-elements): Counts the number of type selectors (p, div) and pseudo-elements (::before, ::after, ::first-line, etc.) in the selector.

Important Notes:

  • The universal selector (*) and combinators (>,+,~, space) do not contribute to specificity.
  • The negation pseudo-class :not() itself doesn't count, but the selectors inside it do.
  • The :where() pseudo-class intentionally has zero specificity (0,0,0,0), allowing you to group selectors without increasing specificity.
  • The :is() and :has() pseudo-classes take on the specificity of their most specific argument selector.

Comparison: Specificity values are compared from left to right (A then B then C then D). The first component where one selector has a higher value determines the winner. A selector with a score of (0,1,0,0) (one ID) beats a selector with (0,0,10,5) (ten classes and five type selectors) because the ID component (B) is higher.

Understanding this calculation helps predict which styles will apply.
