495. Semantics: What is Semantic HTML? - Contrast with Presentational HTML

To fully appreciate semantic HTML, it helps to briefly understand the older, presentational approach it replaced.

Presentational HTML (Historical Context):

  • In the early web, CSS was limited or nonexistent. Developers often relied on HTML elements and attributes primarily for their visual appearance or layout capabilities.
  • Examples:
    • Using <b> solely to make text bold, or <i> solely for italics, without considering semantic emphasis or alternate voice.
    • Using <font> tags (now obsolete) to control font size and color directly in HTML.
    • Using <table> elements extensively for page layout (creating columns, positioning sidebars) – the "layout tables" anti-pattern.
    • Using attributes like bgcolor, align, border directly on elements like <table> or <td>.

The Shift to Semantics & Separation of Concerns:

  • Modern web development strongly emphasizes the separation of concerns:
    • HTML (Structure & Meaning): Defines what the content is (a heading, a paragraph, a list, an article, navigation). Uses semantic elements.
    • CSS (Presentation): Defines how the content looks (colors, fonts, layout, spacing, borders). Uses selectors targeting HTML elements (often via classes).
    • JavaScript (Behavior): Defines how the page interacts and responds dynamically.
  • Semantic HTML is the foundation of this separation for the structural layer. It focuses HTML on meaning, leaving visual styling entirely to CSS.

Why is Presentational HTML Bad?

  • Violates separation of concerns.
  • Poor accessibility (layout tables, non-semantic tags are confusing).
  • Difficult to maintain (styling changes require HTML edits).
  • Inflexible for responsive design.
  • Less meaningful for search engines and other tools.

✅ Observation: No code changes. Contrast the idea of using <h1> because it means "the most important heading" versus using it just because it looks big and bold by default. Contrast using <nav> because it means "navigation block" versus <div class="my-nav-looks">.

✨ Key Takeaway: Semantic HTML focuses on using tags for their meaning, separating structure from presentation (CSS). It replaces older, discouraged practices where HTML elements and attributes were used primarily for visual styling or layout.
