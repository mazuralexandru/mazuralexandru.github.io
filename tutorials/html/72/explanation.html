Other Inline Semantic Tags

  • <small> (Side Comments):
    • Semantic Meaning: Represents side comments such as fine print, legal notices (copyright, disclaimers), or attributions, often rendered smaller. It's not for making arbitrary text smaller (use CSS font-size for that).
    • Default Styling: Renders text one size smaller than its parent (e.g., like applying font-size: smaller;). We added opacity for extra effect.
  • <sup> (Superscript):
    • Semantic Meaning: Represents superscript text, typically rendered raised above the baseline and often smaller. Used for exponents, ordinal indicators, footnote markers.
    • Default Styling: Uses vertical-align: super; to raise the text and font-size: smaller;. Setting line-height: 0; on sup and sub is a common technique to prevent them from increasing the overall line height of the line they appear on.
  • <sub> (Subscript):
    • Semantic Meaning: Represents subscript text, typically rendered lowered below the baseline and often smaller. Used for chemical formulas, mathematical indices.
    • Default Styling: Uses vertical-align: sub; to lower the text and font-size: smaller;. line-height: 0; is also often applied.
