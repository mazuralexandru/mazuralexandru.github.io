412. Forms: The <form> Element - Core Mechanics

Let's examine the structural rules and behavior of the <form> element.

Syntax Review:

  • Requires opening <form> and closing </form> tags.
  • Attributes (like action, method, discussed soon) go in the opening tag.
  • Form-associated content (controls, labels, etc.) goes between the tags.

Element Category:

  • Typically treated as a block-level element by browsers, meaning it usually starts on a new line and occupies available width.
  • Its content model is Flow Content.

Permitted Content:

  • A <form> element can contain most flow content elements you'd normally find in the <body>, such as paragraphs (<p>), headings (<h1>-<h6>), divisions (<div>), lists (<ul>/<ol>), etc., which are often used to structure and label the form controls within it.
  • Crucially, it contains the form-associated elements: <input>, <textarea>, <select>, <button>, <label>, <fieldset>, <legend>, <output>, <progress>, <meter>.

Prohibited Content:

  • A <form> element cannot directly contain another <form> element. Nesting forms is invalid HTML and leads to unpredictable submission behavior.

Behavior Scope:

  • The <form> element defines the boundary for submission. When a submit action is triggered within a form (e.g., by clicking <button type="submit">):
    • The browser gathers data only from the named, non-disabled controls within that specific form.
    • The data is sent to the destination specified by that form's action attribute using that form's method.
  • This allows having multiple independent forms on the same page, each submitting to different locations or with different methods.

✅ Observation: No code changes. Review the empty <form> in index.html. Understand its role as a container defining a submission boundary and its inability to be nested. Recognize that it can hold both structural elements (like <p> or <div> for layout) and the actual form controls (<input>, <label>).

✨ Key Takeaway: <form> is a block-level container for flow content (especially form controls). It defines the scope for data submission and cannot be nested.
