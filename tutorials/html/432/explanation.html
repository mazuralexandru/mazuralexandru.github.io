432. Common <input> Types: type="text"

Let's begin exploring the different values for the type attribute, starting with the most fundamental: text.

type="text"

  • Definition: Creates a basic, single-line text input field.
  • Default: This is the default value if the type attribute is completely omitted from an <input> tag (though explicitly stating type="text" is clearer).
  • Use Case: Used for collecting short, freeform textual input where no specific format (like email or number) is required. Examples: names, usernames, subjects, street addresses (though more specific types exist), simple search terms.
  • Behavior: Allows the user to enter almost any character. Renders as a standard rectangular text box.

Essential Accompanying Elements:

  • name Attribute: Remember to include a name attribute so the data can be submitted (e.g., name="username").
  • id Attribute: Include a unique id (e.g., id="username").
  • <label> Element: Crucially, provide a <label> element associated with the input using the for attribute matching the input's id (<label for="username">). This is essential for accessibility and usability.

Code Change:

We've added our first form control inside the <form> in index.html: an <input type="text"> along with its associated <label>.

<!-- index.html snippet -->
<form action="/submit-data" method="POST">
   <label for="username">Username:</label> <!-- Added Label -->
   <input type="text" id="username" name="username"> <!-- Added input type="text" -->
</form>

✅ Observation:

  1. Examine the code inside the <form>. Note the <label> with for="username" and the <input> with type="text", id="username", and name="username".
  2. Look at the browser preview. You should now see the label "Username:" followed by a standard text input box.
  3. Click inside the box – you can type text.
  4. Click on the label "Username:" – the cursor should jump into the input box because of the for/id association.

✨ Key Takeaway: <input type="text"> creates a basic single-line text field. Always associate it with a <label for="..."> (matching the input's id) and give it a name attribute for submission.
