396. <figure> & <figcaption>: Accessibility

Files Modified: None

Using <figure> and <figcaption> correctly significantly enhances accessibility:

  • Programmatic Association: The browser and assistive technologies understand that the <figcaption> provides a description or title for the content within the <figure>. Screen readers can announce this relationship, for example, reading the caption when the user navigates to the figure's image.
  • Relationship to alt Text:
    • The alt attribute on an <img> inside a <figure> should still describe the image itself (what it visually depicts).
    • The <figcaption> provides broader context, interpretation, or a title for the figure as a whole.
    • They serve complementary purposes. The alt text describes the image; the <figcaption> describes the figure (which contains the image). Avoid simply repeating the alt text in the <figcaption> or vice versa.
    • If the <figcaption> adequately describes the image content (e.g., the caption is the full description needed), the image's alt attribute might sometimes be empty (alt=""), but providing both is often clearer.
  • Landmark Potential: While <figure> isn't a landmark role by default, it provides structure that can be navigated by some assistive technologies.

Using these elements semantically helps create a richer, more understandable experience for all users, especially those relying on assistive technology.
