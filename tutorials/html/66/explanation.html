66. Text Styling: white-space

The white-space property controls how whitespace (spaces, tabs, newlines) inside an element is handled and how lines are broken (wrapped).

white-space Explained

  • Purpose: Controls whitespace collapsing and text wrapping behavior.
  • Values:
    • normal: (Default) Collapses sequences of whitespace into a single space. Newlines are treated like spaces. Text wraps as needed.
    • nowrap: Collapses whitespace like normal, but prevents text from wrapping to the next line. Content will overflow its container if too wide.
    • pre: Preserves sequences of whitespace exactly as written in the source HTML. Preserves newline characters. Text does not wrap by default. This is the default behavior for HTML <pre> tags.
    • pre-wrap: Preserves whitespace and newlines like pre, but allows text to wrap to the next line if needed.
    • pre-line: Collapses sequences of whitespace into a single space (like normal), but preserves newline characters from the source. Text wraps as needed.
    • break-spaces: Similar to pre-wrap but behaves slightly differently with preserved spaces at the end of lines.

In Our Example:

  • The first <p> shows the default normal behavior.
  • The <pre> element uses white-space: pre; (often applied by browser defaults, but explicitly set here). Notice the preservation of extra spaces and the line break. We added overflow-x: auto to allow scrolling if the content is too wide.
  • The .no-wrap-div uses white-space: nowrap;. Multiple spaces in the source HTML still collapse, but the entire text content stays on a single line, potentially overflowing the div (hence overflow-x: auto).
