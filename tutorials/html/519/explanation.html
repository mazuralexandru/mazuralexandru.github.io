519. Headings & Outline: The Implicit Outline (Headings Only)

Before HTML5 introduced explicit sectioning elements, the structure or "outline" of a document was determined solely by the hierarchy of heading elements (<h1> - <h6>). This implicit outline mechanism still plays a role.

How Headings Define Structure:

  • <h1>: Represents the main topic or title of the entire page. Rule: Should generally only be one <h1> per page.
  • <h2>: Represents a major section directly under the <h1>.
  • <h3>: Represents a subsection under the preceding <h2>.
  • <h4>, <h5>, <h6>: Represent progressively deeper levels of subsections.

The Cardinal Rule: Do Not Skip Levels

  • The most important rule when using headings is to maintain a logical, sequential hierarchy. Start with <h1>, use <h2> for main sections, <h3> for subsections of <h2>, etc.
  • Never skip heading levels purely for visual styling. For example, do not jump from an <h2> directly to an <h4> just because you want smaller text. Use CSS to control heading appearance.
    • Correct: <h1> -> <h2> -> <h3> -> <h2> -> <h3>
    • Incorrect: <h1> -> <h3> (Skipped <h2>)
    • Incorrect: <h2> -> <h4> (Skipped <h3>)

Why Skipping Levels is Bad:

  • Accessibility: Screen reader users rely heavily on navigating by heading levels (e.g., jumping to the next H2, then the next H3). Skipping levels breaks this logical structure, making the page confusing and hard to navigate efficiently. They might miss entire sections or misunderstand the content hierarchy.
  • SEO: Search engines use heading structure to understand content organization. While not as severe as missing headings altogether, illogical skipping might slightly hinder their understanding.
  • Maintainability: Creates illogical code structure.
