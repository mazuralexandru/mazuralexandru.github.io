433. Common <input> Types: type="password"

Next, let's look at the input type specifically designed for sensitive data: password.

type="password"

  • Definition: Creates a single-line text input field where the characters entered by the user are masked (obscured).
  • Appearance: Instead of showing the typed characters, the browser typically displays bullet points (●) or asterisks (*) for each character entered.
  • Use Case: Exclusively used for inputting passwords or other sensitive information that should not be visible on the screen (e.g., preventing "shoulder surfing").
  • Behavior: Functionally similar to type="text" regarding allowed characters, but with the crucial visual masking feature. The actual value entered is stored and submitted (usually via POST over HTTPS), just not displayed directly. Browsers also often engage password manager features for these fields.

Essential Attributes:

  • Requires name for submission (e.g., name="password").
  • Requires id for <label> association (e.g., id="pwd").
  • Must have an associated <label for="pwd">.

Code Change:

We've added a second label/input pair to our form in index.html for a password field, using type="password". We've also wrapped each label/input pair in <p> tags for basic vertical spacing.

<!-- index.html snippet -->
<form action="/submit-data" method="POST">
  <p> <!-- Wrap in P for spacing -->
    <label for="username">Username:</label>
    <input type="text" id="username" name="username">
  </p>
  <p> <!-- Wrap in P for spacing -->
    <label for="pwd">Password:</label> <!-- Added Label -->
    <input type="password" id="pwd" name="password"> <!-- Added input type="password" -->
  </p>
</form>

✅ Observation:

  1. Examine the new <input type="password"> and its associated <label>.
  2. Look at the browser preview. You now see a "Password:" label and input field below the username field.
  3. Type something into the password field. Notice how the characters you type are obscured by dots or asterisks.
  4. Type something into the username field – the characters remain visible.

✨ Key Takeaway: Use <input type="password"> for password fields. It masks user input for security against shoulder surfing but submits the actual value. Always use a corresponding <label>.
