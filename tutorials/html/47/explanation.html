47. Introduction to CSS Inheritance

Besides the Cascade deciding between conflicting rules applied directly to an element, CSS also features Inheritance. This means that some CSS properties, when applied to a parent element, are automatically passed down to its child elements unless a more specific rule targets the child directly.

Inheritance Explained

  • Concept: Child elements "inherit" the computed value of certain properties from their parent element in the HTML structure.
  • Which Properties Inherit? Primarily properties related to text:
    • color
    • font-family, font-size, font-weight, font-style
    • line-height
    • text-align
    • list-style properties
    • Also visibility, cursor
  • Which Properties Don't (Usually) Inherit? Most properties related to element boxes and positioning:
    • background-color
    • width, height
    • padding, border, margin
    • position, top, left, etc. (Think: It usually wouldn't make sense for a child element to automatically get the same border or width as its parent).

In Our Example:

  • We applied color: mediumpurple; directly to the parent <div> using an inline style. We also gave it a border and padding (which won't be inherited).
  • We haven't applied any specific CSS rules to the child <p> element in our style.css file.
