343. Common <link> Types: Linking Favicons (rel="icon") - Fundamentals

A favicon (short for "favorites icon") is a small icon file associated with a website or web page. It's used to visually identify your site in various browser UI locations.

Purpose:

  • Branding & Recognition: Provides a recognizable visual marker for your site in browser tabs, the bookmarks bar, history lists, etc.
  • User Experience: Helps users quickly locate your site's tab among potentially many open tabs.

Core Concept:

  • You use the <link> element within the <head> section to tell the browser where to find the icon file(s).
  • The rel attribute value icon is the standard way to indicate that the linked resource is an icon for the page.

Historical Note:

  • Historically, browsers would often automatically look for a file named favicon.ico in the website's root directory (/favicon.ico). While many still do this as a fallback, explicitly declaring your favicon(s) using <link> tags is the reliable and recommended method.
  • The rel value shortcut icon is often seen alongside or instead of icon. This is a non-standard, legacy value originating from Internet Explorer but still widely supported. Using just rel="icon" is sufficient for modern browsers, but sometimes rel="shortcut icon" or even rel="icon shortcut" is included for maximum backward compatibility, although arguably unnecessary now. We'll stick to rel="icon".

✅ Observation: No code changes yet. Understand the purpose of favicons (visual identification in browser UI) and that <link rel="icon"> is the primary way to declare them.

✨ Key Takeaway: Favicons are small icons representing your website, declared using <link rel="icon" href="..."> within the <head> to improve branding and user navigation in browser interfaces.
