520. Headings & Outline: The HTML Outline Algorithm (with Sectioning Content)

HTML5 introduced sectioning content elements (<article>, <section>, <nav>, <aside>) which interact with headings to create a more explicit and potentially more complex document outline.

The Outline Algorithm (Conceptual):

  • The browser (and assistive technologies) uses both headings and sectioning content to build the outline.
  • Each sectioning content element creates a new, independent "section" in the outline. Think of it like starting a new chapter or subsection.
  • Headings within a sectioning element define the hierarchy for that specific section.
  • The rank of the first heading (h1-h6) inside a sectioning element determines its level relative to its parent section in the outline.
  • Benefit: This allows you, for example, to potentially restart heading levels by using an <h1> or <h2> inside each <article> or <section> without necessarily breaking the overall document's logical heading flow according to the algorithm.

Example (Conceptual):

<body>
  <h1>Page Title (Level 1)</h1>
  <section> <!-- Section 1 -->
    <h2>Section 1 Title (Level 2)</h2>
    <section> <!-- Nested Section 1.1 -->
      <h3>Subsection 1.1 Title (Level 3)</h3>
    </section>
  </section>
  <article> <!-- Section 2 (Article) -->
    <h2>Article Title (Level 2)</h2> <!-- H2 is appropriate here -->
    <section> <!-- Nested Section 2.1 -->
       <h3>Article Subsection Title (Level 3)</h3>
    </section>
  </article>
</body>

In this structure, the outline would understand the hierarchy correctly even though <h2> is used within both the first <section> and the <article>.

Practical Recommendation & The <h1> Debate:

  • HTML5 Spec: Allows multiple <h1> elements per page, provided each is the main heading for its own sectioning content element (<article>, <section>, etc.). The outline algorithm theoretically handles this, making each <h1> a top-level heading for its own section.
  • Accessibility/SEO Reality: Historically, and for broadest compatibility and simplicity, using only one <h1> per rendered page for the main visible title is still widely recommended. Many accessibility tools and SEO practices are still optimized around a single, clear <h1>. Using multiple <h1>s, while valid per spec, can sometimes cause confusion for users or tools expecting a single main heading.
  • Safe Approach: Stick to one <h1> for the page title. Start subsequent sections (<article>, <section>) with <h2>, and use <h3>, <h4>, etc., for subsections within them, always following the "don't skip levels within a section" rule. This creates a clear outline that works reliably everywhere.

Outline Checkers:

  • Browser developer tools often have features to inspect the accessibility tree or document outline. Online HTML validators or specific outline checker tools can also help visualize the structure your markup creates.

✅ Observation: No code changes. Understand that sectioning elements (<article>, <section>, etc.) create distinct outline sections, and headings inside them define the hierarchy within that section. While multiple <h1>s are technically possible with sectioning content, using a single <h1> per page and descending logically from there is the most common and recommended best practice.

✨ Key Takeaway: Sectioning content elements (<article>, <section>, <nav>, <aside>) work with headings (<h1>-<h6>) to define the document outline. Use headings hierarchically within each section. For maximum compatibility and clarity, stick to one <h1> per page and use <h2>-<h6> logically within sections.
