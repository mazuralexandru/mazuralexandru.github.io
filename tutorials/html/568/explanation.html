568. Reset: Preparing for <audio> Element

We have thoroughly covered embedding video (<video>), providing multiple sources (<source>), and adding text tracks for accessibility (<track>).

Now, we turn our attention to the element specifically designed for embedding audio content: <audio>. Much of its functionality mirrors <video>, but without the visual component.

To focus clearly on the <audio> element, we will remove the entire <video> example structure from index.html.

Why Remove Code?

  • Focus: Isolate the <audio> element and its specific attributes and use cases.
  • Clean Slate: Start fresh without the visual player of the <video> element.

Code Change:

  1. Removed the entire <video>...</video> block (including sources and tracks) from the <body> of index.html.
  2. Updated the <h1> and paragraph text to introduce the <audio> element.
<!-- index.html -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>HTML Multimedia</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <h1>Embedding Audio (&lt;audio&gt;)</h1> <!-- Updated H1 -->
  <p>Resetting page to demonstrate the 'audio' element.</p> <!-- Adjusted P -->

  <!-- Removed video element -->
  <!-- <video ...> ... </video> -->

  <!-- Audio element goes here -->

  <p>Used for embedding sound content like music or podcasts.</p>

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>
</html>

✅ Observation:

  1. Verify the removal of the <video> element from index.html.

✨ Key Takeaway: Clearing the <video> example provides a clean page structure to focus on the syntax and attributes of the native <audio> element.
