214. Links (<a>) vs. Buttons (<button>): Semantic Distinction

While CSS can make links look like buttons and vice-versa, their underlying semantic purpose in HTML is distinct and should guide your choice.

<a> (Anchor Element): Purpose = Navigation

  • Use <a> primarily when the element's function is to navigate the user to a different location. This includes:
    • Going to another web page (internal or external).
    • Jumping to a different section of the current page (fragment links).
    • Accessing a resource (PDF, image file).
    • Activating a specific protocol handler (mailto:, tel:).
  • The core idea is a change in the user's location or resource context.

<button> Element: Purpose = Action

  • Use <button> primarily when the element's function is to trigger an action within the current page's interface without necessarily navigating away. This includes:
    • Submitting a form (<button type="submit">).
    • Resetting form fields (<button type="reset">).
    • Triggering a JavaScript function to manipulate the current page (e.g., opening/closing a dialog modal, playing/pausing media, toggling a feature, performing a calculation) (<button type="button">).
  • The core idea is performing an operation or changing the state of the current interface.

Simple Rule of Thumb:

  • If clicking it takes the user somewhere else (different URL, page section, file, email client) -> Use <a>.
  • If clicking it does something on the current page (submits data, toggles visibility, starts a process) -> Use <button>.

✅ Observation: No code changes. This is a fundamental conceptual distinction. Think about common website interactions: clicking a navigation item (usually <a>), clicking a "Submit" button on a form (usually <button>), clicking a "Play" icon on a video player (usually <button>).

✨ Key Takeaway: Choose <a> for navigation (going somewhere) and <button> for actions (doing something on the current page). This semantic distinction is crucial, even if styling makes them look similar.
