443. Selection Inputs: Checkbox vs. Radio - Semantic Choice

Let's explicitly compare checkboxes and radio buttons to ensure you choose the semantically correct one.

The Key Difference: Single vs. Multiple Selection

  • type="radio": Use when the user must select exactly one option from a list of two or more mutually exclusive choices. Selecting one deselects others in the same name group.
    • Keywords: Choose one, Select an option, Preferred method.
    • Examples: Yes/No, Gender, Payment Type, Shipping Speed, Single Answer Multiple Choice.
  • type="checkbox": Use when the user can select zero, one, or multiple options from a list. Each checkbox operates independently (within the same name group for submission).
    • Keywords: Select all that apply, Choose any, Optional features, Agree to terms.
    • Examples: Interests, Toppings, Subscribe to lists, Optional settings, Agree to Terms & Conditions (single checkbox).

Choosing Incorrectly:

  • Using Checkboxes for Single Choice: If you use checkboxes when only one option should be selected (e.g., for a Yes/No question), users could check both, leading to invalid data submission unless you add complex JavaScript validation to enforce single selection. Using radio buttons enforces this constraint naturally via HTML.
  • Using Radio Buttons for Multiple Choices: If you use radio buttons when multiple selections should be allowed (e.g., for selecting multiple interests), users will only ever be able to select one, preventing them from providing the intended input. Checkboxes are required for multi-select scenarios.

Accessibility & Usability:

  • Using the correct type ensures that assistive technologies announce the control type correctly ("Radio button, 1 of 3" vs. "Checkbox") and that standard keyboard interactions (arrow keys for radio groups, spacebar for toggling) work as expected.
  • It provides clear visual cues (circle vs. square) that align with user expectations for single or multiple selections.

✨ Key Takeaway: Choose type="radio" for single-choice selections from a group (use the same name). Choose type="checkbox" for zero-or-more selections. This semantic choice is critical for functionality and accessibility.
