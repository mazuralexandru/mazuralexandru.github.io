212. Styling Links with CSS (Pseudo-classes Preview)

While this is an HTML tutorial, it's essential to understand that the visual appearance of links is controlled by CSS. A key CSS feature for styling links is pseudo-classes, which allow you to apply styles based on the link's state.

Common Link Pseudo-classes:

These selectors target <a> elements (usually those with an href) in different states:

  1. :link: Styles links that the user has not yet visited. This is often where the default link color (like blue) is set.
  2. :visited: Styles links that the user has visited previously. Browsers restrict the styles you can apply here for privacy reasons (you can usually only change colors like color, background-color, border-color).
  3. :hover: Styles a link when the user's mouse pointer is hovering over it. Often used to change color or remove the underline for visual feedback.
  4. :active: Styles a link during the brief moment it is being activated (e.g., while the mouse button is pressed down on it). Often used for a subtle visual confirmation like a color flash.
  5. :focus: Styles a link when it has keyboard focus. Users can navigate through links using the Tab key. The :focus state (often shown by default as an "outline" around the link) is critical for keyboard accessibility, as it shows keyboard users which link is currently selected. It's vital not to remove focus outlines without providing an alternative, highly visible focus indicator.

LVHA Order: For these pseudo-classes to work correctly due to CSS specificity rules, they are often defined in the CSS file in this specific order: Link, Visited, Hover, Active. (:focus can often go before or after :hover).

Code Change:

We've added temporary styles within <style> tags in the <head> of index.html to demonstrate these pseudo-classes visually. We've also added a couple of placeholder links (href="#") to apply these styles to. (Remember, styles normally go in style.css).

<head>
  ...
  <style>
    /* Basic link state styling examples */
    a:link { color: #87CEFA; text-decoration: underline; } /* Unvisited */
    a:visited { color: #D8BFD8; } /* Visited */
    a:hover { color: #FFB6C1; text-decoration: none; } /* Hover */
    a:active { color: #FFA07A; font-weight: bold; } /* Active */
    a:focus { outline: 2px solid #FFD700; } /* Focus */
    .link-examples a { display: inline-block; margin: 5px 10px 5px 0; }
  </style>
</head>

✅ Observation:

  1. Examine the CSS rules within the <style> block in the <head>.
  2. Look at the two "Example Link" links in the preview. Interact with them:
    • Unvisited (:link): They should initially appear LightSkyBlue with an underline.
    • Hover (:hover): Move your mouse over them. They should turn LightPink and lose the underline.
    • Active (:active): Click and hold the mouse button down. They should turn LightSalmon and bold.
    • Focus (:focus): Use the Tab key on your keyboard to navigate to the links. When a link receives focus, a gold outline should appear around it. This focus indicator is vital for accessibility.
    • Visited (:visited): After clicking one (like Link 2), it might turn Thistle color (browser privacy settings can affect :visited rendering).

✨ Key Takeaway: CSS pseudo-classes (:link, :visited, :hover, :active, :focus) are used to style links differently based on user interaction. Providing clear :hover, :active, and especially :focus styles is crucial for usability and accessibility.
