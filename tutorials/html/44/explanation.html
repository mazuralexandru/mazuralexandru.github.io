44. Introduction to the CSS Cascade

What happens when multiple CSS rules target the same element and try to set the same property (like color)? The browser needs a system to decide which rule "wins". This system is called the Cascade.

The Cascade Problem

Look at our CSS:

  1. The p rule says the paragraph text should be #ccc (light gray).
  2. The .special rule says the paragraph text should be lightcoral.

Our paragraph (<p class="special">) is matched by both selectors. Which color will it actually be? The Cascade algorithm determines the winner based on three main factors (in order of precedence):

  1. Origin and Importance: Where the style comes from (browser default, user stylesheet, author stylesheet) and whether !important is used. For now, we're only concerned with our own "author" styles. !important provides an override mechanism, but should generally be avoided if possible.
  2. Specificity: How specific is the selector that applies the rule? More specific selectors beat less specific ones.
  3. Source Order: If two rules have the same origin/importance and the same specificity, the rule that appears last in the CSS code wins.

Applying the Cascade (Simplified)

  • Origin/Importance: Both our p and .special rules are author styles and neither uses !important. So this factor is a tie.
  • Specificity: A class selector (.special) is considered more specific than a type selector (p). We'll dive into how specificity is calculated next, but for now, know that .special is stronger than p.
  • Result: Because the .special selector is more specific, its declaration (color: lightcoral;) will win over the color: #ccc; from the p rule.
