132. Description Lists (<dl>): Multiple <dd> per Group

Conversely, description list groups also allow a single term (<dt>) to be associated with multiple consecutive descriptions (<dd>).

Syntax:

You list the single <dt> element, followed by all the <dd> elements that describe or relate to it.

<dl>
  <dt>Term 1</dt>
  <dd>First description or value for Term 1.</dd>
  <dd>Second description or value for Term 1.</dd>
</dl>

Use Case:

This is useful when a single term has multiple definitions, aspects, examples, or associated values that need to be listed separately but are all tied to that one term.

Code Change:

We've modified the first group in our <dl> in index.html. The <dt>HTML</dt> is now followed by two separate <dd> elements.

<dl>
   <dt>HTML</dt>
   <dd>HyperText Markup Language.</dd> <!-- Adjusted first dd -->
   <dd>Used for structuring web content.</dd> <!-- Added second dd for HTML -->

   <dt>CSS</dt>
   <dd>Cascading Style Sheets - used for styling web content.</dd>

   <dt>JS</dt>
   <dt>JavaScript</dt>
   <dd>A programming language for web interactivity.</dd>
</dl>

✅ Observation:

  1. Examine the first group (HTML) in the <dl> in index.html. Notice the single <dt> followed by two <dd> tags.
  2. Look at the browser preview:
    • The term "HTML" appears.
    • Below it, both description lines ("HyperText Markup Language." and "Used for structuring...") appear, one after the other, and both are indented, associating them with the single term "HTML".

✨ Key Takeaway: You can associate multiple consecutive <dd> elements with the preceding <dt> element(s) in a description list group, useful for multiple definitions or values for a single term.
