118. Ordered Lists (<ol>): Best Practices & Anti-Patterns

Let's summarize the best practices for using <ol> and highlight common mistakes.

Best Practices:

  1. Use Semantically: Only use <ol> when the sequence or order of items is inherently significant to the content's meaning (steps, ranks, outlines). If order doesn't matter, use <ul>.
  2. Direct Children: Ensure all list content is wrapped in <li> elements that are direct children of the <ol>.
  3. Prefer CSS for Styling: Use CSS (list-style-type, CSS Counters) for controlling the visual appearance of markers (numbers, letters, Roman numerals). Avoid the HTML type attribute unless the marker type itself carries specific semantic weight in your context.
  4. Use Attributes Semantically: Use start, reversed, and li[value] only when the meaning of the sequence requires a non-standard start, reversed order, or specific numbering overrides. Don't use them purely for visual adjustments where CSS Counters would be more appropriate and flexible.
  5. Accessibility: Remember that correct semantic use directly benefits accessibility by clearly conveying the sequence to screen reader users.

Anti-Patterns (Avoid These):

  1. Using <ol> for Visual Numbering Only: Do not use <ol> just to get automatic numbering if the items themselves have no meaningful sequence. Use <ul> instead. If you need numbers visually on an unordered list, consider using CSS Counters.
  2. Manually Typing Numbers: Never create ordered lists by manually typing numbers (e.g., "1.", "2.") inside <li> elements or at the beginning of <p> tags. This breaks semantics, accessibility, and makes reordering or styling impossible. Let the <ol> element handle the numbering automatically.
    <!-- INCORRECT - DO NOT DO THIS -->
<ul>
  <li>1. First item</li> <!-- Manual number inside li -->
  <li>2. Second item</li>
</ul>

<!-- ALSO INCORRECT -->
<p>1. First item</p>
<p>2. Second item</p>

✨ Key Takeaway: Use <ol> strictly for semantically ordered sequences. Let the browser and attributes handle numbering automatically. Use CSS for styling markers and complex numbering schemes (Counters). Avoid manual numbering and using <ol> purely for visual effect.
