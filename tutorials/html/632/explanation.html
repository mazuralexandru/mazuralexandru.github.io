632. Escaping Reserved Characters: Greater Than (&gt;)

Next is the greater-than sign (>).

The Problem:

  • The > character signifies the end of an HTML tag (e.g., <p>). While browsers are often more lenient with unescaped > characters in text content compared to <, using it literally can still cause issues in certain contexts (especially within code examples or near other markup) and is considered bad practice for robustness.

The Solution: &gt;

  • Use the named entity &gt; (stands for "greater than") to represent the literal > character.
  • The browser will parse &gt; and render it visually as >.

Use Cases:

  • Displaying mathematical comparisons (e.g., 10 > 5).
  • Crucially, showing code examples containing HTML tags. To display the text <p>, you must write &lt;p&gt;.

Code Change:

We've added a new section and paragraphs demonstrating &gt; and combining it with &lt; to show an example tag.

<!-- index.html snippet -->
<body>
  <h1>HTML Entities: Escaping Reserved Characters</h1>
  ...
  <h2>Displaying Greater Than (&gt;)</h2> <!-- Added section -->
  <p>To show the greater than sign, use &amp;gt; like this: 10 &gt; 5.</p> <!-- Used &gt; -->
  <p>Crucial for displaying code like &lt;p&gt; (using &amp;lt;p&amp;gt;).</p> <!-- Combined example -->

  <!-- Example of what NOT to do (less likely to break than <, but still bad practice) -->
  <!-- <p>Potentially okay but unsafe: 10 > 5 </p> -->
  ...
</body>

  1. Locate the new section and the use of &gt; in the first paragraph and &lt;p&gt; in the second paragraph.
  2. Look at the browser preview. The first paragraph should render as: 10 > 5. The second should render as: Crucial for displaying code like

    (using <p>).. The browser correctly replaced &gt; with > and the combination with <p>.

✨ Key Takeaway: While browsers might tolerate unescaped >, always use the entity &gt; to display a literal greater-than sign (>) for robustness, especially when showing code examples (&lt;tagname&gt;).
