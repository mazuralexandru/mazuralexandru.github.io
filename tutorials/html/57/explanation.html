font-weight Explained

  • Purpose: Sets the thickness or boldness of the font glyphs.
  • Values:
    • Keywords:
      • normal: Typically equivalent to the numeric value 400. The standard weight.
      • bold: Typically equivalent to the numeric value 700. Standard bold weight.
    • Numeric Values: Ranges from 100 (Thin) to 900 (Black/Heavy) in increments of 100.
      • 100: Thin
      • 200: Extra Light (Ultra Light)
      • 300: Light
      • 400: Normal (Regular)
      • 500: Medium
      • 600: Semi Bold (Demi Bold)
      • 700: Bold
      • 800: Extra Bold (Ultra Bold)
      • 900: Black (Heavy)
    • Relative Keywords: lighter and bolder make the text lighter or bolder relative to the parent element's weight (less commonly used).
  • Font Availability: The crucial point is that for a specific numeric weight (e.g., 300 or 900) to actually render, the chosen font-family must have that specific weight available. If you request font-weight: 900; but the font only has regular (400) and bold (700) weights loaded, the browser will usually try to synthesize a weight or pick the closest available one (often just rendering the 700 weight). This is why when using web fonts (like Google Fonts), you need to ensure you request and load the specific weights you intend to use (e.g., wght@400;700 in the Google Fonts URL).
  • Variable Fonts: A newer technology allows a single font file to contain a continuous range of weights (and other variations like width or slant), offering more fine-grained control and potentially better performance. This is an advanced topic.

In Our Example:

  • We set the body's font-family to "Roboto" to ensure the font is used.
  • .weight-normal uses the normal keyword (maps to 400).
  • .weight-bold uses the bold keyword (maps to 700).
  • .weight-700 uses the explicit numeric value 700.

Since we loaded Roboto weights 400 and 700, all these should render correctly.
