387. Grouping Content: <thead> - Fundamentals

The <thead> element is used to group the header content of a table.

The Table Head Element (<thead>)

  • Definition: Represents the block of header rows in an HTML table.
  • Purpose: To semantically identify the row(s) that contain column headers (<th>) or other header information applicable to the entire table body.
  • Core Concept: It acts as a container specifically for the <tr> element(s) that constitute the table's header section.

Placement & Structure:

  • Must be a direct child of the <table> element.
  • Must appear before any <tbody> or <tfoot> elements in the HTML source code.
  • A table can have at most one <thead> element.
  • Content Model: Must contain one or more <tr> elements. These rows typically contain <th> cells defining the column headers.

Code Change:

We've wrapped the existing header row (<tr> containing <th>s) inside <thead>...</thead tags in index.html.

<!-- index.html snippet -->
  <table>
    <thead> <!-- Added thead -->
      <tr> <!-- Header Row -->
        <th scope="col">Header 1</th>
        <th scope="col">Header 2</th>
      </tr>
    </thead> <!-- Closed thead -->
    <!-- tbody/tfoot will follow -->
    <tr> <!-- Data Row 1 (Currently implicitly in tbody) -->
      <td>Data A1</td>
      <td>Data B1</td>
    </tr>
    <tr> <!-- Data Row 2 (Currently implicitly in tbody) -->
      <td>Data A2</td>
      <td>Data B2</td>
    </tr>
  </table>

✅ Observation:

  1. Locate the <thead> tags wrapping the first <tr> in index.html.
  2. Look at the browser preview. There might be no significant visual change by default just from adding <thead>. Its primary function is semantic grouping and enabling other features (styling, scrolling, printing). The header row still looks like a header row.

✨ Key Takeaway: Use the <thead> element to wrap the header row(s) (<tr> elements, usually containing <th>) of your table. It must appear before <tbody> and <tfoot> in the source code.
