178. Security Solution: rel="noreferrer"

Another related rel attribute value often used with target="_blank" is noreferrer.

rel="noreferrer":

  • Purpose: Instructs the browser not to send the Referer HTTP header when the user follows the hyperlink.
  • The Referer Header: Normally, when you navigate from Page A to Page B, the browser sends an HTTP header called Referer (yes, misspelled in the original spec) along with the request for Page B. This header contains the URL of Page A, telling Page B where the traffic came from.
  • Effect of noreferrer:
    • Privacy: Prevents the destination site (Page B) from knowing the URL of the page the user clicked the link on (Page A). This can be desirable for privacy reasons or to avoid leaking potentially sensitive information in URLs.
    • Implied noopener: Importantly, adding rel="noreferrer" also provides the same security and performance benefits as rel="noopener". Modern browsers treat noreferrer as implicitly including the noopener behavior (setting window.opener to null).

Code Change:

We've changed the rel attribute value on our link in index.html from "noopener" to "noreferrer".

<p>
  Example Link (_blank + noreferrer):
  <a href="https://developer.mozilla.org/" target="_blank" rel="noreferrer"> <!-- Changed rel to "noreferrer" -->
    MDN Home (New Tab, No Referrer)
  </a>
</p>

✅ Observation:

  1. Note the rel="noreferrer" attribute in the <a> tag.
  2. Clicking the link still opens MDN in a new tab. Visually, the behavior is the same as with noopener. The difference is the absence of the Referer header sent to the MDN server (which we can't easily observe here) and the implied noopener protection.

✨ Key Takeaway: Use rel="noreferrer" on links (especially those with target="_blank") when you want to prevent the destination site from knowing the source URL (for privacy) and gain the security/performance benefits of noopener simultaneously.
