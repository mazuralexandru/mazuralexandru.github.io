656. Validation: What Validation Checks - Syntax

What specific kinds of rules does an HTML validator check? Let's start with fundamental syntax rules.

2.1. Syntax Checks:

Validators meticulously examine the basic construction of your HTML code:

  • Correct Tag Opening/Closing: Ensures that every non-void element (like <p>, <div>, <em>) has a corresponding closing tag (</p>, </div>, </em>) placed correctly. It flags unclosed tags or stray end tags.
    • Example Error: Error: End tag for element "p" which is not open.
  • Correct Void Element Syntax: Verifies that void elements (like <img>, <br>, <hr>, <input>) do not have closing tags. While HTML5 parsers ignore optional self-closing slashes (<br />), validators ensuring strict adherence might flag unnecessary slashes or expect their absence depending on configuration, though the core check is against having a separate closing tag like </br>.
    • Example Error: Error: End tag "br". (if </br> was used)
  • Proper Attribute Syntax: Checks that attributes follow the name="value" pattern (or just name for boolean attributes). Critically, it enforces the use of quotes around attribute values. While browsers might parse unquoted values in simple cases, validation requires quotes for robustness.
    • Example Error: Error: Attribute value must be quoted. (if class=my-class was used)
  • Correct Character Escaping (Entities): Verifies that reserved characters (<, >, &, and sometimes "/' in attributes) are correctly escaped using HTML entities (&lt;, &gt;, &amp;amp;, &quot;, &apos;). Flags unescaped characters that could break parsing.
    • Example Error: Error: "&" did not start a character reference. (if R&D was used instead of R&amp;D)

These syntax checks form the most basic layer of validation, ensuring the code is fundamentally readable by the HTML parser according to the language's grammar.

✅ Observation: No code changes. Review the list of syntax checks. Consider common typing errors like forgetting a closing tag or omitting quotes around an attribute value – these are precisely the kinds of errors a validator excels at finding instantly.

✨ Key Takeaway: HTML validation rigorously checks fundamental syntax rules, including proper tag opening/closing, void element format, attribute quoting, and correct character escaping, catching many common coding errors.
