547. Multimedia: Video (<video>) - autoplay, muted, loop Attributes

Let's explore the boolean attributes that control automatic playback, audio state, and repetition.

1. autoplay Attribute:

  • Purpose: Hints that the video should start playing automatically as soon as possible without waiting for the user to initiate playback.
  • Syntax: <video autoplay>
  • Restrictions (Critical): Modern browsers heavily restrict autoplay with sound to prevent user annoyance. Autoplay usually only works if muted is also present, or based on user interaction history/browser settings. Do not rely on autoplay with sound.
  • Use Cases: Often used with muted and loop for background videos. Otherwise, generally discouraged in favor of user-initiated playback via controls.

2. muted Attribute:

  • Purpose: Mutes the audio track of the video by default.
  • Syntax: <video muted>
  • Behavior: Video starts silent. Users can typically unmute using the player controls (if visible). Often necessary to enable autoplay.

3. loop Attribute:

  • Purpose: Causes the video to automatically loop back to the beginning and play again once it reaches the end.
  • Syntax: <video loop>
  • Use Cases: Background videos, short repeating animations or clips.

Code Change:

We've added autoplay, muted, and loop attributes to our <video> tag. We've also re-added a hypothetical src attribute pointing to a short video file, as these attributes only have an effect if there's a video source.

<!-- index.html snippet -->
  <!-- Added autoplay, muted, loop -->
  <video controls width="480" height="270"
         src="/videos/short-loop.mp4" <!-- Hypothetical short video -->
         autoplay muted loop>
    <p>Sorry, your browser doesn't support embedded videos.</p>
  </video>

✅ Observation:

  1. Locate the autoplay, muted, and loop boolean attributes added to the <video> tag. Note the presence of src again.
  2. Look at the browser preview.
  3. (Conceptual): If /videos/short-loop.mp4 existed and contained a playable video:
    • The video would attempt to start playing automatically (autoplay).
    • It would start silently (muted).
    • When it reached the end, it would restart from the beginning (loop).
    • The controls would still be visible, allowing the user to pause, unmute, etc.
    • (In our environment, it will still likely show an error as the file doesn't exist).

✨ Key Takeaway: Use autoplay (usually requires muted) to attempt automatic playback. Use muted to start the video without sound. Use loop to make the video repeat automatically. Use autoplay sparingly and responsibly.
