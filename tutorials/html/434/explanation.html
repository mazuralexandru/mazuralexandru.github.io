434. background-repeat

Files Modified: style.css

By default, background images tile (repeat) both horizontally and vertically to fill the background area. The background-repeat property controls this behavior.

  • Purpose: To specify if/how a background image should be repeated.
  • Applies To: Each background image layer individually (if multiple images are specified, use comma-separated values).
  • Common Values:
    • repeat (Default): Tiles the image in both directions.
    • repeat-x: Repeats horizontally only.
    • repeat-y: Repeats vertically only.
    • no-repeat: Displays the image only once.
    • space: Repeats as many times as possible without clipping, distributing space evenly between tiles.
    • round: Repeats as many times as possible by scaling the image slightly (if necessary) to fit an integer number of times.

✅ Observe style.css:

  • We've added background-repeat: no-repeat; to the .bg-box rule. Since we have two background layers (gradient and image URL), this value applies to both by default if only one is provided. To be explicit, we could use background-repeat: no-repeat, repeat; to make the gradient not repeat but let the pattern repeat, but for simplicity we'll apply no-repeat to both for now.
.bg-box {
    background-color: transparent; 
    background-image: 
        linear-gradient(to bottom, rgba(0,0,0,0.5), transparent), 
        url('images/placeholder-pattern.png'); 
    background-repeat: no-repeat; /* Stop image/gradient from repeating */
    padding: 2em;
    margin-bottom: 1em;
    border: 1px dashed #fff; 
    min-height: 150px; 
}
