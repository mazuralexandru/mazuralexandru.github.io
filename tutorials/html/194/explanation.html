194. Absolute Units Recap: px

Files Modified: None

Absolute length units represent a fixed physical measurement. Once set, their size isn't directly affected by parent element sizes or user settings (though overall page zoom affects everything). The most common absolute unit on the web is the pixel (px).

  • Pixel (px): Represents one "device pixel" (dot) on the output device.
    • Pros: Provides fine-grained control, often perceived as predictable, widely understood. Useful for elements that should not scale with text size, like thin borders or fixed-size icons.
    • Cons: Not inherently scalable with user font-size preferences. Using px for everything, especially font sizes and layout spacing, can hinder accessibility and responsiveness. A user increasing their base font size in the browser won't affect px-based layouts.
    • Use Cases: Borders (border: 1px solid black;), sometimes small, fixed margins/paddings where scalability isn't desired, occasionally base font size (though rem is often preferred).

Other absolute units exist (cm, mm, in, pt, pc) but are rarely used for screen styling, being more relevant for print stylesheets.

Rule of Thumb: Use px sparingly, primarily for things like borders where a fixed small size is intended. Prefer relative units for font sizes and most layout spacing.
