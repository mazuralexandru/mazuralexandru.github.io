355. Styling Description Lists (<dl>, <dt>, <dd>): Common Layouts

Files Modified: None

Description lists (<dl>) often require specific styling to achieve desired layouts beyond the browser default (term on one line, description indented below).

Common Layout Approaches:

  1. Default Browser Style:

    • <dt> acts like a block element.
    • <dd> acts like a block element with default margin-left for indentation.
    • This is often sufficient for simple glossaries.

  2. Inline Term and Description:

    • Goal: Place the term and its description on the same line, often with the term bolded.
    • Technique: Might involve setting display: inline; on both <dt> and <dd> (potentially problematic if descriptions are long) or, more robustly, using Flexbox or Grid on a wrapper <div> around each dt/dd pair (or directly on the dl if the structure allows).

  3. Two-Column Layout:

    • Goal: Place the term in one column and the description in an adjacent column.
    • Technique: Almost always requires CSS Grid or Flexbox applied to the parent <dl> (or wrapper <div>s).
      • Grid: Define two columns on the <dl> (display: grid; grid-template-columns: auto 1fr;). Each <dt> and <dd> automatically flows into the next available grid cell.
      • Flexbox: Might involve setting display: flex; on wrapper <div>s around each pair, with fixed width on the <dt> and flex-grow: 1; on the <dd>.

Styling <dl> often involves overriding default margins and using more advanced layout techniques depending on the desired visual outcome.
