698. Keyboard Navigation: Interaction

Once an element has keyboard focus, users need standard ways to activate or interact with it using the keyboard.

Standard Keyboard Interactions (Native Elements):

HTML's native interactive elements have built-in, expected keyboard behaviors:

  • Links (<a href="...">):
    • Activated by pressing Enter.
  • Buttons (<button>, <input type="button/submit/reset">):
    • Activated by pressing Enter OR Space. (This is a key difference from links).
  • Checkboxes (<input type="checkbox">):
    • Toggled (checked/unchecked) by pressing Space.
  • Radio Buttons (<input type="radio">):
    • Selected using Space.
    • Crucially, within a radio group (same name), focus often moves between the options using the Arrow Keys (Up/Down or Left/Right), and selecting one automatically deselects the others. Tabbing usually moves focus into and out of the entire group, not between individual radio buttons within it.
  • Select Dropdowns (<select>):
    • Opened using Alt + Down Arrow, Space, or Enter.
    • Options navigated using Up/Down Arrow keys.
    • Option selected using Enter.
  • Text Inputs/Textareas (<input type="text/email/etc.">, <textarea>):
    • Receive text input directly once focused. Enter might trigger form submission if it's the only text field or depending on context.

Importance for Accessibility:

  • Users who rely on keyboards expect these standard interaction patterns. Adhering to them is crucial for usability.

Custom Controls & JavaScript:

  • If you create custom interactive elements using non-native tags (like <div>s or <span>s styled to look like buttons or menus):
    • You must use tabindex="0" to make them focusable.
    • You must use JavaScript addEventListener to listen for keyboard events (keydown, keyup).
    • Your JavaScript must implement the expected interaction logic (e.g., listen for Enter and Space on a fake button, handle arrow key navigation for custom menus or radio groups).
    • This adds significant complexity and potential for error compared to using native HTML elements.

Principle: Use native HTML elements whenever possible, as they provide standard, accessible keyboard interaction "for free".
