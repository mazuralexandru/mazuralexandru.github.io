160. href URL Types: Relative URL (Root-Relative)

A fourth type of relative URL is the root-relative URL.

Definition:

  • A root-relative URL specifies a path starting from the root directory of the website (the top-level directory associated with the domain name).
  • It is distinguished by starting with a forward slash (/).

Syntax:

  • href="/path/from/root/filename.html"

Key Behavior:

  • The leading / tells the browser to ignore the current page's directory path and start resolving the path directly from the website's root.
  • Example: If your domain is http://www.mysite.com, a link like <a href="/assets/style.css"> will always resolve to http://www.mysite.com/assets/style.css, no matter where the page containing the link is located (e.g., whether it's in /products/, /about/team/, or the root itself).

Use Cases & Advantages:

  • Excellent for linking to site-wide resources like main CSS files, JavaScript files, logos, or top-level pages (like the homepage / or /contact.html) from anywhere within the site structure.
  • Provides consistency, as the path doesn't change depending on the current page's depth in the directory structure.

Caveat:

  • Relies on the web server being configured correctly to define the website's "root" directory.
  • Slightly less portable than other relative URLs if you move a subsection of the site to a different location relative to the root (though generally more robust than absolute URLs for internal linking).

Code Change:

We've modified the link in index.html to demonstrate a root-relative path.

<body>
  <h1>Links and Navigation</h1>
  <p>Relative URL: Root-relative paths start from the domain root.</p>

  <p>This link always points to '/images/logo.png' relative to the domain root, regardless of the current page's location:</p>
  <p><a href="/images/logo.png">Site Logo</a></p> <!-- Added root-relative link -->

  <p>Starts with a '/' character.</p>

</body>
