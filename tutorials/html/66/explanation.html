66. Specialized Text: Inline Code (<code>)

While <pre> handles block-level code, <code> is used for marking up short, inline fragments of computer code within the flow of text.

The Inline Code Element (<code>)

  • Purpose: Semantically identifies a short piece of computer code, such as a variable name, function name, method name, keyword, filename, CSS property, or HTML tag name.
  • Semantics: Clearly marks the enclosed text as being "code" or machine-readable text, distinct from the surrounding prose.
  • Type: Inline element. It flows within the line and doesn't cause breaks.
  • Syntax: Uses an opening <code> and closing </code> tag.

Default Rendering:

Browsers typically render text inside <code> using a monospace font, similar to <pre>, to visually distinguish it as code.

Use Cases:

  • Referencing code elements within explanatory text, documentation, or tutorials.
  • Marking up commands or configuration values mentioned in a paragraph.
