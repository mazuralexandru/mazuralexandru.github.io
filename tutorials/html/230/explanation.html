230. <em> & <strong>: CSS Styling

Files Modified: style.css

While browsers provide default italic/bold styling, you can customize the appearance of <em> and <strong> elements using CSS, just like any other element. You might do this to better integrate them with your site's design or to provide alternative visual cues.

✅ Observe style.css:

  • We've added rules specifically targeting em and strong.
  • For em, we're removing the default italic style but adding a background color and padding.
  • For strong, we're keeping the bold weight but changing the color.
/* Styling Semantic Elements */
em {
  font-style: normal; /* Remove default italic */
  background-color: rgba(200, 200, 0, 0.3); /* Add subtle yellow background */
  padding: 0.1em 0.3em;
  border-radius: 3px;
}

strong {
  /* font-weight: bold; <- Already default, no need to redeclare unless changing */
  color: orange; /* Change color from inherited */
}

✨ Observe the Preview:

  • The word "standard" (<em>) in the first paragraph is no longer italic but now has a subtle yellow background highlight.
  • The word "follows" (<strong>) in the second paragraph is still bold, but its color should now be orange instead of the inherited light green.

This demonstrates that while <em> and <strong> have semantic meaning and default styles, you have full control over their final presentation using CSS.
