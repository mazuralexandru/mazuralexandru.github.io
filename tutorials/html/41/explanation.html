41. CSS Combinators: Child Combinator (>)

While the Descendant Combinator (space) selects elements nested anywhere inside, the Child Combinator (>) is more specific: it only selects elements that are direct children of the parent element.

Child Combinator (>)

  • Syntax: Two selectors separated by the "greater than" symbol (>): parent > child.
  • Meaning: Selects all elements matching the child selector that are direct children of an element matching the parent selector. It does not select grandchildren or other deeper descendants.
  • Specificity: More specific than the descendant combinator. Useful when you want to avoid styles "leaking" down into nested structures unintentionally.

In our example:

  • The rule ul > li { ... } targets <li> elements that are immediate children of a <ul> element.
  • It applies an orange color and changes the list-style-type to square.
