376. Common Link Styling: Color/Underline on Hover/Focus

Files Modified: None (Refining existing styles)

Let's refine the common pattern of changing color and adding/removing underlines for interaction states.

Our current setup in style.css already implements this:

  • Base State (:link, :visited):
    • text-decoration: none; (Underline removed)
    • Specific colors are set (cyan for :link, mediumpurple for :visited).
  • Interaction State (:hover):
    • text-decoration: underline; (Underline added back)
    • color: lightcoral; (Color changes)
  • Focus State (:focus-visible):
    • Applies an outline for visibility, doesn't necessarily change text decoration or color unless desired (could be combined with :hover styles if needed: a:hover, a:focus-visible { ... }).

This pattern ensures:

  1. Links are initially distinct (color) but not underlined (common design choice).
  2. Hovering provides clear feedback (color change, underline appears).
  3. Keyboard focus provides clear feedback (outline appears).
