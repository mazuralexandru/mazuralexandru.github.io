527. Semantics: Structuring a Typical Page - Example & Review

Let's synthesize what we've learned by structuring a typical page layout using semantic HTML elements. This example brings together landmarks, sectioning content, headings, and other semantic elements.

Visualizing the Structure:

Imagine a standard blog post page layout:

  1. Top Header: Site title/logo and primary navigation.
  2. Main Content Area: The blog post itself, possibly with multiple sections, and potentially a sidebar with related links or info.
  3. Bottom Footer: Copyright, contact info, secondary links.

Mapping Layout to Semantics:

We will now build this structure in our HTML using the appropriate tags.

Code Change:

We've removed the previous <data> examples and populated the <body> with a common semantic structure representing the layout described above. We've also updated the temporary CSS to visualize all the key structural elements.

<!-- index.html -->
<head>
  ...
  <style>
    /* Basic layout visualization */
    body > header, body > nav, body > main, body > footer, main article, main section, main aside {
      border: 1px dashed skyblue;
      padding: 10px;
      margin: 10px 0;
      min-height: 50px; /* Ensure visibility */
    }
    /* Basic layout for main/aside (Flexbox) */
    main { display: flex; gap: 20px; }
    main article { flex-grow: 1; }
    main aside { flex-basis: 200px; flex-shrink: 0; }
    /* Basic nav styling */
    nav ul { list-style: none; padding: 0; }
    nav li { display: inline-block; margin-right: 10px; }
    /* Basic heading margin */
    h1, h2, h3 { margin: 0 0 0.5em 0; }
  </style>
</head>
<body>
  <!-- Site Header -->
  <header>
    <h1>Site Title</h1>
    <nav aria-label="Primary Navigation">...</nav>
  </header>

  <!-- Main Content Area -->
  <main>
    <!-- Main Article/Post -->
    <article>
      <header> <!-- Article Header -->
        <h2>Main Article Title</h2>
        <p>Published: <time datetime="2024-01-01">Jan 1, 2024</time></p>
      </header>
      <section aria-labelledby="sec1-heading"> <!-- Article Section 1 -->
        <h3 id="sec1-heading">Section One</h3>
        <p>Content for section one...</p>
      </section>
      <section aria-labelledby="sec2-heading"> <!-- Article Section 2 -->
        <h3 id="sec2-heading">Section Two</h3>
        <p>Content for section two...</p>
      </section>
      <footer> <!-- Article Footer -->
         <p>Article tags: <mark>semantic</mark>, <mark>html</mark>.</p>
      </footer>
    </article>

    <!-- Sidebar -->
    <aside aria-labelledby="sidebar-heading">
       <h3 id="sidebar-heading">Related Info</h3>
       <p>Tangential content goes here.</p>
       <nav aria-label="Secondary Navigation">...</nav>
    </aside>
  </main>

  <!-- Site Footer -->
  <footer>
    <p><small>&copy; 2024 Site Name</small></p>
    <address>Contact: <a href="mailto:x@y.com">Email Us</a></address>
  </footer>
  ...
</body>

✅ Observation:

  1. Carefully trace the structure in index.html. Identify the top-level <header>, <main>, and <footer>. Inside <header>, find the <nav>. Inside <main>, find the <article> and the <aside>. Inside the <article>, find its nested <header>, <section>s, and <footer>. Note the appropriate heading levels (h1 for site, h2 for article, h3 for sections/aside). Note the use of aria-label / aria-labelledby for navigation and regions.
  2. Look at the browser preview. The temporary CSS (borders and flex layout) should visually outline these different semantic regions, showing the header, main content area (with article and sidebar side-by-side), and footer as distinct blocks.

✨ Key Takeaway: This example demonstrates how to combine landmark elements (<header>, <nav>, <main>, <footer>) and sectioning content (<article>, <section>, <aside>) with appropriate headings (<h1>-<h6>) and other semantic tags (<time>, <address>, <mark>) to create a well-structured, meaningful, and accessible page layout.
