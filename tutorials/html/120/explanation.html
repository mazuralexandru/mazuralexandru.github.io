120. Nested Lists: Syntax Rule (ul inside li)

The most crucial rule for creating nested lists correctly is placement.

Syntax Rule:

  • A nested list (<ul> or <ol>) must be placed directly inside the <li> element it serves as a sub-list for.
  • It should appear after any introductory text for the parent list item but before the closing </li> tag of that parent item.

Incorrect Placement (Common Mistake): Placing the nested list between <li> elements or directly inside the parent <ul>/<ol> (but not inside an <li>) is invalid HTML.

Code Change:

We have now added a nested <ul> inside the <li> for "Item Two" in index.html. We also added "Item Three" for context.

<ul> <!-- Outer ul -->
  <li>Item One</li>
  <li>Item Two  <!-- Outer li for Item Two STARTS -->
    <ul> <!-- Nested ul START (inside li) -->
      <li>Sub-item A</li>
      <li>Sub-item B</li>
    </ul> <!-- Nested ul END (inside li) -->
  </li> <!-- Outer li for Item Two ENDS -->
  <li>Item Three</li> <!-- Added Item Three -->
</ul>

Pay close attention to where the nested <ul>...</ul> block is placed relative to the opening <li> and closing </li> tags for "Item Two".

✅ Observation:

  1. Carefully examine the structure in index.html. Trace the opening and closing tags to confirm the nested <ul> is fully contained within the second <li>.
  2. Look at the browser preview. You should see:
    • Item One (Level 1 bullet).
    • Item Two (Level 1 bullet).
    • Sub-item A and Sub-item B displayed below "Item Two" and indented further.
    • Sub-item A and Sub-item B will likely have a different bullet style (e.g., circles ○ instead of discs •) due to default browser styling for nested lists.
    • Item Three (Level 1 bullet) appears after the nested list, aligned with Item One and Item Two.

✨ Key Takeaway: The fundamental rule for nesting lists is that the inner (nested) <ul> or <ol> must be placed entirely within the <li> element of the outer (parent) list that it modifies or expands upon.
