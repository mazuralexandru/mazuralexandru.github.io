151. Placeholder Links (<a> without href vs. href="#")

Let's clarify the distinction between an anchor tag without an href attribute and one with href="#".

  1. <a> without href:

    • Semantics: Represents a placeholder where a link might otherwise be placed. It's not considered a hyperlink by the browser or assistive technologies.
    • Appearance: Typically rendered as plain text, without default link styling (no blue color, no underline).
    • Behavior: Not interactive by default (clicking does nothing). It can be targeted by CSS and potentially made interactive with JavaScript, but it doesn't have the built-in link behaviors. It might be focusable via keyboard (Tab).

  2. <a href="#">:

    • Semantics: Represents a hyperlink whose destination is the top of the current page fragment identifier.
    • Appearance: Receives default hyperlink styling (blue, underlined). Looks like a link.
    • Behavior: Interactive. Clicking scrolls the page to the top (unless prevented by JavaScript). It's announced as a "link" by screen readers.

Use Cases:

  • <a> without href: Rarely used. Sometimes seen as placeholders in templates before the actual link destination is known, or perhaps as targets for specific JavaScript interactions where link semantics aren't desired.
  • <a href="#">: Very common as a placeholder during development before the real URL is ready, or for links that will only trigger JavaScript actions (though using <button> is often semantically better for actions that don't navigate).

✨ Key Takeaway: An <a> tag needs the href attribute to be treated as a true hyperlink (receiving default styles and behaviors). href="#" creates a styled link that points to the top of the page, often used as a placeholder. An <a> tag without href is just a placeholder element, not a link.
