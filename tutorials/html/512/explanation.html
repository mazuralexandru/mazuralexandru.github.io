512. Reset: Preparing for Sectioning Content (<article>, <section>, <aside>)

We have established the main landmark structure of a page using <header>, <nav>, <main>, and <footer>.

Now, we need to explore the elements used to structure the content within these landmarks, particularly within <main>. These are the sectioning content elements: <article>, <section>, and <aside> (which we'll also consider as potentially being alongside <main>).

To focus on these, we will simplify the content inside the <main> element from the previous step, removing the placeholder <h2>/<p> to make way for <article> examples. We'll also add an aria-label to the <nav> as discussed for best practice with landmarks.

Why Reset/Adjust?

  • Focus: Isolate the sectioning content elements and their specific roles within the main content area.
  • Clarity: Build up the internal structure (article, section) piece by piece.

Code Change:

  1. Added aria-label="Primary site navigation" to the <nav> element (good practice).
  2. Replaced the content inside <main> with a new placeholder heading and paragraph.
  3. Adjusted the temporary CSS to also visualize article, section, and aside when they are added inside main.
<!-- index.html snippet -->
<head>
  ...
  <style>
    /* Basic landmark/section visualization */
    body > header, body > nav, body > main, body > footer, /* Added main children */ main article, main section, main aside {
      border: 1px dashed skyblue;
      padding: 10px;
      margin: 10px 0;
    }
    nav ul { list-style: none; padding: 0; } /* Basic nav list style */
    nav li { display: inline-block; margin-right: 10px; }
  </style>
</head>
<body>
  <header>
    <h1>Site Title / Logo Area</h1>
    <nav aria-label="Primary site navigation"> <!-- Added aria-label -->
      <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
        <li><a href="/contact">Contact</a></li>
      </ul>
    </nav>
  </header>

  <main> <!-- Content inside main reset -->
    <h1>Main Content Title (Maybe H2 if H1 is Site Title)</h1>
    <p>Introductory paragraph for the main content.</p>

    <!-- Article goes here -->

  </main>

  <footer>
    <p><small>&copy; 2024 Your Website Name. All rights reserved.</small></p>
    <p><a href="/privacy">Privacy Policy</a> | <a href="/terms">Terms of Service</a></p>
  </footer>
  ...
</body>

✅ Observation:

  1. Verify the updated content within the <main> element and the added aria-label on <nav>. Note the updated CSS selector in the <style> block.
  2. The browser preview shows the header, the new main content intro, and the footer, all with dashed borders. The <main> section is ready for sectioning content examples.

✨ Key Takeaway: Resetting the content within <main> prepares a clean space for demonstrating the specific semantic roles of <article>, <section>, and <aside> in structuring the primary content area.
