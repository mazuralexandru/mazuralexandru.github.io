111. CSS Comments

Files Modified: style.css

Just like HTML, CSS also has its own syntax for comments. These are used within <style> blocks in HTML or, more commonly, in external .css files like our style.css.

  • Purpose: Explain CSS rules, document sections of the stylesheet, or temporarily disable specific declarations or entire rulesets.
  • Syntax: CSS comments start with /* and end with */. They can span multiple lines. There is no single-line comment syntax like // in CSS (that's used in JavaScript).

✅ Observe style.css!

  • We've added single-line comments explaining the file's purpose and a specific declaration.
  • We've added a multi-line comment.
  • We've added some actual CSS rules for <h1> and <p>.
  • Crucially, we've used comments to "comment out" the text-decoration for the <h1> and the entire color rule for the <p>.

✨ Observe the Preview:

  • The <h1> ("Main Heading") should now be cornflowerblue. The underline rule is commented out, so it's not applied.
  • The <p> ("This is a standard paragraph.") remains the default light color set by base_styles.css, because its specific color: limegreen; rule is commented out.

CSS comments are ignored by the browser when applying styles but are essential for developers.
