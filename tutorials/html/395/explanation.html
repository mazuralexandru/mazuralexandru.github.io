395. Spanning Columns: The colspan Attribute - Fundamentals

Sometimes, a header or data cell needs to stretch horizontally across more than one column in the table grid. The colspan attribute enables this.

The colspan Attribute:

  • Definition: An attribute applicable to both <th> and <td> elements.
  • Purpose: Specifies the number of columns that a single cell should span (merge across).
  • Core Concept: It allows a cell to occupy the horizontal space normally taken by multiple adjacent cells in the same row.

Syntax:

  • <th colspan="number">...</th>
  • <td colspan="number">...</td>
  • The number value must be a positive integer greater than 0.
    • colspan="1" is the default and has no effect.
    • colspan="2" makes the cell span across two columns.
    • colspan="3" spans three columns, etc.
    • colspan="0" is a special value meaning "span until the last column of the column group", but its use is less common and potentially less reliable than specifying an exact number.

Layout Impact:

  • When a cell uses colspan, you need to reduce the number of subsequent <th> or <td> elements in that specific <tr> to compensate for the columns already occupied by the spanned cell.
  • If a row has 3 columns total, and the first cell has colspan="2", you only need one more <td> or <th> in that row to fill the third column.

✅ Observation: No code changes yet. Understand the purpose of colspan: to make a cell wider, covering multiple columns, and the consequent need to adjust the number of cells in that row.

✨ Key Takeaway: The colspan attribute, applied to <th> or <td>, allows a cell to span horizontally across a specified number of columns. Remember to adjust the number of cells in the row accordingly.
