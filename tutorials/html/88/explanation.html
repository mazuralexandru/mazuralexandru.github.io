88. Whitespace in CSS

Files Modified: None

How does CSS handle whitespace (spaces, tabs, line breaks)?

Generally, CSS is whitespace-insensitive outside of property values. This means you can format your code for readability without affecting how it works.

These are typically equivalent:

/* Compact */
p{color:red;font-size:16px;} 

/* Readable */
p {
  color: red;
  font-size: 16px;
}

/* Also valid, though less conventional */
p 
{
    color:      red;

    font-size:  16px;
}
  • Readability: Use whitespace (indentation, line breaks) consistently to make your CSS easy to read and understand. The second example above is the most common and recommended format.
  • Property Values: Whitespace can be significant within property values (e.g., separating multiple values for margin or font-family, or within a url() value).
  • Selectors: Whitespace can also be significant within complex selectors (e.g., descendant combinator div p vs. divp).

Use a code formatter (like Prettier) to automatically maintain consistent and readable whitespace formatting.
