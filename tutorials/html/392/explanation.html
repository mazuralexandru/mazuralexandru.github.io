392. Table Caption: The <caption> Element - Fundamentals

A data table should almost always have a title or caption to explain its purpose and content. HTML provides the <caption> element specifically for this.

The Table Caption Element (<caption>)

  • Definition: Represents the title or caption of its parent <table> element.
  • Purpose: To provide a concise, descriptive label that identifies the table's content for all users.
  • Semantic Meaning: Explicitly associates the caption text with the table structure, acting as its accessible name.

Placement Rule (CRITICAL):

  • The <caption> element MUST be the very first child placed immediately inside the opening <table> tag.
  • It must appear before any <thead>, <tbody>, <tfoot>, <tr>, or <colgroup> elements.
  • Only one <caption> is allowed per table.

Content Model:

  • Flow Content, but typically contains phrasing content (the caption text). It should generally not contain block-level elements like <h2> or <p> directly inside it.

Code Change:

We've added a <caption> element as the first child inside the <table> tag in index.html.

<!-- index.html snippet -->
  <table>
    <caption>Sample Data Comparison</caption> <!-- Added caption -->
    <thead>
      <tr>
        <th scope="col">Header 1</th>
        <th scope="col">Header 2</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>Data A1</td>
        <td>Data B1</td>
      </tr>
      <tr>
        <td>Data A2</td>
        <td>Data B2</td>
      </tr>
    </tbody>
  </table>

✅ Observation:

  1. Locate the <caption>Sample Data Comparison</caption> element placed immediately after the opening <table> tag and before <thead> in index.html.
  2. Look at the browser preview. You should now see the text "Sample Data Comparison" displayed above the table content, usually centered horizontally. This is the default rendering position for <caption>.

✨ Key Takeaway: Use the <caption> element to provide a title for your data table. It must be the first element directly inside the <table> tag.
