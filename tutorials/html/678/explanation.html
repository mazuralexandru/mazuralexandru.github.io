678. Reset & Common Errors: Duplicate id

As emphasized when discussing the id attribute, its value must be unique within the document. Using the same id on multiple elements is a critical validation error.

Resetting:

Remove the previous <img> examples.

Error: Duplicate ID

  • Problem: Two or more elements within the same HTML document are assigned the exact same value for their id attribute.
  • Impact: Breaks the fundamental purpose of id as a unique identifier. Causes unpredictable behavior for CSS (#id selector usually only targets the first), JavaScript (document.getElementById() only returns the first), fragment links (#id), label association (for="id"), and ARIA relationships (aria-labelledby). Results in invalid HTML.
  • Validator Message (Example): Error: Duplicate ID "item1".

Code Change:

  1. Removed the previous <img> examples from index.html.
  2. Added a "Bad Example" showing two <p> elements with the same id="item1".
  3. Added a "Good Example" showing two <p> elements with unique IDs (itemA, itemB).
<!-- index.html -->
<body>
  <h1>Common Validation Errors</h1>

  <h2>Error: Duplicate ID</h2>

  <!-- Bad Example -->
  <p id="item1">First item with ID 'item1'.</p>
  <p id="item1">Second item conflicting ID 'item1'.</p> <!-- Duplicate ID -->

  <hr> <!-- Separator -->

  <!-- Good Example -->
  <p id="itemA">First item with unique ID 'itemA'.</p>
  <p id="itemB">Second item with unique ID 'itemB'.</p>

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

✅ Observation:

  1. Verify the removal of previous examples and the addition of the id examples. Note the duplicated id="item1" in the "Bad Example".
  2. The browser preview shows the paragraphs, but the duplicate ID issue is structural, not visual.
  3. (Conceptual): Running the "Bad Example" through validator.w3.org would report a "Duplicate ID" error. document.getElementById('item1') in JavaScript would only ever return the first paragraph. A link href="#item1" would have unpredictable behavior.

✨ Key Takeaway: Validators strictly enforce the rule that all id attribute values must be unique within a document. Duplicate IDs are invalid HTML and break styling, scripting, linking, and accessibility features relying on unique identification.
