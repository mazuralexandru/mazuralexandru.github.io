127. Description Lists (<dl>): Fundamentals

Let's introduce the container element for description lists: <dl>.

The Description List Element (<dl>)

  • Definition: The <dl> element represents an association list or description list. It consists of zero or more groups, where each group associates one or more terms/names (<dt>) with one or more descriptions/values (<dd>).
  • Purpose: To structure content that naturally forms pairs of related information, like terms and definitions, metadata keys and values, questions and answers, or speakers and dialogue.
  • Semantic Meaning: Conveys these term-description relationships. It's distinct from a simple list of items (<ul>/<ol>).
  • Type: Block-level element.

Analogy:

  • A glossary section.
  • A dictionary entry structure.
  • Product specification (Feature: Detail).
  • FAQ list (Question: Answer).

Structure Note: Unlike <ul> and <ol> which contain only <li> children, the <dl> element has a specific structure involving two different child elements: <dt> (Description Term) and <dd> (Description Details), which we will introduce next.
