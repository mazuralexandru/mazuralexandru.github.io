229. src URL Types: Relative URL Best Practices Review

Let's quickly reinforce the different types of relative URLs and best practices when using them for the src attribute.

Relative URL Types Recap (Applied to src):

  1. Same Directory: If image.jpg is in the same folder as index.html: <img src="image.jpg" alt="...">
  2. Subdirectory: If image.jpg is inside an images folder, which is in the same folder as index.html: <img src="images/image.jpg" alt="...">
  3. Parent Directory: If index.html is inside a pages folder, and image.jpg is in the parent directory of pages: <img src="../image.jpg" alt="..."> Or, if image.jpg is in a sibling folder named assets: <img src="../assets/image.jpg" alt="...">
  4. Root-Relative: If image.jpg is inside an images folder located directly at the website's root: <img src="/images/image.jpg" alt="..."> (Works from any page on the site).

Best Practices for Image src:

  • Use Relative Paths for Site Assets: Strongly prefer relative paths (any of the types above) for all images that are part of your website's content or design. This ensures portability.
  • Organize Images: Keep images in dedicated folders (e.g., /images/, /assets/img/, /uploads/) rather than scattering them in the same directories as your HTML files. This improves project organization.
  • Choose Appropriate Relative Type:
    • Root-relative paths (/...) are often excellent for common site-wide images like logos or icons accessed from many different pages, as the path is always the same.
    • Document-relative paths (../, images/) are good for images closely associated with a specific section or page.
  • Consistency: Adopt a consistent strategy for your paths (e.g., always use root-relative for layout images, document-relative for content images).
  • Filenames: Use descriptive, lowercase filenames with hyphens (my-product-photo.jpg), not spaces.
