479. Forms: Validation - The required Attribute

The most fundamental validation attribute is required, used to indicate that a form field must be filled out before the form can be submitted.

The required Attribute:

  • Definition: A boolean attribute. Its presence signifies that the user must provide a value for the associated control.
  • Applies To: <input> (most types except hidden, range, color, and button types like submit, reset, button, image), <select>, and <textarea>.
  • Syntax: <input type="..." name="..." required> or <select name="..." required> or <textarea name="..." required>.
  • Behavior (Native Validation):
    • If a control has the required attribute and the user tries to submit the form without providing a value (e.g., leaving a text input empty, not selecting a radio button in a required group, not selecting a file, not selecting a non-placeholder option in a <select>), the browser will:
      1. Block the form submission.
      2. Typically focus on the first invalid required field.
      3. Display a default error message (e.g., "Please fill out this field.") near the field. (Appearance/wording varies by browser).
  • CSS Styling: You can style required fields using the :required pseudo-class and invalid ones using :invalid.

Code Change:

We've added an email input with the required attribute and an optional textarea without it to our form in index.html.

<!-- index.html snippet -->
<form action="/submit-data" method="POST">
  <p>
    <label for="email_req">Email (Required):</label>
    <input type="email" id="email_req" name="email_required" required> <!-- Added required -->
  </p>
   <p>
    <label for="comment_opt">Comment (Optional):</label>
    <textarea id="comment_opt" name="comment_optional"></textarea> <!-- No required -->
  </p>
  <p><button type="submit">Submit</button></p>
</form>

✅ Observation:

  1. Locate the required boolean attribute on the <input type="email">.
  2. In the browser preview, the fields look normal initially.
  3. Try clicking the "Submit" button without entering anything in the email field.
  4. Observe that the form submission is likely blocked. The browser should highlight the email field and display an error message like "Please fill out this field." or "Please enter an email address." near it.
  5. Now, enter a valid email address and click Submit again – the submission would proceed (though it will likely fail in our environment as the action URL doesn't exist).

✨ Key Takeaway: Add the boolean required attribute to form controls that must have a value provided by the user. Native browser validation will prevent submission if required fields are left empty.
