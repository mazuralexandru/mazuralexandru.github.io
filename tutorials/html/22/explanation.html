22. Saving and Viewing HTML Files (Conceptual)

Now that we have a basic HTML file with structure and content, let's discuss how you actually work with these files.

1. Saving Files:

  • File Extension: As we're doing in this tool, HTML files must be saved with an .html (or occasionally .htm) extension. This extension tells your operating system and web browsers that the file contains HTML code. .html is the modern standard.
  • Filename Conventions:
    • Use descriptive, lowercase names (e.g., about-us.html, contact.html).
    • Avoid spaces; use hyphens (-) instead.
    • The name index.html is special. Web servers usually serve this file by default when a user requests a directory (e.g., visiting www.example.com/ might load www.example.com/index.html).
  • Character Encoding: Crucially, ensure your text editor is set to save files using UTF-8 encoding. This matches the <meta charset="UTF-8"> declaration in your HTML and prevents character display issues. Most modern code editors (like VS Code) default to UTF-8.
  • File Organization: For websites with multiple pages, organize your files into folders (directories) for better management (e.g., an images folder, a css folder).

2. Viewing Files:

  • Locally: You can simply double-click an .html file on your computer, or use your browser's "File > Open File..." menu. The browser will open and render the page. You'll typically see a file:///... address in the browser's address bar. (This is similar to how the preview pane in our tool works).
  • Via Web Server: For actual websites, files are placed on a web server. Users access them via http:// or https:// URLs. While basic HTML often renders the same locally vs. on a server, some advanced features (especially involving JavaScript security or server-side interaction) require a server environment.
  • Browser Rendering Process (Simplified):
    1. Browser requests/opens the HTML file.
    2. Reads the HTML code.
    3. Parses tags/content, building the internal DOM tree.
    4. Applies any linked CSS rules (we'll add CSS soon).
    5. Executes any linked JavaScript.
    6. Renders the final visual result on the screen.

✅ Observation: No file changes in this step. This is conceptual information about handling the files we are creating. Our current index.html is viewable in the preview pane because the tool simulates this local viewing process.

✨ Key Takeaway: Save files with .html extension using UTF-8 encoding. You can view them directly in a browser locally (file:///) or deploy them to a web server (http:// or https://).
