222. word-break & overflow-wrap

Files Modified: style.css

These properties control how lines break when text might otherwise overflow its container, especially concerning long words or strings without spaces.

  1. overflow-wrap (formerly word-wrap):

    • Purpose: Specifies whether the browser may break lines within words to prevent overflow if an otherwise unbreakable string (like a long URL or word) is too long to fit on the line.
    • Values:
      • normal: (Default) Only break lines at allowed break points (like spaces).
      • break-word: Allows unbreakable words to be broken at arbitrary points if they would otherwise overflow the container.
      • anywhere: Similar to break-word but with potentially softer breaking opportunities considered.
    • Use Case: Useful for preventing long URLs or user-generated content from breaking layouts.

  2. word-break:

    • Purpose: Specifies more generally how line breaks should occur.
    • Values:
      • normal: (Default) Use default breaking rules.
      • break-all: Allows breaks between any two characters (primarily for CJK - Chinese, Japanese, Korean text, where word boundaries are less distinct). Can look very awkward for English text.
      • keep-all: Prevents breaks between characters for CJK text.
      • break-word: Deprecated alias for overflow-wrap: break-word.

Recommendation: For handling long words/URLs in non-CJK text, prefer using overflow-wrap: break-word;.

  • Let's apply overflow-wrap: break-word; to the first paragraph (.highlight-text) to see how it handles long strings. We'll need to add a long string to the HTML in the next step to see the effect.

.highlight-text {
  /* ... other styles ... */
  word-spacing: 0.5em; 
  overflow-wrap: break-word; /* Allow breaking long words */
}

.container span {
  font-weight: bold; 
  border: inherit; 
  font-size: 1.5rem; 
  /* white-space: nowrap; <- Removed */
  background-color: rgba(0, 0, 255, 0.2); 
}
