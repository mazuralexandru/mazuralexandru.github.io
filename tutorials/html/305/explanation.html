305. Metadata: The <meta> Element - Fundamentals

Having reviewed the <head> container, let's dive into one of its most versatile children: the <meta> element.

Purpose:

  • The <meta> element is used to represent various kinds of metadata that cannot be adequately represented by other specific <head> elements like <title>, <link>, <base>, <style>, or <script>.
  • It acts as a generic container for specifying different types of information about the HTML document using key-value pairs defined through its attributes.

Element Category:

  • <meta> is a void (or empty) element. It represents a single piece of metadata and therefore has no content between tags and no closing tag.

Core Concept:

  • Think of <meta> tags as providing labeled pieces of information about the page. The label (the type of metadata) and the value (the actual information) are defined using specific attributes within the <meta> tag itself. The most common attribute combinations define the character set, viewport settings, page descriptions, keywords (though now ignored by major search engines), author information, and instructions for search robots.

✅ Observation: Locate the existing <meta charset="UTF-8"> tag within the <head>. Note its self-contained nature (no closing tag). Understand that we will be adding more <meta> tags with different attributes to define different pieces of information about the page.

✨ Key Takeaway: The <meta> element is a void element used within <head> to provide various types of metadata not covered by other elements like <title> or <link>. It uses attributes (like charset, name, content) to define the type and value of the metadata.
