238. <s>, <del>, <ins>, <mark>: CSS Styling

Files Modified: style.css

As with other semantic elements, you can customize the default appearance of <s>, <del>, <ins>, and <mark> using CSS.

✅ Observe style.css:

  • We've added rules to style these elements:
    • Making del red.
    • Making ins green and removing its default underline in favor of a background color.
    • Changing the mark background color from the default yellow to light blue.
/* Styling Edit/Annotation Elements */
del {
  color: red;
  background-color: rgba(255, 0, 0, 0.1); /* Subtle red background */
}

ins {
  color: green;
  background-color: rgba(0, 255, 0, 0.1); /* Subtle green background */
  text-decoration: none; /* Remove default underline */
  padding: 0.1em 0.2em;
}

mark {
  background-color: lightblue;
  color: #333; /* Darker text on light blue */
  padding: 0.1em 0.2em;
  border-radius: 3px;
}

✨ Observe the Preview:

  • The deleted text (<del>$50</del>) should now appear red with a faint red background, in addition to the strikethrough.
  • The inserted text (<ins>$40</ins>) should now appear green with a faint green background and padding, and it should not be underlined anymore.
  • The marked text (<mark>CSS</mark>) should now have a light blue background instead of yellow.

This shows how CSS allows you to tailor the presentation of semantic edits and annotations to fit your design system.
