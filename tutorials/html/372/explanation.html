372. Link State: :focus-visible

Files Modified: style.css

A common design request is to have focus styles appear for keyboard users but not when an element (like a button or link) is simply clicked with a mouse. The :focus-visible pseudo-class addresses this.

  • :focus-visible:

    • Selects: An element that has focus, but only when the browser determines (via heuristics) that displaying a visible focus indicator is appropriate (typically when focus was received via keyboard or script). It usually does not match when focus is received via a mouse click on elements that don't typically need a persistent focus ring after clicking (like links or buttons).
    • Purpose: To provide clear focus indicators for keyboard users without adding potentially distracting focus styles for mouse users who just clicked an element.
    • Browser Support: Well-supported in modern browsers.

  • Common Pattern:

    1. Optionally remove the default :focus outline if it appears undesirably on mouse clicks (but only if you provide a :focus-visible style!).
    2. Define distinct, visible styles using :focus-visible.

✅ Observe style.css:

  • We've modified the :focus rule to remove the default outline (this is only acceptable because we are adding a :focus-visible style).
  • We've added a new rule for a:focus-visible that applies the orange dashed outline.
/* Style links when focused (potentially removing default for mouse users) */
a:focus { 
  /* Example: Remove default outline IF providing focus-visible */
   outline: none; 
}

/* Style links ONLY when focus is visibly indicated (keyboard/script) */
a:focus-visible {
  outline: 2px dashed orange; 
  outline-offset: 2px; 
}

✨ Observe the Preview:

  • Keyboard Navigation: Use the Tab key. Links should still get the dashed orange outline when focused via the keyboard (now matched by :focus-visible).
  • Mouse Click: Click on one of the links. It should not retain the dashed orange outline after the click (because :focus-visible typically doesn't match on mouse clicks for links, and we removed the basic :focus outline). The link might briefly flash red due to the :active style.

This pattern provides accessibility for keyboard users while maintaining a potentially cleaner visual appearance for mouse interaction.
