398. Spanning Rows: The rowspan Attribute - Fundamentals

Similar to colspan for horizontal spanning, the rowspan attribute allows a table cell to span vertically across multiple rows.

The rowspan Attribute:

  • Definition: An attribute applicable to both <th> and <td> elements.
  • Purpose: Specifies the number of rows that a single cell should span (merge down across).
  • Core Concept: It allows a cell to occupy the vertical space normally taken by cells in the same column but across multiple adjacent rows.

Syntax:

  • <th rowspan="number">...</th>
  • <td rowspan="number">...</td>
  • The number value must be a positive integer greater than 0.
    • rowspan="1" is the default and has no effect.
    • rowspan="2" makes the cell span down across two rows.
    • rowspan="0" means span until the last row of the row group (<thead>, <tbody>, <tfoot>). Use rarely.

Layout Impact:

  • When a cell uses rowspan="N", you need to omit the corresponding cell (<td> or <th>) from the next N-1 rows in the source code, because the spanned cell already occupies that vertical slot in its column.
  • If a cell in Row 1, Column 1 has rowspan="2", then the <tr> for Row 2 should not include a cell intended for Column 1 – it should start directly with the cell for Column 2.

✅ Observation: No code changes yet. Understand the purpose of rowspan: to make a cell taller, covering multiple rows, and the consequent need to omit cells from subsequent rows to account for the occupied space.

✨ Key Takeaway: The rowspan attribute, applied to <th> or <td>, allows a cell to span vertically down across a specified number of rows. Remember to adjust (omit) cells in subsequent rows accordingly.
