176. Child Combinator (>)

Files Modified: style.css

The Child Combinator is represented by the "greater than" symbol (>) between two selectors.

  • Syntax: parent > child
  • Meaning: Selects any element matching child that is a direct child of an element matching parent. It does not select deeper descendants.

✅ Observe style.css:

  • We've added a new rule: .container > p.
/* Child Combinator Example */
.container > p {
  border-left: 3px solid orange; /* Add orange left border to direct child paragraphs */
  padding-left: 10px;
}

✨ Observe the Preview:

  • Both paragraphs ("This is a standard paragraph." and "This paragraph follows the heading.") should now have an orange left border and some left padding.
  • This happens because both <p> elements are direct children of the <div class="container">.
  • Crucially, if we had a paragraph nested deeper, like <div class="container"><div><p>Deep</p></div></div>, the .container > p rule would not apply to the "Deep" paragraph because it's not a direct child of .container.

The child combinator is more specific than the descendant combinator.
