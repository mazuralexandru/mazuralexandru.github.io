374. Styling Order (LVHA Rule): Problem & Recommendation

Files Modified: None (Reverted index.html and style.css to previous state for context)

We've now seen the main link state pseudo-classes: :link, :visited, :hover, :active, and :focus/:focus-visible. A common pitfall arises because these pseudo-classes all have the same specificity (0,0,1,0 - one pseudo-class).

  • The Problem: When selectors have equal specificity, the source order determines which rule wins. The rule defined later in the stylesheet overrides earlier ones. If you define your link state rules in the wrong order, some styles might never appear. For example, if :link comes after :visited, unvisited links might incorrectly show the visited style. If :hover comes before :link, the hover effect might be immediately overridden by the :link rule.

  • The Recommendation (LVHA Rule): To ensure all states work correctly, follow this specific order when defining your link state rules:

    1. :link (Unvisited)
    2. :visited (Visited)
    3. :hover (Mouse over)
    4. :active (Clicking)

    A common mnemonic is LoVe HAte.

  • Where does :focus/:focus-visible fit? This is slightly more debatable, but common placements are:

    • Between :visited and :hover: Ensures focus styles override visited styles but are overridden by hover/active styles.
    • After :active: Ensures focus styles are the strongest and remain visible even during hover/active states (often preferred for accessibility).

    The key is to be consistent and test the behavior. Placing it last (:link, :visited, :hover, :active, :focus/:focus-visible) is often the safest bet for ensuring the focus style is always visible when needed.
