608. Graphics: SVG (<svg>) - The <text> Element

SVG is not limited to shapes; it can also render real text content using the <text> element.

The <text> Element:

  • Purpose: Defines a text element to be rendered within the SVG graphic.
  • Key Advantage: Text rendered via <text> is actual text, not just pixels or shapes. This means it is:
    • Accessible: Screen readers can read it directly.
    • Selectable: Users can select and copy the text.
    • Searchable: Search engines can index the text content within SVG.
  • Syntax: <text x="..." y="..." ...attributes...> Text Content </text>
  • Requires opening and closing tags. The text content goes between them.

Key Attributes:

  • x, y: Specify the coordinates for positioning the text. The exact point these coordinates refer to depends on factors like text-anchor and dominant-baseline, and can apply to the whole block or individual characters (if multiple values are provided for x/y). Often represents the start or middle of the baseline.
  • dx, dy: Specify relative offsets from the current text position. Useful for fine-tuning or shifting individual characters/spans (<tspan>).
  • text-anchor: Controls horizontal alignment relative to the x coordinate (start, middle, end).
  • dominant-baseline: Controls vertical alignment relative to the y coordinate (middle, hanging, central, alphabetic, etc.).
  • Font Styling: Can use presentation attributes (font-family, font-size, font-weight, fill) or, preferably, apply styles via CSS.

Other Text Elements:

  • <tspan>: Used inside <text> to apply different styles or positioning (dx, dy) to specific parts of the text.
  • <textPath>: Renders text along a predefined <path>.

Code Change:

  1. Removed the <path> element from the previous example.
  2. Adjusted the width/height/viewBox of the <svg> element.
  3. Added a <text> element inside the <svg>.
  4. Added CSS rules to style the text (setting fill, font-size, text-anchor).
<!-- index.html -->
<head>
  ...
  <style>
    svg {
      border: 1px dashed skyblue;
      margin: 10px;
      font-family: sans-serif; /* Set default font for text */
    }
    svg text { /* Style text */
      fill: lightgreen;
      font-size: 16px;
      text-anchor: middle; /* Center text horizontally */
    }
    /* Removed path style */
  </style>
</head>
<body>
  <h1>Inline SVG Example</h1>
  <p>Adding text using the &lt;text&gt; element.</p>

  <svg width="150" height="50" viewBox="0 0 150 50" xmlns="http://www.w3.org/2000/svg">
     <!-- Removed path -->

     <!-- Added text element -->
     <text x="75" y="30">SVG Text!</text>
     <!-- Coordinates (75,30) are the middle point horizontally (due to text-anchor) -->

  </svg>

  <p>Text in SVG is real, selectable, accessible text.</p>
  ...
</body>

✅ Observation:

  1. Locate the <text> element inside the <svg>. Note the x and y attributes and the text content.
  2. Examine the CSS rule for svg text.
  3. Look at the browser preview. Inside the dashed SVG border, you should see the text "SVG Text!" rendered in light green. It should be horizontally centered within the SVG viewport because x="75" (half of width 150) and text-anchor: middle; were used.
  4. Try selecting the text with your mouse – you should be able to select it like normal HTML text.

✨ Key Takeaway: Use the <text> element to include accessible, searchable, and selectable text within SVG graphics. Position using x/y attributes and style using CSS (fill, font-size, text-anchor, etc.).
