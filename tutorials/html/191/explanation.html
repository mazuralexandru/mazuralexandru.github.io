191. The unset Keyword

Files Modified: None

The unset keyword provides a slightly more nuanced way to reset a property. Its behavior depends on whether the property is naturally inherited or not.

  • Purpose: To reset a property to its "natural" state in the cascade.
  • Syntax: property: unset;
  • Behavior:
    • If the property is naturally inherited (like color, font-family), unset acts like inherit. The element will inherit the value from its parent.
    • If the property is not naturally inherited (like background-color, border), unset acts like initial. The property is reset to its CSS-defined initial value.

Example:

div {
  color: blue;
  border: 1px solid red;
}
p {
  color: unset; /* Behaves like 'inherit', p gets color: blue from div */
  border: unset; /* Behaves like 'initial', p gets border: none (initial value) */
}

<div>
  <p>This text will be blue (inherited via unset), with no border (reset via unset).</p>
</div>

unset is useful when you want to reset multiple properties (perhaps using the all: unset; shorthand) back to their default inherited or initial state without needing to know which ones inherit and which don't.
