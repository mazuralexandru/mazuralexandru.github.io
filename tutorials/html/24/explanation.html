24. CSS Selectors: Type Selectors

We've already been using the most basic type of CSS selector without formally naming it: the Type Selector (also known as an Element Selector or Tag Selector).

What's New?

  • We added a new element type, <h2>, to index.html.
  • We added a new CSS rule in style.css using a type selector to target this <h2>.
<!-- index.html snippet -->
<body>
  <h1>Styling with External CSS</h1>
  <p>This paragraph will be styled externally.</p>
  <h2>This is a Sub-heading</h2> <!-- Added H2 -->

  <!-- Commented out paragraph remains -->
  <!-- <p>This paragraph is commented out and won't appear.</p> -->

  <script src="script.js"></script>
</body>

/* style.css snippet */
  color: #ccc;
}

/* Added H2 rule using Type Selector */
h2 {
  color: lightcoral;
  font-weight: normal; /* Make H2 not bold */
}

/* CSS files can also have comments! */

Type Selectors Explained

  • Syntax: Simply the name of the HTML element you want to target, without the angle brackets (e.g., h1, p, body, img, a, h2).
  • Scope: Selects all instances of that specific HTML element type within the document (or within the scope defined by more complex selectors, which we'll see later).
  • Use Cases:
    • Applying very general, base styles to fundamental elements (like we did with body).
    • Applying global resets (e.g., setting default margins/paddings to 0 for all elements, although dedicated reset stylesheets are often used for this).
    • Styling all instances of a particular element consistently when specific variations aren't needed.

In our example, the selector h2 targets every <h2> element on the page. We've set its color to lightcoral and its font-weight to normal (overriding the browser's default bold styling for headings).
