226. src URL Types: Absolute vs. Relative Recap

Just like the href attribute for links (<a>), the src attribute for images (<img>) can accept different types of URLs to locate the image file.

URL Types Recap (from Module 4):

  1. Absolute URLs:

    • Provide the full, complete web address, including the scheme (https:// or http://) and domain name.
    • Use Case for src: Used to display images hosted on different websites or external servers, such as Content Delivery Networks (CDNs) or image hosting services.
    • Example: <img src="https://cdn.some-service.com/images/photo123.jpg" alt="...">
    • Considerations: Creates dependency on the external server being available. Potential for cross-origin issues if interacting with the image via scripts (less common for simple display).

  2. Relative URLs:

    • Provide a partial path relative to the current HTML document's location (or the site root). Do not include scheme or domain.
    • Use Case for src: The standard and highly recommended method for displaying images hosted within your own website structure. Ensures portability – your images will still load if you move your site to a different domain or run it locally, as long as the internal folder structure is maintained.
    • Examples:
      • Same Directory: <img src="logo.png" alt="...">
      • Subdirectory: <img src="images/banner.webp" alt="...">
      • Parent Directory: <img src="../assets/icon.svg" alt="...">
      • Root-Relative: <img src="/user/avatars/profile.jpg" alt="...">

Which to Use?

  • For images that are part of your own website, always prefer relative URLs (images/my-image.jpg, /assets/logo.png, etc.).
  • Use absolute URLs only when you need to display an image hosted on a completely separate domain or CDN.

✨ Key Takeaway: Understand the difference between absolute and relative URLs for the src attribute. Use relative URLs for images within your own website for portability and maintainability. Use absolute URLs for externally hosted images.
