562. Reset: Preparing for <track> Element

We have explored using <source> to provide multiple video formats and responsive versions.

Now, we turn to a crucial element for multimedia accessibility: the <track> element, used to add timed text tracks like captions, subtitles, and audio descriptions.

To focus on <track> and its attributes, we will reset our <video> element slightly, removing the media attribute example and reverting to the standard dual-source (WebM/MP4) setup.

Why Reset?

  • Focus: Isolate the <track> element and its role in adding text-based accessibility layers.
  • Clarity: Demonstrate adding <track> elements within the standard <video> structure containing <source> elements.

Code Change:

  1. Removed the media attribute from the first <source> tag in index.html.
  2. Adjusted the src values back to simple examples.
  3. Updated the <h1> and paragraph text.
