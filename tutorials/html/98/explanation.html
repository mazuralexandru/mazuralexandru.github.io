98. List Items (<li>): Content Model (Inline Content)

What kind of content can you put inside an <li> element? The answer is: quite a lot! <li> elements are very flexible containers.

Content Model: Flow Content

Technically, <li> accepts flow content. This is a broad category that includes most elements you'd typically use within the <body> of a page.

Simple Content: Text & Inline Elements

The most common content for list items is:

  • Plain Text: As we've seen (<li>First item</li>).
  • Phrasing Content (Inline Elements): You can freely use inline elements learned in Module 2 within an <li> to add semantic meaning or formatting to parts of the item's text. Examples include:
    • <em>, <strong>
    • <i>, <b>, <s>
    • <code>
    • <a> (links - Module 4)
    • <img> (images - Module 5)
    • <span> (for CSS styling hooks)
    • <sub>, <sup>
    • And others...

Code Change:

We've modified the content of the <li> elements in index.html to show plain text and text including <em> and <strong> tags.

<ul>
  <li>Just plain text.</li> <!-- Simple text content -->
  <li>Text with <em>inline emphasis</em>.</li> <!-- Inline element inside li -->
  <li>Content can be quite <strong>flexible</strong>.</li> <!-- Inline element inside li -->
</ul>

✅ Observation:

  1. Examine the content inside the <li> tags in index.html.
  2. Look at the browser preview. You should see:
    • The first item as plain text.

✨ Key Takeaway: <li> elements can contain plain text and any inline/phrasing elements (<em>, <strong>, <a>, <img>, <code>, etc.) to structure and format the item's content.
