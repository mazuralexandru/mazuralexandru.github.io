426. Forms: The target Attribute (on <form>)

Similar to the target attribute on anchor (<a>) tags, the <form> element also supports a target attribute.

Definition & Purpose:

  • The target attribute on a <form> specifies the browsing context (window, tab, or iframe name) where the response received from the server after submitting the form should be displayed.

Values (Same as <a> target):

  • _self (Default): The response loads in the same window/tab/frame where the form was submitted, replacing the current page.
  • _blank: The response loads in a new, unnamed window or tab.
  • _parent: The response loads in the parent frame of the current frame.
  • _top: The response loads in the top-level window, breaking out of all frames.
  • <framename>: The response loads into the <iframe> with the matching name attribute.

Use Cases:

  • Setting target="_blank" can be used if you want the submission result (e.g., a generated report or a confirmation page) to appear in a new tab, leaving the original form page open.
  • Using target="<framename>" was common in older frame-based sites to submit a form in one frame and display results in another.
  • Modern Context: With the prevalence of Single-Page Applications (SPAs) and JavaScript techniques like AJAX/Fetch for handling form submissions without full page reloads, directly using target on the <form> element itself has become less common for typical web applications. It's more frequently seen in specific scenarios like submitting to payment gateways that require opening in a new context or interacting with legacy framed systems.

Code Change:

We've removed the name attribute and added target="_blank" to our <form> tag in index.html.

<!-- index.html snippet -->
<form action="/submit-data" method="POST" target="_blank"> <!-- Removed name, added target -->
   <!-- Form content -->
   <!-- Response will open in a new tab -->
</form>

✅ Observation:

  1. Locate the target="_blank" attribute in the <form> tag.
  2. There is no visual change to the form itself.
  3. If this form were submitted, the browser would send the POST request to /submit-data. When the server sends back a response (e.g., a "Thank You" HTML page), the browser would display that response page in a new tab or window due to target="_blank".

✨ Key Takeaway: The target attribute on <form> controls where the server's response is displayed after submission (_self, _blank, etc.). Its use is less common in modern AJAX-driven applications but can be useful for specific cross-context submission scenarios.
