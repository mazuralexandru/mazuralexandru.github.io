168. Universal Selector: Use Cases

Files Modified: None

Despite its broad scope and potential performance impact, the universal selector (*) has a few specific, common use cases:

  1. CSS Resets (Box-sizing): The most famous and widely recommended use is setting the box-sizing property globally. This changes how the browser calculates element widths and heights to include padding and borders, making layout math much more intuitive.
    *,
*::before,
*::after {
  box-sizing: border-box;
}
    (Note: Including ::before and ::after ensures pseudo-elements are also included in this reset).
  2. Basic Resets (Margin/Padding): Sometimes used in simple resets to remove default browser margins and padding from all elements, although targeting specific elements (like body, h1, p, ul etc.) is often preferred for more control.
    * {
  margin: 0;
  padding: 0;
  /* box-sizing: border-box; Often included here */
}
  3. Debugging: Temporarily applying a highly visible style like * { outline: 1px solid red; } can help visualize the boundaries of all elements on the page to debug layout issues. Remember to remove this after debugging!

Avoid using * for general styling (like setting default font colors or families) as it's inefficient and usually better handled by styling the <body> or using more specific selectors.
