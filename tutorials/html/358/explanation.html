358. Other <head> Elements: The <script> Element - Fundamentals

The <script> element is used to embed or reference executable client-side scripts, most commonly JavaScript.

Purpose:

  • To include code that adds interactivity, dynamic content updates, animations, form validation, communication with servers (AJAX/Fetch), and many other dynamic behaviors to web pages. It brings the "action" to the structure (HTML) and presentation (CSS).

Core Concept:

  • The <script> tag tells the browser that the content inside it (or the file referenced by its src attribute) is code that should be executed by the browser's scripting engine (typically the JavaScript engine).

Two Main Usage Modes:

  1. Embedding Inline Script: Writing JavaScript code directly between the opening <script> and closing </script> tags.
    <script>
  // JavaScript code goes here
  console.log('Hello from inline script!');
</script>
  2. Linking External Script File: Referencing an external file containing JavaScript code using the src attribute. This is the strongly preferred method for non-trivial amounts of code.
    <script src="path/to/your/script.js"></script>
    • Important: When the src attribute is present, the content between the <script> and </script> tags is ignored. You cannot have both a src attribute and inline code in the same <script> tag.

Element Category & Placement:

  • The <script> element itself can be placed in either:
    • The <head> section.
    • The <body> section.
  • Its placement has significant implications for page loading and rendering performance (discussed later).

✨ Key Takeaway: The <script> element is used to include JavaScript, enabling dynamic behavior. It can either contain inline code or link to an external script file via the src attribute (preferred). It can be placed in <head> or <body>.
