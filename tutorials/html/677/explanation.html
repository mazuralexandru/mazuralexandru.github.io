677. Reset & Common Errors: Missing Required Attribute (alt)

Some HTML elements require specific attributes to be present for the element to be valid and function correctly, especially for accessibility. Forgetting these is a common validation error.

Resetting:

Remove the previous nesting error examples.

Error: Missing Required Attribute

  • Problem: An element is used without one of its mandatory attributes. The most frequent example is omitting the alt attribute on an <img> element.
  • Impact: Directly harms accessibility (screen readers have no alternative for the image), prevents fallback text display for broken images, and results in invalid HTML. Other examples include missing kind on <track> or potentially label on <optgroup>.
  • Validator Message (Example): Error: Element "img" is missing required attribute "alt".

Code Change:

  1. Removed the previous nesting examples from index.html.
  2. Added a "Bad Example" showing an <img> tag missing the alt attribute.
  3. Added a "Good Example" showing the <img> tag with a proper alt attribute, and another example using the valid empty alt="" for decorative images.
<!-- index.html -->
<body>
  <h1>Common Validation Errors</h1>

  <h2>Error: Missing Required Attribute</h2>

  <!-- Bad Example (Missing alt) -->
  <p>An image without alt text: <img src="/images/placeholder.png"></p>

  <hr> <!-- Separator -->

  <!-- Good Example -->
  <p>An image with alt text: <img src="/images/placeholder.png" alt="Placeholder image"></p>
  <p>A decorative image: <img src="/images/decor.png" alt=""></p>

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

(Note: Image paths are hypothetical and will show as broken).

✅ Observation:

  1. Verify the removal of previous examples and the addition of the <img> examples. Note the missing alt in the first image.
  2. The browser preview shows broken image icons.
  3. (Conceptual): Running the "Bad Example" through validator.w3.org would report an error because the alt attribute is missing. The "Good Example" code would validate correctly because alt is present (even if empty for the decorative case).

✨ Key Takeaway: Always include required attributes defined by the HTML specification for elements like <img> (alt), <track> (kind), <optgroup> (label), etc. Validators are essential for catching these omissions, which often have significant accessibility implications.
