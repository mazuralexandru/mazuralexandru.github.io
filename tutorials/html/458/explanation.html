458. background Shorthand: Syntax

Files Modified: None

Let's look at the syntax structure again, highlighting the key parts:

background: [background-color] [background-image] [background-repeat] [background-attachment] [background-position] / [background-size] [background-origin] [background-clip];

  • Order: While technically flexible for many parts before the /, sticking to a consistent order (like the one above) improves readability.
  • background-position / background-size: If background-size is specified in the shorthand, it must come immediately after background-position, separated by a /.
  • background-origin & background-clip: These can be set using the shorthand. If both are set, background-origin should typically precede background-clip. Note that if only one <box> value (padding-box, border-box, content-box) is present, it sets both origin and clip. If two <box> values are present, the first sets origin and the second sets clip.
  • Omitted Values: Reset to their defaults (e.g., transparent, none, repeat, scroll, 0% 0%, auto, padding-box, border-box).
  • Multiple Layers: Use commas to separate the property groups for each layer, remembering that background-color applies only to the last layer specified.
/* Example with multiple layers and size/position */
div {
  background: 
    url(fg.png) no-repeat left top / 50px, /* Layer 1: image, no-repeat, top left, 50px wide */
    url(bg.png) repeat-x center / auto 100px, /* Layer 2: image, repeat-x, center, auto wide/100px tall */
    linear-gradient(to bottom, blue, green); /* Layer 3: gradient (acts as image), color implicitly added here */
}

/* Example setting origin and clip */
div {
  background: url(img.png) no-repeat 10px 10px / 50px content-box padding-box; 
  /* image, no-repeat, 10px 10px pos, 50px wide size, origin=content-box, clip=padding-box */
}
