657. Validation: What Validation Checks - Structure

Beyond basic syntax, validators also enforce rules about the overall structure of the document and how elements are nested.

2.2. Structural Rules Checks:

  • Element Nesting (Parent-Child Relationships): HTML defines strict rules about which elements can legally contain which other elements (the element's "content model"). Validators check these rules.
    • Example Errors:
      • Error: Element "ul" not allowed as child of element "p" in this context. (Block element <ul> inside inline-like <p>)
      • Error: Element "li" must be contained within parent element "ul" or "ol". (Stray <li> outside a list)
      • Error: Element "p" cannot be nested inside element "p".
      • Error: Element "dt" must be followed by "dd" or "dt". (Incorrect <dl> structure)
  • Element Placement: Ensures elements are in their correct sections of the document.
    • Example Errors:
      • Error: Element "head" is missing a required instance of child element "title".
      • Error: Element "p" is not allowed as child of element "head".
      • Error: Element "caption" must be the first child of its parent "table" element.
  • Document Structure: Verifies the overall required structure (<!DOCTYPE html>, <html>, <head>, <body>).
    • Example Error: Error: Start tag seen without seeing a doctype first.

Ensuring correct structure is vital because browsers and assistive technologies rely on this predictable hierarchy to build the DOM tree and interpret the relationships between different parts of the content. Invalid nesting can lead to a broken or nonsensical DOM, causing rendering glitches and accessibility issues.

Think back to the rules learned for lists (<li> inside <ul>), description lists (<dt>/<dd> inside <dl>), tables (<caption> first, <th>/<td> inside <tr>), and the overall page structure (<head> then <body> inside <html>). Validators automatically check that these structural rules are followed correctly.

✨ Key Takeaway: Validation checks ensure correct element nesting according to HTML's content models and verify the proper placement of key structural elements like <head>, <body>, <caption>, etc., which is crucial for a well-formed DOM.
