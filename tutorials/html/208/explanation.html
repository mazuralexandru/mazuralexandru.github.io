208. Other <a> Attributes: rel (Relationship)

Beyond rel="noopener noreferrer" used with target="_blank", the rel attribute has a broader purpose: describing the relationship between the current document and the linked document/resource.

The rel Attribute:

  • Purpose: Provides semantic hints about the nature of the link. This information can be used by:
    • Browsers (e.g., for UI hints or prefetching).
    • Search engines (e.g., nofollow influences ranking signals).
    • Assistive technologies (potentially, though support varies).
    • Web crawlers and other automated tools.
  • Syntax: rel="value1 value2 ..."
    • Values are space-separated if multiple relationships apply.

Common rel Values (Examples):

  • nofollow: Instructs search engines not to follow this link or pass ranking authority (often used for paid links or user-generated content).
  • external: Indicates the link targets an external resource (outside the current site). Often used alongside target="_blank".
  • next: Indicates the linked document is the next part of a sequence (e.g., next page in an article or gallery).
  • prev: Indicates the linked document is the previous part of a sequence.
  • help: Links to a help document related to the main content.
  • license: Links to the license governing the current document.
  • author: Links to information about the author.
  • bookmark: Provides a permalink for the nearest ancestor section.
  • search: Links to a search tool for the document/site.
  • tag: Indicates the link targets a tag/keyword description relevant to the document.
  • (And others...)

  Common 'rel' values (space-separated if multiple):
  <ul>
    <li><a href="external-site.com" rel="external nofollow" target="_blank">External Link (nofollow)</a></li> <!-- Added rel="external nofollow" -->
    <li><a href="/page-2" rel="next">Next Page</a></li> <!-- Added rel="next" -->
    <li><a href="/page-1" rel="prev">Previous Page</a></li> <!-- Added rel="prev" -->
    <li><a href="/help" rel="help">Help Section</a></li> <!-- Added rel="help" -->

    <!-- Remember: Use noopener noreferrer with target="_blank" -->

(Note: The reminder comment about noopener noreferrer is included because the first example uses target="_blank").

✅ Observation:

  1. Examine the different rel="..." attributes in the example links. Note how multiple values can be space-separated (rel="external nofollow").
  2. The links appear normally in the preview. The rel attribute primarily provides non-visual, semantic information.

✨ Key Takeaway: The rel attribute adds semantic meaning about the relationship between the current page and the linked resource, used by browsers, search engines, and tools. Choose values that accurately describe the link's purpose.
