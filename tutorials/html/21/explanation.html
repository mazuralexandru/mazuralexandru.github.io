21. Adding Visible Content: <h1> and <p>

We finally have our complete minimal HTML document structure! Now, let's add some actual visible content inside the <body> section.

Recap: The <body> element is the container for everything the user sees on the page.

Common Content Elements:

  • <h1> to <h6>: Heading elements. <h1> represents the main heading of the page or section, with <h2>, <h3>, etc., representing subheadings of decreasing importance. They are block-level elements (meaning they typically start on a new line and take up the full available width).
  • <p>: Paragraph element. Used for blocks of regular text. Also a block-level element.

Code Change:

We've added an <h1> (main heading) element and a <p> (paragraph) element inside the <body>. We've also updated the comments slightly.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>My First Web Page</title>
  <!-- Link to CSS will go here -->
</head>
<body>
  <h1>Hello World!</h1> <!-- Added H1 heading -->
  <p>This is the body content of my first web page.</p> <!-- Added paragraph -->
  <!-- Visible content goes here --> <!-- Comment retained for clarity -->
</body>
</html>

✅ Observation:

  1. Examine the index.html file. Note the new <h1> and <p> elements nested within the <body>.
  2. Look at the browser preview pane! You should now see visible content:
    • "Hello World!" displayed in a large, bold font (default <h1> styling).
    • "This is the body content of my first web page." displayed below it in a regular font size (default <p> styling).
    • Notice the vertical space between the heading and the paragraph – this is default browser styling for these block-level elements.

✨ Experiment: Try changing the text inside the <h1> or <p> tags. You can also try adding another <p> element below the first one. Observe how the preview updates.
