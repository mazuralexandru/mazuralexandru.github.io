162. Reset: Removing Relative URL Example

We've explored the different forms of relative URLs used for internal linking. Now, we'll move on to another type of linking: creating links that jump to specific sections within a page (using fragment identifiers).

To prepare, let's remove the root-relative link example.

Why Remove Code?

  • Focus: Shift attention from linking between pages/resources to linking within a single page.
  • Clear Space: Make room in the <body> for adding elements with id attributes that will serve as targets for our fragment links.

Code Change:

We have removed the <a> tag containing the root-relative link example from index.html.

<body>
  <h1>Links and Navigation</h1>
  <p>Preparing to demonstrate linking to page fragments (sections).</p> <!-- Adjusted P -->

  <!-- Removed relative link example -->
  <!-- <p><a href="/images/logo.png">Site Logo</a></p> -->

  <p>This uses the '#' symbol and element IDs.</p> <!-- Adjusted P -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

✅ Observation:

  1. Confirm that the anchor tag example is gone from index.html.
  2. The browser preview shows only the heading and introductory paragraphs for the next topic.

✨ Key Takeaway: Removing the previous relative link example clears the stage for demonstrating fragment identifier links and their corresponding target elements.
