38. CSS Selectors: Introduction to Grouping Selectors

Often, you want to apply the exact same style rules to several different elements. Instead of writing separate rulesets for each selector, you can group them together using commas. This follows the DRY (Don't Repeat Yourself) principle.

The Need for Grouping

Imagine you want all your headings (<h1>, <h2>, <h3>) to share the same font family and maybe a base color, but retain their different default sizes. Without grouping, you might write:

/* Without Grouping (Repetitive) */
h1 {
  font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
  color: #ddd; /* Base heading color */
  /* ... other h1 specific styles ... */
}
h2 {
  font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
  color: #ddd; /* Base heading color */
  /* ... other h2 specific styles ... */
}
h3 {
  font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
  color: #ddd; /* Base heading color */
  /* ... other h3 specific styles ... */
}

This is very repetitive! Grouping selectors allows us to apply the common styles much more efficiently.
