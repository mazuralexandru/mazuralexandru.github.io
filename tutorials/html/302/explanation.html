302. Metadata: The <head> Element Revisited - Core Mechanics

Let's review the structural rules and permitted contents of the <head> element.

Placement & Context:

  • The <head> element must be the first element directly inside the root <html> element.
  • It is always followed immediately by the <body> element.
  • Only one <head> element is allowed per document.
  • Syntax: It requires opening <head> and closing </head> tags.

Permitted Content (Metadata Elements):

The <head> element is designed to contain specific types of child elements that define metadata or link resources. The primary ones are:

  1. <title> (Required): Exactly one <title> element defining the document's title must be present within the <head>.
  2. <meta> (Zero or more): Used for various types of metadata (character set, viewport, description, keywords, etc.). We'll focus heavily on this element soon.
  3. <link> (Zero or more): Used to link external resources, most commonly CSS stylesheets (rel="stylesheet") but also icons (rel="icon"), fonts, alternate page versions, etc.
  4. <style> (Zero or more): Used to embed CSS styles directly within the HTML document (internal styles). Generally less preferred than external stylesheets via <link>.
  5. <script> (Zero or more): Used to embed JavaScript code or link to external JavaScript files. While allowed in <head>, scripts can also appear in <body> (often placed just before the closing </body> tag for performance reasons).
  6. <base> (Zero or one): Specifies a base URL against which all relative URLs in the document are resolved.
  7. <noscript> (Zero or more): Contains content that is rendered only if JavaScript is disabled in the browser. Can contain link, style, and meta elements when used within <head>.

Prohibited Content:

It's crucial to remember that elements intended for the main visible content of the page, such as <p>, <h1>-<h6>, <img>, <ul>, <div>, etc., are invalid if placed directly inside the <head> element. They belong in the <body>.

✅ Observation: Look again at the index.html file structure. Verify the placement of <head> between <html> and <body>. Identify the current children within <head> (<meta>, <title>, <link>) – these all fit the permitted content model.

✨ Key Takeaway: The <head> contains specific metadata elements (<title>, <meta>, <link>, <style>, <script>, <base>, <noscript>) and must not contain visible content elements like <p> or <h1>.
