71. Character Encoding (<meta charset="UTF-8">)

Files Modified: None

The second piece of essential metadata is the character encoding declaration. This tells the browser how to interpret the bytes that make up your HTML file into readable text characters.

Computers store text as sequences of bytes (numbers). A character encoding (like ASCII, ISO-8859-1, or UTF-8) is a mapping that defines which character corresponds to which byte or sequence of bytes.

If the browser uses a different encoding to interpret the file than the one it was saved with, you can get garbled text (often called mojibake), where special characters, accents, or non-English characters display incorrectly (e.g., Â© instead of ©, or Ã± instead of ñ).

We need to explicitly tell the browser which encoding to use.
