647. Context: CSS and JavaScript

It's important to remember that HTML entity syntax (&name; or &#...;) is specific to HTML parsing. Other contexts, like CSS and JavaScript, have their own mechanisms for handling special characters.

CSS Context (<style> block or .css file):

  • No HTML Entities: CSS does not parse or recognize HTML entities. Writing content: "&copy;"; or content: "&#169;"; in CSS will likely result in the literal text &copy; or &#169; being displayed, not the Â© symbol.
  • CSS Escaping: CSS uses backslash (\) escaping, primarily for:
    • Including special CSS syntax characters literally (e.g., \: to match an element with id="a:b").
    • Representing characters using their hexadecimal Unicode code point: \<hex-code> (e.g., content: '\A9'; or content: '\00A9'; for Â©). Note the leading backslash and optional padding zeros. A space is needed after the hex code if the following character could also be part of the hex sequence.
  • Direct UTF-8: As with HTML, the easiest way to include special characters in CSS (e.g., within content properties or font names) is to ensure your CSS file is saved as UTF-8 and type the character directly.

JavaScript Context (<script> block or .js file):

  • No HTML Entities: Standard JavaScript code does not parse HTML entities. let symbol = "&copy;"; creates a string containing the literal text &copy;, not the Â© symbol.
  • JavaScript String Escaping: JavaScript uses backslash (\) escaping within string literals for its own special characters or for Unicode representation:
    • '\'' (single quote inside single-quoted string)
    • "\"" (double quote inside double-quoted string)
    • '\\' (literal backslash)
    • '\n' (newline)
    • '\uXXXX' (Basic Multilingual Plane Unicode escape using 4 hex digits, e.g., '\u00A9' for Â©).
