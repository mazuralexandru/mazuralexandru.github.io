322. The overflow Property: Purpose

Files Modified: index.html, style.css

What happens when the content inside an element is larger than the element's box (defined by its width and height)? The overflow property controls this behavior.

  • Purpose: Specifies how to handle content that overflows the boundaries of its container element (its padding edge).
  • Applicability: Applies to block-level containers that have a defined size (or constraints like max-height) which their content might exceed. It generally doesn't apply to inline elements whose size is determined by their content.

Let's set up an example where content overflows.

.overflow-box {
  width: 200px;
  height: 100px;
  padding: 10px;
  border: 2px solid orange;
  background-color: rgba(0, 0, 100, 0.2);
  margin-bottom: 1em; /* Space below */

  /* overflow: visible; <- Default */
}

✨ Observe the Preview: You should see an orange-bordered box. The text content inside is longer than can fit within the 100px height (considering padding). Because the default overflow is visible, the text should spill out of the bottom of the box, potentially overlapping subsequent content (if any existed).
