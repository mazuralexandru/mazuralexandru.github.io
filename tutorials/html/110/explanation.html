110. Ordered Lists (<ol>): Removing type & CSS Preference

Having demonstrated the type attribute, let's remove it to revert to the default numbering and strongly reinforce the best practice of using CSS for styling.

Why Remove type?

  • Best Practice: To emphasize that controlling marker appearance should primarily be done via CSS (list-style-type) for better separation of concerns and maintainability.
  • Default Behavior: To clearly show the default decimal numbering (1, 2, 3...) again before introducing other attributes like start and reversed.
  • Simplicity: Keep the HTML cleaner as we move forward.

Code Change:

We have removed the type="I" attribute completely from the opening <ol> tag in index.html.

<p>Removing 'type' attribute. Default is decimal numbers.</p> <!-- Adjusted P -->

<ol> <!-- Removed type attribute -->
  <li>First step (now 1)</li>
  <li>Second step (now 2)</li>
  <li>Third step (now 3)</li>
</ol>

<p>CSS (list-style-type) is preferred for styling markers.</p> <!-- Adjusted P -->

✅ Observation:

  1. Confirm that the <ol> tag in index.html no longer has a type attribute.
  2. Look at the browser preview. The list markers should have reverted back to the default decimal numbers: 1., 2., 3..

✨ Key Takeaway: Removing the type attribute reverts the list to its default decimal numbering. Always prefer using the CSS list-style-type property over the HTML type attribute for controlling the visual style of list markers.
