209. font-weight

Files Modified: style.css

The font-weight property specifies the weight (or boldness) of the font.

  • Purpose: To control how thick or thin the characters appear.
  • Values:
    • Keywords:
      • normal: Normal font weight. Equivalent to 400. (Default for most elements).
      • bold: Bold font weight. Equivalent to 700. (Default for <strong>, <b>, <h1>-<h6>).
      • lighter: One step lighter than the parent element's weight.
      • bolder: One step bolder than the parent element's weight.
    • Numeric Values: 100 to 900 (in increments of 100).
      • 100: Thin
      • 400: Normal/Regular
      • 700: Bold
      • 900: Black/Heavy
  • Font Availability: The numeric values and keywords only work if the selected font-family actually has glyphs available for that specific weight. If you request font-weight: 300; (light) but the font only has normal and bold variants, the browser will usually try to synthesize a weight or fall back to the nearest available one (often normal). This is why loading specific weight files with @font-face or from font services is important when using numeric weights other than 400/700.
  • Variable Fonts: Newer "variable fonts" can support weights fluidly between the standard increments (e.g., 451), but this is a more advanced topic.

✅ Observe style.css:

  • We previously set font-weight: bold; on .container span.
  • Let's explicitly set the weight for the second paragraph (h1 + p) to lighter.
/* Adjacent Sibling Combinator Example */
h1 + p {
  margin-top: 5px; 
  color: lightgreen; 
  font-size: 0.9rem; 
  font-weight: lighter; /* Make this paragraph lighter */
}

✨ Observe the Preview: The second paragraph ("This paragraph follows the heading.") should now appear noticeably thinner or lighter than the first paragraph, assuming the default sans-serif font on your system has a lighter variant available relative to the normal weight inherited from the body.
