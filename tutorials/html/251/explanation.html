251. Quotations: CSS Styling

Files Modified: style.css

You can customize the appearance of quotations using CSS.

Common Styling for <blockquote>:

  • Adjusting margins/padding for different indentation levels.
  • Adding borders (especially a left border is common).
  • Changing background color.
  • Styling the attribution (<footer> or <cite>) within it.

Styling <q>:

  • While browsers add quotes, you can customize them using the CSS quotes property and ::before/::after pseudo-elements. This gives you control over using single vs. double quotes, or even custom characters.
    /* Example: Use French guillemets for <q> */
q {
  quotes: "«" "»" "‹" "›"; /* Outer pair, inner pair */
}
q::before { content: open-quote; }
q::after { content: close-quote; }
  • You can also style the text within <q> (e.g., font-style: italic;).

Styling <cite>:

  • You can override the default italic style or change its color, etc.

✅ Observe style.css:

  • We've added styles for blockquote and cite.
  • The blockquote gets a left border, different padding/margins, and a background color.
  • The cite element within the blockquote's footer gets a specific color.
/* Styling Quotations */
blockquote {
  background-color: rgba(255, 255, 255, 0.05);
  border-left: 4px solid cornflowerblue;
  margin: 1.5em 10px; /* Adjust default margins */
  padding: 0.5em 10px;
}

blockquote p {
  margin-bottom: 0.5em; /* Adjust paragraph margin inside quote */
}

blockquote footer {
  text-align: right;
  font-size: 0.9em;
  color: #aaa; /* Lighter color for attribution */
}

blockquote cite {
  font-style: normal; /* Override default italic if desired */
  color: #ccc; /* Slightly brighter color for title */
}

/* We won't override <q> quotes for now */

✨ Observe the Preview:

  • The blockquote should now have a distinct appearance with a cornflower blue left border, slightly different spacing, and a faint background.
  • The citation ("Some Work Title") within the blockquote footer should now be a brighter gray color and no longer italicized.
  • The inline quote (<q>) remains unchanged, still showing the browser's default quotation marks.
