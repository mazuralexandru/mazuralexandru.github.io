15. Linking the JavaScript File

We've covered basic HTML structure and CSS styling. The third core web technology is JavaScript, used for adding interactivity and behavior to web pages. To use JavaScript code stored in an external file, we need to link it to our HTML document, similar to how we linked the CSS file.

What's New?

We added a <script> tag near the end of the <body> section in index.html.

<!-- index.html snippet -->
  <!-- <p>This paragraph is commented out and won't appear.</p> -->

  <script src="script.js"></script> <!-- Added script tag -->
</body>
</html>

The <script> Element

  • Purpose: Used to embed or reference executable code, almost always JavaScript.
  • Linking External Files:
    • The src attribute (short for "source") specifies the path/URL to an external JavaScript file. In this case, src="script.js" tells the browser to load and execute the code found in script.js (located in the same directory).
    • When src is used, the content between <script> and </script> tags is ignored (though the closing tag </script> is still required!).
  • Placement (Best Practice): Placing <script> tags just before the closing </body> tag is a common best practice. Why?
    • HTML Parsing: The browser parses HTML from top to bottom. If a script is placed in the <head> without special attributes (defer or async, which we'll cover much later), the browser will stop parsing the HTML, download the script, execute it, and then continue parsing the rest of the HTML. If the script tries to interact with HTML elements that haven't been parsed yet (because they are lower down in the file), it will fail.
    • Perceived Performance: Placing scripts at the end allows the browser to render the visible HTML content first, making the page appear to load faster for the user, even if the script takes a moment to download and run.
  • Internal Scripts: You can also write JavaScript directly between <script> and </script> tags if you omit the src attribute, but keeping JS in separate .js files is much better for organization and maintainability, similar to using external CSS files.
