694. Accessibility (A11y): Image alt Text Deep Dive

Providing appropriate alternative text (alt) for images (<img>) is one of the most fundamental and impactful aspects of web accessibility. Let's consolidate best practices.

Recap: alt text provides the meaning/information/function of an image for users who cannot see it (e.g., screen reader users) and acts as fallback text if the image fails to load. It is mandatory.

Writing Effective alt Text - Process:

  1. Determine the Image's Purpose: Why is this image here?

    • Is it Informative? Does it convey content, data, or information relevant to understanding the context? (e.g., photo, chart, diagram illustrating a point).
    • Is it Functional? Does the image perform an action, primarily as the content of a link or button? (e.g., logo link, icon button).
    • Is it Purely Decorative? Does it add no information and only serves aesthetic purposes or visual formatting? (e.g., background flourish, spacer image).

  2. Write Text Based on Purpose:

    • Informative Images:
      • Describe the information or content conveyed by the image as concisely as possible.
      • Focus on what's important in the context of the surrounding text.
      • Example (Photo of a specific bird): alt="Blue jay perched on a snowy branch."
      • Example (Chart): alt="Bar chart showing sales increased 50% in Q4." (Consider providing detailed data in text nearby or a linked table if complex).
      • Avoid starting with "Image of...", "Picture of...".
    • Functional Images (Links/Buttons):
      • Describe the action or destination of the link/button.
      • Example (Logo link): alt="Example Corp Homepage"
      • Example (Search icon button): alt="Search"
      • Example (Print icon link): alt="Print this page"
    • Decorative Images:
      • Use an empty alt attribute: alt="".
      • This tells screen readers to ignore the image completely.
      • Crucial: Do not omit the alt attribute entirely (invalid HTML). Use alt="". Do not put spaces inside (alt=" ").
      • Use only if the image truly adds no information and its absence wouldn't change the meaning of the page. If unsure, err on the side of providing a brief description.

  3. Review Context: Does the surrounding text already describe the image adequately? If so, the image might be decorative (alt="") or need only very brief alt text to avoid redundancy.

  4. Keep it Concise: Aim for descriptions that convey the essential information efficiently. Long descriptions are better placed in the main page text or a linked resource, potentially associated using aria-describedby.

✅ Observation: No code changes. Think about different images you see online: photos in news articles, icons on buttons, logos, charts, background patterns. Apply the decision process: what is the purpose (informative, functional, decorative)? How would you write the alt text (or use alt="") for each?

✨ Key Takeaway: Writing effective alt text depends entirely on the image's purpose in context. Describe information for informative images, action/destination for functional images, and use alt="" for purely decorative images. alt is mandatory for all <img> tags.
