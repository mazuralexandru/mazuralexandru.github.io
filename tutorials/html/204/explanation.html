204. Link Content: Best Practices for Writing Link Text

Based on the previous examples, let's summarize some key best practices for writing effective link text:

  1. Be Specific and Descriptive: The link text should clearly and accurately describe the content the user will find or the action that will occur when they click the link. Integrate the description naturally into the sentence where possible.

    • Instead of: "Click here for the report."
    • Try: "View the Quarterly Sales Report."

  2. Be Concise (But Clear): While descriptive, avoid overly long or verbose link text. Find a balance between clarity and brevity.

    • Okay: "Read the full minutes from the committee meeting held on Tuesday discussing budget allocations."
    • Better: "Read the full minutes from the budget committee meeting." (Assuming context makes date clear).

  3. Avoid Generic Phrases: Actively avoid uninformative phrases like "Click Here," "More," "Read More," "Link," "Info," etc., unless the surrounding context or an ARIA attribute provides the necessary description (prefer making the link text itself descriptive).

  4. Indicate File Types/Actions: If linking to a non-HTML resource, inform the user.

    • "Download Application Form (PDF)"
    • "View Product Image (JPG)"
    • "Email Support"
    • "Call Sales"

  5. Front-Load Information: Place the most important identifying words at the beginning of the link text. This helps users scanning the page or listening via screen reader grasp the link's purpose faster.

    • Less Good: "Information about our company history can be found here."
    • Better: "Read about our company history."

  6. Maintain Consistency: Use a consistent style and tone for link text throughout your website.

✅ Observation: No code changes. Review the "Good Examples" list from the previous step (index.html lesson 203) and see how they align with these best practices (descriptive, specific, indicate file type).

✨ Key Takeaway: Writing effective link text involves being specific, descriptive, reasonably concise, avoiding generic terms, indicating file types/actions, and placing key information early.
