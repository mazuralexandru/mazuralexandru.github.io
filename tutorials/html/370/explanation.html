370. Module 7 Recap: Looking Ahead to Module 8 (Tables)

We have now completed our detailed exploration of the HTML <head> section and its crucial metadata elements (<meta>, <link>, <base>, <style>, <script>, <noscript>), adding to our knowledge of structure, text, lists, links, images, and global attributes.

Connecting to the Next Module:

Our next module, Module 8: Tables, shifts back to structuring content within the <body>. We will learn how to represent tabular data – information best presented in rows and columns – using HTML table elements.

Module 8 will cover:

  • The core table structure: <table>, <tr> (table row), <td> (table data cell).
  • Table headers: <th> (table header cell) and its scope (col, row).
  • Grouping table content: <thead>, <tbody>, <tfoot>.
  • Adding captions: <caption>.
  • Spanning rows and columns: rowspan and colspan attributes.
  • Accessibility considerations for tables.
  • Important Distinction: Using tables for their semantic purpose (displaying tabular data) versus the outdated (and inaccessible) practice of using tables for page layout (which should always be done with CSS).

The global attributes (id, class, style, title, lang, dir) learned in Module 6 can, of course, be applied to table elements (<table>, <tr>, <td>, <th>, etc.) to provide hooks for styling and scripting these data structures. The CSS linked via <link rel="stylesheet"> (covered in this module) is what we use to style the appearance of tables.
