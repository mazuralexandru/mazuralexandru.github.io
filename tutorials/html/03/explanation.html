03. Setting the Character Encoding

Now, let's add a crucial piece of metadata inside the <head> section: the character encoding declaration.

What's New?

We added a <meta> tag inside the <head> element.

<!-- index.html snippet -->
<head>
  <meta charset="UTF-8"> <!-- Added charset meta tag -->
</head>
<body>

Understanding Character Encoding (<meta charset="...">)

  • Purpose: This tag tells the browser how the text characters in your HTML file are encoded into bytes. Computers store text as numbers (bytes), and the character encoding defines the mapping between those numbers and the actual characters (letters, symbols, emojis, etc.).
  • Why UTF-8? UTF-8 is the universal standard for the web. It can represent virtually any character from any language, including emojis. Using it prevents garbled text (often called "mojibake," where characters display incorrectly like ï¿½ or strange symbols).
  • Syntax:
    • <meta>: The tag used for specifying various kinds of metadata. This is a void element (or empty element), meaning it doesn't have content between opening and closing tags, and therefore doesn't need a closing tag (</meta>).
    • charset: The attribute specifying the type of metadata â€“ in this case, the character set.
    • UTF-8: The value, indicating the specific encoding standard.
  • Placement: It's critical to place the <meta charset="UTF-8"> tag as early as possible within the <head> section (ideally, the very first element). The browser needs to know the encoding before it starts parsing content that might contain special characters.
