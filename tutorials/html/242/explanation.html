242. Code & Preformatted Text (<code>, <pre>)

Files Modified: None

HTML provides elements specifically for marking up computer code and preformatted text.

  • <code> (Code Fragment):
    • Semantic Meaning: Represents a fragment of computer code (inline). Can be a variable name, keyword, filename, code snippet, etc.
    • Default Styling: Typically rendered in the browser's default monospace font.
    • Usage: Used inline within paragraphs or other text blocks.
    • Example: The JavaScript keyword <code>const</code> declares a constant.
  • <pre> (Preformatted Text):
    • Semantic Meaning: Represents a block of text where whitespace (spaces, tabs, newlines) is significant and should be preserved exactly as written in the HTML source.
    • Default Styling: Typically rendered in a monospace font. Crucially, it respects whitespace and newlines from the source (white-space: pre; by default).
    • Usage: Used for displaying blocks of code, poetry, ASCII art, or any text where formatting and line breaks are important.
  • Combining <pre> and <code>: It's common and semantically correct practice to wrap code blocks within a <pre> element with a <code> element inside it. The <pre> handles the preservation of whitespace and block formatting, while the <code> provides the semantic meaning that the content is code.
    <pre><code>function greet(name) {
  console.log(`Hello, ${name}!`);
}</code></pre>
  • Escaping HTML: If your code snippet itself contains HTML characters like < or >, you must escape them using HTML entities (&lt; for <, &gt; for >) within the <code> block to prevent the browser from interpreting them as actual HTML tags.
    <code>&lt;div class="example"&gt;This is HTML code.&lt;/div&gt;</code>
