453. Forms: The value Attribute - Buttons (<input>)

The value attribute also has a specific meaning when used with button-like input types: submit, reset, and button.

value on Button Inputs:

  • Purpose: For <input> elements with type="submit", type="reset", or type="button", the value attribute determines the text displayed visually on the button itself.
  • Default Labels: If the value attribute is omitted:
    • type="submit" defaults to "Submit" (or a localized equivalent).
    • type="reset" defaults to "Reset" (or localized equivalent).
    • type="button" typically displays no text (or might have a default depending on the browser/OS), making value usually necessary for this type.
  • Submission Behavior (for Submit/Reset): The value attribute for type="submit" and type="reset" primarily controls the label. While a submit button's name=value pair can be submitted (if name is present) to indicate which button was pressed, the primary submitted data comes from the other form controls. Reset buttons don't submit data.
  • Contrast with <button> Element: Remember that for the <button> element (which we'll cover separately), the text between the opening and closing tags (<button>Label Here</button>) defines the button's label, and its value attribute has a different meaning (the value submitted with its name).

Code Change:

We've removed the previous fieldsets and added examples of <input> buttons, using the value attribute to set their visible labels. We've also added a simple inline onclick to the generic button for demonstration.

<!-- index.html snippet -->
<form action="/submit-data" method="POST">
  <!-- Removed checkbox/radio fieldsets -->
  <p>
     <input type="submit" value="Send Data"> <!-- Value sets label -->
  </p>
   <p>
     <input type="reset" value="Clear Form"> <!-- Value sets label -->
  </p>
   <p>
     <input type="button" value="Do Something (JS)" onclick="alert('Button clicked!');"> <!-- Value sets label -->
  </p>
</form>

✅ Observation:

  1. Examine the three <input> elements. Note how the value attribute corresponds to the desired button text.
  2. Look at the browser preview. You should see three buttons labeled "Send Data", "Clear Form", and "Do Something (JS)", respectively. The text comes directly from their value attributes.
  3. Click the "Do Something (JS)" button – an alert box should pop up. This shows the button is functional via its onclick. Clicking the others would attempt to submit or reset the (currently empty) form.

✨ Key Takeaway: For <input type="submit">, <input type="reset">, and <input type="button">, the value attribute specifies the text displayed on the button face.
