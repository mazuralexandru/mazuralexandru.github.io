225. The src Attribute - Syntax & Requirement

Let's add the src attribute to our <img> tag.

Syntax:

  • The attribute is added inside the <img> tag, like other attributes.
  • It takes the URL of the image file as its value, enclosed in quotes.
    <img src="url-to-image-file">

Requirement:

  • As stated, src is mandatory. An <img> tag without it is invalid HTML.

Placeholder URL:

  • In a real project, you would provide the correct path to your image file (e.g., images/logo.png, /assets/photo.jpg, or an absolute URL).
  • Since we don't have actual image files in this tutorial environment, we will use a placeholder path. This allows us to demonstrate the attribute's presence, even though it will still result in a broken image visually in the preview.

Code Change:

We've added a src attribute with a placeholder path (/images/nonexistent.jpg) to the <img> tag in index.html. Note that the alt attribute is still missing, so the element is still technically invalid, but we are focusing on src first.

<body>
  <h1>Embedding Images</h1>
  <p>Welcome to Module 5! Let's learn how to add images to web pages.</p>

  <img src="/images/nonexistent.jpg"> <!-- Added src attribute (still INVALID without alt) -->

  <p>The src attribute specifies the image file location.</p> <!-- Adjusted P -->


  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

✅ Observation:

  1. Locate the src="/images/nonexistent.jpg" attribute within the <img> tag.
  2. Look at the browser preview. You should now see a more distinct broken image icon where the <img> tag is. The browser tried to load the image specified by src, failed (because the file doesn't exist), and is showing the broken image placeholder. This confirms the src attribute was processed.

✨ Key Takeaway: Add the src="url" attribute to your <img> tag, providing the path to the image file. Even with src, the image won't display if the path is incorrect or the file is missing. The alt attribute is also required for validity.
