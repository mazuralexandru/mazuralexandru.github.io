400. Spanning Columns & Rows: Combining colspan and rowspan

It is technically possible for a single cell (<th> or <td>) to have both a colspan attribute and a rowspan attribute simultaneously.

Syntax:

<td rowspan="number_rows" colspan="number_cols">...</td>

Behavior:

  • The cell will span horizontally across number_cols columns AND vertically down across number_rows rows, effectively creating a merged rectangular block within the table grid.
  • This requires careful adjustment (omission) of cells in both the current row (for the colspan) AND subsequent rows (for the rowspan).

Complexity & Use Cases:

  • Combining spans significantly increases the complexity of the table structure, making it harder to write correctly, maintain, and ensure accessibility.
  • Use Case Example: Might be needed for a complex header structure where a single top-level category spans multiple sub-columns and also applies to multiple rows below it, or for certain types of calendar or scheduling layouts.
  • Caution: Use this combination only when absolutely necessary due to the complexity involved. Always double-check the number of cells omitted in affected rows.

Accessibility Implications:

  • Tables with combined spans are often the most challenging for assistive technologies to interpret correctly.
  • It becomes almost essential to use the headers attribute on the affected <td> data cells, explicitly listing the ids of all relevant <th> elements (including the doubly-spanned ones) to ensure screen readers can accurately announce the context. Relying solely on scope might become insufficient.

Code Change:

We won't create a complex combined example here, as it can quickly become difficult to visualize without specific styling and data. The key is to understand that it's possible but requires meticulous planning of the cells in each affected row.

✨ Key Takeaway: While possible, combining colspan and rowspan on a single cell should be done with extreme caution due to increased complexity in structure and accessibility. If used, ensure header associations are explicitly defined (often using the headers attribute).
