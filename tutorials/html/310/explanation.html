310. Centering with Auto Margins: Behavior

Files Modified: None

Let's recap the behavior and requirements for margin: auto centering:

  • Horizontal Only: This technique only works for horizontal centering. Setting margin-top: auto; and margin-bottom: auto; does not typically vertically center an element in the same way (vertical auto margins usually resolve to 0). Vertical centering requires different techniques (Flexbox, Grid, absolute positioning with transforms, etc.).
  • Requires Explicit Width: The element must have a width defined that is less than its container's width. If width is auto (the default for block elements), the element will take up the full available width, leaving no space for the auto margins to distribute, and the element will remain left-aligned.
  • Block-Level: The element must be block-level or behave like one (e.g., display: block;). It doesn't work directly on inline elements.

This margin: 0 auto; pattern (or similar with vertical margins) is a classic and reliable way to center fixed-width or percentage-width block containers horizontally.
