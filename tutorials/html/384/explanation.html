384. Horizontal Nav (Inline Block): Whitespace Issue

Files Modified: None

A well-known quirk of the display: inline-block; technique relates to whitespace in the HTML source code.

  • The Issue: Any whitespace (spaces, tabs, newlines) between the closing </li> tag of one item and the opening <li> tag of the next item in your HTML source code is treated like a single space by the browser when rendering the inline-block elements.

  • Effect: This often results in small, unwanted visual gaps between the inline-block list items, even if you set their margins to zero. The size of this gap depends on the current font-size.

  • Example HTML causing the gap:

    <ul>
  <li>Item 1</li> <!-- Newline/space here -->
  <li>Item 2</li> <!-- Newline/space here -->
  <li>Item 3</li>
</ul>

  • Common Workarounds (often feel like hacks):

    1. Remove Whitespace in HTML: Write the HTML with no spaces or newlines between the <li> tags (hard to read and maintain).
      <ul><li>Item 1</li><li>Item 2</li><li>Item 3</li></ul>
    2. HTML Comments: Use HTML comments to bridge the gap.
      <ul>
  <li>Item 1</li><!--
  --><li>Item 2</li><!--
  --><li>Item 3</li>
</ul>
    3. Negative Margins: Apply a negative margin-left to the <li> elements to counteract the space (value depends on font size, feels fragile).
    4. Font Size Zero: Set font-size: 0; on the parent <ul> (making the whitespace characters have zero width) and then reset the desired font-size on the child <li> elements. This is quite effective but can feel indirect.

  • Why Flexbox is Better: Modern layout methods like Flexbox inherently avoid this whitespace issue, making them a much cleaner solution for horizontal lists.
