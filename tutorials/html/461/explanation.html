461. Forms: Text Areas (<textarea>) - Fundamentals

While <input type="text"> is for single lines, the <textarea> element allows users to enter multi-line plain text.

The <textarea> Element:

  • Definition: Creates a multi-line text input control.
  • Use Case: Ideal for longer freeform text like comments, messages, descriptions, postal addresses (though address-specific fields also exist), or code snippets entered by users.
  • Key Difference from <input>: It is not a void element. It requires both an opening <textarea> and a closing </textarea> tag.

Core Mechanics & Syntax:

  • Syntax: <textarea name="..." id="..." rows="num" cols="num">Initial content</textarea>
  • Initial Value: Text placed between the opening <textarea> and closing </textarea> tags serves as the default content when the page loads. Important: Any whitespace (including leading/trailing spaces and line breaks) between the tags is considered part of the initial value. Keep the tags snug around the content or empty if no default is desired.
  • name Attribute [Required]: Identifies the textarea's data for submission.
  • id Attribute [Required]: For associating with a <label for="...">.
  • <label> [Required]: Always provide a <label> for accessibility.

Sizing Attributes (Hints):

  • rows: Specifies the visible number of text lines (approximate height).
  • cols: Specifies the visible width in average character widths (approximate width).
  • CSS Preferred: The rows and cols attributes provide basic sizing hints, but using CSS width and height properties (or min-height, max-height, etc.) is the standard and more flexible way to control the dimensions and appearance of a <textarea>. CSS also controls whether the user can resize the textarea (resize property).

Code Change:

We've added a <textarea> element inside the form in index.html, along with a <label> and some attributes, including initial content between the tags.

<!-- index.html snippet -->
<form action="/submit-comment" method="POST"> <!-- Updated action -->
  <p>
    <label for="user_comment">Your Comment:</label><br> <!-- Added label -->
    <textarea id="user_comment" name="comment" rows="5" cols="40">Enter your thoughts here...</textarea> <!-- Added textarea -->
  </p>
  <p><button type="submit">Post Comment</button></p>
</form>

(Note: <br> used after label for simple layout in this basic example).

✅ Observation:

  1. Examine the <textarea> element. Note the opening/closing tags, the text content between them, and the rows/cols attributes.
  2. Look at the browser preview. You should see the "Your Comment:" label followed by a multi-line text box.
  3. The box should be pre-filled with "Enter your thoughts here...".
  4. The size should be approximately 5 rows high and 40 characters wide (based on the rows/cols hints, though actual rendering depends on font/browser).
  5. The textarea is likely resizeable by the user by default (usually a drag handle in the bottom-right corner). You can type multiple lines of text into it.

✨ Key Takeaway: Use <textarea> for multi-line text input. Place initial value text directly between the opening and closing tags. Use rows and cols as hints, but prefer CSS for reliable sizing and styling. Always include a <label> and name.
