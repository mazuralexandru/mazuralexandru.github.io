446. Forms: Labels (<label>) - Core Mechanics

Let's look at the basic mechanics of the <label> element.

Syntax:

  • Requires opening <label> and closing </label> tags.
  • The visible label text goes between the tags.
    <label>Label Text Here</label>

Element Category:

  • Inline element by default.
  • Phrasing content.

Content Model:

  • Generally contains phrasing content (the label text itself, potentially including inline elements like <span> or <em> if needed for styling or emphasis within the label).
  • Important Restriction: A <label> element must not contain another <label> element.
  • Association Method Impact:
    • If using explicit association (for attribute), the label should not contain the form control it is labeling.
    • If using implicit association (wrapping the control), the label does contain the form control.

Association Methods (Brief Overview):

How do we link the <label> to its control?

  1. Explicit Association (for/id): The label's for attribute points to the control's id attribute. (Strongly Recommended - Covered next).
  2. Implicit Association (Wrapping): The input control is placed inside the <label> element. (Less flexible, less recommended).

We will focus on the preferred explicit method first.

✅ Observation: No code changes yet. Understand the basic tag structure (<label>...</label>) and that its content is typically the visible text description. Recognize that the association with the input is key and will be achieved via attributes.

✨ Key Takeaway: The <label> element wraps the visible text description for a form control. Explicit association using for and id attributes is the recommended way to link them.
