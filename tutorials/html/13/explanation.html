13. Styling Elements with External CSS

We removed the inline styles because they mix presentation with structure. The preferred way to style elements is using CSS rules in our external stylesheet (style.css). Let's style the <h1> we just added.

What's New?

We added a new CSS ruleset to style.css targeting the <h1> element.

/* style.css snippet */
  min-height: 150px; /* Ensure space for preview */
}

/* Added H1 style rule */
h1 {
  color: tomato; /* Change H1 text color */
  border-bottom: 2px solid #eee; /* Add a light bottom border */
  padding-bottom: 5px; /* Add space below text before border */
}

/* CSS files can also have comments! */

CSS Rulesets Recap

Remember the basic structure from Step 8:

  1. Selector: h1 - This targets all <h1> elements in the linked HTML document. This is an Element Type Selector.
  2. Declaration Block: { ... } - Contains the style definitions.
  3. Declarations: property: value; - The actual style rules.

New CSS Properties Introduced:

  • color: tomato;: We're using the color property again, this time setting the <h1> text color to the keyword tomato. This overrides the default light gray inherited from the body style.
  • border-bottom: 2px solid #eee;: This is a shorthand property to define the border on the bottom edge of the element.
    • 2px: Sets the border width to 2 pixels.
    • solid: Sets the border style to a solid line (other options include dotted, dashed, double).
    • #eee: Sets the border color to a very light gray (#eeeeee).
  • padding-bottom: 5px;: Adds 5 pixels of padding (internal space) specifically to the bottom edge of the <h1> element, between the text content and the bottom border. This prevents the border from touching the text.
