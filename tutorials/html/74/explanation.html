74. Semantic HTML for Quotations: <blockquote>, <q>, <cite>

HTML offers tags to semantically mark up quotations: <blockquote> for long, block-level quotations, <q> for short inline quotations, and <cite> to identify the title of a cited work.

Quotation Tags Explained

  • <blockquote> (Block Quotation):
    • Semantic Meaning: Represents content quoted from another source, rendered as a distinct block.
    • Attributes: Can have a cite attribute whose value is a URL pointing to the source of the quotation. This attribute is not displayed visually but can be used by browsers or scripts.
    • Default Styling: Typically indented by browsers (using margin). Often styled further with borders, background colors, or different fonts.
  • <q> (Inline Quotation):
    • Semantic Meaning: Represents a short inline quotation.
    • Default Styling: Browsers automatically add appropriate quotation marks around the content (e.g., " " for English). The specific quotes used depend on the language context (defined by the lang attribute on a parent element) and nesting level. You should not manually type quotes inside the <q> tag. Can also have a cite attribute.
  • <cite> (Citation / Work Title):
    • Semantic Meaning: Represents the title of a creative work (book, paper, essay, poem, score, song, script, film, TV show, game, sculpture, painting, play, opera, musical, exhibition, legal case report, etc.). It's not for quoting people.
    • Default Styling: Typically rendered in italics.
