649. Entities: Best Practices & Recommendations

Let's consolidate the best practices for using HTML entities.

Best Practices Summary:

  1. Use UTF-8 Encoding: Save your HTML files as UTF-8. Declare <meta charset="UTF-8"> first in <head>. Configure your server to send UTF-8 headers. This is the foundation.
  2. Use Direct Characters (with UTF-8): For any character other than HTML's reserved characters (<, >, &, ", '), prefer typing or pasting the character directly into your UTF-8 encoded HTML file.   3. Escape Reserved Characters: Always use entities when you need to display the literal reserved characters:
    • < -> &lt;
    • > -> &gt;
    • & -> &amp;amp;
  4. Escape Quotes in Attributes (Conditionally):
    • Use &quot; for " inside an attribute value delimited by ".
    • Use &apos; for ' inside an attribute value delimited by '.
    • (Or switch the delimiter quotes if possible).
  5. Prefer Named Entities (When Used): If you do need an entity (for reserved chars or sometimes common symbols), prefer named entities (&copy;) over numeric ones (&#169;) for better source code readability.
  6. Use Numeric Entities Sparingly: Only use numeric entities (&#...; or &#x...;) if a named entity doesn't exist and direct UTF-8 input isn't feasible (rare).
  7. Use &nbsp; Purposefully: Use non-breaking spaces only to prevent unwanted line breaks within text. Never use them for visual layout spacing (use CSS).
  8. Always Use Semicolons: Include the trailing semicolon (;) for all named and numeric entities.
  9. Validate: Use an HTML validator to catch errors like unescaped reserved characters or invalid entity syntax.
  10. Be Context-Aware: Remember that entity syntax (&...;) is for HTML parsing. CSS and JavaScript use different escaping mechanisms (\) or direct UTF-8 characters.
