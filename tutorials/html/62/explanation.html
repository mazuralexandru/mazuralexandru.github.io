62. Text Styling: line-height

The line-height property controls the amount of space between lines of text, essentially setting the height of a line box. It's crucial for readability.

line-height Explained

  • Purpose: Sets the height of the box that contains a line of text. It controls the vertical distance between baselines of consecutive lines.
  • Values:
    • normal: The browser's default value. Usually around 1.2, but can vary slightly.
    • <unitless number> (Recommended): e.g., 1.5, 1.6. This number is multiplied by the element's own font-size to calculate the line height. This is the preferred method because it scales proportionally if the font-size changes (e.g., due to inheritance or user settings).
    • <length>: e.g., 24px, 1.5em. Sets a fixed line height. Using px can lead to accessibility issues or overlapping lines if the font size increases significantly without the line height also increasing. Using em makes the line height relative to the element's font size (like the unitless number), but can be affected by inheritance differently in some edge cases.
    • <percentage>: e.g., 150%. Similar to the unitless number, calculated based on the element's font-size. 150% is equivalent to 1.5. The unitless value is generally preferred over percentages for line-height.
  • Readability: Adequate line height (often between 1.4 and 1.8) is crucial for making text comfortable to read, especially for longer paragraphs. Too little spacing makes lines run together; too much can make them feel disconnected.
  • Inheritance: Yes, line-height is inherited. When a unitless number is inherited, the child element multiplies its own font size by that number. If a length or percentage is inherited, the child inherits the computed pixel value from the parent, which might not scale correctly if the child has a different font size. This is another reason unitless values are preferred.
