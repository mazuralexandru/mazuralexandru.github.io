169. Reset: Removing Fragment Link Examples

We have thoroughly explored linking to page fragments using id attributes and the # syntax in href.

Now, we will move on to another important link attribute: target, which controls where the linked document opens (e.g., same tab, new tab). To focus on this new attribute, we will remove the fragment linking examples and the associated structural/styling elements from index.html.

Why Remove Code?

  • Focus: Shift attention from where a link points (href fragments) to how the link opens (target attribute).
  • Simplify HTML: Remove the extra headings, paragraphs, spacer divs, temporary styles, and navigation elements that were needed for the fragment demo, but are not needed for demonstrating target.
  • Clean Slate: Provide a minimal structure ready for adding simple links where the target attribute's effect can be clearly observed.

Code Change:

We have removed the <nav> section, the spacer <div>s, the <h2> elements with their ids and content, the "Back to Top" links, and the temporary <style> block from the <head>. We also updated the <h1> and paragraph text.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>HTML Links</title> <!-- Updated title -->
  <link rel="stylesheet" href="style.css">
  <!-- Removed temporary styles -->
</head>
<body>
  <h1>Links and Navigation</h1> <!-- Updated H1 -->
  <p>Preparing to discuss the 'target' attribute.</p> <!-- Adjusted P -->

  <!-- Removed fragment links and target elements/spacers -->

  <p>This attribute controls where linked documents open.</p> <!-- Adjusted P -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>
</html>

✅ Observation:

  1. Verify that the index.html file now contains only the basic page structure (html, head, body), the <h1>, and two paragraphs. The fragment-related elements and temporary styles are gone.
  2. The browser preview should show a very simple page, ready for the target attribute examples.

✨ Key Takeaway: Removing the fragment link examples and associated demo structure allows us to focus clearly on the next topic: the target attribute and its effect on link behavior.
