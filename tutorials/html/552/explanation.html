552. Multimedia: Video (<video>) - Global Attributes & Styling

Remember that <video> (and <audio>) are standard HTML elements, so global attributes (like those covered in Module 6) can be applied to them.

Common Global Attributes on <video>:

  • id: Assigns a unique identifier. Crucial if you need to target this specific video player with JavaScript (e.g., document.getElementById('my-video').play();).
  • class: Assigns one or more classes. Used extensively for applying CSS styles to the video player container or for selecting multiple video players with JavaScript (document.querySelectorAll('.product-video')).
  • style: Applies inline styles (generally discouraged; use classes).
  • title: Provides advisory text (tooltip on hover). Less useful here than the accessible title attribute on the <iframe> if using an iframe embed, or relying on accessible controls for native video.
  • lang/dir: Can specify language/direction if relevant to surrounding text or custom controls, though less common on the video tag itself.
  • Accessibility Attributes (aria-*): Might be used in conjunction with custom JavaScript controls to enhance accessibility if the native controls attribute is hidden.

Styling the <video> Element:

  • You use CSS to style the appearance of the <video> element itself (the container). Common properties include:
    • width, height, max-width (for responsive sizing)
    • display (often block or inline-block to control spacing)
    • margin, padding
    • border, border-radius
    • box-shadow
    • object-fit, object-position (to control how the video scales within its container if aspect ratios don't match perfectly).
  • Styling Native Controls: Styling the browser's default controls (shown when the controls attribute is present) with CSS is very limited and inconsistent across browsers. Achieving a fully custom player look requires building controls with HTML/CSS and controlling the video via the JavaScript API.

Code Change:

We've added id and class attributes to our <video> element and added corresponding example styles in style.css to demonstrate targeting. We removed the src and crossorigin again to focus on the container.

<!-- index.html snippet -->
  <!-- Added class and id -->
  <video controls width="480" height="270"
         id="promo-video" class="video-player featured">
    <!-- src/source needed for actual video -->
    <p>Sorry, your browser doesn't support embedded videos.</p>
  </video>

/* style.css - Added rules */
/* Example styles targeting video via class/id */
.video-player {
  display: block; /* Make it block to control margins */
  margin: 15px auto; /* Center horizontally */
  border: 2px solid lightgreen; /* Example border */
}

#promo-video {
  box-shadow: 0 0 10px rgba(144, 238, 144, 0.5); /* Example shadow */
}

✅ Observation:

  1. Examine the id and class attributes on the <video> tag.
  2. Examine the new CSS rules in style.css targeting .video-player and #promo-video.
  3. Look at the browser preview. The empty video player should now:
    • Have a light green border (.video-player class style).
    • Be centered horizontally (.video-player class style).
    • Have a subtle green glow (#promo-video id style).
  4. This demonstrates using class and id to apply CSS styles directly to the video container element.

✨ Key Takeaway: Apply global attributes like id and class to <video> elements for JavaScript targeting and CSS styling. Use CSS to control the player container's dimensions, position, and appearance. Styling native controls is limited; custom controls require JS.
