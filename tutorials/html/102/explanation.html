102. Unordered Lists (<ul>): Best Practices & Anti-Patterns

Let's summarize the best practices for using <ul> and common mistakes to avoid.

Best Practices:

  1. Use Semantically: Only use <ul> when the order of the items genuinely does not matter for the meaning of the list. If the sequence is important (e.g., steps in a process), use <ol> instead.
  2. Direct Children: Ensure all list content is wrapped in <li> elements that are direct children of the <ul>.
  3. Use CSS for Styling: Rely exclusively on CSS (list-style-type, padding, margin, ::marker, etc.) to control the visual appearance:
    • Marker type (bullets, circles, squares, none, images)
    • Indentation
    • Spacing
    • Layout (e.g., using display: flex for horizontal navigation)
  4. Accessibility Considerations: While <ul> itself is accessible, ensure the content within the <li> items is accessible (e.g., meaningful link text if used for navigation).

Anti-Patterns (Avoid These):

  1. Using <ul> for Indentation: Do not use a <ul> (often with list-style-type: none;) simply to achieve visual indentation for a block of text. This is semantically incorrect. Use CSS padding-left or margin-left on the appropriate element (<p>, <div>, etc.) instead.
  2. Using <p> with Manual Bullets: Avoid creating fake lists by manually typing hyphens (-), asterisks (*), or other characters at the start of paragraph lines. This completely lacks semantic structure and is inaccessible.
    <!-- INCORRECT - DO NOT DO THIS -->
<p>* Item one</p>
<p>* Item two</p>
  3. Using <br> to Separate List Items: Don't use line breaks within a single <p> or other element to simulate list items. Use distinct <li> elements.
    <!-- INCORRECT - DO NOT DO THIS -->
<p>Item one<br>Item two<br>Item three</p>
