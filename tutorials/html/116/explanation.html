116. Preparing for Accessibility: Removing li value Example

We've now seen how attributes on both <ol> (type, start, reversed) and <li> (value) can affect numbering.

Before discussing the accessibility implications of ordered lists and some final best practices, let's simplify our example again by removing the value attribute from the <li> and reverting to a basic, standard ordered list.

Why Reset?

  • Focus: To discuss general <ol> accessibility without the complication of overridden numbering.
  • Simplicity: A standard list (1, 2, 3) provides the clearest baseline for understanding accessibility announcements.

Code Change:

We have removed the value="10" attribute from the second <li> and removed the fourth <li> element in index.html.

<p>Resetting list before discussing accessibility.</p> <!-- Adjusted P -->

<ol>
  <li>Item 1</li> <!-- Removed value attribute -->
  <li>Item 2</li>
  <li>Item 3</li>
  <!-- Removed fourth li -->
</ol>

<p>Attributes removed.</p> <!-- Adjusted P -->

✅ Observation:

  1. Verify that the value attribute is gone from the second <li> and the fourth <li> has been removed in index.html.
  2. The browser preview should show a standard ordered list numbered 1., 2., 3..

✨ Key Takeaway: Removing the specific li value attribute example allows us to discuss the general accessibility characteristics of standard <ol> lists more clearly.
