122. Nested Lists: Nesting <ol> inside <ul>

You are not restricted to nesting the same type of list. You can nest an ordered list (<ol>) inside an unordered list item (<ul><li>), or vice-versa.

Scenario: Ordered Sub-items in an Unordered List

Imagine a list of general topics (<ul>), where one topic needs to be broken down into specific, ordered steps (<ol>).

Code Change:

We've modified the nested list example in index.html. The outer list is still a <ul>, but the nested list inside the second <li> is now an <ol> with type="a".

<h1>Nested Lists</h1>
<p>Nesting different list types (ol inside ul):</p> <!-- Adjusted P -->

<ul> <!-- Outer ul -->
  <li>Topic Alpha</li>
  <li>Topic Beta <!-- Outer li STARTS -->
    <ol type="a"> <!-- Nested ol START (inside li) -->
      <li>Sub-point a</li>
      <li>Sub-point b</li>
    </ol> <!-- Nested ol END (inside li) -->
  </li> <!-- Outer li END -->
  <li>Topic Gamma</li>
</ul>

✅ Observation:

  1. Examine the structure in index.html. Note the <ul> containing an <li> which, in turn, contains an <ol>.
  2. Look at the browser preview:
    • "Topic Alpha" and "Topic Gamma" have the default <ul> bullet point marker (•).
    • "Topic Beta" also has the default <ul> bullet point marker.
    • The sub-items "Sub-point a" and "Sub-point b" appear below "Topic Beta", indented further.
    • Crucially, the sub-items are marked with sequential letters (a., b.) because they are inside an <ol> with type="a". The <ol> controls their markers.

✨ Key Takeaway: You can nest <ol> inside <ul><li> (or vice-versa) when the hierarchy requires changing between ordered and unordered sub-items. The nested list follows its own rules for markers (<ol> uses numbers/letters, <ul> uses bullets). Remember the placement rule: the nested list always goes inside the parent <li>.
