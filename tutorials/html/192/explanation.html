192. The revert Keyword

Files Modified: None

The revert keyword is another reset value, but it specifically targets the styles defined by the user-agent (browser) stylesheet.

  • Purpose: To undo any styles applied by Author or User stylesheets and explicitly revert the property's value back to whatever the browser's default style for that element would be.
  • Syntax: property: revert;
  • Behavior: Rolls back the cascade for the specific property to the user-agent level. If the user-agent stylesheet has a specific style for that element/property combination, that value is used. Otherwise, it behaves like unset.

Example: Browsers typically style <h1> elements with display: block;, font-size: 2em;, margin-top: 0.67em;, margin-bottom: 0.67em;, font-weight: bold;.

h1 {
  /* Author styles overriding browser defaults */
  font-size: 1.5rem; 
  margin: 0;
  font-weight: normal;
  color: blue;
}

.special-h1 {
  /* Revert some properties back to browser defaults */
  font-size: revert; /* Reverts to browser's default h1 size (e.g., 2em) */
  margin: revert; /* Reverts to browser's default h1 margins (e.g., 0.67em top/bottom) */
  font-weight: revert; /* Reverts to browser's default h1 weight (e.g., bold) */
  color: revert; /* Reverts to browser's default text color (likely black) */
}

<h1>This H1 uses author styles (1.5rem, no margin, normal weight, blue).</h1>
<h1 class="special-h1">This H1 reverts to browser defaults (2em, margins, bold, black).</h1>

revert is useful when you want to selectively undo your own "reset" or base styles and rely on the browser's native styling for a particular element instance. It's newer than initial and unset but has good modern browser support.
