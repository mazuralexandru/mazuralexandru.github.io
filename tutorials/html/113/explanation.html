113. CSS Comments: Syntax

Files Modified: None

The syntax for CSS comments is consistent:

  • Start: /* (Forward slash, asterisk)
  • End: */ (Asterisk, forward slash)
  • Content: Anything between the start and end delimiters is the comment.

Key Points:

  • Multi-line: CSS comments can span multiple lines naturally.
    /* This comment
   spans multiple
   lines easily. */
  • No Nesting: Like HTML comments, standard CSS comments cannot be nested reliably. The first */ encountered will close the comment.
    /* Outer /* Inner */ Still Outer? */ /* Behavior is undefined/error */
  • Placement: Comments can generally appear anywhere in a CSS file where whitespace is allowed – before, after, or between rulesets, and even within a declaration block (though commenting out parts of a value is usually invalid). Commenting out entire declarations or rulesets is the common use case.

✅ Review the comments in style.css to see the /* ... */ syntax used for both single and multi-line comments, and for disabling code.
