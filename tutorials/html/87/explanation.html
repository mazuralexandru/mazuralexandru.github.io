87. Other Inline Semantics: <s> vs. <del> & Best Practices

It's important to distinguish <s> from another element that can look similar: <del>. We haven't formally covered <del> (which often pairs with <ins> for representing edits), but understanding the semantic difference helps clarify when to use <s>.

Semantic Distinction:

  • <s> (Strikethrough): Represents content that is no longer correct or relevant. It implies the information itself is outdated or inaccurate in the current context (like an old price, a wrong statement).
  • <del> (Deleted Text): Represents content that has been deleted or removed from the document, often as part of tracking changes or edits. It indicates removal from a previous version.

Think of <s> as "this is wrong/obsolete now" and <del> as "this used to be here but isn't anymore (in this version)". While both might render with a strikethrough by default, their underlying meaning differs. For cases like showing a previous price alongside a new price, <s> is generally more appropriate than <del>.

Best Practices:

  1. Use Semantically: Choose <s> when the meaning is "no longer accurate/relevant". Consider <del> if the meaning is explicitly "deleted content" (often in conjunction with <ins> for inserted content).
  2. Avoid for Styling: Do not use <s> simply to achieve a visual strikethrough effect if the text isn't actually inaccurate or obsolete. For purely decorative strikethrough, use the CSS text-decoration: line-through; property (applied via a class or selector).
  3. Accessibility: Screen readers may announce struck-through text (either from <s> or <del>), potentially indicating its status to users. Using the tags semantically provides this potential benefit, whereas purely visual CSS strikethrough offers no such information to non-visual users.

✨ Key Takeaway: Use <s> specifically for content that is no longer correct or relevant. Distinguish it from <del> (deleted content) and use CSS (text-decoration: line-through;) for purely stylistic strikethrough effects.
