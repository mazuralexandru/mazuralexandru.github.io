292. The lang Attribute - Revisited (Element-Level Use)

We know the lang attribute is crucial on the <html> tag to declare the primary language of the entire document. However, lang is also a global attribute, meaning you can apply it to any element to indicate that the content of that specific element (and its children, unless overridden again) is in a different language.

Purpose (Element-Level):

  • To correctly identify the language of specific words, phrases, paragraphs, or sections when they differ from the main document language.

Why It Matters (Recap):

  • Accessibility: Enables screen readers to switch pronunciation rules and voices, ensuring accurate rendering of foreign words or quotes.
  • Browser Behavior: Can influence spell checking, hyphenation, font selection, and translation tool behavior for that specific piece of content.
  • SEO: Helps search engines understand the multilingual nature of your content.

Syntax & Values (Recap):

  • lang="language-tag" (e.g., lang="fr", lang="es-MX", lang="ja"). Uses BCP 47 language tags.

Code Change:

We've added examples to the paragraphs in index.html, marking specific phrases with the lang attribute using <span> as a neutral inline wrapper.

<!-- index.html -->
<body>
  <h1>Global Attributes - Lang & Dir</h1>
  <p>The document language is English (`lang=html:<html lang="en">`). But we can mark specific parts in other languages, like this French phrase: <span lang="fr">C'est la vie</span>.</p> <!-- Added lang="fr" -->

  <p>This helps screen readers use correct pronunciation. Another example in Spanish: <span lang="es">Hola Mundo</span>.</p> <!-- Added lang="es" -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

✅ Observation:

  1. Locate the <span> elements with lang="fr" and lang="es" attributes in index.html.
  2. Look at the browser preview. Visually, the text "C'est la vie" and "Hola Mundo" will likely appear identical to the surrounding English text (unless you have specific fonts installed or browser settings that trigger different rendering based on lang).
  3. Crucially: Although there's no visual change, the semantic information has been added. A screen reader encountering <span lang="fr">C'est la vie</span> would know to switch to French pronunciation rules for that phrase, significantly improving the experience for a listener.

✨ Key Takeaway: Use the lang attribute on specific inline or block elements (often using a <span> for inline phrases) to declare the language of content that differs from the primary document language set on the <html> tag. This is essential for accessibility.
