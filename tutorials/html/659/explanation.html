659. Validation: What Validation Checks - Uniqueness & Deprecated

Validators also check for specific constraints like uniqueness and warn about outdated features.

2.4. Uniqueness Constraints:

  • The most prominent example is the id attribute. Validators enforce the rule that every id value used within a single HTML document must be unique.
    • Example Error: Error: Duplicate ID "main-content".
  • This is critical because duplicate IDs break fragment linking, label association (for/id), ARIA relationships, and document.getElementById() functionality.

2.5. Character Encoding Declaration:

  • Checks that a character encoding is declared correctly (ideally <meta charset="UTF-8">).
  • Flags missing declarations, multiple declarations, incorrect syntax, or placement too late in the document (not within the first 1024 bytes).
    • Example Error: Error: A "charset" attribute on a "meta" element found after the first 1024 bytes.

2.6. Deprecated Features:

  • Validators check against a specific version of the HTML standard (determined by the DOCTYPE, usually HTML5/Living Standard). They will flag the use of elements or attributes that were part of older HTML versions but are now obsolete or deprecated in the current standard.
  • These usually appear as Warnings rather than Errors, encouraging developers to update to modern, standard practices.
  • Examples:
    • Warning: Warning: The "center" element is obsolete. Use CSS instead.
    • Warning: Warning: The "font" element is obsolete. Use CSS instead.
    • Warning: Warning: The "border" attribute on the "table" element is obsolete. Use CSS instead.
    • Warning: Warning: The "bgcolor" attribute is obsolete. Use CSS instead.
    • Warning: Warning: The "align" attribute on the "p" element is obsolete. Use CSS instead.

Addressing these warnings by replacing deprecated presentational markup with modern semantic HTML and CSS improves code quality, maintainability, accessibility, and future compatibility.

✅ Observation: No code changes. Recall the importance of unique id attributes (Lesson 269) and the rule about placing <meta charset> first (Lesson 310). Understand that validators automatically enforce these. Recognize that validators help identify and discourage the use of outdated HTML features.

✨ Key Takeaway: Validation enforces critical uniqueness constraints (like id) and helps identify the use of deprecated elements and attributes, encouraging adherence to modern HTML standards and best practices.
