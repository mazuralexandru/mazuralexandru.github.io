457. Forms: Submit Buttons (type="submit") - Behavior & Interactions

Let's clarify the behavior triggered by submit buttons.

Submission Process Trigger:

  • Clicking a <input type="submit"> or <button type="submit"> initiates the form submission sequence:
    1. Validation: If the form does not have the novalidate attribute, the browser first performs its native validation checks based on attributes like required, type, pattern, min, max, minlength, maxlength, step.
    2. Validation Success: If all controls are valid (or validation is disabled), the browser proceeds to package and send the data.
    3. Validation Failure: If any control fails validation, the submission is blocked, and the browser typically displays error messages and focuses the first invalid field.
    4. Data Packaging: Data from all named, non-disabled controls within the form is collected and encoded according to the form's method and enctype.
    5. Button Data: If the submit button that was clicked has a name attribute, its name=value pair is included in the submitted data. This allows the server to potentially determine which submit button initiated the action if there are multiple.
    6. HTTP Request: The browser sends the HTTP request (GET or POST) containing the packaged data to the URL specified in the form's action attribute.
    7. Response Handling: The browser waits for and then displays the response from the server (unless target specifies otherwise).

Keyboard Interaction:

  • Pressing the Enter key when focus is inside certain form fields (typically single-line text inputs like type="text", type="search", type="email", type="password") usually triggers the first submit button (<input type="submit"> or <button type="submit">) found within that form in the source code.

Multiple Submit Buttons:

  • A form can contain multiple submit buttons, each potentially having a different name and/or value attribute.
  • When the user clicks a specific submit button with a name, only that button's name=value pair is included in the submission.
  • This allows the server-side script to differentiate actions based on which button was pressed (e.g., <button name="action" value="save">Save Draft</button> vs. <button name="action" value="publish">Publish</button>).
