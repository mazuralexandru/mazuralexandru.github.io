199. Relative Units: Viewport Units (vw, vh, vmin, vmax)

Files Modified: None

Viewport units define sizes relative to the dimensions of the browser's viewport (the visible area of the web page).

  • vw (Viewport Width): 1vw is equal to 1% of the viewport's width.
  • vh (Viewport Height): 1vh is equal to 1% of the viewport's height.
  • vmin (Viewport Minimum): 1vmin is equal to 1% of the smaller of the viewport's width or height.
  • vmax (Viewport Maximum): 1vmax is equal to 1% of the larger of the viewport's width or height.

Use Cases:

  • Creating elements that fill the full height or width of the screen (e.g., height: 100vh; for a full-height hero section).
  • Making font sizes scale directly with the viewport size (e.g., font-size: 4vw;). Use with caution, as text can become too large or too small on extreme screen sizes; often combined with calc() and min()/max() or CSS locks for better control.
  • Creating responsive layouts where spacing or element sizes adapt fluidly to the viewport dimensions.

Pros: Allows direct scaling based on the viewport, useful for full-screen elements and fluid typography/spacing. Cons: Can lead to extremely large or small sizes on very wide or narrow/short screens if used without constraints. vh units can sometimes behave unexpectedly on mobile browsers due to dynamic browser UI elements (address bar).

Example:

.full-screen-section {
  width: 100vw;
  height: 100vh;
}

.responsive-heading {
  /* Font size is 5% of viewport width, but minimum 20px, maximum 50px */
  font-size: clamp(20px, 5vw, 50px); 
}

Viewport units are powerful tools for responsive design but require careful consideration of potential scaling issues at extreme viewport sizes.
