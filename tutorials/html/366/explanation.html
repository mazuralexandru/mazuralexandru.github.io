366. Removing Default Link Styles

Files Modified: style.css

Now let's dive into styling links with CSS. One of the first things developers often do is modify or remove the browser's default link styling (blue color, underline).

  • Removing Underline: The most common reset is to remove the underline using text-decoration: none;.
  • Changing Color: You can set a custom color for links to match your site's design. It's important to ensure links still have sufficient contrast and are visually distinguishable from regular text.
  • Re-adding Styles for Interaction: While you might remove the default underline, it's crucial to add back visual cues for :hover and :focus states (like re-adding the underline or changing the color/background) so users know the text is interactive.

✅ Observe style.css:

  • We've added a rule targeting all <a> elements.
  • It sets the color to inherit from the parent (so links match surrounding text color by default) and removes the text-decoration.
/* Section 4: Styling Links Deep Dive */

a { /* Target all anchor tags */
  color: inherit; /* Make links inherit text color from parent */
  text-decoration: none; /* Remove default underline */
}

✨ Observe the Preview:

  • Look at the links in the first paragraph and the special links (mailto, tel) in index.html.
  • They should no longer be blue but should match the default light text color set on the body.
  • They should also no longer have an underline.
  • They might now be harder to distinguish from regular text, highlighting the need for hover/focus styles, which we'll add next.
