319. display Value: none

Files Modified: style.css

The display: none; value is used to completely hide an element.

  • Behavior:
    1. Removed from Flow: The element generates no box and takes up no space in the document layout. It's as if the element doesn't exist in the HTML structure for rendering purposes.
    2. Removed from Accessibility Tree: The element and its descendants are typically removed from the accessibility tree, meaning screen readers will not announce them.
    3. Descendants Also Hidden: All child elements of an element with display: none; are also hidden.
  • Use Cases:
    • Conditionally hiding/showing elements using JavaScript (by adding/removing a class that sets display: none;).
    • Hiding elements in specific contexts (e.g., hiding certain elements in a print stylesheet).
    • Implementing dropdown menus or modals that are initially hidden.

✅ Observe style.css:

  • We've added a new rule targeting the third span specifically (using :nth-of-type(3)) and set its display to none.
/* Display Inline-Block Example */
.inline-span {
  display: inline-block; 
  background-color: rgba(255, 192, 203, 0.4); 
  border: 1px solid pink;
  padding: 10px; 
  margin: 10px; 
  width: 100px; 
  height: 50px; 
  text-align: center; 
}

/* Display None Example */
.inline-span:nth-of-type(3) { /* Target the 3rd span with this class */
  display: none; /* Hide this element completely */
}

(Note: :nth-of-type() is a pseudo-class selector we haven't formally covered, but here it selects the third element matching .inline-span within its parent).

  • The third pink box ("Span 3") should now be completely gone.
  • Crucially, the space it occupied is also gone; "Span 4" should now appear immediately after "Span 2" (or on the next line, depending on wrapping), as if "Span 3" never existed in the layout.
