163. Attribute Selectors: Case Sensitivity Modifier (i)

Files Modified: None

By default, attribute selector values (the part inside the quotes, like "value") are case-sensitive.

  • [type="text"] will match type="text" but NOT type="TEXT".

However, CSS provides a case-insensitivity modifier flag (i) that can be added before the closing square bracket ] to make the value matching case-insensitive.

  • Syntax: [attribute="value" i]
  • Example:
    /* Matches type="text", type="TEXT", type="Text", etc. */
input[type="text" i] {
  border: 1px solid green;
}

/* Matches data-status="Complete", data-status="complete", etc. */
[data-status="complete" i] {
  opacity: 0.5;
}

This modifier works with the attribute value matching selectors:

  • [attribute="value" i]
  • [attribute~="value" i]
  • [attribute|="value" i]
  • [attribute^="value" i]
  • [attribute$="value" i]
  • [attribute*="value" i]

It does not apply to the attribute name itself (which is case-insensitive in HTML anyway) or the presence selector ([attribute]).

Use Case: Useful when dealing with HTML where attribute value casing might be inconsistent (though consistent casing in HTML is always preferred).
