223. The Image Element (<img>) - Category & Basic Syntax

Let's look at how the <img> element is categorized and its basic syntax structure.

Element Category:

  • Void (Empty) Element: The <img> element is a void element. This means it represents a single, self-contained item (the image) and cannot contain any other HTML elements or text content between opening and closing tags. Therefore, it has no closing tag.
  • Embedded Content: It belongs to the category of elements that embed external resources into the document.
  • Phrasing Content: It's also considered phrasing content, meaning it can typically be placed within sentences or paragraphs, similar to inline elements like <em> or <strong>.
  • Default Layout Behavior: While technically phrasing content (inline), its display is often influenced by its dimensions (width and height attributes or CSS). By default, text may try to wrap around it, but it often behaves more like an inline-block element in practice.

Basic Syntax:

  • The basic syntax requires the tag name img and the mandatory src attribute. The alt attribute is also mandatory for validity and accessibility.
    <img src="path/to/image.jpg" alt="description">
  • Self-Closing Slash (Optional): In HTML5, the trailing slash (/) common in XHTML (<img ... />) is optional and ignored by parsers. Consistency is key; omitting it is more common in modern HTML5 (<img ... >). We will generally omit it.
