121. Nested Lists: Default Rendering (Indentation & Markers)

Let's focus on the default visual rendering behavior we observed when nesting lists.

Default Browser Rendering:

When you correctly nest a list inside an <li>:

  1. Indentation: Browsers automatically apply additional indentation (usually via padding-left) to the nested list (<ul> or <ol>). This visually pushes the sub-items further to the right than their parent item, clearly indicating the hierarchy.
  2. Marker Changes (<ul>): For nested unordered lists (<ul> inside <li> of another <ul>), browsers often cycle through default marker styles for different nesting levels. A common pattern is:
    • Level 1: disc (•)
    • Level 2: circle (○)
    • Level 3+: square (▪︎) (This default cycle helps visually distinguish nesting levels but should be controlled explicitly with CSS for consistent design).
  3. Marker Changes (<ol>): For nested ordered lists, the numbering usually restarts for each nested list by default (e.g., 1, 2, then a, b, or i, ii). We'll see this later. Advanced nested numbering (e.g., 1.1, 1.2) requires CSS Counters.

CSS Control:

Remember, these default behaviors (indentation amount, specific marker styles) can vary slightly between browsers. For consistent and precise control over the appearance of nested lists, you should always use CSS:

  • Use selectors targeting specific nesting levels (e.g., ul ul, ol li ul, ul > li > ul > li) to apply padding-left for indentation.
  • Use list-style-type applied to the appropriate nested list level selectors to control marker styles explicitly.
  • Use CSS Counters for complex nested numbering in <ol>.
