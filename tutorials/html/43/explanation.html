43. CSS Combinators: General Sibling Combinator (~)

The General Sibling Combinator (~) is similar to the Adjacent Sibling Combinator (+), but less strict. It selects sibling elements that follow a specific element, even if they don't immediately follow it.

What's New?

  • HTML Addition: Added a <div> element between the <h2> and the first <p>. Added a third <p> after the second one.
  • CSS Pruning: Removed the h2 + p adjacent sibling rule.
  • CSS Addition: Added a new rule using the general sibling combinator: h2 ~ p.
<!-- index.html snippet -->
<body>
  <h1>General Sibling Combinator Demo</h1> <!-- Changed H1 text -->

  <h2>Another Heading</h2>
  <div>This div is between H2 and P.</div> <!-- Added Div -->
  <p>This paragraph follows the H2 (after the div).</p> <!-- Target for ~ -->
  <p>This paragraph also follows the H2.</p> <!-- Target for ~ -->
  <p>This paragraph follows too.</p> <!-- Target for ~ -->


  <!-- Commented out paragraph remains -->
  <!-- <p>This paragraph is commented out and won't appear.</p> -->

  <script src="script.js"></script>
</body>

/* style.css snippet */
/* Removed adjacent sibling combinator rule */
/*
h2 + p { ... }
*/

/* General Sibling Combinator Example */
h2 ~ p {
  color: plum; /* Style P tags that follow (anywhere after) an H2 */
  text-decoration: underline;
}

/* CSS files can also have comments! */

General Sibling Combinator (~)

  • Syntax: Two selectors separated by a tilde (~): element1 ~ element2.
  • Meaning: Selects any element matching element2 that comes after an element matching element1 in the HTML source code, and both elements must share the same parent element. Unlike the + combinator, element2 does not need to be immediately after element1. It selects all subsequent siblings that match.
  • Use Cases: Styling all paragraphs that follow a heading within the same section, applying styles to elements based on a preceding sibling's state (often used with pseudo-classes like :checked).

In our example:

  • The rule h2 ~ p { ... } targets any <p> element that is a later sibling of an <h2> (sharing the same parent, <body>).
  • It applies a plum color and an underline.
