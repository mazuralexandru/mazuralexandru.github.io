437. Common <input> Types: type="tel"

For telephone number input, HTML5 provides type="tel".

type="tel"

  • Definition: Creates a single-line text input field semantically designated for entering telephone numbers.
  • Use Case: Any field requiring a phone number.
  • Behavior & Benefits:
    • Mobile Keyboard Optimization: Its primary benefit is that on most mobile devices, it triggers a telephone keypad (digits, *, #, +, etc.), making number entry much easier for users.
    • NO Format Validation: Unlike email or url, type="tel" does not perform any automatic validation on the format of the entered number. This is because phone number formats vary drastically worldwide (e.g., parentheses, hyphens, spaces, extensions).
  • Validation using pattern: To enforce a specific phone number format (if necessary for your application), you must use the pattern attribute with a suitable regular expression.
  • Key Attributes: Supports standard text attributes like placeholder, required, pattern, minlength, maxlength.

Code Change:

We've removed the URL field and added a label/input pair using type="tel". We've also included an example pattern attribute to demonstrate format validation (this specific pattern matches a simple XXX-XXX-XXXX US format).

<!-- index.html snippet -->
<form action="/submit-data" method="POST">
  <!-- URL field removed for focus -->
  <p> <!-- Added Tel Field -->
     <label for="phone">Phone Number:</label> <!-- Added Label -->
     <input type="tel" id="phone" name="phone" placeholder="123-456-7890" pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}"> <!-- Added type="tel" with pattern -->
     <!-- Pattern is optional but recommended for format validation -->
  </p>
</form>

✅ Observation:

  1. Examine the <input type="tel"> and its attributes, particularly the pattern.
  2. Look at the browser preview. It renders as a standard text input field, showing the placeholder.
  3. Keyboard (Conceptual): On a mobile device, tapping this field would likely bring up the telephone keypad.
  4. Validation (Conceptual): If required was added and a submit button present, native validation would check against the pattern. Entering "1234567890" would fail (missing hyphens), but "123-456-7890" would pass this specific pattern. Without the pattern, any text would be considered valid by the browser for type="tel".

✨ Key Takeaway: Use <input type="tel"> primarily for its mobile keyboard hint benefit. It performs no format validation itself; use the pattern attribute if you need to enforce a specific format. Always include a <label>.
