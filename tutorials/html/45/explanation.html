45. Cascade Factor: Specificity Calculation

Specificity is a weight or score that a browser calculates for each CSS selector to determine which rule prevails in the cascade when rules conflict. Higher specificity scores win.

What's New?

  • HTML Modification: Added an id="unique-para" attribute to our paragraph, alongside its existing class="special".
  • CSS Addition: Added a third conflicting rule using the ID selector #unique-para.
<!-- index.html snippet -->
<body>
  <h1>CSS Cascade Demo</h1>

  <p id="unique-para" class="special">This paragraph has conflicting styles.</p> <!-- Added ID -->

  <!-- Pruned previous sibling examples -->

  <!-- Commented out paragraph remains -->
  <!-- <p>This paragraph is commented out and won't appear.</p> -->

  <script src="script.js"></script>
</body>

/* style.css snippet */
/* Updated P rule */
p {
  font-style: italic;
  color: #ccc; /* Rule 1: Type Selector */
}

/* Existing conflicting rule */
.special {
  color: lightcoral; /* Rule 2: Class Selector */
  font-weight: bold;
}

/* Added ID selector rule */
#unique-para {
  color: cyan; /* Rule 3: ID Selector */
}

/* CSS files can also have comments! */

Calculating Specificity (Simplified)

Browsers calculate specificity using (typically) four components, often represented like (A, B, C, D):

  1. A (Inline Styles): Score 1 here if the style is applied using the style="..." attribute directly on the element. Otherwise, 0. (We have no inline styles here).
  2. B (IDs): Count the number of ID selectors (#idname) in the selector.
  3. C (Classes, Attributes, Pseudo-classes): Count the number of class selectors (.classname), attribute selectors ([attr=val]), and pseudo-classes (like :hover, :nth-child(), which we haven't covered yet).
  4. D (Types, Pseudo-elements): Count the number of type selectors (p, div) and pseudo-elements (like :before, :after, not covered yet).

Universal selector (*) and combinators ( , >, +, ~) do not add to specificity.

Comparing Our Rules:

  • p: Has one type selector. Specificity = (0, 0, 0, 1)
  • .special: Has one class selector. Specificity = (0, 0, 1, 0)
  • #unique-para: Has one ID selector. Specificity = (0, 1, 0, 0)

Determining the Winner:

Specificity values are compared component by component from left to right (A, then B, then C, then D). The first component where one selector has a higher value determines the winner.

  • Comparing #unique-para (0,1,0,0) and .special (0,0,1,0):

    • A is tied (0 vs 0).
    • B: #unique-para has 1, .special has 0. #unique-para wins! We don't need to compare C and D.

  • Comparing .special (0,0,1,0) and p (0,0,0,1):

    • A is tied (0 vs 0).
    • B is tied (0 vs 0).
    • C: .special has 1, p has 0. .special wins!

So, for the color property, the #unique-para rule has the highest specificity and will determine the final color.

Mastering specificity calculation is key to understanding why styles apply (or don't apply!) as expected. Aim for the lowest necessary specificity by favouring class selectors over IDs for general styling.
