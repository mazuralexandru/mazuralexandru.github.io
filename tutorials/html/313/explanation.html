313. Why border-box is Preferred

Files Modified: None

The preference for box-sizing: border-box; stems directly from the difficulties caused by the default content-box model, especially in responsive design and grid layouts.

Imagine creating a two-column layout where each column should be 50% wide.

  • With content-box (Default):

    • You set width: 50%; on both columns.
    • You then add padding: 15px; for internal spacing.
    • Now, each column's actual rendered width is 50% + 30px (15px left + 15px right padding).
    • The total width is 100% + 60px, which is wider than the container! The layout breaks, and the second column wraps below the first. You'd need complex calculations (calc(50% - 30px)) for the width to compensate.

  • With border-box:

    • You set box-sizing: border-box; (usually globally).
    • You set width: 50%; on both columns.
    • You add padding: 15px;.
    • The actual rendered width of each column remains exactly 50%. The browser automatically reduces the content area width to make space for the 30px of horizontal padding within that 50%.
    • The total width is 50% + 50% = 100%. The columns fit perfectly side-by-side.

border-box makes defining element dimensions and adding padding/borders independent, leading to much simpler and more predictable layout code.
