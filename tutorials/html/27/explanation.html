27. Class Selector Reusability and Multiple Classes

A key advantage of class selectors is their reusability. You can apply the same class to different elements. You can also apply multiple classes to a single element to combine styles.

What's New?

  • We added a <span> element inside the first paragraph and gave it the highlight class.
  • We added a third paragraph with two classes: highlight and important.
  • We added a new CSS rule in style.css targeting the .important class.
<!-- index.html snippet -->
<body>
  <h1>Styling with External CSS</h1>
  <p>This paragraph will be styled externally, but <span class="highlight">this part</span> is highlighted.</p> <!-- Added span.highlight -->
  <h2>This is a Sub-heading</h2>
  <p class="highlight">This paragraph has a highlight class.</p>
  <p class="highlight important">This paragraph is highlighted AND important.</p> <!-- Added P with multiple classes -->

  <!-- Commented out paragraph remains -->
  <!-- <p>This paragraph is commented out and won't appear.</p> -->

  <script src="script.js"></script>
</body>

/* style.css snippet */
.highlight {
  background-color: rgba(255, 255, 0, 0.2); /* Semi-transparent yellow background */
  border: 1px solid yellow;
  padding: 5px;
}

/* Added rule for second class */
.important {
  font-weight: bold;
  color: tomato; /* Overrides the 'p' color */
}

/* CSS files can also have comments! */

Class Reusability

  • Notice we applied class="highlight" to both a <p> element and a <span> element. The .highlight styles (background, border, padding) will apply to both, demonstrating how classes can be reused across different element types.

Multiple Classes

  • HTML: You assign multiple classes by listing them in the class attribute, separated by spaces: class="highlight important". The order doesn't matter.
  • CSS: The element will receive styles from both the .highlight rule and the .important rule.
  • Combining Selectors (Chaining): If you needed to style only elements that have both classes, you could chain the selectors together without a space: .highlight.important { ... }. This is more specific than targeting either class alone. We haven't used this chained selector here, but it's a useful technique.
