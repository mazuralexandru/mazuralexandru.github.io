690. Accessibility (A11y): Assistive Technologies (AT) Overview

To design accessible experiences, it's helpful to understand the types of Assistive Technologies (AT) that people with disabilities use to interact with the web. Well-structured, semantic HTML is the key interface between your website and these technologies.

Common Categories of Assistive Technologies:

  1. Screen Readers:

    • Function: Software that reads the content of the screen aloud (text-to-speech) or outputs it to a refreshable Braille display. Users navigate using keyboard commands.
    • Users: Primarily people who are blind, but also users with low vision or certain cognitive/learning disabilities.
    • Relies On: Semantic HTML structure (headings, landmarks, lists, tables with scope), alt text for images, accessible names for controls (<label>, ARIA), keyboard focus management.
    • Examples: JAWS (Windows), NVDA (Windows, free), VoiceOver (macOS/iOS), TalkBack (Android).

  2. Screen Magnifiers:

    • Function: Software that enlarges a portion (or all) of the screen content.
    • Users: People with low vision.
    • Relies On: Content that reflows well when zoomed, sufficient color contrast, clear visual layout. rem/em units for text sizing help. Avoid fixed-size containers that hide content when zoomed.

  3. Keyboard-Only Navigation:

    • Function: Using only the keyboard (Tab, Shift+Tab, Enter, Space, Arrow Keys) to navigate and interact with all page elements.
    • Users: People with motor impairments, screen reader users, power users, users with temporary injuries.
    • Relies On: All interactive elements being natively focusable or made focusable (tabindex="0"), a visible focus indicator, a logical focus order (matching visual order), standard key interactions for controls.

  4. Speech Recognition / Voice Control:

    • Function: Software allowing users to control the computer and dictate text using voice commands (e.g., "Click Submit button," "Scroll down").
    • Users: People with motor impairments, users performing hands-free tasks.
    • Relies On: Clear, unambiguous visible labels for controls and links (so users know what command to speak), well-defined interactive elements.

  5. Switch Devices:

    • Function: Input devices (buttons, sip-and-puff controllers, etc.) used by people with severe motor impairments, often emulating keyboard input (like Tab and Enter).
    • Relies On: Everything that keyboard-only navigation relies on (focusable elements, visible focus, logical order).

  6. Other: Braille displays, alternative keyboards, reading aids, color contrast adjustment tools, etc.

✅ Observation: No code changes. Consider how each type of AT relies on different aspects of well-built HTML and CSS (semantics, focus management, contrast, responsive layout) to function effectively.

✨ Key Takeaway: Assistive technologies like screen readers, magnifiers, and keyboard navigation depend heavily on semantic, well-structured, and keyboard-accessible HTML to provide access for users with disabilities.
