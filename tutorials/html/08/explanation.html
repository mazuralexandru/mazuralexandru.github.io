08. Adding Base CSS Styles

Now that style.css is linked, let's add some fundamental styles to it. These "base styles" will provide a consistent, pleasant visual foundation for our examples.

What's New?

We've added CSS rules to the previously empty style.css file.

/* style.css */
/* Base styles for visual consistency */
body {
  background-color: #2e2e2e; /* Dark background */
  color: #E0E0E0; /* Light default text */
  font-family: sans-serif; /* Basic readable font */
  padding: 15px; /* Breathing room */
  min-height: 150px; /* Ensure space for preview */
}

/* CSS files can also have comments! */

CSS Syntax Breakdown (First Look):

  • /* ... */ (CSS Comments): Similar to HTML comments, these are notes ignored by the browser. They start with /* and end with */.
  • Ruleset: The core unit of CSS. It consists of a selector and a declaration block.
    • body (Selector): This targets the HTML element(s) the styles should apply to. Here, it targets the <body> element.
    • { ... } (Declaration Block): Contains one or more declarations, enclosed in curly braces.
    • property: value; (Declaration): Defines a specific style.
      • background-color: The CSS property to change the background color.
      • #2e2e2e: The value for the background color (a dark gray hex code).
      • color: The property for the foreground (text) color.
      • #E0E0E0: The value for the text color (a light gray hex code).
      • font-family: Specifies the font. sans-serif asks the browser to use its default sans-serif font (like Arial, Helvetica, etc.).
      • padding: Adds space inside the element's border. 15px adds 15 pixels of padding on all sides of the body content.
      • min-height: Ensures the body takes up at least this much vertical space, helpful for small examples. 150px sets a minimum height of 150 pixels.
      • ; (Semicolon): Separates multiple declarations within a block. Crucial!
