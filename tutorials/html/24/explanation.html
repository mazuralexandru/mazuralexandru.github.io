24. Comments in HTML

We saw a comment briefly in an earlier step. Let's formally cover HTML comments.

Purpose:

Comments allow you to add notes, explanations, or reminders directly within your HTML code without affecting how the page is displayed by the browser. They are purely for humans reading the code.

Syntax:

  • Starts with <!--
  • Ends with -->
  • Anything between these markers is treated as a comment.

Characteristics:

  • Ignored by Browser: The browser completely ignores comments when rendering the page. They do not appear visually, nor do they affect the layout or functionality (unless they are commenting out actual HTML code).
  • Visible in Source Code: Importantly, comments are visible to anyone who views the page source ("View Page Source").
  • Multi-line: Comments can span multiple lines.

Common Uses:

  1. Explaining Code: Clarifying complex or non-obvious sections of HTML.
    <!-- This section handles user profile display -->
<div>...</div>
  2. Adding Reminders: Leaving notes for yourself or colleagues about future work.
    <!-- TODO: Update the image link when the new asset is ready -->
  3. Documenting Structure: Outlining different parts of the page.
    <!-- Header Section Start -->
<header>...</header>
<!-- Header Section End -->
  4. Temporarily Disabling Code ("Commenting Out"): A very common technique during development or debugging is to wrap a section of HTML in comment tags to temporarily remove it from the page without deleting the code itself.
    <!--
<div class="old-feature">
   <p>This feature is currently disabled.</p>
</div>
-->

Code Change:

We've added several examples of comments to index.html, both inside the <head> and <body>. Notice the example of commenting out a paragraph.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>My First Web Page</title>
  <!-- Link to CSS will go here -->

  <!-- This is an HTML comment. It's ignored by the browser --> <!-- Added -->
  <!-- Comments can span
       multiple lines. --> <!-- Added -->

</head>
<body>
  <h1>Hello World!</h1>
  <p>This is the body content of my first web page.</p>

  <!-- TODO: Add more content later --> <!-- Added -->

  <!-- <p>This paragraph is temporarily commented out.</p> --> <!-- Added -->
</body>
</html>

✅ Observation:

  1. Examine index.html to see the new comments.
  2. Look at the browser preview pane. It should look exactly the same as in the previous step. The comments themselves, and the paragraph commented out, have no visual effect.

⚠️ Pitfalls & Best Practices:

  • NEVER put sensitive information (passwords, private data, internal notes you wouldn't want public) in HTML comments, as they are easily viewable in the page source.
  • Avoid over-commenting obvious code. Aim for clear HTML structure first.
  • Keep comments up-to-date if the code changes.
  • HTML comments cannot be nested according to the standard (e.g., <!-- outer <!-- inner --> --> is invalid; the comment ends at the first -->).

✨ Key Takeaway: Use <!-- ... --> to add non-displayed notes to your code or to temporarily disable sections of HTML. Remember they are visible in the source code.
