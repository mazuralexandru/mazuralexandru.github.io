128. Description Term (<dt>): Fundamentals

Inside a <dl>, the first essential element is <dt>, used to define a term or name.

The Description Term Element (<dt>)

  • Definition: Represents a term, name, label, or key within a description list group.
  • Purpose: To identify the item that will be described or defined by the subsequent <dd> element(s). It's the "name" part of the name-value pair.
  • Syntax: <dt>...</dt> (Closing tag is mandatory).
  • Placement: Must be a direct child of a <dl> element (or a specific <div> wrapper within a <dl>, which we'll discuss later).
  • Content Model: Generally contains phrasing content (the term itself). It should typically not contain block-level elements like <p>.

Default Rendering:

Browsers often render <dt> elements as block-level, usually without indentation, and sometimes with a bold font weight, but this varies and should not be relied upon for styling.

Code Change:

We have added the first <dt> element, containing the term "HTML", inside our <dl> in index.html.

<dl>
   <dt>HTML</dt> <!-- Added dt -->
   <!-- Description (dd) will follow -->
</dl>

✅ Observation:

  1. Verify the presence of <dt>HTML</dt> inside the <dl> in index.html.
  2. Look at the browser preview. You should now see the text "HTML" displayed. It might appear slightly offset (due to <dl> margins) and potentially bolded, depending on your browser's defaults. It doesn't have a description yet.

✨ Key Takeaway: The <dt> element defines the term, name, or key within a description list (<dl>). It is typically followed by one or more <dd> elements containing its description or value.
