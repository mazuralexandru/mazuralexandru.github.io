563. Multimedia: Video Tracks (<track>) - Fundamentals

To make video and audio content accessible and more useful, HTML provides the <track> element for adding timed text tracks.

The <track> Element:

  • Definition: Represents a supplementary timed text track (like captions or subtitles) for its parent media element (<video> or <audio>).
  • Purpose: To provide text-based alternatives or enhancements synchronized with the media playback. Essential for accessibility.
  • Core Concept: Links an external text track file (typically in WebVTT format, .vtt) containing timed text cues to the media player. The browser then displays this text at the appropriate times during playback.

Placement & Syntax:

  • Placement: Must be a direct child of a <video> or <audio> element.
  • Order: Must appear after any <source> elements but before the fallback content.
  • Void Element: <track> is a void element; it has no closing tag.
  • Basic Syntax: <track kind="..." src="..." srclang="..." label="..." ...>

Key Attributes (Covered Next):

Attributes like kind, src, srclang, label, and default are crucial for defining the track's purpose, source, language, and behavior.
