380. Navigation Structure: Common Pattern & Accessibility

Files Modified: None

Let's examine the common HTML pattern we used for navigation and its accessibility implications:

<nav>
    <ul>
        <li><a href="#home">Home</a></li>
        <li><a href="#about">About</a></li>
        <li><a href="#services">Services</a></li>
        <li><a href="#contact">Contact</a></li>
    </ul>
</nav>
  • <nav>: Provides the landmark role (navigation), signaling this is a primary navigation block.
  • <ul>: Semantically groups the navigation items as an unordered list. This is appropriate because the order of main navigation links usually isn't strictly sequential or hierarchical in meaning (though visual order matters). Using a list helps screen readers announce the number of items ("List, 4 items").
  • <li>: Each list item contains a single navigation link.
  • <a>: The actual hyperlink element.

Accessibility Considerations:

  • Landmark: The <nav> element itself acts as a landmark, allowing users of assistive technology to quickly jump directly to the navigation.
  • List Semantics: The <ul> and <li> structure provides context about the group of links.
  • Multiple <nav> Elements: If a page has multiple <nav> elements (e.g., main site navigation, secondary sidebar navigation, breadcrumbs), it's crucial to distinguish them using aria-label or aria-labelledby.
    <nav aria-label="Main Site">
  <ul>...</ul>
</nav>

<nav aria-label="Table of Contents">
  <ul>...</ul>
</nav>
  • Current Page Indication: Visually highlighting the link corresponding to the current page is important. This is often done by adding a specific class (e.g., class="active") to the <a> or <li> element of the current page and styling that class differently in CSS. Sometimes aria-current="page" is also added for better screen reader feedback.

Using this semantic structure (nav > ul > li > a) provides a solid, accessible foundation before we even apply CSS for visual styling.
