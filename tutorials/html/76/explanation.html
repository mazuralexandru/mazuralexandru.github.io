76. Character Encoding: Interaction with HTTP Headers

Files Modified: None

It's worth briefly mentioning that the character encoding can also be specified by the web server via an HTTP header called Content-Type.

Example HTTP Header: Content-Type: text/html; charset=utf-8

  • Priority: If a Content-Type header with a charset is sent by the server, it generally takes precedence over the <meta charset="..."> tag within the HTML document.
  • Best Practice: Even if the server sends the header (which is good practice for server configuration), you should still include the <meta charset="UTF-8"> tag in your HTML.
    • It ensures correct interpretation when the page is viewed locally (via file:///) where there's no server sending headers.
    • It serves as clear documentation within the file itself.
    • It acts as a fallback if the server header is missing or incorrect.

Takeaway: Configure your server to send the Content-Type header with charset=utf-8, AND always include <meta charset="UTF-8"> early in your HTML <head>.
