277. The class Attribute - Core Mechanics (Syntax & Values)

Let's look at the syntax for applying classes and the rules for class name values.

Syntax:

  • The class attribute is added to the opening tag of an element.
  • The value is a string containing one or more class names, enclosed in quotes.
  • Multiple Classes: If an element belongs to multiple classes, their names are listed within the same attribute value, separated by space characters.
    • Example: class="alert error-message urgent"

Value Requirements & Constraints:

  • No Spaces within Names: A single class name itself cannot contain spaces. Spaces are used only as separators between different class names.
    • Invalid: class="error message" (This represents two classes: error and message).
    • Correct: class="error-message" (This is a single class name).
  • Naming Rules (CSS Identifiers): Class names must follow the rules for CSS identifiers. This is generally similar to the best practices for id values, but the practical conventions are often more varied:
    • They cannot typically start with a digit, two hyphens, or a hyphen followed by a digit (though escaping in CSS can sometimes work around this, it's best avoided).
    • Characters allowed include letters, numbers, hyphens (-), and underscores (_).
    • Special characters generally require escaping in CSS selectors.
    • Recommendation: Stick to letters, numbers, hyphens, and underscores. Start with a letter.
  • Naming Conventions (Common Practices):
    • Lowercase, Hyphen-Separated: Very common and easy to read (e.g., product-card, is-active).
    • Underscore-Separated: Also used (e.g., user_avatar).
    • BEM (Block, Element, Modifier): A popular methodology using double underscores (__) to separate blocks from elements and double hyphens (--) for modifiers (e.g., class="card__title card__title--highlighted").
    • CamelCase: Sometimes seen, especially when generated by JavaScript frameworks (e.g., isActive). Less common in manually written CSS/HTML.
  • Reusability: The core idea is that the same class name can be applied to many different elements across the page or site.
  • Case Sensitivity: Class names are case-sensitive in HTML, CSS, and JavaScript. class="Error" and class="error" refer to different classes. By strong convention, class names are almost always written in lowercase.

Code Change:

We've added class attributes to the paragraphs in index.html.

<!-- index.html -->
<body>
  <h1>Global Attributes - Class</h1>
  <p class="intro-paragraph highlight">Now let's explore the 'class' attribute.</p> <!-- Added class attribute -->

  <p class="info-text">Classes are used to group elements for styling and scripting.</p> <!-- Added class attribute -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>
  • The first paragraph now belongs to two classes: intro-paragraph and highlight.
  • The second paragraph belongs to the info-text class.

✅ Observation:

  1. Locate the class attributes added to the <p> tags. Note the space separating the two class names in the first paragraph.
  2. Look at the browser preview. There is no visual change yet. Adding classes only provides hooks; it doesn't apply any styling until we define CSS rules for those classes.

✨ Key Takeaway: Use the class attribute with space-separated names (class="name1 name2") to classify elements. Class names are case-sensitive (use lowercase convention) and cannot contain spaces within a single name. Adding a class doesn't change appearance until corresponding CSS rules are defined.
