397. Reset: Preparing for rowspan

We have successfully demonstrated using colspan to make a header cell span multiple columns.

Next, we will explore the complementary attribute, rowspan, which makes cells span vertically across multiple rows.

To clearly illustrate rowspan, we will reset our table again to a simpler structure, removing the multi-level header and colspan introduced in the previous step. We'll revert to a basic 2x3 table (two rows of data, three columns including a row header).

Why Reset?

  • Focus: Isolate the rowspan attribute and its effect on vertical cell layout.
  • Clarity: A simple grid makes it easier to visualize how rows are merged vertically.
  • Remove Complexity: Eliminate the colspan and multi-row header which aren't needed for the basic rowspan demonstration.

Code Change:

  1. Simplified the <thead> back to a single row with three simple column headers (<th> scope="col").
  2. Adjusted the <tbody> rows to have a row header (<th> scope="row") followed by two data cells (<td>) each.
  3. Updated the paragraph text.
