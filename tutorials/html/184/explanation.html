184. Specificity: Examples

Files Modified: style.css

Let's illustrate specificity conflicts. We'll add a new rule targeting the second paragraph using just a type selector and give it a conflicting color.

✅ Observe style.css:

  • We added a new rule p { color: hotpink; } at the end of the file.
/* Specificity Example */
p {
  color: hotpink; /* Low specificity (0,0,0,1) */
}

Analysis:

  • The second paragraph (<p>This paragraph follows the heading.</p>) is matched by:

    • h1 + p (Specificity 0,0,0,2) - Sets color: lightgreen;
    • h1 ~ p (Specificity 0,0,0,2) - Doesn't set color.
    • .container > p (Specificity 0,0,1,1) - Doesn't set color.
    • p (Specificity 0,0,0,1) - Sets color: hotpink;

  • Comparing the selectors that set color:

    • h1 + p has specificity (0,0,0,2).
    • p has specificity (0,0,0,1).

  • Since (0,0,0,2) is more specific than (0,0,0,1), the rule from h1 + p wins.

✨ Observe the Preview: The second paragraph should remain light green, not hot pink. Even though the p { color: hotpink; } rule appears later in the source code, the h1 + p { color: lightgreen; } rule wins because its selector (h1 + p) is more specific.

Now, let's try overriding with the class selector.

✅ Modify style.css again:

  • Add color: yellow; to the .container > p rule.
/* Child Combinator Example */
.container > p {
  border-left: 3px solid orange; 
  padding-left: 10px;
  color: yellow; /* Added color (Specificity 0,0,1,1) */
}

Analysis:

  • Now we compare:

    • h1 + p (Specificity 0,0,0,2) - Sets color: lightgreen;
    • .container > p (Specificity 0,0,1,1) - Sets color: yellow;
    • p (Specificity 0,0,0,1) - Sets color: hotpink;

  • (0,0,1,1) is more specific than (0,0,0,2) and (0,0,0,1).

✨ Observe the Preview: The second paragraph (and the first one, since it's also matched by .container > p) should now be yellow. The rule with the class selector (.container > p) wins because it has the highest specificity score among the rules setting the color property for that element.
