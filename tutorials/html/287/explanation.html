287. Longhand Border Properties: border-style

Files Modified: style.css

The border-style property is required to make a border visible. It defines the pattern of the line.

  • Purpose: Sets the style of the border line(s).
  • Syntax: border-style: value; (shorthand for all four sides) or border-top-style: value;, border-right-style: value;, etc.
  • Values:
    • none: (Default) No border is displayed, regardless of width or color.
    • hidden: Same as none, except for border conflict resolution in tables.
    • solid: A single, solid line. (Most common)
    • dashed: A series of short dashes.
    • dotted: A series of dots.
    • double: Two parallel solid lines. The border-width specifies the total width including the space between the lines.
    • groove: A 3D grooved effect (depends on border-color).
    • ridge: A 3D ridged effect (opposite of groove).
    • inset: A 3D inset effect. Makes the box look embedded.
    • outset: A 3D outset effect (opposite of inset). Makes the box look embossed.
  • Shorthand Application: Like padding, the border-style shorthand can accept 1, 2, 3, or 4 values to set top, right, bottom, and left styles respectively.

✅ Observe style.css:

  • We've added border-style: dashed; to the .box rule.
.box {
  width: auto; 
  height: auto; 
  min-width: 200px; 
  max-width: 400px; 
  min-height: 150px; 
  max-height: 200px; 
  overflow: auto; 
  background-color: cornflowerblue; 
  padding: 20px 5px 15px 30px; 
  margin: 1em auto; 
  border-width: 3px; 
  border-style: dashed; /* Set border style */
  /* Still need border-color (will default to text color) */
}

✨ Observe the Preview: A border should now be visible around the blue box! It should be 3px thick (from border-width) and composed of dashes (from border-style). The color will likely be the default text color (light gray/white from our base_styles.css) because we haven't explicitly set border-color yet.
