458. Forms: Reset Buttons (type="reset") - <input> vs. <button>

The type="reset" attribute creates a button that, when clicked, reverts all interactive controls within the same form to their initial default states.

Purpose:

  • To allow users to quickly clear any changes they've made to a form and restore its original values (as defined by value attributes, checked attributes, or default selections).
  • It does not trigger form submission.

Syntax Options:

  1. <input type="reset">:

    • Syntax: <input type="reset" value="Button Label">
    • Label: Defined by the value attribute (defaults to "Reset").
    • Content: Void element.

  2. <button type="reset">:

    • Syntax: <button type="reset">Button Label Content</button>
    • Label: Defined by the content between the tags.
    • Flexibility: Allows richer content (images, formatting) in the label. This is generally preferred for consistency with other button types.

Behavior:

  • Clicking a reset button targets all interactive controls (<input>, <textarea>, <select>) within the same parent <form>.
  • It resets their current values back to the state they were in when the page initially loaded:
    • Text inputs revert to their HTML value attribute (or become empty if no value was set).
    • Checkboxes/Radio buttons revert to their default checked state.
    • Select boxes revert to their default selected option.
    • Textareas revert to their initial content (between the tags).

Usability Warning (Critical):

  • Use reset buttons with extreme caution, or avoid them altogether.
  • Users frequently click them accidentally, especially when placed near the submit button.
  • Accidentally resetting a long or complex form causes significant frustration and data loss for the user.
  • Modern UX practice generally avoids reset buttons. If users want to clear fields, they can do so manually or navigate away. Providing clear "Cancel" links or actions is usually a better approach than a potentially destructive "Reset".

Code Change:

We've replaced the submit buttons from the previous example with reset buttons of both types (<input> and <button>) in index.html. We also added an initial value to the text input.

<!-- index.html snippet -->
<form action="/submit-data" method="POST">
  <p>
    <label for="data">Data:</label>
    <input type="text" id="data" name="data_field" value="Initial Value"> <!-- Added value -->
  </p>

  <!-- Method 1: Input Reset -->
  <p>
     <input type="reset" value="Reset via Input"> <!-- Input reset -->
  </p>

  <!-- Method 2: Button Reset -->
   <p>
     <button type="reset">Reset via Button</button> <!-- Button reset -->
  </p>
</form>

✅ Observation:

  1. Examine the <input type="reset"> and <button type="reset">.
  2. In the preview, you see buttons labeled "Reset via Input" and "Reset via Button".
  3. Test:
    • Type something different into the "Data" field (e.g., "New Text").
    • Click either of the reset buttons.
    • The text field should revert back to displaying "Initial Value".

✨ Key Takeaway: <input type="reset"> and <button type="reset"> create buttons that reset form fields to their initial values. <button> is preferred for label flexibility. Due to high risk of accidental data loss, reset buttons are generally discouraged in modern web forms.
