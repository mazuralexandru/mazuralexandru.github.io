414. Forms: The action Attribute - Specifying the Destination

Now let's add the first crucial attribute to our <form> tag: the action attribute.

Definition & Purpose Recap:

  • The action attribute specifies the URL where the collected form data should be sent for processing when the form is submitted.
  • It defines the destination endpoint – typically a server-side script.

Core Concept:

  • When the user submits the form, the browser packages the data and sends an HTTP request to the URL provided in the action attribute.

Syntax:

  • <form action="url"> ... </form>
  • The url value can be absolute or relative.

Code Change:

We've added an action attribute to the <form> tag in index.html, using a hypothetical relative URL /submit-form-data.

<!-- index.html snippet -->
<form action="/submit-form-data"> <!-- Added action attribute -->
  <!-- Placeholder URL -->
  <!-- Form controls will go here -->
</form>

✅ Observation:

  1. Locate the action="/submit-form-data" attribute added to the opening <form> tag in index.html.
  2. Look at the browser preview. There is still no visual change. The action attribute defines behavior (where data goes on submission), not appearance.
  3. If we were to add a submit button and click it (without any input fields yet), the browser would attempt to navigate to (or send data to) /submit-form-data relative to the current location (likely resulting in a "Not Found" error in our current setup).

✨ Key Takeaway: The action attribute is essential for specifying the server-side URL that will receive and process the submitted form data.
