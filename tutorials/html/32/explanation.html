32. Text: Paragraphs (<p>) - Mechanics

Let's delve a bit deeper into the mechanics of the <p> element.

Core Mechanics:

  1. Syntax: As we've seen, it uses an opening <p> tag and a closing </p> tag, wrapping the paragraph content.
  2. Block-Level: Being a block-level element means it inherently creates structure by starting on a new line and occupying available horizontal space, pushing subsequent content downwards.
  3. Content Model: What can go inside a <p>?
    • Allowed: Phrasing content. This includes plain text and most inline elements like <em>, <strong>, <a> (links), <img> (images), <span>, <code>, etc. We'll cover many of these soon.
    • Not Allowed: Generally, you cannot nest other block-level elements directly inside a <p>. This includes elements like <div>, <h1>-<h6>, other <p> tags, lists (<ul>, <ol>), etc. Putting a block element inside a <p> tag will implicitly close the first paragraph before the block element starts, which can lead to unexpected structure and validation errors.
  4. Whitespace Handling: Like most HTML elements (except <pre>), browsers collapse whitespace within a <p> element. This means:
    • Multiple consecutive spaces or tabs in your source code are rendered as a single space.
    • Line breaks within the source code inside a <p> tag are treated as single spaces (they don't create visual line breaks in the output, unless you use a <br> tag).

Example (Illustrative - Not Code Change):

If your HTML source code looked like this:

<p>
  This   paragraph   has
  lots of       extra


  whitespace in the source.
</p>

The browser would likely render it like this (whitespace collapsed):

This paragraph has lots of extra whitespace in the source.

✅ Observation: No code changes in this step. Review the index.html file and the preview. Observe how the text within each paragraph flows naturally, and the paragraphs themselves form distinct blocks. The whitespace rules explain why simple line breaks in your source code editor within a <p> don't show up in the browser preview.

✨ Key Takeaway: <p> elements are block-level containers for inline content and text. They enforce structure and have specific rules about what can be nested inside them and how whitespace is handled.
