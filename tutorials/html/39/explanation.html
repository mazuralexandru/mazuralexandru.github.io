39. Syntax Variations for Void Elements

Files Modified: None

You might sometimes see void elements written with a trailing slash, like <br /> or <hr />. This comes from XHTML, an older, stricter version of HTML based on XML rules.

  • XHTML: Required all elements, including void ones, to be explicitly closed. The trailing slash (/) was the syntax for self-closing tags.
  • HTML5: Is more lenient. The trailing slash on void elements is optional and has no functional effect. <br> and <br /> are treated identically by modern browsers interpreting HTML5.

Best Practice / Convention:

  • For HTML5, the common convention is to omit the trailing slash for void elements (e.g., use <br>, <img>, <input>).
  • Consistency is key. Choose one style (with or without the slash) and stick to it within your project. Omitting it is generally preferred in modern HTML5 code.

We will follow the convention of omitting the trailing slash for void elements.
