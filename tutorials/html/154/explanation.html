154. HTML id Attribute Revisited

Files Modified: None

Let's refresh our memory on the HTML id attribute:

  • Purpose: The id attribute assigns a unique identifier to an HTML element. This ID can be used by CSS to apply specific styles or by JavaScript to select and manipulate that particular element. It's also used to create fragment identifiers (page anchors/bookmarks) in URLs (e.g., page.html#section-id).
  • Syntax: id="unique-name"
    <div id="main-content">...</div>
<h2 id="error-summary">Error Summary</h2>
<form id="contact-form">...</form>
  • Uniqueness Constraint: This is the most critical rule. The value assigned to an id attribute MUST be unique across the entire HTML document. You cannot have two elements with id="main-content". Browsers might render the page even with duplicate IDs, but CSS and JavaScript behavior becomes unpredictable and invalid.
  • Naming Conventions:
    • Similar to classes: Use meaningful names.
    • Can contain letters, numbers, hyphens, underscores.
    • Must contain at least one character.
    • Cannot contain spaces.
    • Should generally start with a letter for broadest compatibility, although HTML5 is more lenient than older versions.
    • Lowercase with hyphens (kebab-case) is a common convention.

Because IDs must be unique, they are used for targeting elements that appear only once per page, like a main navigation bar, a specific section header, or a form.
