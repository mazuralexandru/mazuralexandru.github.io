379. Semantic HTML Structure for Navigation (<nav>)

Files Modified: index.html, style.css

Now let's combine our knowledge of lists and links to build a common UI pattern: a navigation bar.

First, it's crucial to use semantic HTML to structure the navigation. The <nav> element is specifically designed for this.

  • Purpose: The <nav> element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents. Common examples include main site navigation, menus, tables of contents, or indexes.
  • Semantics & Accessibility:
    • Using <nav> clearly identifies major navigation blocks to browsers, assistive technologies (like screen readers, which announce it as a "navigation landmark"), and search engines.
    • Not all groups of links need to be in a <nav> element ‚Äì a simple list of links in a page footer, for example, doesn't necessarily warrant a <nav>. Use it for primary navigation sections.
  • Display: By default, <nav> is a block-level element.
