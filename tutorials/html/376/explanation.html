376. Basic Structure: Table Data Cells (<td>) - Fundamentals

Now that we have a row (<tr>), we need cells within that row to hold the actual data. The element for standard data cells is <td>.

The Table Data Cell Element (<td>)

  • Definition: Represents a standard data cell within an HTML table row.
  • Purpose: To contain one piece of data belonging to a specific row and column intersection in the table. It holds the primary content values.
  • Core Concept: Each <td> creates one cell in the current row (<tr>). The sequence of <td> elements within a <tr> defines the data for the columns in that row.

Placement & Structure:

  • A <td> element must be a direct child of a <tr> element.
  • Content Model: Accepts flow content. This means a <td> is very flexible and can contain plain text, paragraphs, images, links, lists, inline elements, etc. – essentially any content you'd normally put in the <body>.

Code Change:

We've added a single <td> element, containing simple text, inside our existing <tr>.

<!-- index.html snippet -->
  <table>
    <tr>
      <td>Cell 1, Row 1</td> <!-- Added first data cell -->
      <!-- More cells go here -->
    </tr>
  </table>

✅ Observation:

  1. Note the <td>Cell 1, Row 1</td> nested inside the <tr> in index.html.
  2. Look at the browser preview. You should now see the text "Cell 1, Row 1" displayed on the page. It might have some default padding around it. This is the content of our first table cell. The table structure itself (borders, etc.) is still invisible by default.

✨ Key Takeaway: The <td> element defines a standard data cell within a table row (<tr>). It holds the actual content for that specific cell and can contain rich flow content.
