248. Quotations (<blockquote>, <q>, <cite>)

Files Modified: None

HTML provides elements for marking up quotations, distinguishing between block-level quotes and inline quotes, and citing sources.

  • <blockquote> (Block Quotation):
    • Semantic Meaning: Represents content quoted from another source, typically displayed as a distinct block, often indented.
    • Usage: For longer quotations that form their own paragraph(s).
    • cite Attribute: Can optionally include a cite attribute whose value is a URL pointing to the source of the quotation. This attribute is not displayed visually but provides metadata.
  • <q> (Inline Quotation):
    • Semantic Meaning: Represents a short quotation inline within surrounding text.
    • Default Styling: Browsers typically add appropriate quotation marks (e.g., ‚Äú ‚Äù in English) around the content automatically, based on the document's language (lang attribute) and nesting level.
    • cite Attribute: Can also have a cite attribute pointing to the source URL.
  • <cite> (Citation / Work Title):
    • Semantic Meaning: Represents the title of a creative work (book, article, movie, song, etc.). It does not represent the name of a person.
    • Default Styling: Typically rendered in italics.
    • Usage: Often used within or alongside <blockquote> or <q> to cite the source work, or on its own when mentioning a title.

Using these elements correctly adds semantic value, indicating to browsers and assistive technologies that the content is a quotation or a work title.
