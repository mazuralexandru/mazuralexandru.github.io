526. Other Semantics: <data> - Fundamentals

The <data> element provides a way to associate a machine-readable value with human-readable content.

The <data> Element:

  • Definition: Links a given piece of content with a machine-readable translation or equivalent.
  • Purpose: To embed data that is primarily intended for processing by scripts or other software agents, while still presenting a user-friendly version to humans.

Core Mechanics & Syntax:

  • Syntax: <data value="machine-readable-value">Human-readable content</data>
  • Requires opening <data> and closing </data> tags. Inline element.
  • Content: Phrasing content (the human-readable version).
  • value Attribute [Required]: This attribute must be present and holds the machine-readable version of the content.

Distinction from <time>:

  • <time> is specifically for dates, times, and durations, using standardized formats in its datetime attribute.
  • <data> is for other types of machine-readable values where a standardized format might not exist or isn't time-related.

Use Cases:

  • Product IDs: Associating a user-friendly product name with its internal database ID. <data value="sku-12345">Awesome Gadget</data>
  • Numerical Values: Providing a standard numerical format alongside a formatted version. Sales: <data value="10523.50">10,523.50</data> USD
  • Scientific Data: Linking common names to taxonomic IDs.
  • Game Scores/User IDs: Linking display names or scores to internal identifiers.

Interaction:

  • Browser Rendering: Browsers render only the content between the <data> tags; the value attribute is not displayed visually by default.
  • JavaScript/Tools: The primary consumer is usually JavaScript, which can easily select the element and access its machine-readable value via the element.value property (or element.getAttribute('value')). This avoids having to parse the human-readable content string.

✅ Observation:

  1. Locate the <data> elements. Note the value attribute containing the machine-readable form and the human-readable text between the tags.
  2. Look at the browser preview. You only see the human-readable content ("Super Widget", "1,024"). The value attribute's content ("P12345", "1024") is hidden but available programmatically. The <data> tag adds no default visual styling.

✨ Key Takeaway: Use the <data value="...">...</data> element to associate human-readable content with its machine-readable equivalent (like IDs or standardized numerical values). This is primarily useful for consumption by JavaScript or other automated tools.
