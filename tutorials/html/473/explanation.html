473. Forms: Hidden Inputs (<input type="hidden">) - Fundamentals

The <input type="hidden"> element allows you to include data with a form submission that is not visible to the user on the page.

type="hidden"

  • Definition: Creates an input control that is completely invisible on the rendered page and cannot be interacted with directly by the user.
  • Purpose: To send predefined or dynamically generated data along with the user-entered data when the form is submitted.

Core Mechanics & Syntax:

  • Syntax: <input type="hidden" name="data_name" value="data_value">
  • Void Element: No closing tag.
  • name Attribute [Required]: Identifies the piece of hidden data being sent.
  • value Attribute [Required]: Contains the actual data value to be submitted. This value is usually set directly in the HTML or dynamically populated using server-side templating or client-side JavaScript.
  • Behavior: Although invisible, its name=value pair is included in the form submission just like any other named input. It requires no <label> as it's not user-interactive.

Use Cases:

  • Tracking/Session IDs: Sending user session identifiers or marketing campaign tracking codes.
  • Security Tokens: Submitting anti-CSRF (Cross-Site Request Forgery) tokens to validate the form submission origin.
  • State Management: Passing specific state information required by the server (e.g., the ID of the item being edited, a specific action code).
  • JavaScript Data: Storing data calculated or retrieved by JavaScript that needs to be submitted with the form.

Security Note: "Hidden" only means visually hidden. The name and value are present in the HTML source code and can be easily viewed and potentially modified by users using browser developer tools. Never use hidden inputs to store sensitive information that should not be exposed client-side.

Code Change:

We've added two <input type="hidden"> elements to our form in index.html, representing common use cases (a product ID and a security token). We also added some mock product info and a submit button for context.

<!-- index.html snippet -->
<form action="/process-item" method="POST">
   <!-- User might see product details here -->
   <p>Product: Super Widget</p>
   <p>Price: $19.99</p>

   <!-- Hidden input sending the product ID -->
   <input type="hidden" name="product_id" value="SW9001"> <!-- Added hidden input -->

   <!-- Hidden input potentially for security -->
   <input type="hidden" name="csrf_token" value="a1b2c3d4e5f67890"> <!-- Added hidden input -->

   <p><button type="submit">Buy Now</button></p> <!-- Added submit -->
</form>

✅ Observation:

  1. Locate the two <input type="hidden"> elements in the HTML source. Note their name and value attributes.
  2. Look carefully at the browser preview. You see the product info and the "Buy Now" button, but the hidden inputs are completely invisible.
  3. (Conceptual): If you clicked "Buy Now", the submitted POST data would include product_id=SW9001 and csrf_token=a1b2c3d4e5f67890, even though the user didn't type them in.

✨ Key Takeaway: Use <input type="hidden"> with name and value attributes to include non-visible data in form submissions (e.g., IDs, tokens, state). Remember this data is visible in the source code and not suitable for sensitive secrets.
