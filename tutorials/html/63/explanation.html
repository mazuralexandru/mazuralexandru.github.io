63. Specialized Text: <pre> and HTML Escaping

A critical consideration when using <pre>, especially for displaying code examples, is HTML escaping.

The Problem:

The browser parses all the content in your HTML file, including the text inside <pre> tags. If you put raw HTML code (like <p> or <div>) directly inside a <pre> block, the browser will interpret those tags as actual HTML elements, potentially breaking the structure of your page or rendering the code incorrectly, instead of displaying the tag characters literally.

The Solution: HTML Entities (Escaping)

To display characters that have special meaning in HTML (like <, >, and &) as literal text, you need to replace them with their corresponding HTML character entities:

  • < becomes &lt; (less-than)
  • > becomes &gt; (greater-than)
  • & becomes &amp; (ampersand)

(Other entities exist, like &quot; for " and &apos; for ', but < > & are the most critical ones to escape when displaying code).

Why it Works:

When the browser encounters an entity like &lt; during parsing, it renders it as the literal < character on the page but does not interpret it as the start of an HTML tag.

Code Change:

We've replaced the previous content of the <pre> block in index.html with an example showing how to display HTML code literally by using escaped entities.

<p>Displaying HTML code within a pre block requires escaping:</p>

<pre>
<!-- How to represent <p> tags literally: -->

&lt;p&gt;This is a paragraph.&lt;/p&gt;
&lt;div class="example"&gt;
  Some content &amp; more text.
&lt;/div&gt;
</pre> <!-- Added escaped HTML example -->

<p>This paragraph follows the preformatted text.</p>
