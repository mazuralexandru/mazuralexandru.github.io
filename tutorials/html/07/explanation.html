07. Linking the CSS File

Our HTML structure exists, but it uses default browser styles. To apply our own custom styles (colors, fonts, layout), we need to link our style.css file to the index.html document.

What's New?

We added a <link> tag inside the <head> section.

<!-- index.html snippet -->
<head>
  <meta charset="UTF-8">
  <title>My First Web Page</title>
  <link rel="stylesheet" href="style.css"> <!-- Added link to CSS file -->
</head>
<body>

The <link> Element

  • Purpose: Defines a relationship between the current HTML document and an external resource. Most commonly used to link to external stylesheets.
  • Placement: Typically placed within the <head> section.
  • Void Element: Like <meta>, <link> is a void element and doesn't need a closing tag.
  • Key Attributes:
    • rel="stylesheet": This attribute specifies the relationship of the linked file to the HTML document. stylesheet indicates that the linked file is a Cascading Style Sheet (CSS). This is crucial for the browser to understand how to use the file.
    • href="style.css": This attribute specifies the Hypertext Reference, or the path/URL to the external resource. Here, style.css tells the browser to look for a file named style.css in the same directory as the index.html file.

Why External CSS?

While we'll see other ways to apply CSS later, linking an external stylesheet is the most common and recommended method because:

  • Separation of Concerns: Keeps structure (HTML) and presentation (CSS) separate, making code cleaner and easier to manage.
  • Maintainability: Changes to styles only need to be made in one place (the .css file) to affect multiple HTML pages that link to it.
  • Caching: Browsers can download the .css file once and reuse it for multiple pages, improving loading speed.
