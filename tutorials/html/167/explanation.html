167. Linking to Fragments: Behavior & id Requirements Review

Let's review the behavior of fragment links and the rules for id attributes.

Fragment Link Behavior:

  • Scrolling: When a fragment link is activated, the browser scrolls the viewport to bring the target element (the one with the matching id) into view. The exact alignment (top, middle) can sometimes vary slightly between browsers or be influenced by CSS (scroll-margin-top).
  • URL Update: The browser's address bar typically updates to include the #fragment identifier. This allows users to bookmark or share links that point directly to specific sections.
  • No Page Reload (Same Page): Clicking a fragment link pointing to an id on the same page does not cause the page to reload. It's purely an intra-page navigation.
  • CSS :target Pseudo-class: You can use CSS to style the element that is currently targeted by the URL fragment. For example:
    /* Style the currently targeted element */
:target {
  background-color: yellow; /* Just an example */
  /* Be mindful of accessibility with color changes */
}
    If the URL is page.html#section1, the element with id="section1" would match :target.

id Attribute Requirements:

  • Uniqueness: MUST be unique within the entire HTML document. Duplicating ids leads to invalid HTML and unpredictable behavior (the browser usually only targets the first element it finds with that id). This is a very common error source.
  • Naming Conventions (HTML5):
    • Must contain at least one character.
    • Cannot contain whitespace characters (spaces, tabs, etc.).
    • Can contain letters, numbers, -, _, ., :.
    • Best Practice: Stick to letters, numbers, hyphens (-), and underscores (_). Start with a letter for maximum compatibility with CSS selectors and JavaScript. Use hyphens for separating words (e.g., section-contact-info). Make them descriptive but reasonably concise. id values are case-sensitive.
