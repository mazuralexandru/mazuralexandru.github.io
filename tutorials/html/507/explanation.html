507. Landmarks: <header> Element - Fundamentals

Let's start building our semantic page structure with the <header> element.

Definition & Purpose:

  • The <header> element represents introductory content for its nearest sectioning ancestor (or the page itself if it's a direct child of <body>).
  • It typically contains elements like:
    • The main site title or logo (<h1>-<h6>, <img>).
    • Primary site navigation (<nav>).
    • A search form.
    • Introductory paragraphs or taglines.
    • Author information (for an <article> header).

Scope & Usage:

  • Page Header: When used as a direct child of <body>, it usually defines the main header for the entire website, consistent across pages.
  • Section Header: Can also be used inside <article>, <section>, <aside>, or <nav> to provide introductory content specifically for that section.
  • Multiple Headers: A document can have multiple <header> elements (e.g., one for the page, one for each article).

Core Mechanics:

  • Block-level element.
  • Syntax: <header> ... </header>
  • Content: Flow content, but cannot directly nest another <header> or <footer>.

Accessibility:

  • When used as a top-level header (direct child of <body>, or sometimes sectioning elements depending on context), it maps to the ARIA banner landmark role. This allows screen reader users to quickly jump to the site header area.

Code Change:

We've added a top-level <header> element directly inside the <body> of index.html, containing a placeholder <h1> and <p>. We've also added temporary CSS to visualize landmark boundaries.

<!-- index.html -->
<head>
  ...
  <style>
    /* Basic landmark visualization */
    body > header, body > nav, body > main, body > footer {
      border: 1px dashed skyblue;
      padding: 10px;
      margin: 10px 0;
    }
  </style>
</head>
<body>
  <!-- Added header -->
  <header>
    <h1>Site Title / Logo Area</h1>
    <p>Introductory content or tagline.</p>
  </header>

  <p>The &lt;header&gt; element contains introductory content.</p>
  ...
</body>

✅ Observation:

  1. Locate the new <header> element and its content in index.html. Note the added temporary CSS targeting direct children of <body>.
  2. In the browser preview, you should see the <h1> and <p> content enclosed within a dashed sky blue border. This border visually represents the <header> landmark area.

✨ Key Takeaway: Use the <header> element to semantically group introductory content (logos, titles, navigation, search) for the page or specific sections like <article> or <section>. Top-level headers provide an important banner landmark for accessibility.
