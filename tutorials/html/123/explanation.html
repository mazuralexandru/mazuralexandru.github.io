123. Nested Lists: Nesting <ul> inside <ol>

Let's look at the reverse scenario: nesting an unordered list (<ul>) inside an ordered list item (<ol><li>).

Scenario: Unordered Details within Ordered Steps

Imagine outlining steps (<ol>), where one step involves several non-sequential details or options (<ul>).

Code Change:

We've changed the outer list in index.html to be an <ol>. The nested list inside the second <li> is now a <ul>.

<h1>Nested Lists</h1>
<p>Nesting different list types (ul inside ol):</p> <!-- Adjusted P -->

<ol> <!-- Outer ol -->
  <li>Step One</li>
  <li>Step Two <!-- Outer li STARTS -->
    <ul> <!-- Nested ul START (inside li) -->
       <li>Detail A (bullet)</li>
       <li>Detail B (bullet)</li>
    </ul> <!-- Nested ul END (inside li) -->
  </li> <!-- Outer li END -->
  <li>Step Three</li>
</ol>

✅ Observation:

  1. Examine the structure in index.html: <ol> containing an <li> which contains a <ul>.
  2. Look at the browser preview:
    • "Step One" and "Step Three" have the default <ol> numerical markers (1., 3.). Note how the numbering continues correctly around the nested list.
    • "Step Two" also has its numerical marker (2.).
    • The sub-items "Detail A (bullet)" and "Detail B (bullet)" appear below "Step Two", indented further.
    • The sub-items have bullet point markers (likely ○, the second-level <ul> default) because they are inside a <ul>. The parent <ol> does not affect the marker type of the nested <ul>.

✨ Key Takeaway: Nesting <ul> inside <ol><li> allows you to include unordered sub-items within an ordered sequence. The nested list retains its characteristic markers (bullets), while the parent list continues its numbering appropriately.
