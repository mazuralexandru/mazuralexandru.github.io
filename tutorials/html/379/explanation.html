379. Reset: Preparing for Table Headers (<th>)

We have successfully created a basic 2x2 table structure using <table>, <tr>, and <td>. However, this table lacks proper headers to explain what the data in the columns (and potentially rows) represents.

To introduce table header cells (<th>) and their importance for semantics and accessibility, we will slightly reset our table structure. We'll keep the <table> container but clear the existing rows to rebuild it with headers.

Why Reset?

  • Focus: To clearly demonstrate adding a header row (<tr> containing <th> elements) first, before adding data rows (<td>).
  • Clarity: Starting with headers makes the semantic distinction between <th> and <td> more obvious.

Code Change:

  1. Removed the two <tr> elements (and their contained <td>s) from inside the <table> in index.html.
  2. Updated the paragraph text to introduce headers.
<!-- index.html snippet -->
<body>
  <h1>Structuring Tabular Data</h1>
  <p>Preparing to add table header cells (&lt;th&gt;).</p> <!-- Adjusted P -->

  <table>
    <!-- Removed previous tr/td content -->
    <!-- Table rows will be added here, starting with headers -->
  </table>

  <p>Headers provide labels for columns or rows.</p> <!-- Adjusted P -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

✅ Observation:

  1. Verify that the <table> element in index.html is now empty again.
  2. The browser preview should revert to showing no visible table content, just the surrounding text.

✨ Key Takeaway: Clearing the basic table data allows us to focus on adding semantic header cells (<th>) next, which are crucial for understanding table structure.
