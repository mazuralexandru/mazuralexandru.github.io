393. <figure> & <figcaption> Review

Files Modified: None

Sometimes, an image (or other content like a code block, diagram, or quote) is self-contained and requires a caption directly associated with it. For this purpose, HTML provides the <figure> and <figcaption> elements.

  • <figure>:
    • Semantic Meaning: Represents self-contained content, often with a caption, that is referenced from the main text and could potentially be moved to a different part of the page (like an appendix) without affecting the main flow. It's not just for images, but commonly used for them.
    • Display: Block-level element. Browsers might add default margins.
  • <figcaption>:
    • Semantic Meaning: Represents the caption or legend for the content of its parent <figure> element.
    • Placement: Must be placed as the first or last child directly inside the <figure> element. A figure can only contain one <figcaption>.
    • Display: Block-level element.

Structure:

<figure>
  <img src="photo.jpg" alt="Description of photo." width="400" height="300">
  <figcaption>Fig. 1: A descriptive caption for the photo.</figcaption>
</figure>

<figure>
  <figcaption>Listing 1: Example Code Block</figcaption>
  <pre><code>
  // Code goes here
  </code></pre>
</figure>

Using <figure> and <figcaption> creates a strong semantic association between the content and its caption, which is beneficial for accessibility and content understanding.
