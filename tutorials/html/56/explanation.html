Common font-size Units

  • Absolute Units:
    • px (Pixels): Defines the size in fixed screen pixels. It's precise but not recommended as the primary unit for text size. Why? Because it doesn't scale if the user changes their browser's default font size setting for accessibility reasons. Users with visual impairments might set a larger default size, but px-defined text won't respect that setting.
  • Relative Units (Preferred for Text):
    • rem (Root EM): Relative to the font-size of the root (<html>) element. This is generally the best choice for font-size. It scales predictably across the entire page and respects the user's base font size preference. If the browser default is 16px, then 1.5rem computes to 1.5 * 16 = 24px.
    • % (Percentage): Relative to the parent element's font-size. Can suffer from the same compounding issues as em if nested. 80% here means 80% of the body's default font size (e.g., 0.8 * 16px = 12.8px).
    • em: Relative to the parent's font-size (or the element's own size if inherited). Prone to compounding issues for font-size. Better suited for properties like padding or margin that should scale with the text inside the element.
  • Keywords: Predefined values like small, medium, large, x-large, etc. Less precise control than numeric units.

Accessibility: Using relative units like rem for font sizes is crucial for accessibility, as it allows users to adjust text size globally via browser settings.
