380. Header Cells: The <th> Element - Fundamentals

Most data tables require header cells to label the columns and/or rows, providing context for the data cells (<td>). The element used for this is <th>.

The Table Header Cell Element (<th>)

  • Definition: Represents a header cell in a table.
  • Purpose: To define a label or title for a group of data cells – typically an entire column or an entire row.
  • Semantic Distinction: Crucially distinguishes header information from regular data (<td>). This semantic difference is vital for assistive technologies and data processing.

Core Concept:

  • Use <th> instead of <td> within a <tr> specifically for cells that act as labels for other cells.

Placement & Structure:

  • Like <td>, a <th> element must be a direct child of a <tr> element.
  • Commonly found in the first row of a table (for column headers, often within <thead>) or in the first cell of subsequent rows (for row headers, often within <tbody>).
  • Content Model: Flow Content, but typically contains concise Phrasing Content (the header text). Abbreviations (<abbr>) are sometimes useful within <th> for providing shorter versions of long headers.

Code Change:

We've added a header row (<tr>) inside our <table>, using <th> elements for the cells.

<!-- index.html snippet -->
  <table>
    <tr> <!-- Header Row -->
      <th>Column Header 1</th> <!-- Added th -->
      <th>Column Header 2</th> <!-- Added th -->
    </tr>
    <!-- Data rows (tr with td) will follow -->
  </table>

✅ Observation:

  1. Note the new <tr> containing two <th> elements inside the <table> in index.html.
  2. Look at the browser preview. You should see the text "Column Header 1" and "Column Header 2". Notice how the browser typically renders <th> content differently from <td> content by default:
    • The text is usually bold.
    • The text is often centered horizontally within the cell. (Exact styling depends on the browser, but the visual distinction highlights the semantic difference).

✨ Key Takeaway: Use the <th> element instead of <td> for cells that function as headers for columns or rows. Browsers typically style them differently (bold, centered) by default, and they provide crucial semantic meaning for accessibility.
