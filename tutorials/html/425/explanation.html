425. Forms: The name Attribute (on <form>)

Besides action, method, and enctype, the <form> element can also have a name attribute.

Definition & Purpose:

  • The name attribute assigns a name to the <form> element itself.
  • Its primary historical purpose was to allow the form to be referenced via JavaScript using the legacy document.forms collection, which allows accessing forms by their name (e.g., document.forms['your_form_name'] or document.forms.your_form_name).
  • It can also potentially be used as a hook for CSS attribute selectors (form[name="your_form_name"]), although using id or class is far more common for styling.

Modern Relevance:

  • While still valid, the name attribute on <form> is less commonly used in modern JavaScript development.
  • Selecting forms using their id (document.getElementById('form-id')) or other more specific CSS selectors (document.querySelector(...)) is generally preferred for clarity and consistency.
  • The document.forms collection still exists and works, so name remains functional, but it's often seen as a slightly older pattern.

Syntax:

  • <form action="..." method="..." name="unique_form_name"> ... </form>
  • The name should ideally be unique within the page if used for scripting, although uniqueness isn't strictly enforced like id.

Code Change:

We've added a name="contact_form" attribute to our <form> tag in index.html.

<!-- index.html snippet -->
<form action="/submit-data" method="POST" name="contact_form"> <!-- Added name -->
   <!-- Form content -->
</form>

✅ Observation:

  1. Locate the name="contact_form" attribute in the <form> tag.
  2. Look at the browser preview. There is no visual change. The name attribute is purely for identification/referencing.

✨ Key Takeaway: The name attribute provides a name for the <form> element, primarily usable for referencing it via the legacy document.forms JavaScript collection. Using id for selection is generally preferred in modern practice.
