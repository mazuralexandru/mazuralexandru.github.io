35. Using Attribute Selectors: Substring Matching

Besides exact matches, attribute selectors can match based on substrings within the attribute's value. This is useful for targeting links based on the protocol (like https://) or file extension.

Let's style the PDF link using a selector that matches href attributes ending with .pdf.

What's New?

  • We added a new CSS rule using the "ends with" attribute selector [href$=".pdf"] to style.css.
/* style.css snippet */
[target="_blank"] {
  /* ... existing styles ... */
  background-size: 12px 12px; /* Control icon size */
}

/* Style links ending with .pdf */
[href$=".pdf"] {
  /* Add a different icon */
  padding-left: 20px; /* Space on the left */

Substring Matching Attribute Selectors

There are several types:

  • [attribute^="value"]: Selects elements whose attribute value starts with the specified value (e.g., [href^="https://"] for secure links).
  • [attribute$="value"]: Selects elements whose attribute value ends with the specified value (e.g., [href$=".pdf"] for PDF links, [src$=".jpg"] for JPEG images).
  • [attribute*="value"]: Selects elements whose attribute value contains the specified substring anywhere within it (e.g., [href*="example.com"] for links containing "example.com").
  • Others like [attribute~="value"] (contains value as a whole word in a space-separated list) and [attribute|="value"] (starts with "value-") exist but are less common for general styling.

In our example, [href$=".pdf"] targets any element whose href attribute value ends exactly with .pdf. We're adding a different icon (a document symbol) to the left of the text and changing the link color to light red.
