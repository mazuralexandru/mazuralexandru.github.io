353. Other <head> Elements: The <style> Element - Fundamentals

While linking external stylesheets (<link rel="stylesheet">) is the standard practice, HTML also provides the <style> element to embed CSS rules directly within the HTML document itself. This is known as an internal stylesheet or embedded styles.

Purpose:

  • The <style> element contains CSS rules that apply only to the current HTML document.

Core Concept:

  • Instead of putting CSS rules in a separate .css file, you write them directly between the opening <style> and closing </style> tags.
  • These rules are parsed and applied by the browser just like external styles, following standard CSS syntax and cascade rules.

Element Category & Placement:

  • The <style> element itself is metadata content.
  • It is typically placed within the <head> section of the document.
  • It can technically appear within the <body> (scoped styles were proposed but aren't widely supported/used standardly), but placing it in the <head> is the conventional and most reliable approach.

Syntax:

  • Requires opening <style> and closing </style> tags.
  • The content between the tags is standard CSS syntax (selectors and declaration blocks).
<head>
  ...
  <style>
    /* CSS rules go here */
    p {
      color: lime;
    }
    .highlight {
      background-color: yellow;
    }
  </style>
  ...
</head>

✅ Observation: No code changes yet. Understand the purpose: <style> allows you to write CSS rules directly inside your HTML file, typically within the <head>.

✨ Key Takeaway: The <style> element allows embedding CSS rules directly within an HTML document (usually in the <head>), creating an internal stylesheet that applies only to that page.
