158. ID Selectors: Specificity Note

Files Modified: None

A crucial difference between ID selectors and class/type selectors is their specificity. Specificity is the algorithm browsers use to decide which CSS rule applies when multiple rules target the same element and property.

  • ID Selectors: Have a high specificity value.
  • Class Selectors: Have a medium specificity value.
  • Type Selectors: Have a low specificity value.

(We'll cover the exact calculation later).

This means that a style applied via an ID selector (#my-id { color: red; }) will override a conflicting style applied via a class selector (.my-class { color: blue; }) or a type selector (p { color: green; }) if they all target the same element.

<p id="intro-para" class="highlight">...</p>

#intro-para { color: red; }    /* Wins due to higher specificity */
.highlight { color: blue; }   /* Overridden by ID */
p { color: green; }           /* Overridden by ID and Class */

This high specificity is why IDs should be used carefully for styling. Overusing them can make it difficult to override styles later without resorting to even more specific selectors or the !important flag (which should generally be avoided).
