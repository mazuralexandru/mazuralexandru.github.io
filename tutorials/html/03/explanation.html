03. HTTP/HTTPS Basics (Request/Response Cycle)

Files Modified: None

The HyperText Transfer Protocol (HTTP) is the set of rules governing communication between web clients and servers. HTTPS is the secure version, encrypting the data exchanged.

Here's the basic Request/Response Cycle:

  1. Request: Your browser (client) sends an HTTP request to the server specified in the URL. This request includes:
    • The method (e.g., GET to retrieve data, POST to send data).
    • The path to the requested resource (e.g., /path/to/page.html).
    • Headers (additional info like browser type, accepted formats).
  2. Response: The server processes the request and sends back an HTTP response. This response includes:
    • A status code (e.g., 200 OK for success, 404 Not Found if the resource doesn't exist).
    • Headers (additional info like content type, server details).
    • The requested resource itself (e.g., the HTML content of the page).

The browser then receives this response and renders the content (like an HTML page) for you to see.
