206. @font-face Rule: Embedding Custom Fonts

Files Modified: None

What if you want to use a specific font that users likely don't have installed on their systems? You can embed custom web fonts using the CSS @font-face at-rule.

  • Purpose: Allows you to define a custom font family name and specify the location (URL) of the font file(s) to be downloaded by the browser. Once defined, you can use this custom font family name in your font-family declarations just like any other font name.
  • Placement: @font-face rules are typically placed at the top of your CSS file, before any rulesets that use the custom font.

We won't implement a full @font-face example here as it requires actual font files, but understanding the concept and syntax is important.

/* Example @font-face rule */
@font-face {
  font-family: 'My Custom Sans'; /* Name you'll use in CSS */
  src: url('fonts/my-custom-sans-regular.woff2') format('woff2'), /* Preferred format */
       url('fonts/my-custom-sans-regular.woff') format('woff');   /* Fallback format */
  font-weight: normal; /* Or 400 */
  font-style: normal;
  font-display: swap; /* Tells browser how to handle font loading */
}

@font-face {
  font-family: 'My Custom Sans'; /* Same family name */
  src: url('fonts/my-custom-sans-bold.woff2') format('woff2'),
       url('fonts/my-custom-sans-bold.woff') format('woff');
  font-weight: bold; /* Or 700 - Defines the bold variant */
  font-style: normal;
  font-display: swap; 
}

/* --- Usage --- */
body {
  font-family: 'My Custom Sans', Arial, sans-serif; /* Use the custom font */
}

strong {
  font-weight: bold; /* Browser will try to use the bold @font-face variant */
}

Key parts include font-family (defining your custom name), src (pointing to font files - woff2 is the modern standard), format (hinting), font-weight/font-style (defining which variant this file represents), and font-display (controlling rendering during load - swap shows fallback text then swaps, which is common).
