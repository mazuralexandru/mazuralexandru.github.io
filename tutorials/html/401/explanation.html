401. Positioning Image within its Frame (object-position)

Files Modified: style.css

When using object-fit values like cover or contain (which might result in cropping or empty space), the object-position property allows you to control which part of the image is shown or how it's aligned within its box.

  • Purpose: Defines the alignment of the replaced element's content box (the image itself) within the element's box (the container defined by width/height).
  • Syntax: Similar to background-position. Accepts one or two values.
    • Keywords: top, bottom, left, right, center.
    • Lengths: px, em, etc. Offset from the top-left corner.
    • Percentages: %. 0% 0% is top-left, 50% 50% is center, 100% 100% is bottom-right.
  • Default: 50% 50% (center).

✅ Observe style.css:

  • We've added object-position: center top; to the .sample-image rule.
.sample-image {
  max-width: 100%; 
  width: 200px; 
  height: 200px; 
  display: block; 
  border: 2px solid yellow; 
  margin: 1em auto; 
  object-fit: cover; 
  object-position: center top; /* Align image content to top edge */
}

✅ Switch to index.html to see the HTML structure.

✨ Observe the Preview:

  • The image area is still a 200x200 square, and the image content is still using object-fit: cover.
  • However, instead of centering the image content vertically within the square (the default), the object-position: center top; should align the top edge of the image content with the top edge of the container. Since the image is wider than it is tall relative to the square container, this means the cropping now happens entirely at the bottom of the image, rather than being split between the top and bottom.
  • Try changing the value in DevTools to bottom right or 25% 75% to see how it shifts the visible portion of the image within its frame.
