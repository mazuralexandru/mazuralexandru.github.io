Now, let's shift our focus back to HTML elements, specifically those used for creating lists. Lists are fundamental for organizing related items, such as navigation links, feature lists, or steps in a process.

The first type we'll look at is the unordered list, created using the <ul> element.

  • Structure: The <ul> element acts as a container. The actual list items are placed inside it using <li> (list item) elements.
  • Rule: The only permitted direct children of a <ul> element are <li> elements (or script-supporting elements like <script> and <template>). You cannot place text or other block elements like <p> directly inside a <ul>.
  • Default Styling: Browsers typically render unordered lists with bullet points (like discs â€¢) preceding each list item and some default indentation.
