508. Landmarks: <nav> Element - Fundamentals

Next, let's add the <nav> element, used specifically for major navigation blocks.

Definition & Purpose:

  • The <nav> element represents a section of a page whose purpose is to provide navigation links.
  • It's intended for major navigation blocks, such as the primary site menu, secondary menus, breadcrumbs, tables of contents, or pagination controls. It's not meant for every single group of links on a page (e.g., simple links within footer text don't usually warrant a <nav>).

Scope & Placement:

  • Can appear in various places, often:
    • Inside the main page <header>.
    • Directly under <body> (e.g., if header doesn't contain the main nav).
    • Within an <aside> for sidebar navigation.
    • Within an <article> or <section> for intra-page navigation (like a table of contents).
  • A document can have multiple <nav> elements.

Core Mechanics:

  • Block-level element.
  • Syntax: <nav> ... </nav>
  • Content: Flow content, but typically contains an unordered list (<ul>) of links (<li><a>...</a></li>). Can also include a heading (<h2>, etc.) to label the navigation section, especially if there are multiple <nav> elements on the page.

Accessibility:

  • Maps to the ARIA navigation landmark role. Allows screen reader users to quickly find and jump to navigation sections.
  • If multiple <nav> elements exist, use aria-label or aria-labelledby to give each one a unique accessible name (e.g., <nav aria-label="Primary site navigation">, <nav aria-label="Pagination">).

Code Change:

We've added a <nav> element containing a basic <ul> of placeholder links inside the existing <header> element in index.html. We've also added minimal CSS to style the list horizontally for a typical nav appearance.

<!-- index.html snippet -->
<head>
  ...
  <style>
    /* Basic landmark visualization */
    body > header, body > nav, body > main, body > footer { ... }
    /* Basic nav list style */
    nav ul { list-style: none; padding: 0; }
    nav li { display: inline-block; margin-right: 10px; }
  </style>
</head>
<body>
  <header>
    <h1>Site Title / Logo Area</h1>
    <!-- Added nav inside header -->
    <nav>
      <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
        <li><a href="/contact">Contact</a></li>
      </ul>
    </nav>
  </header>

  <p>The &lt;nav&gt; element groups major navigation links.</p> <!-- Adjusted P -->
  ...
</body>

✅ Observation:

  1. Locate the <nav> element nested inside the <header>. Note the common pattern of using <ul><li><a> for the links. Note the added CSS for basic horizontal styling.
  2. In the browser preview, the "Home", "About", "Contact" links should appear horizontally below the "Site Title" area, all within the dashed border of the <header>. The <nav> element itself doesn't add a border here, but it groups the navigation links semantically.

✨ Key Takeaway: Use the <nav> element to wrap major blocks of navigation links (site menus, pagination, etc.). Typically contains a list (<ul>) of links (<a>). Provides an essential navigation landmark for accessibility.
