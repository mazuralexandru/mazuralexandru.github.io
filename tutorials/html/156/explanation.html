156. href URL Types: Relative URLs (Definition & Purpose)

After absolute URLs, the next crucial type is the relative URL.

Definition:

  • A relative URL specifies the location of a resource based on its position relative to the current document's location.
  • It's a partial path and does not include the scheme (http/https) or the domain name.

Purpose & Use Case:

  • Relative URLs are primarily used for creating links between pages and resources within the same website.
  • Key Advantage: Portability. Because relative links don't contain the domain name, you can move your entire website (all its files and folders) to a different domain or server, or even run it locally, and the internal links will still work as long as the internal file/folder structure remains the same relative to each other. This makes them essential for internal linking.

How they Work (Conceptual):

When the browser encounters a relative URL in an href, it combines that relative path with the URL of the current page (specifically, its directory path) to figure out the full, absolute URL it needs to request.
