348. Other <head> Elements: The <base> Element - Fundamentals

Now let's look at the <base> element, which allows you to specify a base URL for all relative URLs within a document.

Purpose:

  • The <base> element sets the base URL that the browser should use when resolving relative URLs found in attributes like href (links), src (images, scripts), and others throughout the entire document.
  • It can also define a default target browsing context for links that don't have their own target attribute.

Core Concept:

  • Normally, a relative URL (like images/pic.jpg or ../about.html) is resolved based on the URL of the current HTML document.
  • If a <base> tag is present with an href attribute, the browser uses the URL specified in the <base> tag's href as the starting point for resolving all subsequent relative URLs in the document, instead of the document's own URL.

Element Category & Placement:

  • <base> is a void (empty) element (no closing tag).
  • It must be placed within the <head> section.
  • There can be at most one <base> element per document. If multiple are present, only the first one's href and target values are used.

✨ Key Takeaway: The <base> element, placed in the <head>, specifies a base URL and/or a default target for resolving relative URLs and links throughout the document. Only one <base> element is allowed.
