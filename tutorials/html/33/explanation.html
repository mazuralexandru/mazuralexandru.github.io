33. Rule: Proper Nesting

Files Modified: None

When nesting HTML elements, there's one crucial rule: Elements must be closed in the reverse order they were opened.

Think of it like parentheses in math or brackets in programming â€“ you must close the innermost one first.

  • Correct: <div><p>Some text</p></div>

    • <p> was opened last, so it's closed first (</p>).
    • <div> was opened first, so it's closed last (</div>).

  • Incorrect: <div><p>Some text</div></p> (Tags overlap / improperly nested)

Browsers often try to correct improperly nested tags, but the results can be unpredictable and lead to invalid HTML structure, potentially breaking your layout or styles. Always ensure your tags are nested correctly. Code editors often help with indentation, which visually reinforces the nesting structure.
