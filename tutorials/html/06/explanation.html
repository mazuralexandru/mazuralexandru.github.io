06. Elements & Tags: Defining Elements and Tags

Now we start diving into the actual syntax of HTML. The core building blocks are elements and tags.

What is an HTML Element?

An element is a fundamental conceptual unit of an HTML document. It represents a distinct piece of content and its semantic meaning or purpose. Think of elements as the different types of building blocks you can use:

  • A paragraph of text
  • A main heading
  • An image
  • A list of items
  • A hyperlink

What is an HTML Tag?

Tags are the specific text codes you write in your HTML file to create an element. They act as markers, telling the browser where an element begins and ends, and what type of element it is.

  • Tags are always enclosed in angle brackets (< and >).
  • The text inside the angle brackets indicates the type of element (e.g., p for paragraph, h1 for level-1 heading).

First Code!

We've added a special type of text to our index.html file:

<!-- Content will go here soon! -->

This is an HTML comment. We'll discuss comments in detail later (Lesson 9), but for now, know that anything between <!-- and --> is ignored by the browser when displaying the page. It's a note for humans reading the code.

✅ Observation: Check the index.html file in your editor. You should see the comment line. The browser preview pane will still be blank because comments are not visible content.

✨ Key Takeaway: Elements are the conceptual building blocks (paragraph, heading). Tags (<tagname>) are the code you write to create those elements.
