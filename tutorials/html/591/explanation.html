591. Graphics: Canvas (<canvas>) - Core Mechanics (HTML Element)

Let's add the <canvas> element to our HTML and examine its core structure and essential attributes in the markup.

Syntax Recap:

  • <canvas id="..." width="..." height="..."> Fallback Content </canvas>
  • Requires opening and closing tags. The content between the tags is fallback content for non-supporting browsers.

Key HTML Attributes:

  1. id [Essential for JS]: A unique identifier used by JavaScript to select this specific canvas element (e.g., document.getElementById('myFirstCanvas')) and get its drawing context.
  2. width [Essential]: Specifies the width of the canvas's actual drawing buffer in CSS pixels. This defines the horizontal dimension of the coordinate system used for drawing. Default is 300.
  3. height [Essential]: Specifies the height of the canvas's drawing buffer in CSS pixels. Defines the vertical dimension of the coordinate system. Default is 150.

Important Note on Sizing:

  • Always set the desired drawing resolution using the width and height attributes.
  • Use CSS (width, height, max-width) to control the displayed size of the canvas element on the page if you need it to be responsive or different from its drawing buffer resolution. Styling the element size via CSS without setting the attributes appropriately will scale the bitmap, often causing blurriness.

Fallback Content:

  • Provide meaningful content between the tags for browsers that don't support canvas. This could be descriptive text, a static image representing the graphic, or a link to an alternative format (like a data table for a chart).

Code Change:

We've added a <canvas> element to index.html with id, width, height, and fallback content. We've also added a temporary CSS border to make the canvas area visible.

<!-- index.html -->
<head>
  ...
  <style>
    canvas { border: 1px dashed skyblue; } /* Visualize canvas */
  </style>
</head>
<body>
  <h1>The &lt;canvas&gt; Element</h1>
  <p>Adding the canvas element with width, height, and ID.</p> <!-- Adjusted P -->

  <!-- Added canvas element -->
  <canvas id="myFirstCanvas" width="300" height="150">
    <!-- Fallback Content -->
    Your browser does not support the canvas element. Maybe display static image or data table here instead?
  </canvas>

  <p>Drawing requires JavaScript targeting the canvas ID.</p> <!-- Adjusted P -->
  ...
</body>

  2. Look at the browser preview. You should now see an empty rectangular box with a dashed sky blue border, 300 pixels wide and 150 pixels high. This is the canvas drawing surface. The fallback content is not displayed because your browser supports canvas.
