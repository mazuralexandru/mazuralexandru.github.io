326. overflow Value: auto

Files Modified: style.css

The auto value is often the most practical choice for handling overflow.

  • Behavior: Content that extends beyond the element's padding edge is clipped. The browser displays scrollbars only if the content actually overflows in that direction (vertically or horizontally).
  • Effect: Provides access to overflowing content via scrolling when needed, but avoids displaying unnecessary scrollbars if the content fits within the box. This generally leads to a cleaner appearance than scroll.
  • Use Case: The most common value used when you want to contain content within a fixed-size box but allow users to access it if it's too large. Ideal for scrollable content areas, code blocks, etc.

✅ Observe style.css:

  • We've changed overflow: scroll; to overflow: auto; in the .overflow-box rule.
.overflow-box {
  width: 200px;
  height: 100px;
  padding: 10px;
  border: 2px solid orange;
  background-color: rgba(0, 0, 100, 0.2);
  margin-bottom: 1em; 
  /* overflow: scroll; */
  overflow: auto; /* Show scrollbars only if needed */
}

✨ Observe the Preview:

  • The text content is still clipped.
  • You should see a vertical scrollbar because the text content overflows the 100px height.
  • You should not see a horizontal scrollbar (unless a word is exceptionally long and overflow-wrap isn't set), because the text wraps naturally within the 200px width and doesn't overflow horizontally. auto intelligently shows scrollbars only where necessary.
