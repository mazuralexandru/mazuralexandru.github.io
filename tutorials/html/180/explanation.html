180. Security Note: Modern Browser Defaults

It's worth noting that browser vendors are aware of the security risks associated with target="_blank".

Implicit noopener Behavior:

  • Most major modern browsers (including Chrome, Firefox, Safari, Edge) have changed their default behavior.
  • Now, if an <a> tag has target="_blank" but does not explicitly have rel="noopener" (or rel="noreferrer"), the browser often implicitly applies the noopener behavior anyway. It sets window.opener to null by default for _blank targets as a security measure.

Should You Still Use rel="noopener noreferrer"?

Yes, absolutely! Here's why relying on the implicit browser behavior is not sufficient:

  1. Backward Compatibility: Older browsers or less common browsers may not have implemented this implicit noopener behavior. Explicitly adding rel="noopener noreferrer" ensures protection across a wider range of user agents.
  2. Explicit Intent: It clearly communicates your intention in the code to disable the opener access and referrer, making the code more understandable and self-documenting.
  3. noreferrer Benefit: The implicit behavior usually only covers noopener. If you also want the privacy benefit of not sending the Referer header, you must explicitly include noreferrer.
  4. Future Changes: Browser defaults could theoretically change again, although unlikely for this specific security feature. Explicit attributes are more future-proof.

✅ Observation: No code changes. Understand that while modern browsers try to help by implicitly adding noopener to target="_blank", you should always explicitly include rel="noopener noreferrer" in your HTML for maximum safety, compatibility, and clarity.

✨ Key Takeaway: Do not rely on implicit browser defaults. Always explicitly add rel="noopener noreferrer" to links using target="_blank".
