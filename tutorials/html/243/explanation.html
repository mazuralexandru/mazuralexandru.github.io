243. Common Web Image Formats: JPEG / JPG

Let's start our overview of common web image formats with one of the most ubiquitous: JPEG (or JPG).

JPEG (Joint Photographic Experts Group)

  • File Extensions: .jpg, .jpeg
  • Best Use Case: Photographs and complex images with millions of colors, smooth gradients, and subtle tonal variations.
  • Compression: Uses lossy compression. This means it reduces file size by discarding some image data that the human eye might not easily perceive.
    • Advantage: Can achieve significant file size reductions, leading to faster page loads.
    • Disadvantage: Some image quality is permanently lost. The level of compression (and thus quality loss) is usually adjustable when saving the image. Higher compression = smaller file size = lower quality. Lower compression = larger file size = higher quality.
  • Transparency: Does not support transparency. Any transparent areas will typically be filled with a solid color (often white) when saved as a JPEG.
  • Color: Supports millions of colors (24-bit).

When to Use JPEG:

  • Ideal for displaying photographs on the web where file size is a major concern and perfect pixel-for-pixel fidelity isn't the absolute top priority.
  • Good for complex illustrations with lots of color blending.

When NOT to Use JPEG:

  • Images requiring transparency (logos, icons on non-solid backgrounds).
  • Images with sharp lines, crisp text, or large areas of flat color (like logos, diagrams, screenshots). The lossy compression can create noticeable artifacts (blurriness, noise) around sharp edges in these cases. Use PNG or SVG instead.
  • Simple animations (use GIF, WebP, or AVIF).

✅ Observation: No code changes. This lesson provides conceptual information about the JPEG format.

✨ Key Takeaway: JPEG (.jpg) is best for photographs due to its effective lossy compression, but it doesn't support transparency and can introduce artifacts in images with sharp lines or text.
