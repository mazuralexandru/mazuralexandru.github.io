39. Using Grouping Selectors in CSS

Let's apply the DRY principle and use a Grouping Selector to set a common font-weight and border-bottom for all our heading levels (<h1>, <h2>, <h3>).

What's New?

  • We modified the existing h1 rule to only set the color.
  • We added a new grouped selector rule h1, h2, h3 to apply common styles to all three heading levels.
/* style.css snippet */
/* Updated H1 rule */
h1 {
  color: lightblue; /* H1 specific color */
  /* Common styles moved to grouped selector below */
}

/* Updated P rule */
p {
  font-style: italic;
  color: #ccc;
}

/* Grouped Selector for common heading styles */
h1,
h2,
h3 {
  font-weight: normal; /* Make all headings normal weight */
  border-bottom: 1px solid #555; /* Common underline */
  padding-bottom: 3px; /* Space between text and underline */
  margin-bottom: 10px; /* Space below headings */
}

/* Removed Universal Selector rule */

/* CSS files can also have comments! */

Grouping Selectors (selector1, selector2, ...)

  • Syntax: List the selectors you want to group, separated by commas (,). Whitespace (including newlines) around the commas is ignored, so you can format the list for readability as shown above.
  • Functionality: The declarations within the following block ({ ... }) are applied to all elements matched by any of the selectors in the comma-separated list.
  • Combining with Specific Rules: You can still have separate rulesets for individual selectors within the group (like our h1 { color: lightblue; } rule) to apply styles specific only to that element type, in addition to the common styles applied by the group.

In this example:

  • h1, h2, h3 { ... } applies font-weight: normal, a gray bottom border, padding, and bottom margin to all <h1>, <h2>, and <h3> elements.
  • The separate h1 { color: lightblue; } rule also applies specifically to the <h1>, setting its color. The <h2> and <h3> will inherit their color from the body or have browser default colors unless explicitly set.
