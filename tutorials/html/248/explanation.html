248. Common Web Image Formats: AVIF

An even newer image format aiming for further compression improvements is AVIF.

AVIF (AV1 Image File Format)

  • File Extension: .avif
  • Development: Developed by the Alliance for Open Media (AOMedia), based on the AV1 video codec.
  • Purpose: Aims to provide significantly better compression efficiency than even WebP, JPEG, PNG, and GIF for both lossy and lossless images.
  • Compression: Supports both lossy and lossless compression. Often achieves even smaller file sizes than WebP for the same perceptual quality, especially at lower quality settings.
  • Features: Supports:
    • High Dynamic Range (HDR) and Wide Color Gamut (WCG).
    • Alpha Transparency.
    • Animation.
  • Browser Support: Support is growing rapidly in modern browsers (Chrome, Firefox, Opera, Safari 16+), but it's generally less widespread than WebP at this moment. Edge support is emerging. Check current compatibility tables (like caniuse.com) before relying solely on it.
  • Use Cases: Ideal where maximum compression efficiency is desired, especially for high-quality images or animations, provided target browsers support it or fallbacks are used.
  • Fallback: Like WebP, often delivered using the <picture> element to provide AVIF first, then WebP, then JPEG/PNG as fallbacks for older browsers.

When to Use AVIF:

  • When you need the absolute smallest file sizes possible for raster images without sacrificing quality significantly.
  • For HDR images.
  • When targeting primarily the latest browser versions or when implementing robust fallbacks.

When NOT to Use AVIF:

  • If vector graphics are more appropriate (use SVG).
  • If broad compatibility with older browsers is needed without using <picture> fallbacks. WebP currently has wider support.

✅ Observation: No code changes. This lesson provides conceptual information about the AVIF format.

✨ Key Takeaway: AVIF (.avif) is a cutting-edge image format offering potentially superior compression compared to WebP and older formats. It supports HDR, transparency, and animation. Browser support is good and growing but currently less universal than WebP, often requiring fallbacks.
