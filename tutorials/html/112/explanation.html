112. CSS Comments: Purpose Explained

Files Modified: None

CSS comments (/* ... */) serve similar purposes to HTML comments, but within the context of stylesheets:

  1. Explanation & Documentation: Add notes to explain complex selectors, the purpose of specific rules, color palette choices, or layout strategies. Organize your CSS file into logical sections using comments as headings.
    /* --- Header Styles --- */
.site-header { ... }

/* --- Footer Styles --- */
.site-footer { ... }
  2. Debugging / Temporarily Disabling Styles: Quickly disable a single declaration or an entire ruleset to test its effect or isolate styling conflicts. This is extremely common during development.
    .widget {
  border: 1px solid red;
  /* padding: 15px; */ /* Temporarily remove padding */
  margin-bottom: 20px; 
}

/*
.old-style {
  color: gray;
  font-style: italic;
} 
*/ /* Temporarily disable the whole rule */

Effective commenting makes your CSS much easier to understand, maintain, and debug.
