21. Applying CSS with External Stylesheets (Preferred)

While internal stylesheets (<style>) are better than inline styles, the most maintainable, reusable, and standard way to apply CSS is using external stylesheets. We already linked style.css back in Lesson 7 using the <link> tag in the <head>. Now, let's move the styles we just created from the internal <style> block into our external style.css file.

What's New?

  • Removal: The entire <style>...</style> block and its contents have been removed from the <head> in index.html. We are pruning the internal styles now that we are focusing on external ones. Look for the comment <!-- Removed internal <style> block -->.
  • Focus Shift: We are now preparing to add these rules (or similar ones) to the external style.css file.
<!-- index.html snippet -->
<head>
  <meta charset="UTF-8">
  <title>My First Web Page</title>
  <link rel="stylesheet" href="style.css"> <!-- External CSS link (remains) -->
  <!-- Removed internal <style> block -->
</head>
<body>
  <h1>Styling with External CSS</h1> <!-- Text Changed slightly -->
  <p>This paragraph will be styled externally.</p> <!-- Text Changed slightly -->

  <!-- Commented out paragraph remains -->
  <!-- <p>This paragraph is commented out and won't appear.</p> -->

  <script src="script.js"></script>
</body>

External Stylesheets (.css files)

  • Recap: We link external CSS files using the <link> tag in the <head>:
    <link rel="stylesheet" href="style.css">
  • Benefits (Why Preferred):
    • Separation of Concerns: Keeps HTML (structure) and CSS (presentation) completely separate. Files are cleaner and easier to understand.
    • Maintainability: Need to change a style across your entire website? Edit just one .css file! Changes automatically apply to all HTML pages linking to it.
    • Reusability: The same .css file can be linked from hundreds or thousands of HTML pages.
    • Performance (Caching): When a user visits your site, their browser downloads style.css once. When they navigate to other pages that use the same file, the browser uses the cached (stored) version instead of downloading it again, making subsequent page loads faster.
