208. font-size Recap & Accessibility

Files Modified: style.css

We've already discussed font-size and its units (px, em, rem, %, keywords). Let's re-emphasize the accessibility aspect.

  • User Control: Users, especially those with visual impairments, often configure their browser's default font size to be larger than the standard 16px.
  • px Issue: If you define all your font sizes using absolute px units, they will not scale when the user changes their browser's base font size setting. This can make your site difficult or impossible to read for these users.
  • rem & em Advantage: Relative units like rem (relative to root <html> size) and em (relative to parent size) do scale relative to the base font size. If the user increases their default size, text sized with rem or em will increase proportionally, maintaining readability.

Best Practice for Accessibility:

  • Set your base font-size on the <html> or <body> element using a relative unit or allowing the browser default (often 16px).
  • Define most other font-size values throughout your site using rem units. This provides consistent scaling based on the root font size, respecting user preferences.
  • Use em units when you specifically need an element's size (like padding or margin) to scale relative to its own text size, not the root size.

✅ Observe style.css:

  • We've changed the font-size for the .container span from 0.8em (relative to parent <a>) to 1.5rem. Assuming a default root size of 16px, this would make the span's font size 1.5 * 16px = 24px.
  • We also added a font size to the second paragraph using rem.
/* Descendant Combinator Example */
.container span {
  font-weight: bold; 
  border: inherit; 
  font-size: 1.5rem; /* Changed to rem unit */
}

/* Adjacent Sibling Combinator Example */
h1 + p {
  margin-top: 5px; 
  color: lightgreen; 
  font-size: 0.9rem; /* Added font-size using rem */
}

✨ Observe the Preview:

  • The text "This is inside the paragraph." within the first paragraph should now be significantly larger (likely 24px).
  • The second paragraph ("This paragraph follows the heading.") should now be slightly smaller than the default text size (0.9 * 16px = 14.4px).

Using rem ensures these sizes would scale if the user changed their browser's base font setting.
