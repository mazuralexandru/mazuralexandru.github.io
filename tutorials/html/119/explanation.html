119. Nested Lists: Fundamentals

We've explored unordered (<ul>) and ordered (<ol>) lists individually. Now, let's see how to create nested lists – lists within lists – to represent hierarchical structures or sub-items.

Definition & Purpose:

  • Nesting: Placing one list (<ul> or <ol>) inside a list item (<li>) of another list.
  • Purpose: To create outlines, multi-level navigation, sub-steps in procedures, or any structure where items have sub-items.

Core Concept:

This demonstrates the flexibility of the <li> content model. Since <li> can contain flow content, and lists (<ul>/<ol>) are flow content, you can naturally place a list inside a list item to create a hierarchy.
