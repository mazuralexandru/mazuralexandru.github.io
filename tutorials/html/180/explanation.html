180. Cascade Factors (Order of Importance)

Files Modified: None

The cascade resolves conflicts by considering three main factors, in order of importance:

  1. Origin and Importance: Where the style comes from and whether it's marked as important.
  2. Specificity: How specific the selector targeting the element is.
  3. Source Order: The order in which the conflicting rules appear in the CSS code.

Let's break down Origin and Importance first. Stylesheets originate from different sources:

  • Browser (User-Agent) Stylesheets: Every browser has built-in default styles (e.g., headings are bold and large, links are blue and underlined). These have the lowest precedence.
  • User Stylesheets: Users can define their own custom stylesheets in their browser settings (e.g., for accessibility reasons, like always using a specific font or high contrast).
  • Author Stylesheets: These are the stylesheets you write as the website developer (inline styles, internal <style> blocks, external .css files). This is what we primarily work with.

Within these origins, declarations can be marked with !important to increase their weight:

  • Normal Declarations: Standard rules without !important.
  • Important Declarations: Rules ending with !important (e.g., color: red !important;).

The general order of precedence (from lowest to highest) is:

  1. Browser default styles (normal)
  2. User styles (normal)
  3. Author styles (normal) - Our primary focus
  4. Author styles (!important)
  5. User styles (!important)
  6. Browser default styles (!important)

Key Takeaway for Developers: Your normal author styles will override browser defaults. Using !important in your author styles makes them override almost everything else, including other author styles, but it should be used very sparingly as it makes debugging harder and breaks the natural cascade. Avoid it unless absolutely necessary to override external library styles or for very specific utility classes.
