124. DevTools: Styles Pane (Brief Look)

Files Modified: None

When you select an element in the Elements panel (using the methods from the previous step), another pane within DevTools becomes very important: the Styles pane (it might be called "Rules" in Firefox).

  • Location: Usually appears to the side or below the Elements panel DOM tree.
  • Purpose: Shows all the CSS rules that are currently being applied to the selected element.
  • Features:
    • Lists rules from different sources (e.g., base_styles.css, style.css, browser default styles).
    • Shows which properties are being applied and which might be overridden by other rules (often shown with a strikethrough).
    • Allows you to temporarily edit existing styles (e.g., change color values, sizes) or add new declarations directly in the pane to see the effect live on the page (again, these changes are temporary).
    • Shows the filename and line number where each rule is defined.

✅ First, ensure you have the <h1> element selected in the Elements panel (viewing index.html). ✅ Now, look for the Styles pane in DevTools. You should see the rule h1 { color: cornflowerblue; } from style.css listed. You might also see other styles inherited from the body (like font-family) or default browser styles. Switch to view style.css to confirm the source of the cornflowerblue rule.

The Styles pane is essential for debugging CSS issues – understanding why an element looks the way it does and experimenting with changes.
