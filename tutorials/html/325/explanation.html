325. overflow Value: scroll

Files Modified: style.css

The scroll value provides a way to access overflowing content.

  • Behavior: Content that extends beyond the element's padding edge is clipped, but the browser always displays scrollbars (both horizontal and vertical) allowing the user to scroll and view the hidden content.
  • Effect: Scrollbars are shown regardless of whether the content actually overflows or not. This can sometimes reserve space for the scrollbars, preventing the layout from jumping when content dynamically changes and causes overflow.
  • Use Case: When you want scrollbars to be permanently visible, or in situations where reserving space for potential scrollbars is important for layout stability.

✅ Observe style.css:

  • We've changed overflow: hidden; to overflow: scroll; in the .overflow-box rule.
.overflow-box {
  width: 200px;
  height: 100px;
  padding: 10px;
  border: 2px solid orange;
  background-color: rgba(0, 0, 100, 0.2);
  margin-bottom: 1em; 
  /* overflow: hidden; */
  overflow: scroll; /* Always show scrollbars */
}

✨ Observe the Preview:

  • The text content is still clipped within the orange-bordered box.
  • However, you should now see both a vertical and a horizontal scrollbar appear within the box.
  • The vertical scrollbar allows you to scroll down and see the rest of the paragraph text.
  • The horizontal scrollbar is likely present but might not do anything unless there was also horizontally overflowing content (like a very long word without overflow-wrap or an image wider than the box). scroll shows both unconditionally.
