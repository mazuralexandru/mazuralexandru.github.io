224. text-overflow

Files Modified: index.html, style.css

The text-overflow property specifies how overflowing inline content (text) that doesn't wrap should be signaled to the user. It's commonly used to display an ellipsis (...) to indicate hidden text.

  • Purpose: To provide a visual cue when text is clipped because it doesn't fit horizontally within its container.
  • Important: For text-overflow to have any effect, three other conditions must typically be met on the element:
    1. overflow: hidden; (Content must be clipped).
    2. white-space: nowrap; (Text must be prevented from wrapping to the next line).
    3. The element must be a block-level or inline-block element with a constrained width.
  • Values:
    • clip: (Default) Simply clips the text at the container's edge. No visual indicator is shown.
    • ellipsis: Displays an ellipsis ('…') to represent the clipped text.
    • string: (Less common, experimental) Allows specifying a custom string to show for overflow.
    • fade: (Non-standard, Firefox only) Fades out the text near the edge.

✅ Observe index.html:

  • We've added a new paragraph with the class ellipsis-example and some longer text content.
    <p class="ellipsis-example">This is a long line of text that will likely overflow its container and demonstrate the ellipsis effect.</p>

✅ Now, switch to style.css:

  • We've added a new ruleset for .ellipsis-example.
  • Crucially, it includes white-space: nowrap;, overflow: hidden;, and text-overflow: ellipsis;. We also give it a fixed width and a border to make the effect obvious.
/* Text Overflow Example */
.ellipsis-example {
  border: 1px dashed skyblue;
  width: 250px; /* Constrain width */
  padding: 5px;
  white-space: nowrap; /* Prevent text wrapping */
  overflow: hidden; /* Hide the overflowing text */
  text-overflow: ellipsis; /* Show ... for overflow */
}

✨ Observe the Preview:

  • You should see the new paragraph with the dashed sky blue border.
  • Because its width is limited to 250px and white-space is nowrap, the text "This is a long line of text..." doesn't fit.
  • Instead of just being cut off (clip), you should see an ellipsis (...) at the end of the visible text, indicating that more text is hidden due to overflow.
