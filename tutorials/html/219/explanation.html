219. text-transform

Files Modified: style.css

The text-transform property controls the capitalization of text within an element, regardless of how it's written in the HTML source.

  • Purpose: To enforce consistent capitalization for headings, labels, buttons, etc., without needing to type them that way in the HTML.
  • Inherited: Yes.
  • Values:
    • none: (Default) No transformation is applied; text renders as written in the HTML.
    • uppercase: Transforms all characters to uppercase (e.g., "Main Heading" becomes "MAIN HEADING").
    • lowercase: Transforms all characters to lowercase (e.g., "Main Heading" becomes "main heading").
    • capitalize: Capitalizes the first letter of each word (e.g., "main heading" becomes "Main Heading"). Be aware that its definition of a "word" boundary can sometimes be simplistic.
    • full-width: Transforms characters to be rendered within a square, primarily for aligning East Asian and Latin scripts (less common).
    • full-size-kana: Transforms Kana characters for ruby annotations (specific East Asian typography use case).

✅ Observe style.css:

  • We've added text-transform: uppercase; to the #main-heading ruleset.
#main-heading {
    /* ... other styles ... */
    letter-spacing: 2px; 
    text-transform: uppercase; /* Make heading uppercase */
}

✨ Observe the Preview: The "Main Heading" text should now be displayed entirely in uppercase letters ("MAIN HEADING"), even though it's written with initial caps in the index.html file.
