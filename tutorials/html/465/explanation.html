465. Forms: Options (<option>) - Fundamentals

To populate a <select> dropdown with choices, we use the <option> element for each available item.

The <option> Element:

  • Definition: Defines an option within a <select> list (or an <optgroup> or <datalist>).
  • Purpose: Represents one specific choice that the user can select from the list.
  • Placement: Must be a direct child of <select> or <optgroup>.

Syntax & Core Mechanics:

  • Syntax: <option value="submitted_value">Displayed Text</option>
  • Requires opening <option> and closing </option> tags.
  • Displayed Text: The text content placed between the tags (e.g., "United States") is what the user sees in the dropdown list.
  • value Attribute [Recommended]: Specifies the value that is sent to the server if this option is selected. It's highly recommended to provide a distinct, often shorter or coded, value (e.g., "US") separate from the display text.
    • If value is omitted: The browser submits the text content of the <option> instead. While functional, using explicit value attributes is generally better practice for consistency and easier server-side processing.
  • Content Model: Can contain only text (and character entities).

Code Change:

We've added four <option> elements inside the <select> element in index.html, each with a value attribute and display text. We also added a submit button.

<!-- index.html snippet -->
<form action="/submit-data" method="POST">
  <p>
    <label for="country-select">Choose a country:</label>
    <select name="country" id="country-select">
      <option value="US">United States</option> <!-- Added option -->
      <option value="CA">Canada</option> <!-- Added option -->
      <option value="GB">United Kingdom</option> <!-- Added option -->
      <option value="AU">Australia</option> <!-- Added option -->
    </select>
  </p>
  <p><button type="submit">Submit</button></p> <!-- Added submit -->
</form>

✅ Observation:

  1. Examine the <option> elements nested within <select>. Note the value attribute and the text content for each.
  2. Look at the browser preview. The dropdown box now likely displays the first option ("United States") by default.
  3. Click the dropdown. You should see a list containing "United States", "Canada", "United Kingdom", and "Australia".
  4. Select an option (e.g., "Canada") and conceptually submit. The data sent would be country=CA. If you selected "United Kingdom", it would send country=GB.

✨ Key Takeaway: Use <option> elements inside <select> to define the choices. The text between <option> tags is displayed; the value attribute defines what's submitted. Provide explicit value attributes for clarity and robustness.
