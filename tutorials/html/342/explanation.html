342. Reset: Preparing for Favicon Linking

We've thoroughly examined linking external stylesheets using <link rel="stylesheet">.

Now, let's move on to another common use of the <link> element: specifying favicons (the small icons displayed in browser tabs and bookmarks).

To isolate this topic, we will remove the conditional stylesheet links (media="print" and media="(max-width: 600px)") from the previous example, keeping only the main stylesheet link.

Why Remove Code?

  • Focus: Concentrate on the attributes and syntax specific to linking icons (rel="icon", type, sizes, href).
  • Simplify <head>: Reduce the number of <link> tags to make the favicon examples clearer.

Code Change:

  1. Removed the <link> tags with media attributes from the <head> of index.html.
  2. Updated the <h1> and paragraph text.
<!-- index.html -->
<head>
  <meta charset="UTF-8">
  <title>HTML Metadata - Favicons</title> <!-- Updated title -->
  <!-- Default stylesheet for all media -->
  <link rel="stylesheet" href="style.css">
  <!-- Removed print and mobile specific links -->

  <!-- Favicon links will be added here -->
</head>
<body>
  <h1>Metadata: Linking Favicons</h1> <!-- Updated H1 -->
  <p>The 'link' element is also used to specify favicons.</p> <!-- Adjusted P -->

  <p>Favicons appear in browser tabs, bookmarks, etc.</p> <!-- Adjusted P -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>
</html>

✅ Observation:

  1. Verify the removal of the <link> tags with media attributes from index.html. The <head> now contains charset, title, and the single main stylesheet link.
  2. The browser preview shows the updated heading and introductory paragraphs, ready for the favicon examples.

✨ Key Takeaway: Removing the conditional stylesheet examples simplifies the <head> and allows a clear focus on learning how to link favicons using the <link> element.
