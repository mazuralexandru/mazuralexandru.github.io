419. Styling <figure>: Sizing

Files Modified: style.css

You can control the size of the <figure> element itself, though often its size is implicitly determined by the content (like the image) inside it.

  • Implicit Sizing: If you don't set a width or height on the <figure>, it will typically shrink-wrap its content (like the image).
  • Explicit Sizing: You can set width, max-width, height, min-height, etc., on the <figure> element just like any other block-level element. This is useful if you want all figures to have a consistent maximum width, for example.

✅ Observe style.css:

  • We've added a max-width to the figure rule to constrain its size.
figure {
  margin: 0; 
  margin-bottom: 1.5em; 
  max-width: 500px; /* Constrain figure width */
  /* Other styles will be added */
}

/* ... rest of styles ... */

✅ Switch to index.html to see the HTML structure.

✨ Observe the Preview: If the available viewport width is greater than 500px, the figure (containing the image and caption) should now be constrained to a maximum width of 500px. The image inside, having max-width: 100%;, will scale down to fit within this 500px container. If the viewport is narrower than 500px, the figure will shrink (as its default width is auto), and the image will shrink with it.
