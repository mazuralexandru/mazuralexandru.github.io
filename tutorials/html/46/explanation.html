46. Inline Semantics: Alternate Voice (<i>)

Now let's look at the <i> element. Historically, this tag simply meant "italic," but HTML5 redefined its semantic purpose.

The <i> Element (HTML5 Semantics):

  • Purpose: Represents text that is set off from the normal prose in some way, often indicating an alternate voice or mood. It's used for things that aren't necessarily emphasized (<em>) or strongly important (<strong>).
  • Specific Use Cases (Examples):
    • Technical terms (when introduced or defined).
    • Taxonomic designations (like genus/species names: Homo sapiens).
    • Fictional character thoughts.
    • Foreign words or phrases (if not part of the main text language).
    • Ship names in historical text.
    • Transliterations.
  • Type: Inline element.
  • Syntax: <i>...</i>

Default Rendering:

Browsers continue to render <i> text in an italic font style by default, matching its historical usage.

<i> vs. <em>:

  • Use <em> when you want to apply stress emphasis that changes the meaning of the sentence (like vocal stress).
  • Use <i> when you want to mark text as being in an "alternate voice" or fitting one of the specific use cases listed above, without implying stress emphasis or strong importance. If none of the specific <i> use cases fit, and you just want italics for presentation, use CSS instead.
