468. Forms: Option Groups (<optgroup>)

For long <select> lists, it's helpful to group related options under headings. The <optgroup> element provides this functionality.

The <optgroup> Element:

  • Definition: Creates a grouping for a set of <option> elements within a <select> list.
  • Purpose: Improves the organization and scannability of long dropdown lists by visually separating options into logical categories.
  • Placement: Must be a direct child of a <select> element. Contains <option> elements as its children. Cannot be nested inside another <optgroup>.

Syntax & Core Mechanics:

  • Syntax: <optgroup label="Group Heading Text"> ... <option> elements ... </optgroup>
  • Requires opening <optgroup> and closing </optgroup> tags.
  • label Attribute [Required]: Specifies the text that will be displayed as the heading for the group within the dropdown list. This heading is typically rendered visually distinct (e.g., bold or italic) and is not selectable itself.
  • disabled (Boolean Attribute): If present on the <optgroup>, it disables all the <option> elements nested within that specific group, making them unselectable.

Code Change:

We've restructured the <select> element in index.html to demonstrate <optgroup>:

  1. Reverted back to a single-select dropdown (removed multiple and size).
  2. Added <optgroup> elements with label attributes ("North America", "Europe", "Asia").
  3. Placed <option> elements within their respective <optgroup>s.
  4. Added a disabled <option> within a group and a disabled entire <optgroup>.
<!-- index.html snippet -->
<form action="/submit-data" method="POST">
  <p>
    <label for="city-select">Choose a city:</label><br>
    <select name="city" id="city-select"> <!-- Reset to single select -->
      <option value="">-- Select City --</option>
      <optgroup label="North America"> <!-- Added optgroup -->
        <option value="nyc">New York</option>
        <option value="tor">Toronto</option>
      </optgroup>
      <optgroup label="Europe"> <!-- Added optgroup -->
        <option value="lon">London</option>
        <option value="par">Paris</option>
        <option value="ber" disabled>Berlin (Coming Soon)</option> <!-- Disabled option -->
      </optgroup>
       <optgroup label="Asia" disabled> <!-- Disabled optgroup -->
        <option value="tok">Tokyo</option>
      </optgroup>
    </select>
  </p>
  <p><button type="submit">Submit</button></p>
</form>

✅ Observation:

  1. Examine the <optgroup> elements wrapping sets of <option>s within the <select>. Note the required label attribute and the use of disabled.
  2. Look at the browser preview and click the dropdown:
    • You should see the options grouped under the non-selectable headings "North America", "Europe", and "Asia".
    • The headings might be visually distinct (e.g., bold).
    • The "Berlin" option under Europe should be disabled (greyed out).
    • The entire "Asia" group (including "Tokyo") should be disabled because the disabled attribute is on the <optgroup> itself.

✨ Key Takeaway: Use <optgroup label="..."> inside <select> to group related <option> elements under a non-selectable heading, improving organization for long lists. The disabled attribute can apply to individual options or entire groups.
