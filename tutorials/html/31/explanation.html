Files Modified: None

Are HTML tags and attributes case-sensitive?

According to the HTML specification, tag names and attribute names are case-insensitive. This means <p>, <P>, and <p> are technically treated the same by the browser. Likewise, class="info" and CLASS="info" are equivalent.

However:

  • Best Practice: The universally accepted convention and strong recommendation is to write all HTML tag and attribute names in lowercase.
  • Readability: Lowercase is generally easier to read.
  • Consistency: Sticking to lowercase ensures consistency within your code and when working with others.
  • XHTML: Older standards like XHTML required lowercase. While HTML5 is more lenient, maintaining lowercase is good practice.
  • Attribute Values: Note that the values of some attributes (like id and class) can be case-sensitive, especially when referenced by CSS or JavaScript.

Rule: Always use lowercase for HTML tags and attributes.
