347. Reset: Preparing for Other <head> Elements

We have now covered the <meta> element (focusing on charset, description, keywords, author, viewport) and the <link> element (focusing on rel="stylesheet" and rel="icon").

Before exploring the remaining common elements found within the <head> (<base>, <style>, <script>, <noscript>), let's clean up our <head> section by removing the favicon link examples.

Why Remove Code?

  • Focus: Shift attention to the next set of distinct <head> elements (base, style, etc.).
  • Simplify <head>: Keep the <head> section uncluttered as we introduce new concepts one by one. The favicon examples have served their purpose.

Code Change:

  1. Removed the <link> tags related to favicons (rel="icon", rel="apple-touch-icon") from the <head> of index.html.
  2. Updated the <h1> and paragraph text in the <body>.
<!-- index.html snippet -->
<head>
  <meta charset="UTF-8">
  <title>HTML Metadata - Other Head Elements</title> <!-- Updated title -->
  <!-- Default stylesheet for all media -->
  <link rel="stylesheet" href="style.css">
  <!-- Removed Favicon Examples -->
  <!-- <link rel="icon" href="/favicon.ico" sizes="any"> -->
  <!-- <link rel="icon" type="image/png" sizes="32x32" href="/icons/favicon-32x32.png"> -->
  <!-- <link rel="apple-touch-icon" sizes="180x180" href="/icons/apple-touch-icon.png"> -->

  <!-- Other head elements will be added here -->
</head>
<body>
  <h1>Other Head Elements</h1> <!-- Updated H1 -->
  <p>Exploring other elements found in the head: base, style, script, noscript.</p> <!-- Adjusted P -->

  <!-- Removed previous body text -->

  <p>Starting with the 'base' element.</p> <!-- Adjusted P -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

✅ Observation:

  1. Verify the removal of the favicon <link> tags from the <head> of index.html. The <head> now contains only charset, title, and the CSS link.
  2. The browser preview shows the updated heading and introductory paragraphs, ready for the <base> element discussion.

✨ Key Takeaway: Removing the favicon examples streamlines the <head> section, allowing us to focus clearly on the remaining <head> elements like <base>.
