442. background-size

Files Modified: style.css

The background-size property determines the size of the background image(s).

  • Purpose: To scale or set explicit dimensions for background images.
  • Syntax: background-size: value;
    • Accepts one or two values.
    • Applies independently to each background layer if multiple images are present (use comma separation).
  • Common Values:
    • auto (Default): Uses the image's intrinsic size. If both values are auto, the intrinsic size is used. If one is auto and the other is a length/percentage, the auto dimension is calculated to maintain the aspect ratio.
    • <length> or <percentage>: Sets the width; height defaults to auto (maintaining aspect ratio).
    • <length> <length> or <percentage> <percentage>: Sets explicit width and height. Percentages are relative to the background positioning area.
    • cover: Scales the image (maintaining aspect ratio) to be as large as possible so the background area is completely covered by the image. Some parts of the image may be clipped.
    • contain: Scales the image (maintaining aspect ratio) to be as large as possible while ensuring the entire image is visible within the background area. May leave empty space.

✅ Observe style.css:

  • We've added background-size: auto, 50px; to the .bg-box rule.
    • The first value (auto) applies to the gradient (gradients don't have an intrinsic size, so auto usually means they fill the area).
    • The second value (50px) applies to the pattern image, setting its width to 50px and letting the height adjust automatically.
.bg-box {
    background-color: transparent; 
    background-image: 
        linear-gradient(to bottom, rgba(0,0,0,0.5), transparent), 
        url('images/placeholder-pattern.png'); 
    background-repeat: no-repeat; 
    background-position: center top, center center; 
    background-size: auto, 50px; /* Size gradient auto, size pattern 50px wide */
    padding: 2em;
    margin-bottom: 1em;
    border: 1px dashed #fff; 
    min-height: 150px; 
}
