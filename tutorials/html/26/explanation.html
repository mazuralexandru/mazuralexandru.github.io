26. Using Class Selectors in CSS

Now that we have an element with the highlight class in our HTML, let's write a CSS rule in style.css to target it using a Class Selector.

What's New?

  • We added a new CSS ruleset to style.css using the class selector .highlight.
/* style.css snippet */
h2 {
  color: lightcoral;
  font-weight: normal; /* Make H2 not bold */
}

/* Added Class Selector rule */
.highlight {
  background-color: rgba(255, 255, 0, 0.2); /* Semi-transparent yellow background */
  border: 1px solid yellow;
  padding: 5px;
}

/* CSS files can also have comments! */

Class Selectors Explained

  • Syntax: A dot (.) followed immediately by the class name (e.g., .highlight, .button-primary, .card). No space between the dot and the class name.
  • Scope: Selects all elements in the document that have the specified class name assigned to their class attribute.
  • The Workhorse: Class selectors are the most common and versatile way to apply styles in CSS. They allow you to create reusable "components" or "utilities" that can be applied to any element needing that specific styling.

In our example, the rule .highlight { ... } will apply a semi-transparent yellow background, a solid yellow border, and some internal padding to any element with class="highlight".

Specificity Note: Class selectors (.highlight) are more specific than type selectors (p). If we had conflicting rules (e.g., p { color: blue; } and .highlight { color: red; }), the .highlight rule would win for elements that are both a paragraph and have the highlight class.
