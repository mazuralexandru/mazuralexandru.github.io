157. ID Selectors: Use Cases

Files Modified: None

Given their uniqueness and high specificity, ID selectors have specific use cases, but should generally be used more sparingly than class selectors for styling:

  1. Targeting Unique Page Landmarks: Identifying major sections of a page that occur only once, such as the main site header (id="site-header"), main content area (id="main-content"), or site footer (id="site-footer"). This can be useful for both styling and creating page anchors/links (<a href="#main-content">Skip to content</a>).
  2. JavaScript Hooks: Providing a reliable way for JavaScript to find and manipulate a specific element on the page (e.g., document.getElementById('user-form')). Using IDs for JS hooks is very common.
  3. High-Specificity Overrides (Use with Caution): Occasionally used to override styles from less specific selectors (like those from third-party libraries or frameworks) when other methods are difficult. However, relying heavily on IDs for styling can lead to specificity wars and make CSS harder to manage. It's often better to use more specific class combinations or adjust the source order if possible.

General Recommendation: Prefer classes for reusable styling. Use IDs primarily for unique page landmarks and JavaScript hooks. Avoid using IDs solely for styling if a class would suffice, as classes offer more flexibility and lower specificity, making your CSS easier to override and maintain.
