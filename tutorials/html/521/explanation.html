521. Headings & Outline: Accessibility Impact

Let's explicitly focus on why a logical heading structure, whether implicit (just headings) or explicit (headings + sectioning elements), is so vital for accessibility.

Screen Reader Navigation:

  • Headings provide the primary means of structural navigation for screen reader users on most web pages.
  • Users can:
    • List all headings: Get a quick "table of contents" overview of the page structure and topics.
    • Jump by level: Press keys (e.g., '1' for next H1, '2' for next H2, '3' for next H3) to quickly navigate between sections of a specific importance level.
    • Jump to next/previous heading: Use shortcuts (e.g., 'H' / 'Shift+H') to move sequentially through the heading structure regardless of level.

Consequences of Poor Heading Structure:

  1. No Headings: The page appears as one large, undifferentiated block of text. Users cannot skim, understand the structure, or navigate efficiently. This is a major accessibility barrier.
  2. Skipped Levels (e.g., H1 -> H3): Breaks the logical outline. Users jumping by level (e.g., expecting an H2 after the H1) might miss content entirely or become confused about the information hierarchy. The perceived structure doesn't match the intended structure.
  3. Using Non-Heading Elements for Headings: Styling text (e.g., using <p style="font-size: 2em; font-weight: bold;">) to look like a heading provides zero semantic information. Screen readers will just read it as a paragraph, and it won't appear in the heading list or be navigable using heading shortcuts. Always use actual <h1>-<h6> tags.
  4. Using Headings for Non-Heading Text: Applying <h1>-<h6> tags to regular text simply for visual styling (making it big/bold) clutters the document outline with irrelevant entries and makes navigation confusing. Use CSS for styling.

Headings and Sectioning Elements Together:

  • While the outline algorithm considers sectioning elements, screen readers still primarily rely on the heading levels themselves for user navigation commands.
  • Therefore, even when using <article> and <section>, maintaining a logical heading hierarchy within those sections (e.g., an <article> starting with <h2>, its subsections using <h3>) remains crucial for a good screen reader experience. This reinforces the recommendation to generally stick to one <h1> per page.
