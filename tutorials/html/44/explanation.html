44. DOCTYPE: Syntax Details

Files Modified: None

Let's look closely at the syntax: <!DOCTYPE html>

  • <! and >: These delimit the declaration. It looks like a tag but isn't technically an HTML element itself.
  • DOCTYPE: The keyword indicating this is a Document Type Declaration. Case-insensitive, but uppercase is conventional.
  • html: Specifies the type of document – in this case, HTML. For the HTML5 DOCTYPE, this is all that's needed. It's case-insensitive, but lowercase is conventional.

Key Rules:

  • Must be first: Nothing, not even whitespace or comments, can precede the DOCTYPE declaration on the first line.
  • Only one: A document must have exactly one DOCTYPE declaration.

It's simple, but getting it right and placing it correctly is the essential first step.
