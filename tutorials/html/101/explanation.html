101. Unordered Lists (<ul>): Behavior (Markers & CSS)

Let's revisit the <ul> element and discuss its default rendering behavior, specifically the list markers, and how CSS is used for styling.

Default Rendering Behavior Recap:

  • Block Element: <ul> creates a block formatting context.
  • Margins/Padding: Browsers add default vertical margins around the <ul> and default horizontal padding (usually padding-left) to indent the list items. The exact values vary between browsers.
  • Markers: The most distinctive feature is that browsers render a marker (typically a bullet point) before each <li> item within the <ul>.

Default Unordered List Markers:

  • The default marker style for the top level of a <ul> is usually a solid disc (•).
  • For nested <ul> elements (which we'll see later), the default marker often cycles through different styles, commonly:
    • Level 1: disc (•)
    • Level 2: circle (○)
    • Level 3+: square (▪︎) (This cycle can vary by browser).

Styling with CSS (Overview):

While browsers provide defaults, you should use CSS for reliable and customized list styling. Key CSS properties include:

  • list-style-type: Controls the marker style. For <ul>, common values are:
    • disc (default)
    • circle
    • square
    • none (removes the marker entirely - very common for navigation menus)
  • list-style-position: Controls whether the marker sits outside (default, in the padding area) or inside the list item's text flow.
  • list-style-image: Allows using a custom image as the marker (url(...)).
  • list-style: A shorthand for the above three properties.
  • padding-left: Often adjusted on the <ul> element to control the amount of indentation. (Resetting default browser padding is common for consistent results).
  • margin: Controls space around the entire list block.
  • ::marker: A pseudo-element for directly styling the marker itself (color, size, etc.).

Historical Note: Older HTML versions had a type attribute directly on the <ul> tag (<ul type="circle">). This attribute is deprecated in HTML5 and should not be used. Always use CSS for styling markers.
