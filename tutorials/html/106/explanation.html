106. Ordered Lists (<ol>): Adding the First Item (<li>)

Just like <ul>, the <ol> element requires <li> child elements to hold the actual list items. Let's add the first item to our ordered list.

Core Mechanics Recap:

  • Container: <ol> defines the ordered list context.
  • Item: <li> defines an individual item within that list.
  • Placement: <li> must be a direct child of <ol> (or <ul>).
  • Syntax: <ol> <li>...</li> </ol>

Code Change:

We have added the first <li> element inside the <ol> in index.html.

<ol>
  <li>First step</li> <!-- Added first li -->
  <!-- More list items will go here -->
</ol>

✅ Observation:

  1. Verify the presence of <li>First step</li> inside the <ol> in index.html.
  2. Look at the browser preview. This is where <ol> differs visibly from <ul>. You should see:
    • The text "First step".
    • A numerical marker, specifically the number 1. (or "1" followed by a period), appearing to the left of the text. This is the default marker for the first item in an <ol>.
    • The item will likely be indented, similar to items in a <ul>.

✨ Key Takeaway: Adding an <li> inside an <ol> creates the first item, which browsers render by default with the numerical marker "1.".
