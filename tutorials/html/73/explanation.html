73. Semantic HTML for Code & Preformatted Text: <code> and <pre>

HTML provides tags specifically for marking up computer code and preformatted text: <code> for inline code snippets and <pre> for blocks of text where whitespace (spaces, tabs, line breaks) is significant and should be preserved.

<code> and <pre> Explained

  • <code> (Code Fragment):
    • Semantic Meaning: Represents a short fragment of computer code (variable names, keywords, filenames, code snippets).
    • Default Styling: Typically rendered in the browser's default monospace font. Often styled further with CSS (background, padding) for better visual distinction.
  • <pre> (Preformatted Text):
    • Semantic Meaning: Represents a block of text in which structure is defined by typographic conventions (like fixed-width characters and significant whitespace) rather than HTML elements. Primarily used for code blocks, ASCII art, or any text where preserving exact spaces and line breaks is crucial.
    • Default Styling: Uses a monospace font and preserves whitespace and line breaks (white-space: pre; behavior). Often styled with CSS for background, borders, padding, and overflow handling.
  • <pre><code>...</code></pre> Combination: It's very common and recommended to wrap code blocks inside a <pre> element with a <code> element nested directly inside it. The <pre> handles the whitespace preservation and block formatting, while the <code> provides the semantic meaning that the content is code. This also provides a good hook for styling and syntax highlighting libraries.
  • Escaping HTML: If you need to display literal <, >, or & characters within your code examples inside <code> or <pre>, you must escape them using HTML entities:
    • < becomes &lt;
    • > becomes &gt;
    • & becomes &amp; (Notice this in the <code>&lt;code&gt;</code> examples in the first paragraph and the <code> block inside the <pre> block).

Syntax Highlighting:

  • While CSS can apply basic styles, achieving proper syntax highlighting (coloring different parts of the code like keywords, strings, comments) usually requires a JavaScript library (like Prism.js or highlight.js). These libraries typically look for the <pre><code> structure and add CSS classes to different parts of the code (tokens), which you then style with CSS (like the basic .language-css .token.* rules added as an example). We won't implement the JS part here, but the CSS provides a hint.
