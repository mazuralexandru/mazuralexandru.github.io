272. The id Attribute - Behavior & Interactions: Fragment Identifiers

We briefly encountered fragment identifiers when discussing links in Module 4. The id attribute is the key component that makes them work.

Mechanism Recap:

  1. Target Definition: You assign a unique id attribute to the element you want to be able to link to.
    <h2 id="section-two">Section Two</h2>
  2. Link Creation: You create an anchor (<a>) element where the href attribute value consists of a hash symbol (#) followed by the exact id value of the target element.
    <a href="#section-two">Go to Section Two</a>
  3. Browser Action: When the user clicks the link:
    • The browser looks for the element within the current document (if the href is just #id) or the target document (if the href is page.html#id) that has the matching id.
    • If found, the browser scrolls the page to bring that target element into view.
    • The URL in the address bar is usually updated to include the fragment identifier (e.g., current-page.html#section-two).

Purpose:

  • Enables deep linking directly to specific sections of a web page.
  • Facilitates intra-page navigation (e.g., tables of contents, "skip to content" links, "back to top" links).

Adding Elements for Demonstration:

To demonstrate this, we need:

  1. More content to make the page scrollable.
  2. A target element with an id.
  3. A link pointing to that id.

Code Change:

  1. Added temporary <style> for spacing (normally in style.css).
  2. Added a second heading (<h2>) with id="content-section".
  3. Added spacer divs and extra paragraphs to create scrollable height.
  4. Added a "Skip to Content" link near the top targeting #content-section.
<!-- index.html -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>HTML Global Attributes - ID</title> <!-- Updated title -->
  <link rel="stylesheet" href="style.css">
  <style>
    /* Temp styles for scrolling demo */
    .spacer { height: 60vh; border-bottom: 1px dashed #555; margin-bottom: 1em; }
    #content-section { margin-top: 1em; }
    nav a { margin-right: 15px;}
  </style>
</head>
<body>
  <nav> <!-- Added nav for link placement -->
    <a href="#content-section">Skip to Content</a> <!-- Link to fragment -->
  </nav>

  <h1 id="main-page-title">Global Attributes - ID & Fragments</h1> <!-- Updated H1 -->
  <p>The 'id' attribute enables linking directly to page sections (fragments).</p> <!-- Adjusted P -->

  <div class="spacer">Scroll down to find the target section...</div>

  <h2 id="content-section">Main Content Section</h2> <!-- Target element with id -->
  <p>This is the section the "Skip to Content" link should jump to.</p>
  <p>Adding more text here just to ensure there is enough content below the target point for the scroll effect to be noticeable.</p>
  <p>Without sufficient content below, the browser might not appear to scroll much as the target is already near the bottom of the viewport.</p>

✅ Observation:

  1. Examine the added elements: the <nav> with the link href="#content-section", the spacer divs, and the <h2> with id="content-section".

✨ Key Takeaway: The id attribute serves as the target for fragment identifier links (href="#id"), enabling direct navigation to specific elements within a page. This is crucial for accessibility ("skip links") and user navigation (tables of contents).
