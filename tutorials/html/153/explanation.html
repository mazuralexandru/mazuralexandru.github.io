153. href URL Types: Absolute URLs (Definition & Syntax)

The first type of URL we'll use in the href attribute is an absolute URL.

Definition:

  • An absolute URL provides the full, complete web address required to locate a resource on the internet. It contains all the information needed, independent of the current page's location.

Syntax / Structure:

An absolute URL typically follows this structure (though some parts are optional):

scheme://domain:port/path?query#fragment

Let's break that down:

  1. scheme: The protocol used to access the resource. Common schemes for web links are:
    • https (HyperText Transfer Protocol Secure - Preferred for web)
    • http (HyperText Transfer Protocol - Less secure)
    • Other schemes exist like ftp:, file:, etc., but http/https are most common for web links.
  2. ://: Standard separator characters following the scheme.
  3. domain: The domain name (e.g., www.example.com, developer.mozilla.org) or IP address of the server hosting the resource.
  4. :port (Optional): The specific network port number on the server. Usually omitted as standard ports (80 for HTTP, 443 for HTTPS) are assumed.
  5. /path (Optional): The specific path to the resource (file or directory) on the server, starting from the server's root for that domain (e.g., /docs/Web/HTML/Element/a).
  6. ?query (Optional): A query string, starting with ?, used to pass parameters to the server (e.g., ?search=html&page=2). Parameters are often key=value pairs separated by &.
  7. #fragment (Optional): A fragment identifier, starting with #, pointing to a specific section (element with a matching id) within the target HTML document.
