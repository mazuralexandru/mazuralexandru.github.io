390. src Attribute: Path & Formats

Files Modified: None

The most critical attribute for the <img> element is src (source).

  • Purpose: Specifies the path or URL to the image file that should be displayed.
  • Syntax: src="path/to/image.format"
  • Value Types: Similar to the href attribute for links:
    • Absolute URL: A full web address (e.g., src="https://example.com/images/logo.png"). Used for images hosted on other servers or CDNs.
    • Relative URL: A path relative to the current HTML file's location (e.g., src="images/photo.jpg", src="../assets/icon.svg"). Most common for project assets.
    • Data URLs: Embed image data directly (less common, generally only for very small images).
  • Common Image Formats:
    • JPEG/JPG: Best for photographs, complex images with many colors. Lossy compression (quality loss for smaller size).
    • PNG: Good for graphics with transparency, logos, icons, screenshots. Lossless compression (no quality loss, potentially larger files than JPEG).
    • GIF: Best for simple animations. Limited color palette.
    • SVG: Vector format. Scalable without quality loss. Ideal for logos, icons, illustrations. Defined using XML.
    • WebP: Modern format offering both lossy and lossless compression, often smaller file sizes than JPEG/PNG with good quality. Excellent browser support.
    • AVIF: Newer format, often providing even better compression than WebP, especially for HDR images. Browser support is growing rapidly.
  • Optimization: It's crucial to optimize images for the web:
    • Resize: Serve images at the dimensions they will be displayed, don't rely on the browser to shrink huge images.
    • Compress: Use appropriate formats and compression tools (online or offline) to reduce file size without significant visual degradation. Smaller files mean faster page loads.

Choosing the right format and optimizing images are key aspects of web performance.
