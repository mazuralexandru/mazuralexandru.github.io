304. Margin Collapsing: Scenarios - Adjacent Siblings

Files Modified: index.html, style.css

The most common scenario where margin collapsing occurs is between adjacent sibling elements.

  • Scenario: The bottom margin of an earlier block element touches the top margin of the immediately following block element (assuming no clearance, padding, or borders separate them).
  • Result: The two margins collapse into a single margin whose size is the maximum of the two original margins.

Let's illustrate this.

✅ Observe index.html:

  • We've simplified the content to have two adjacent paragraphs, p1 and p2.

✅ Switch to style.css:

  • We've removed the previous .box styles.
  • We've added rules for .p1 and .p2.
    • .p1 has margin-bottom: 30px;.
    • .p2 has margin-top: 20px;.
  • We added background colors to make them distinct.
/* Margin Collapse Example: Adjacent Siblings */
.p1 {
  background-color: rgba(0, 255, 0, 0.2); /* Faint green */
  margin-bottom: 30px; 
  padding: 10px;
}

.p2 {
  background-color: rgba(255, 165, 0, 0.2); /* Faint orange */
  margin-top: 20px;
  padding: 10px;
}

/* Expected space between p1 and p2 will be max(30px, 20px) = 30px, NOT 30px + 20px = 50px */

✨ Observe the Preview:

  • You see two paragraphs.
  • Measure or estimate the vertical space between the green box (.p1) and the orange box (.p2).
  • This space should correspond to 30px (the larger of the 30px bottom margin of .p1 and the 20px top margin of .p2), not 50px. This is margin collapsing in action.
