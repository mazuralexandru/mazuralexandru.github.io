405. Module 8 Recap: Structuring Tabular Data with Tables

This module focused on using HTML table elements correctly to structure and present tabular data semantically and accessibly.

Key Concepts Reviewed:

  • Purpose of Tables: Exclusively for structuring tabular data (information logically organized in rows and columns).
  • Data vs. Layout (CRITICAL): Never use tables for page layout; use CSS (Grid, Flexbox, etc.). Layout tables are inaccessible and outdated.
  • Core Structure:
    • <table>: The main container for the entire table.
    • <tr>: Defines a table row. Contains <th> or <td> cells.
    • <td>: Defines a standard data cell. Can contain flexible flow content.
    • <th>: Defines a header cell (for columns or rows). Semantically distinct from <td>. Browsers usually style bold/centered.
  • Header Association (scope):
    • The scope attribute on <th> is essential for accessibility.
    • scope="col": Header applies to cells below it in the column.
    • scope="row": Header applies to cells to its right in the row.
    • Explicitly defines header-data relationships for screen readers.
  • Structural Grouping:
    • <thead>: Groups header row(s). Max one per table, must appear first (after <caption>).
    • <tbody>: Groups data row(s). One or more allowed. Must be used if <thead> or <tfoot> is present. Implicitly created if no grouping elements are used.
    • <tfoot>: Groups footer/summary row(s). Max one per table. Appears after <thead>, renders at the bottom.
    • Benefits: Semantic structure, styling hooks, potential scrolling/printing features.
  • Table Caption (<caption>):
    • Provides a title/description for the entire table.
    • Essential for accessibility (provides the table's accessible name).
    • Must be the first child element inside <table>.
  • Spanning Cells:
    • colspan="number": Makes a <td> or <th> span horizontally across number columns. Requires adjusting cell counts in that row.
    • rowspan="number": Makes a <td> or <th> span vertically down number rows. Requires adjusting (omitting) cells in subsequent rows.
    • Use sparingly; increases complexity. Crucial to maintain clear header associations (consider headers attribute for very complex spans).
  • Styling: Use CSS for all visual presentation (borders, padding, alignment, colors, etc.). Avoid deprecated presentational HTML attributes.

Core Principles Reinforced:

  • Semantics First: Use table elements for their intended purpose (tabular data).
  • Accessibility: Prioritize <caption>, <th>, and scope for screen reader users.
  • Structure over Presentation: Build the correct HTML structure before applying CSS styles. Separate concerns.
