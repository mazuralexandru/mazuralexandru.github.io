01. The Minimal HTML Document

Welcome to your web development journey! We start with the absolute minimum structure required for a valid HTML document.

What is HTML?

HTML stands for HyperText Markup Language. It's the standard language used to create the structure and content of web pages. Think of it as the skeleton of a webpage.

  • Markup: HTML uses special markers called "tags" (like <html> or <body>) to describe different parts of the content.
  • Not Programming: HTML describes structure, it doesn't perform complex logic like a programming language.

The Files:

  • index.html: This is the core file the browser reads to display the page. The name index.html is special; web servers often look for this file by default when someone visits a website directory.
  • style.css: This file will hold our styling rules (CSS - Cascading Style Sheets) to make the page look good. It's empty for now.
  • script.js: This file will hold our JavaScript code for adding interactivity. It's also empty.

Code Breakdown (index.html):

  • <!DOCTYPE html>: This is the Document Type Declaration. It must be the very first thing in the file. It tells the browser, "This document is written in HTML5," ensuring the browser uses modern rendering standards (avoids "quirks mode").
  • <html>: This is the root element. All other HTML elements go inside this tag. It signifies the start and end of the HTML document.
  • <head>: This element contains metadata about the HTML document â€“ information about the page, not the visible content itself. Examples include the page title, character set declaration, links to stylesheets, etc. It's currently empty.
  • <body>: This element contains all the visible content of the web page â€“ the text, images, links, etc., that the user sees in the browser window. It's also empty right now.
