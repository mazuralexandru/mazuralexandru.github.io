406. Image Spacing (margin)

Files Modified: style.css

Regardless of whether an image is inline, inline-block, or block, you can use the margin property (or its longhand versions) to create space around the image, pushing other elements away.

  • Purpose: Controls the external spacing around the image element's box (outside any border).
  • Applicability:
    • Horizontal margins (margin-left, margin-right) work reliably on inline, inline-block, and block elements.
    • Vertical margins (margin-top, margin-bottom) are generally only effective on block-level or inline-block elements; they don't typically affect line height spacing for purely inline elements.

✅ Observe style.css:

  • We are currently using margin: 1em auto; on the .block-image. This gives it 1em of margin on the top and bottom, and auto on the left and right (for centering).
  • Let's change the shorthand to give it different vertical and horizontal margins.
/* Block Image Alignment */
.block-image {
    display: block; 
    max-width: 100%; 
    height: auto; 
    width: 300px; 
    /* margin: 1em auto; <- Changed from this */
    margin: 2em auto 3em auto; /* 2em Top, Auto R, 3em Bottom, Auto L */
    border: 2px solid lightgreen; 
}

✅ Switch to index.html to see the HTML structure.

✨ Observe the Preview:

  • The image should still be centered horizontally (due to auto left/right margins).
  • The vertical space above the image should have increased to 2em.
  • The vertical space below the image (before the paragraph "Some text following...") should have increased significantly to 3em.

Margins are the standard way to control spacing between elements.
