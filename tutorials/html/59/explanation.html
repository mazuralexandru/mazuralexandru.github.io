59. Text Styling: font Shorthand Property

Instead of setting font-style, font-variant, font-weight, font-size, line-height, and font-family individually, you can use the font shorthand property to set several of them in a single declaration.

font Shorthand Explained

  • Purpose: Combines multiple font-related properties into one line for brevity.
  • Syntax: font: [font-style] [font-variant] [font-weight] [font-stretch] font-size[/line-height] font-family;
    • Values in square brackets ([]) are optional.
    • font-stretch is less common, controls condensation/expansion of font.
    • CRITICAL: font-size and font-family must be specified.
    • CRITICAL: The order shown above must be followed for the values you include.
    • If line-height is included, it must come immediately after font-size, separated by a forward slash (/).
    • Any omitted optional values are reset to their initial values (e.g., normal for style/variant/weight/stretch, implementation-specific default for line-height). This means using the shorthand can unintentionally reset properties you didn't explicitly include.

In Our Example:

  • font: italic bold 1.3rem/1.6 "Roboto", sans-serif; sets:
    • font-style: italic
    • font-variant: normal (reset to initial)
    • font-weight: bold
    • font-stretch: normal (reset to initial)
    • font-size: 1.3rem
    • line-height: 1.6 (unitless)
    • font-family: "Roboto", sans-serif

Caution with Shorthand:

Because omitted values are reset to their initial defaults, using the font shorthand can sometimes have unexpected side effects if you were relying on inherited values for, say, font-weight or font-style. Be mindful of what it resets. Often, setting properties individually provides more explicit control.
