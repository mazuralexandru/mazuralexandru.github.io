514. Sectioning Content: <section> Element - Fundamentals

While <article> represents a whole, self-contained piece, the <section> element is used to group content thematically within a larger context.

Definition & Purpose:

  • The <section> element represents a generic thematic grouping of content.
  • It's used when a part of the document can be considered a distinct section based on its theme or topic, but doesn't warrant a more specific semantic element (like <article>, <nav>, or <aside>).

Core Concept & Heading Requirement:

  • Each <section> should logically group related content.
  • Crucially, a <section> should almost always begin with a heading (<h1>-<h6>) that identifies the section's topic. This heading provides the semantic label for the section. Sections without headings lose much of their semantic value and can be confusing for assistive technology users trying to understand the document outline.

Placement & Structure:

  • Can be used within <body>, <main>, <article>, or even nested within other <section>s to create subsections.
  • Syntax: <section> <h2>Section Title</h2> ... related content ... </section>
  • Content: Flow content.

Accessibility:

  • A <section> element maps to the generic ARIA region landmark role only if it has an accessible name (provided by a heading linked via aria-labelledby or directly via aria-label). Without an accessible name, it doesn't typically expose a distinct landmark role.
  • Its primary accessibility benefit comes from structuring the content under clear headings, contributing to the document outline used for navigation.

Distinctions:

  • vs. <article>: Use <section> for thematic groups within a larger piece (like chapters in an article). Use <article> for the entire, self-contained piece.
  • vs. <div>: Use <section> only when the grouping is thematic and logically part of the document outline (and has a heading). Use <div> for non-semantic grouping purely for styling or layout. Don't just use <section> as a generic wrapper instead of <div>.

Code Change:

We've added a <section> element inside the <article> in index.html, representing a specific part of the blog post. Note the required <h3> heading inside it.

<!-- index.html snippet -->
<head>
  ...
  <style>
    ...
    article header h2, section h3 { margin: 0 0 0.5em 0; } /* Basic heading style */
  </style>
</head>
<body>
  ...
  <main>
    ...
    <article>
      ...
      <p>This is the introductory paragraph of the blog post...</p>

      <!-- Added section element -->
      <section>
        <h3>Subsection: Key Features</h3> <!-- Heading REQUIRED for section -->
        <p>This section details the key features discussed in the blog post.</p>
        <ul>
          <li>Feature A</li>
          <li>Feature B</li>
        </ul>
      </section>

      <p>Concluding paragraph for the blog post...</p>
    </article>
  </main>
  ...
</body>

✅ Observation:

  1. Locate the <section> nested inside the <article>. Note the <h3> directly inside it.
  2. In the browser preview, the "Subsection: Key Features" content appears within the main article. The <section> itself might gain a dashed border from our temporary styles, showing it as a distinct block within the article's border.

✨ Key Takeaway: Use <section> to group content thematically within a document or article. Always provide a heading (<h1>-<h6>) to identify the section's topic. Use it only for semantic grouping, not as a generic replacement for <div>.
