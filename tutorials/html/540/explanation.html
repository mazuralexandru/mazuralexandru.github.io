540. Native Multimedia: Contrast with <img> and <iframe>

It's useful to distinguish <video> and <audio> from other embedding elements we've already discussed.

<img> vs. <video>/<audio>:

  • <img>: Embeds static raster or vector images (JPEG, PNG, GIF, SVG, WebP, AVIF). It represents a single visual frame. Void element.
  • <video>/<audio>: Embed non-static, time-based media (video with/without sound, standalone audio). They represent content that progresses over time and typically have playback controls. Not void elements (they have fallback content).

<iframe> vs. <video>/<audio>:

  • <iframe>: Embeds an entire, independent HTML document within the current page. This embedded document might contain <video> or <audio> elements (like a YouTube embed page), but the <iframe> itself isn't the media element. It creates a nested browsing context.
  • <video>/<audio>: Embed the media stream directly into the current document's context, allowing for more direct control via JavaScript (within same-origin policy) and integration with the parent page's accessibility features (like tracks).

When to Choose Which:

  • Need to display a static picture? Use <img>.
  • Need to play a video file hosted on your server or a compatible source? Use <video>.
  • Need to play an audio file? Use <audio>.
  • Need to embed a complex widget, interactive application, or player from a third-party service (like YouTube, Vimeo, Google Maps) that provides an embeddable page URL? Use <iframe> (with title and sandbox).

✅ Observation: No code changes. Solidify the distinctions: <img> for static visuals, <video>/<audio> for direct playback of time-based media files, <iframe> for embedding separate HTML documents (which might contain media).

✨ Key Takeaway: Use <video> and <audio> for direct embedding and playback of video/audio files. Use <img> for static images. Use <iframe> for embedding external documents or complex third-party players.
