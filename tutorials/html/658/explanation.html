658. Validation: What Validation Checks - Element/Attribute Validity

Validators also check if the specific elements and attributes you use are valid according to the HTML standard and if their values are appropriate.

2.3. Element & Attribute Validity Checks:

  • Valid Element Names: Ensures that the tags used correspond to actual elements defined in the HTML specification (e.g., <p>, <img>, <section>). Flags typos or use of non-standard/made-up tags.
    • Example Error: Error: Element "paragaph" not allowed as child of element "body". (Typo in tag name)
  • Valid Attribute Names: Checks if an attribute used on a specific element is either a global attribute or an attribute specifically allowed for that element type.
    • Example Error: Error: Attribute "href" not allowed on element "p" at this point.
  • Required Attributes: Verifies that elements requiring certain attributes for validity and function actually have them.
    • Example Error: Error: Element "img" is missing required attribute "alt".
    • Example Error: Error: Element "track" is missing required attribute "kind".
  • Valid Attribute Values: Checks if the value provided for an attribute conforms to the expected format or belongs to a predefined set of allowed values (enumeration).
    • Example Error: Error: Bad value "center" for attribute "scope" on element "th". (Should be col, row, etc.)
    • Example Error: Error: Bad value "ja_JP" for attribute "lang": Syntax of language tag "ja_JP" is not valid. (Should use hyphen: ja-JP)
    • Example Error: Error: Bad value "-10" for attribute "colspan" on element "td": Must be greater than zero.
  • Boolean Attributes: Ensures boolean attributes (like required, checked, disabled, readonly) are used correctly (just the name, no value needed in HTML5). Flags incorrect usage like required="true".
    • Example Warning/Error: Error: Attribute "required" should not have value "true". (Depending on validator strictness)

These checks ensure that you are using the building blocks of HTML correctly and providing necessary information (like alt text) or valid configurations (like scope values).

✅ Observation: No code changes. Recall attributes like alt on <img>, scope on <th>, type on <input>, method on <form>. Validators check that these attributes are present when required, allowed on the element they're used on, and have valid values from the permitted set.

✨ Key Takeaway: Validation verifies the existence and correctness of element names, attribute names, required attributes, and attribute values according to the HTML specification.
