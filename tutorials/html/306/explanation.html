306. Metadata: The <meta> Element - Core Mechanics

Let's examine the core syntax and the different attribute combinations used to define metadata with the <meta> element.

Basic Syntax:

  • As a void element, it consists of just the tag name and its attributes: <meta attribute1="value1" ... >

Key Attribute Patterns:

There are three main ways attributes are combined on <meta> tags to specify different kinds of metadata:

  1. charset Attribute:

    • Purpose: Declares the character encoding of the HTML document.
    • Syntax: <meta charset="encoding-name">
    • Example: <meta charset="UTF-8"> (This is the standard and preferred method for encoding declaration).
    • Uniqueness: A document must not contain more than one <meta> element with a charset attribute.

  2. name and content Attributes (Used Together):

    • Purpose: Provides document-level metadata in the form of name-value pairs. Used for descriptions, keywords, author info, viewport settings, robot instructions, etc.
    • Syntax: <meta name="metadata-name" content="metadata-value">
    • name: Defines the type or name of the metadata (e.g., description, keywords, author, viewport, robots). The available names include standardized values and can also be extended.
    • content: Specifies the actual value or content associated with the name.
    • Example: <meta name="description" content="Learn about HTML metadata...">

  3. http-equiv and content Attributes (Used Together):

    • Purpose: Used to simulate an HTTP response header. This is sometimes called a "pragma directive".
    • Syntax: <meta http-equiv="header-name" content="header-value">
    • http-equiv: Specifies the name of the HTTP header being simulated (e.g., content-type, refresh, default-style, x-ua-compatible).
    • content: Specifies the value for the simulated header.
    • Usage: Less common for standard metadata now. <meta charset> is preferred over <meta http-equiv="Content-Type">. <meta http-equiv="refresh"> (for auto-redirecting or refreshing) is generally discouraged due to poor usability and accessibility. <meta http-equiv="X-UA-Compatible" content="IE=edge"> was used historically to control rendering modes in older versions of Internet Explorer.

Placement:

  • All <meta> tags must be placed within the <head> section.
  • As noted previously, the <meta charset="UTF-8"> tag should ideally be the very first element within <head>. The order of other <meta> tags (like description or viewport) is generally less critical, though placing viewport early is also common practice.
