205. Link Content: Image Links and alt Text

Sometimes, the entire content of a link is an image (<img>) rather than text. This is common for logos linking to a homepage or icons representing actions.

Syntax:

<a href="destination.html">
  <img src="path/to/image.png" alt="Crucial description here">
</a>

Accessibility Challenge:

If there is no text within the <a> tags, a screen reader needs alternative text to understand what the link is for. Simply describing the image visually might not be enough.

The Role of alt Text:

  • When an <img> element is the sole content of an <a> element, the alt attribute of the image must describe the destination or function of the link, not just the image itself.
  • The screen reader will typically announce the alt text as the link's accessible name.

Good vs. Bad alt Text for Image Links:

  • Scenario: A company logo image (logo.png) links to the homepage (/).
    • BAD: alt="Example Corp Logo" (Describes the image, but not where the link goes). A screen reader would announce "Link: Example Corp Logo".
    • GOOD: alt="Example Corp Homepage" or alt="Go to Example Corp Homepage" (Describes the link's destination/function). A screen reader would announce "Link: Example Corp Homepage".
