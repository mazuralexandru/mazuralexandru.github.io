80. Other Inline Semantics: <small> Best Practices

While <small> is straightforward, it's important to use it correctly according to its semantic purpose.

Use Cases:

As mentioned, the primary intended uses for <small> are for content that is supplementary or legally required fine print:

  • Copyright statements: (As shown in the <footer> example).
  • Disclaimers: Short notices limiting liability or scope.
  • Legal attributions: Citing required licenses or sources in brief.
  • Side comments: Text that is tangential to the main flow, like a brief annotation.

Anti-Pattern: Using <small> for Visual Size Reduction

  • Mistake: Using the <small> tag simply because you want a piece of text to appear smaller, without it actually representing small print or a side comment semantically.
  • Why it's Wrong: This is misusing a semantic element for purely presentational purposes. It adds incorrect meaning to the document structure.
  • Correct Approach: If you need text to be smaller for visual design reasons only (and it doesn't fit the semantic definition of <small>), use CSS. Apply the font-size property (e.g., font-size: 0.8em; or font-size: 12px;) using a CSS class or by targeting the specific element directly.

Example (Conceptual):

  • Incorrect: <p>Main text and <small>just some smaller text</small>.</p> (if "just some smaller text" isn't actually small print).
  • Correct (using CSS):
    • HTML: <p>Main text and <span class="visually-smaller">just some smaller text</span>.</p>
    • CSS: .visually-smaller { font-size: 0.8em; }

✅ Observation: No code changes. Review the examples in index.html. The text "small print or side comments" fits the definition, and the copyright notice in the <footer> is a classic, appropriate use case for <small>.

✨ Key Takeaway: Use <small> semantically for actual small print, side comments, or legal notices. For purely visual font size reduction, use the CSS font-size property.
