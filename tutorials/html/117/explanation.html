117. File Organization (Folders)

Files Modified: None

For anything beyond a single-page site, organizing your files into folders (directories) is essential for maintainability.

Common Structures:

  • Simple:
project-root/
├── index.html
├── about.html
├── styles.css
└── script.js
  • Using Folders:
project-root/
├── index.html
├── about.html
├── contact.html
├── css/ 
│   ├── base.css
│   └── main.css
├── js/
│   ├── vendor/ 
│   │   └── jquery.js 
│   └── main.js
└── images/
    ├── logo.png
    └── banner.jpg

Benefits:

  • Clarity: Separates different types of files (HTML, CSS, JS, images).
  • Scalability: Easier to manage as the project grows.
  • Maintainability: Finding specific files is much easier.

Impact on Paths: When using folders, remember that the href attribute in <link> tags and the src attribute in <img> or <script> tags need to reflect the correct relative path.

  • Example: If main.css is inside a css folder, the link in index.html (which is in the root) would be <link rel="stylesheet" href="css/main.css">.

In our current setup, all files (index.html, base_styles.css, style.css) are assumed to be in the same directory, so the paths are just the filenames.

✅ Our current lesson structure keeps things simple for now, with all files at the same level.
