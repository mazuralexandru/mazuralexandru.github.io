31. Using Attribute Selectors: Presence

The simplest attribute selector targets elements based merely on the presence of an attribute, regardless of its value. Let's use this to style the input field that has the required attribute.

What's New?

  • CSS Pruning: We've commented out the rules for h2 and #main-content as those elements were removed.
  • CSS Addition: We added a new CSS rule using the presence attribute selector [required] to style.css.
/* style.css snippet */
/* Removed H2 rule as element was pruned */
/* Removed ID Selector rule as element was pruned */

/* Added Presence Attribute Selector rule */
[required] {
  border: 2px solid orange;
  background-color: #444; /* Darker background for input */
}

/* CSS files can also have comments! */

Attribute Presence Selector ([attribute])

  • Syntax: The attribute name enclosed in square brackets [] (e.g., [required], [disabled], a[target]).
  • Scope: Selects all elements that have the specified attribute present, no matter what its value is (or even if it's a boolean attribute like required that doesn't technically need a value).
  • Use Cases: Styling elements based on their state (e.g., disabled buttons, required fields), targeting elements that have a specific attribute set (like links with targets).

In our example, [required] targets any element with the required attribute. We're applying an orange border and a darker background color to it.
