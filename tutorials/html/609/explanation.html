609. Graphics: SVG (<svg>) - Grouping (<g>) Element

To organize complex SVG graphics and apply styles or transformations to multiple shapes simultaneously, SVG provides the <g> (group) element.

The <g> Element:

  • Purpose: Acts as a container to group related graphical elements (<rect>, <circle>, <path>, <text>, other <g> elements, etc.).
  • Core Concept: It doesn't draw anything itself, but attributes and styles applied to the <g> element are inherited by its child elements.
  • Benefits:
    • Organization: Structures complex drawings logically.
    • Styling: Apply common fill, stroke, opacity, font-size, etc., to the group, and children will inherit unless they override it.
    • Transformation: Apply transformations (transform="translate(...) rotate(...) scale(...)") to the <g> element, and all children within the group will be transformed together relative to the group's origin.
    • Targeting: Easier to target related elements with CSS (g.my-component circle) or JavaScript.

Syntax:

  • <g ...attributes...> ... child elements ... </g>
  • Requires opening and closing tags.

Code Change:

  1. Removed the <text> element from the previous example.
  2. Adjusted the <svg> dimensions.
  3. Added a <g> element with an id="my-group".
  4. Placed a <rect> and a <circle> inside the <g>.
  5. Added a CSS rule targeting #my-group to apply fill, stroke, and a transform.
<!-- index.html -->
<head>
  ...
  <style>
    svg {
      border: 1px dashed skyblue;
      margin: 10px;
      stroke-width: 2;
       /* Removed text styles */
    }
    /* Styles for the group */
    #my-group {
        fill: lightcoral; /* Applied to group */
        stroke: darkred; /* Applied to group */
        transform: translate(20px, 10px) rotate(15); /* Apply transform to group */
    }
  </style>
</head>
<body>
  <h1>Inline SVG Example</h1>
  <p>Grouping elements with &lt;g&gt;.</p>

  <svg width="200" height="100" viewBox="0 0 200 100" xmlns="http://www.w3.org/2000/svg">
     <!-- Removed text element -->

     <!-- Added g element with shapes inside -->
     <g id="my-group"> <!-- Added group -->
       <rect x="10" y="10" width="60" height="40"/> <!-- Inherits fill/stroke from group -->
       <circle cx="100" cy="50" r="20"/> <!-- Inherits fill/stroke from group -->
     </g>

  </svg>

  <p>Styles and transforms applied to 'g' affect its children.</p>
  ...
</body>

✅ Observation:

  1. Locate the <g id="my-group"> element wrapping the <rect> and <circle>.
  2. Examine the CSS rule for #my-group, noting the fill, stroke, and transform properties.
  3. Look at the browser preview:
    • You should see both the rectangle and the circle rendered.
    • Both shapes should be filled with light coral and have a dark red outline, inheriting these styles from their parent <g> element via the CSS rule.
    • The entire group (rectangle and circle together) should be shifted right 20px, down 10px, and rotated slightly (15 degrees clockwise around the origin 0,0) due to the transform applied to the <g> element.

✨ Key Takeaway: Use the <g> element to group related SVG elements. This simplifies applying shared styles (fill, stroke, etc.) and transformations (transform) which are inherited by the child elements.
