605. Graphics: SVG (<svg>) - The <svg> Root Element

Let's start building an SVG graphic by adding the root <svg> element directly into our HTML (inline SVG).

The <svg> Element:

  • Purpose: Acts as the container for all elements defining an SVG graphic when embedded inline or as the root element of an .svg file. It establishes the viewport (the display area) and the internal coordinate system.
  • Syntax: <svg ...attributes...> ... SVG content (shapes, text, etc.) ... </svg>
  • Requires opening and closing tags.

Key Attributes for <svg> Root:

  1. width & height:
    • Specify the size of the SVG viewport as rendered on the HTML page.
    • Can use CSS units (px, em, %) or be unitless numbers (interpreted as pixels).
    • Similar to <img> dimensions, defines the space the SVG occupies.
  2. viewBox="min-x min-y width height" [Crucial for Scaling]:
    • Defines the boundaries and aspect ratio of the SVG's internal coordinate system. It specifies a rectangular region in user space that is mapped onto the viewport defined by width/height.
    • min-x, min-y: Coordinates of the top-left corner of the viewbox.
    • width, height: The width and height of the viewbox in user units. These numbers define the scale of your drawing coordinates.
    • Example: viewBox="0 0 100 50" means the internal coordinate system goes from (0,0) to (100,50). If the <svg> element has width="200" height="100", the internal 100x50 drawing will be scaled up 2x to fit the 200x100 viewport.
    • Using viewBox allows you to draw using convenient internal coordinates and have the browser automatically scale the graphic to fit the width/height attributes, maintaining aspect ratio by default.
  3. preserveAspectRatio [Optional]: Controls alignment and scaling if the viewBox aspect ratio doesn't match the width/height aspect ratio. (Defaults to xMidYMid meet - center the graphic and scale it down to fit entirely).
  4. xmlns="http://www.w3.org/2000/svg" [Often Included]: XML namespace declaration. Technically required for XML validity, browsers usually handle inline SVG in HTML5 without it, but including it is good practice for robustness and tool compatibility.

Code Change:

We've added a basic, empty inline <svg> element to index.html with width, height, and viewBox. We've also added temporary CSS to visualize its boundary.

<!-- index.html -->
<head>
  ...
  <style>
    svg { /* Basic SVG styling */
      border: 1px dashed skyblue;
      margin: 10px;
    }
  </style>
</head>
<body>
  <h1>Inline SVG Example</h1> <!-- Updated H1 -->
  <p>Placing &lt;svg&gt; markup directly in HTML.</p>

  <!-- Added Inline SVG -->
  <svg width="100" height="100" viewBox="0 0 100 100" xmlns="http://www.w3.org/2000/svg">
     <!-- Shapes will go inside here -->
     <!-- Title and Desc for accessibility added later -->
  </svg>

  <p>Requires width, height attributes. ViewBox controls coordinate system.</p>
  ...
</body>

✅ Observation:

  1. Locate the <svg> element in index.html. Note the width, height, and viewBox attributes. (Here, the aspect ratios match).
  2. Look at the browser preview. You should see an empty square box (100x100 pixels) with a dashed sky blue border. This is the viewport defined by the <svg> element, ready for drawing inside.

✨ Key Takeaway: The <svg> element is the container for SVG graphics. Use width and height to set its display size and viewBox to define the internal coordinate system and enable scaling.
