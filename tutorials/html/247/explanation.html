247. Syntax Highlighting (JS Libraries)

Files Modified: None

While our CSS makes the code block look like a code block, the code itself is still plain text. For enhanced readability, especially with complex code, syntax highlighting (coloring different parts of the code like keywords, strings, comments, etc.) is highly desirable.

Syntax highlighting is not typically done with pure CSS. It requires analyzing the code structure and applying different styles (usually via CSS classes) to different tokens within the code. This analysis is almost always performed by JavaScript libraries.

Common JavaScript Syntax Highlighting Libraries:

  • Prism.js: Popular, lightweight, extensible, supports many languages. Uses semantic HTML5 (<pre><code class="language-xyz">...</code></pre>) and applies classes like .keyword, .string, etc.
  • highlight.js: Another very popular library, automatically detects language, supports many themes.
  • Shiki: Generates highly accurate highlighted HTML on the server or at build time, often used in static site generators and documentation tools.

How they generally work:

  1. Include the library's JavaScript and CSS theme file(s) in your HTML page.
  2. Wrap your code blocks in the expected HTML structure (often <pre><code class="language-yourlanguage">...</code></pre>). Remember to escape HTML within the code!
  3. Initialize the library with JavaScript (e.g., Prism.highlightAll();).
  4. The JavaScript library finds the code blocks, parses the code, wraps different parts (tokens) in <span> tags with specific CSS classes, and the library's CSS theme file provides the colors and styles for those classes.

Implementing full syntax highlighting is beyond the scope of this basic HTML/CSS section, but it's important to know that it's usually achieved with JavaScript libraries working in conjunction with CSS themes.
