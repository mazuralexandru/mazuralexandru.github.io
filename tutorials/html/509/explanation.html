509. Landmarks: <main> Element - Fundamentals

The <main> element identifies the primary, dominant content area of the document.

Definition & Purpose:

  • The <main> element represents the main content of the <body> of a document or application.
  • It encapsulates content that is unique to that specific page and directly relates to its central topic or core functionality. It excludes content that is repeated across pages, such as site headers, footers, primary navigation, and site-wide sidebars (unless the sidebar is the main content, which is rare).

Scope & Placement:

  • Uniqueness: There should only be one non-hidden <main> element per document.
  • Placement: It should not be a descendant of an <article>, <aside>, <footer>, <header>, or <nav> element. It represents the primary content area, separate from these other landmark or sectioning elements. It's usually a direct child of <body> or perhaps within a top-level layout wrapper <div>.

Core Mechanics:

  • Block-level element.
  • Syntax: <main> ... primary page content ... </main>
  • Content: Flow content. Contains the core articles, sections, forms, etc., that make up the page's unique purpose.

Accessibility:

  • Maps to the ARIA main landmark role.
  • This is extremely important for accessibility. It provides the target for "skip to main content" links (allowing keyboard and screen reader users to bypass introductory/navigation blocks) and lets users jump directly to the core content area using landmark navigation shortcuts.

Code Change:

We've added a <main> element after the <header> in index.html, containing placeholder content (<h2>, <p>).

<!-- index.html snippet -->
<body>
  <header>
    ... Site Header Content ...
  </header>

  <!-- Added main element -->
  <main>
    <h2>Main Content Area</h2>
    <p>This is where the primary, unique content of the page resides, distinct from headers, footers, and navigation.</p>
  </main>

  <!-- Footer will go here -->
  ...
</body>

✅ Observation:

  1. Locate the <main> element placed after <header> in index.html.
  2. In the browser preview, the content within <main> ("Main Content Area" heading and paragraph) appears below the header content. It should also have the dashed sky blue border applied by our temporary CSS, visually identifying it as a distinct landmark region.

✨ Key Takeaway: Use the <main> element (only one per page) to wrap the primary, unique content of the document. It provides a crucial main landmark for accessibility ("skip to content" links, landmark navigation). Do not nest it within other landmarks like <header> or <nav>.
