281. Reset: Preparing for style Attribute

We have explored the class attribute and its primary role in linking HTML structure to CSS styles and JavaScript behavior.

Next, we will discuss the style attribute, which allows applying CSS directly within an HTML tag (inline styles). While generally discouraged, it's important to understand how it works and its limited use cases.

To focus on the style attribute, we will remove the class attributes and their associated CSS rules from our example.

Why Remove Code?

  • Focus: Isolate the concept of inline styles applied via the style attribute.
  • Clarity: Avoid confusion between styles applied via classes (external/internal CSS) and styles applied directly inline.
  • Clean Slate: Start with unstyled elements to clearly see the effect of adding inline styles.

Code Change:

  1. Removed the class attributes from both <p> tags in index.html.
  2. Removed the CSS rules for .intro-paragraph, .highlight, and .info-text from style.css.
  3. Updated the paragraph text in index.html.
