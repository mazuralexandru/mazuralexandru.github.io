381. Horizontal Nav (Inline Block): Reset List Styles

Files Modified: style.css

Let's start styling our navigation to appear horizontally instead of vertically. We'll first explore the older inline-block technique (though Flexbox is generally preferred now).

The first step, regardless of the technique, is usually to remove the default list styling (bullets and indentation) that we don't want in a horizontal navigation bar.

✅ Observe style.css:

  • We've added rules targeting the nav ul and nav li elements.
  • We set list-style: none; and reset margin and padding to 0 on the ul to remove bullets and default spacing/indentation.
  • We also reset the margin on the li elements.
/* Section 5: Navigation Bar Implementation */

/* Technique 1: Inline Block */

/* Reset list defaults within nav */
nav ul {
    list-style: none;
    padding: 0;
    margin: 0;
    background-color: rgba(0, 0, 0, 0.2); /* Add background to see ul area */
    text-align: center; /* Center the list items container */
}

nav li {
    margin: 0; /* Reset li margin */
}

/* Basic link styling within nav */
nav a {
    text-decoration: none;
    color: lightblue; /* Basic link color */
    padding: 0.5em 1em; /* Add padding for click area */
}

nav a:hover, 
nav a:focus-visible {
    background-color: rgba(255, 255, 255, 0.1);
    color: white;
    outline: none;
}

(Note: We also added some basic link styling and a background to the ul for context).

✨ Observe the Preview:

  • The navigation links ("Home", "About", etc.) should now appear without bullet points and without the default list indentation.
  • They are still stacked vertically because the <li> elements are still behaving like block-level items within the list.
  • The <ul> now has a faint dark background, showing its extent, and the list items inside are centered horizontally due to text-align: center.
