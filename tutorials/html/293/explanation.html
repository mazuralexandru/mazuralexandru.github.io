293. The dir Attribute - Fundamentals & Syntax

Now let's introduce the dir global attribute, which controls text directionality.

Purpose:

  • The dir attribute specifies the base text direction for the content of the element it's applied to: Left-to-Right (LTR) or Right-to-Left (RTL).

Core Concept:

  • Different languages are written in different directions (e.g., English is LTR, Arabic and Hebrew are RTL). This attribute ensures the browser renders the text flow correctly according to the language's conventions.
  • It's also crucial for handling bidirectional text (mixing LTR and RTL scripts within the same block).

Syntax:

  • dir="direction"
  • Placed within the opening tag of an element.

Attribute Values:

  1. ltr
    • Specifies Left-to-Right text direction.
    • This is the default direction for most browsers and for HTML itself if dir is not specified. You usually only need to set it explicitly if overriding an inherited RTL direction.
  2. rtl
    • Specifies Right-to-Left text direction.
    • Essential for correctly displaying languages like Arabic, Hebrew, Persian, Urdu, etc.
  3. auto
    • Lets the browser determine the direction based on the first strongly-typed character within the element's content (using the Unicode Bidirectional Algorithm).
    • Useful for user-generated content where the language/direction is unknown beforehand.
    • Caution: Can be unreliable if the content starts with directionally neutral characters (like punctuation or numbers).

Inheritance:

  • Like lang, the dir attribute is inherited by child elements unless explicitly overridden on a descendant.
