679. Reset & Common Errors: Invalid Attribute Value

Many HTML attributes can only accept specific, predefined values (enumerated values) or values matching a certain format (like integers, valid URLs, language tags). Using an incorrect value triggers a validation error.

Error: Invalid Attribute Value

  • Problem: An attribute is given a value that is not allowed according to the HTML specification for that specific attribute.
  • Impact: The browser might ignore the attribute, fall back to a default behavior, or behave unpredictably. The semantic meaning conveyed by the attribute might be lost. Results in invalid HTML.
  • Validator Message (Example): Error: Bad value "sideways" for attribute "scope" on element "th". or Error: Bad value "required" for attribute "disabled" on element "input". (for trying disabled="required" instead of just disabled).

Common Examples:

  • scope on <th>: Must be col, row, colgroup, or rowgroup.
  • type on <input>: Must be one of the defined input types (text, email, number, checkbox, etc.).
  • method on <form>: Usually GET or POST.
  • target on <a>/<form>: Usually _self, _blank, _parent, _top, or a framename.
  • lang on any element: Must be a valid BCP 47 language tag.
  • colspan/rowspan: Must be a positive integer.
  • Boolean attributes: Should just be present, not have values like "true" or "required".

✨ Key Takeaway: Ensure attribute values match the allowed format or set of predefined keywords specified in the HTML standard (e.g., scope="col", method="POST"). Validators catch incorrect or misspelled attribute values. Refer to documentation (like MDN) for allowed values.
