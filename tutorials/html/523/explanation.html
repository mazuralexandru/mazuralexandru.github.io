523. Other Semantics: <address> - Fundamentals

The <address> element is used to provide contact information.

Definition & Purpose:

  • Represents contact information for its nearest <article> or <body> ancestor.
  • It identifies the authorship or contact point related to the content of that specific article or the document as a whole.

Core Concept & Scope:

  • If placed within an <article>, it provides contact info for that article's author.
  • If placed outside any <article> (e.g., directly in the <body> or commonly within a page <footer>), it provides contact info for the entire document or site.
  • Important: It's not for arbitrary postal addresses (like listing multiple store locations). It's specifically for contact details about the document/article content.

Syntax & Content:

  • <address> ... contact info ... </address>
  • Requires opening and closing tags. Typically block-level.
  • Content: Can contain names, physical addresses, phone numbers, email addresses, links to contact pages or social media profiles, etc. Often structured using <p> or <br> tags.
  • Restrictions: Cannot contain headings (h1-h6), sectioning elements (article, section, nav, aside), <header>, <footer>, or another <address>.

Default Rendering:

  • Browsers often render the content of <address> in italics. This can be changed with CSS.

Code Change:

We've added an <address> element inside a conceptual <footer> in index.html, containing typical contact information.

<!-- index.html snippet -->
<body>
  <h1>Other Semantic Elements</h1>
  <p>The &lt;address&gt; element provides contact information.</p>

  <footer> <!-- Commonly used in footer -->
    <address> <!-- Added address element -->
      Contact us at:<br>
      Example Corp<br>
      123 Web St, Suite 100<br>
      Codeville, WD 54321<br>
      <a href="mailto:info@example.com">info@example.com</a><br>
      <a href="tel:+15551234567">(555) 123-4567</a>
    </address>
  </footer>

  <p>Applies to nearest article or body ancestor.</p>
  ...
</body>

✅ Observation:

  1. Locate the <address> element within the <footer>. Note the typical content (name, address, email/phone links).
  2. Look at the browser preview. The contact information should be displayed, likely rendered in italics by default.

✨ Key Takeaway: Use the <address> element to provide contact information specifically related to the document or article it's within (often placed in a <footer>). Do not use it for arbitrary postal addresses. It often renders in italics by default.
