124. Nested Lists: Nesting <ol> inside <ol>

Finally, let's look at nesting ordered lists within other ordered lists, which is very common for outlines and complex procedures.

Scenario: Multi-Level Outlines or Steps

This structure is used when you have main steps or points that have ordered sub-steps or sub-points.

Default Numbering Behavior:

  • When you nest an <ol> inside an <li> of another <ol>, the numbering for the nested list restarts by default (from 1, a, or i, depending on the type attribute or browser defaults).
  • You can control the marker type for each nesting level independently using the type attribute (or preferably CSS list-style-type).

Code Change:

We've created a nested <ol> structure in index.html, going three levels deep, using different type attributes for clarity.

<h1>Nested Lists</h1>
<p>Nesting ordered lists (ol inside ol):</p> <!-- Adjusted P -->

<ol> <!-- Outer ol (Level 1: 1, 2, 3...) -->
  <li>Main Point 1</li>
  <li>Main Point 2 <!-- Level 1 li STARTS -->
    <ol type="a"> <!-- Nested ol (Level 2: a, b, c...) STARTS -->
      <li>Sub-point a</li>
      <li>Sub-point b <!-- Level 2 li STARTS -->
         <ol type="i"> <!-- Inner-nested ol (Level 3: i, ii, iii...) STARTS -->
            <li>Detail i</li>
            <li>Detail ii</li>
         </ol> <!-- Inner-nested ol END -->
      </li> <!-- Level 2 li END -->
    </ol> <!-- Nested ol END -->
  </li> <!-- Level 1 li END -->
  <li>Main Point 3</li>
</ol>

<p>Numbering typically restarts and type can change.</p> <!-- Adjusted P -->

Trace the nesting carefully: <ol><li><ol><li><ol><li>...</li></ol></li></ol></li></ol>

✅ Observation:

  1. Examine the multi-level nested <ol> structure in index.html.
  2. Look at the browser preview:
    • Level 1 items ("Main Point 1", "Main Point 2", "Main Point 3") are numbered 1., 2., 3..
    • Level 2 items ("Sub-point a", "Sub-point b") are nested under "Main Point 2", indented further, and numbered a., b. (due to type="a").
    • Level 3 items ("Detail i", "Detail ii") are nested under "Sub-point b", indented even further, and numbered i., ii. (due to type="i").

✨ Key Takeaway: Nesting <ol> inside <ol><li> creates hierarchical outlines. Numbering restarts for each nested level by default, and you can control the marker type independently for each level using attributes or CSS. For complex nested numbering like "1.1.a", CSS Counters are required.
