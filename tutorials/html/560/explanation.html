560. Multimedia: Multiple Sources (<source>) - Practical Application (MP4/WebM)

Let's solidify the most common practical application of using multiple <source> elements for video: providing both MP4 and WebM formats.

Why MP4 and WebM?

  • MP4 (Container Format):
    • Video Codec: Typically H.264 (AVC).
    • Audio Codec: Typically AAC.
    • Compatibility: Excellent, supported natively by virtually all modern browsers, including Safari/iOS (which historically didn't support WebM well). This makes it the essential fallback format.
    • Licensing: H.264 involved patent licensing, although widespread use makes this less of a direct issue for most web developers now.
  • WebM (Container Format):
    • Video Codec: Typically VP8 or VP9 (newer, more efficient).
    • Audio Codec: Typically Vorbis or Opus (newer, more efficient).
    • Compatibility: Supported by Chrome, Firefox, Edge, Opera, and newer versions of Safari. Not supported by older Safari/iOS or Internet Explorer.
    • Efficiency: VP9/Opus often provide better compression (smaller file size for similar quality) compared to H.264/AAC.
    • Licensing: Open and royalty-free codecs.

Standard Implementation:

By providing both, you cater to almost all users:

  1. List the WebM source first (more efficient if supported).
  2. List the MP4 source second as the fallback.
<!-- Standard Compatibility Pattern -->
<video controls>
  <source src="my_video.webm" type="video/webm">
  <source src="my_video.mp4" type="video/mp4">
  <!-- Fallback text -->
  Your browser doesn't support HTML video.
</video>

Encoding Tools:

  • You typically need to use video encoding software (like HandBrake, FFmpeg) or online services to convert your original video file into both optimized WebM and MP4 formats.

✅ Observation: No code changes. Our example from lesson 558 already demonstrates this pattern (<source src="...webm" type="video/webm"> followed by <source src="...mp4" type="video/mp4">). Recognize this as the standard approach for ensuring broad video playback compatibility on the web.

✨ Key Takeaway: Providing both WebM (type="video/webm") and MP4 (type="video/mp4") sources using <source> elements (listing WebM first) offers the best combination of efficiency and broad browser compatibility for video content.
