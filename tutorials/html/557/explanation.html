557. Multimedia: Multiple Sources (<source>) - Fundamentals

When you need to support different browsers that might handle different video or audio formats, the <source> element is essential.

The <source> Element:

  • Definition: Allows you to specify multiple alternative media resources for a parent media element (<video> or <audio>).
  • Purpose: To provide the browser with a list of different file formats (e.g., MP4, WebM, Ogg) or different versions (e.g., based on media queries) of the same media content. The browser can then choose the first format it recognizes and supports.
  • Core Concept: Instead of linking one file via the parent's src attribute, you list multiple options using <source> tags, letting the browser pick the best fit.

Placement & Syntax:

  • Placement: Must be placed inside a <video> or <audio> element.
  • Order: Must appear before any <track> elements and before the fallback content.
  • Void Element: <source> is a void element; it has no content and no closing tag.
  • Basic Syntax: <source src="path/to/media.ext" type="mime/type">

Why Needed? Format Compatibility:

  • Different browsers have varying levels of native support for different video and audio codecs (the algorithms used to compress/decompress media) and container formats (like MP4, WebM, Ogg).
  • For video, the most common scenario requiring <source> is providing both:
    • MP4 (H.264 video + AAC audio): Has the widest historical compatibility across browsers (including Safari/iOS).
    • WebM (VP8/VP9 video + Vorbis/Opus audio): An open, royalty-free format often offering better compression, supported by Chrome, Firefox, Edge, Opera.
  • By providing both via <source>, you ensure most modern browsers can play your video.

✅ Observation: No code changes yet. Understand the reason for <source>: browsers don't all support the same media file types, so <source> lets you offer choices.

✨ Key Takeaway: The void <source> element, placed inside <video> or <audio>, allows you to specify multiple alternative media files, enabling the browser to choose a format it supports. This is crucial for cross-browser video/audio compatibility.
