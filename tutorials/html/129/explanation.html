129. Description Details (<dd>): Fundamentals

Following a <dt> (term), we use the <dd> element to provide the corresponding description or value.

The Description Details Element (<dd>)

  • Definition: Represents the description, definition, value, or explanation associated with the preceding term(s) (<dt>) in a description list group.
  • Purpose: To provide the details corresponding to the term. It's the "value" or "description" part of the name-value pair.
  • Syntax: <dd>...</dd> (Closing tag is mandatory).
  • Placement: Must be a direct child of a <dl> element (or a <div> wrapper). Must follow one or more <dt> elements (or another <dd> within the same group).
  • Content Model: Accepts flow content, meaning it's very flexible. It can contain simple text, paragraphs (<p>), inline elements, images, and even other lists.

Default Rendering:

Browsers typically render <dd> elements as block-level and apply indentation (e.g., margin-left) to visually associate them with the preceding, unindented <dt>.

Code Change:

We have added a <dd> element immediately following the <dt>HTML</dt> inside our <dl> in index.html.

<dl>
   <dt>HTML</dt>
   <dd>HyperText Markup Language - used for structuring web content.</dd> <!-- Added dd -->
</dl>

✅ Observation:

  1. Verify the structure in index.html: <dl> containing <dt> followed immediately by <dd>.
  2. Look at the browser preview:
    • You see the term "HTML" (possibly bold, unindented).
    • Below it, you should see the description "HyperText Markup Language..." indented to the right. This indentation visually pairs the description with its term.

✨ Key Takeaway: The <dd> element provides the description or value for the preceding term(s) (<dt>) in a description list (<dl>). It typically renders as an indented block by default.
