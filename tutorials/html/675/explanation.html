675. Common Errors: Unclosed Tags & Stray Tags

Forgetting to close elements or having extra closing tags are very common mistakes.

Error 1: Unclosed Element

  • Problem: An opening tag (like <em>, <div>, <a>) is missing its corresponding closing tag (</em>, </div>, </a>).
  • Impact: The browser tries to guess where the element ends, often incorrectly including subsequent content within the unclosed element. This breaks structure and can lead to unexpected styling or layout issues.
  • Validator Message (Example): Error: Unclosed element "em". or Error: End tag for "body" seen, but there were open elements. (if unclosed elements exist when </body> is reached).

Error 2: Stray End Tag

  • Problem: A closing tag (e.g., </div>) appears without a matching opening tag preceding it in the correct scope.
  • Impact: Confuses the parser; the tag is usually ignored, but it indicates a structural mistake.
  • Validator Message (Example): Error: Stray end tag "div".

Code Change:

We've added examples to index.html demonstrating both an unclosed <em> tag and a stray </div> tag, along with corrected versions for comparison.

<!-- index.html -->
<body>
  <h1>Common Validation Errors</h1>

  <h2>Error: Unclosed/Stray Tags</h2>

  <!-- Bad Example (Unclosed <em>) -->
  <p>This text has <em>emphasis that never ends... <!-- Missing </em> -->
  <p>...and affects this paragraph too.</p>

  <hr> <!-- Separator -->

  <!-- Good Example -->
  <p>This text has <em>emphasis that ends correctly</em>.</p> <!-- Correctly closed -->
  <p>This paragraph is unaffected.</p>

  <!-- Stray Tag Example (will be removed next step) -->
  <p>Oops, an extra closing tag here.</div></p> <!-- Stray </div> -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

✅ Observation:

  1. Examine the "Bad Example". Note the missing </em>. In the preview, notice how both paragraphs might appear italicized because the browser extends the <em> scope incorrectly.
  2. Examine the "Good Example" and see how only the intended text is italicized.
  3. Examine the last paragraph with the stray </div>. Visually, it might look fine, but a validator would flag this structural error.
  4. (Conceptual): Running the "Bad Example" code through validator.w3.org would report errors related to the unclosed <em> and potentially the subsequent <p> tag placement. It would report the stray </div>.

✨ Key Takeaway: Always ensure every non-void element has a corresponding closing tag. Check for stray closing tags that don't match an open one. Validators excel at catching these.
