216. Links vs. Buttons: The <a href="#"> Anti-Pattern

A common anti-pattern, especially in older code or when developers are less familiar with semantics, is using an anchor element with a placeholder href (like href="#", href="javascript:void(0);" or even no href) and attaching JavaScript click event handlers to make it behave like a button.

Example Anti-Pattern:

<!-- INCORRECT - DO NOT DO THIS FOR ACTIONS -->
<a href="#" class="button-style" onclick="doSomething(); return false;">Perform Action</a>

Why This is Bad Practice:

  1. Incorrect Semantics: It declares the element as a link (navigation) when its actual purpose is an action. This confuses assistive technologies and search engines.
  2. Poor Accessibility:
    • Screen readers announce "Link, Perform Action", setting the wrong expectation.
    • It doesn't activate with the Space key like a native button, breaking standard keyboard interaction patterns.
    • Requires extra JavaScript (return false; or event.preventDefault()) to stop the link from navigating (scrolling to top), adding unnecessary complexity.
  3. Less Robust: Relies entirely on JavaScript being enabled and running correctly for the element to function. A native <button> often has meaningful default behavior (like form submission) even without JavaScript.

The Correct Approach for Actions:

Use a <button> element, typically with type="button" if it's not submitting a form. Attach JavaScript event listeners to the button if needed. Style it with CSS.

<!-- CORRECT APPROACH FOR ACTIONS -->
<button type="button" class="button-style" onclick="doSomething();">Perform Action</button>

(Or better yet, attach the event listener using JavaScript's addEventListener rather than the inline onclick attribute).

✅ Observation: No code changes. Understand why using <a href="#"> with JavaScript to simulate button behavior is an anti-pattern that harms semantics and accessibility.

✨ Key Takeaway: Always use <button type="button"> (or <input type="button">) for elements that trigger actions on the current page. Reserve <a> for actual navigation. Do not use placeholder href values on <a> tags to mimic buttons.
