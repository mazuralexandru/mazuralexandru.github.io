131. Description Lists (<dl>): Multiple <dt> per Group

The structure of a description list group is flexible. One common variation is having multiple terms (<dt>) associated with the same description (<dd>).

Syntax:

You simply list the consecutive <dt> elements first, followed by the single <dd> element that applies to all of them.

<dl>
  <dt>Term 1</dt>
  <dt>Term 2</dt>
  <dd>Description that applies to both Term 1 and Term 2.</dd>
</dl>

Use Case:

This is useful when several different terms or labels refer to the same concept or have the same definition (e.g., synonyms, acronyms and full names).

Code Change:

We've added a new group to our <dl> in index.html where two <dt> elements ("JS" and "JavaScript") are followed by a single <dd>.

<dl>
   <dt>HTML</dt>
   <dd>HyperText Markup Language - used for structuring web content.</dd>

   <dt>CSS</dt>
   <dd>Cascading Style Sheets - used for styling web content.</dd>

   <dt>JS</dt> <!-- Added third dt -->
   <dt>JavaScript</dt> <!-- Added fourth dt -->
   <dd>A programming language for web interactivity.</dd> <!-- Single dd for preceding two dt -->
</dl>

✅ Observation:

  1. Examine the last group in the <dl> in index.html. Notice the two <dt> tags followed by one <dd> tag.
  2. Look at the browser preview:
    • The terms "JS" and "JavaScript" should appear one after the other (both unindented, maybe bold).
    • The single description ("A programming language...") should appear below them, indented, clearly associated with both preceding terms.

✨ Key Takeaway: You can associate multiple consecutive <dt> elements with the subsequent <dd> element(s) in a description list group.
