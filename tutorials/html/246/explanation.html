246. Styling Code Blocks (<pre>)

Files Modified: style.css

While the default monospace font and preserved whitespace of <pre> are useful, code blocks often benefit from additional styling for readability.

Common styles applied to <pre> elements (or often the <code> element inside the <pre>) include:

  • background-color: A distinct background (often dark or light gray) to visually separate the code block.
  • padding: Adds space between the code text and the edge of the background/border.
  • border / border-radius: Adds a border and potentially rounded corners.
  • font-family: Explicitly setting a preferred monospace font stack (e.g., Consolas, Monaco, 'Courier New', monospace).
  • overflow-x: auto;: Adds a horizontal scrollbar only if a line of code is too long to fit within the container's width (often combined with white-space: pre; which is the default for <pre>).
  • white-space: pre-wrap;: An alternative to scrolling. Preserves whitespace but allows long lines to wrap onto the next line within the block.

✅ Observe style.css:

  • We've added a new rule targeting pre elements.
  • It sets a background color, padding, border-radius, and enables horizontal scrolling if needed.
  • We also added a rule for pre code to reset some styles that might be inherited from the general code rule we added previously, ensuring the code inside pre doesn't get double backgrounds or incorrect font sizes.
/* Styling for code blocks */
pre {
  background-color: #1e1e1e; /* Darker background for block */
  padding: 1em;
  border-radius: 5px;
  overflow-x: auto; /* Add scrollbar if code is too wide */
  font-size: 0.9rem; /* Slightly smaller base size for code blocks */
}

/* Reset specific styles for code inside pre if needed */
pre code {
  background-color: transparent; 
  padding: 0;
  border-radius: 0;
  font-size: 1em; /* Use the font-size set on the pre tag */
  /* The font-family will be inherited from pre (monospace default) */
}

✨ Observe the Preview: The HTML code block displayed within the <pre><code> tags should now have a dark background, padding around it, and rounded corners, making it look much more like a typical styled code block. If a line were extremely long, a horizontal scrollbar would appear.
