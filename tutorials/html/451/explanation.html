451. Forms: The value Attribute - Text Inputs

Let's start examining the value attribute by looking at its behavior with text-based input types (text, password, email, url, tel, search, etc.).

value on Textual Inputs:

  • Purpose: Specifies the initial value displayed in the input field when the page first loads.
  • Behavior:
    • If the value attribute is present, its content pre-populates the input field.
    • If the user submits the form without changing this initial value, the value specified in the value attribute is the one that gets sent to the server.
    • If the user does type something into the field, their input overrides the initial value, and the user's input is what gets submitted.
    • If the value attribute is omitted, the text field starts empty.

Use Cases:

  • Providing default suggestions (e.g., a default city or country).
  • Pre-filling fields with known user data (e.g., loading saved preferences, though often done dynamically with server-side languages or JavaScript).
  • Hidden inputs (type="hidden") require a value attribute to hold the data they submit.

Code Change:

We've added two text input fields to the form in index.html. The first ("City") has a value attribute set, while the second ("Country") does not.

<!-- index.html snippet -->
<form action="/submit-data" method="POST">
  <p>
    <label for="city">City:</label>
    <!-- Sets the initial text displayed in the field -->
    <input type="text" id="city" name="city" value="Default City"> <!-- Value on text input -->
  </p>
  <p>
    <label for="country">Country:</label>
    <!-- Field starts empty if value attribute is omitted -->
    <input type="text" id="country" name="country">
  </p>
</form>

✅ Observation:

  1. Examine the two <input type="text"> elements. Note the presence of value="Default City" on the first and the absence of the value attribute on the second.
  2. Look at the browser preview:
    • The "City" input field is pre-filled with the text "Default City".
    • The "Country" input field is empty.
  3. You can edit the text in the "City" field. If submitted without changes, "Default City" would be sent for the city parameter. If submitted empty, an empty string would be sent for the country parameter.

✨ Key Takeaway: For text-based inputs (text, password, email, etc.), the value attribute sets the initial content of the field. If omitted, the field starts empty.
