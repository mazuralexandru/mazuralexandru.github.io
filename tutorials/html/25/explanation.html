25. Module 1 Recap

We've covered the absolute fundamentals of HTML in this first module! Let's quickly recap the key takeaways:

  • What HTML Is: HyperText Markup Language, used to define the structure and semantic meaning of web content. It's a markup language, not a programming language.
  • HTML vs. CSS vs. JS: HTML provides the structure (skeleton), CSS handles presentation (styling), and JavaScript manages behavior (interactivity). Keeping them separate ("Separation of Concerns") is crucial.
  • Elements & Tags: Elements are the conceptual building blocks (paragraph, heading). Tags (<tagname>... </tagname>) are the syntax used to create elements. Use lowercase for tags.
  • Nesting: Elements can be placed inside others, but must be closed in the reverse order they were opened.
  • Void Elements: Some elements like <br> and <hr> are empty, have no content, and use only a single tag (no closing tag).
  • Minimal HTML5 Document: Requires <!DOCTYPE html>, <html> (with lang attribute), <head> (containing <meta charset="UTF-8"> and <title>), and <body> (containing visible content).
  • <title> Element: Defines the page title shown in browser tabs, bookmarks, and search results. Crucial for usability, accessibility, and SEO.
  • Saving & Viewing: Save files as .html with UTF-8 encoding. View locally (file:///) or via a web server (http://).
  • Comments: Use <!-- ... --> for non-displayed notes or to temporarily disable code. Visible in page source.

We now have a valid, albeit simple, HTML page!
