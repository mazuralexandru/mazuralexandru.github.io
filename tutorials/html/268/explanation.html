268. The id Attribute - Value Requirements

While the uniqueness rule is paramount, there are also specific rules and best practices for the value you assign to an id attribute.

Value Requirements & Constraints:

  • HTML5 Naming Rules (Technical Minimum):
    • Must contain at least one character.
    • Must not contain any space characters (spaces, tabs, line breaks, etc.).
  • Best Practices & Compatibility Naming Rules: While HTML5 is quite permissive, following stricter rules ensures better compatibility with CSS selectors, JavaScript, and older standards. It's highly recommended to:
    • Start the value with a letter (a-z, A-Z).
    • Use only letters, digits (0-9), hyphens (-), and underscores (_) for subsequent characters.
    • Avoid using characters like colons (:) or periods (.) which, while technically allowed in HTML5 IDs, can require special escaping in CSS selectors (\:, \.) and are best avoided.
    • Avoid starting with a digit or a hyphen, as this can also complicate CSS selection in some older contexts or require escaping.
    • Recommendation: Use descriptive, meaningful, lowercase names, with hyphens separating words.
      • Good: id="main-navigation", id="product-summary-table", id="contact-form-submit"
      • Okay (but less common): id="user_profile_section"
      • Avoid: id="1stSection", id="-some-id", id="section:header", id="my ID" (contains space - invalid!)
  • Case Sensitivity: Remember, id values are case-sensitive. id="mainNav" and id="mainnav" would be treated as different IDs. Sticking to a consistent case (usually lowercase) is recommended.

Code Example (Illustrative - No Code Change):

<!-- Good ID examples -->
<div id="user-settings">...</div>
<table id="results-table-2">...</table>
<button id="confirm-deletion-button">...</button>

<!-- Technically valid HTML5 but potentially problematic/less readable IDs -->
<!-- <div id="section:header">...</div> -->
<!-- <div id="1stPlace">...</div> -->

<!-- INVALID IDs -->
<!-- <div id="my id">...</div> <!-- Contains space -->
<!-- <div id="">...</div> <!-- Empty -->

✅ Observation: No changes to index.html. The ID we used (main-page-title) follows the recommended best practices (starts with letter, lowercase, hyphen-separated). Review the naming rules and recommendations above.

✨ Key Takeaway: id values must be unique and contain no spaces. Follow best practices: start with a letter, use only letters, numbers, hyphens, or underscores, and prefer lowercase hyphen-separated names for readability and compatibility.
