38. Text: Headings (<h1> to <h6>) - Structure vs. Style

A crucial principle when working with headings (and indeed, most HTML elements) is to distinguish between structure and style.

Semantic Priority: Use Headings for Structure ONLY

  • Purpose: <h1> through <h6> elements exist to define the structural hierarchy and outline of your document content. Their primary role is semantic.
  • Anti-Pattern: Do not choose a heading level simply because you want the text to look a certain way (e.g., picking <h2> just because you want large, bold text, even if it doesn't represent a major section heading).
  • Why it's Wrong: Misusing headings for purely visual reasons breaks the logical structure of the document, harming accessibility and SEO (as discussed in the previous lesson). If you used an <h2> for something that wasn't actually a major section title, a screen reader user navigating by headings would be misled about the document's organization.

Controlling Appearance: Use CSS for Style

  • Solution: If you need text to be larger, smaller, bold, italic, a different color, etc., for purely presentational reasons without it representing a structural heading, you should use CSS.
  • How:
    • Apply styles directly using CSS properties like font-size, font-weight, color, etc.
    • Often, you'll wrap the text in a semantically neutral element like <span> (for inline text) or <div> (for block-level text) and apply a CSS class to it. Then, define the styles for that class in your style.css file.

Example (Conceptual):

  • Incorrect (Using heading for style): <h3>Make this text big and bold!</h3> (if it's not actually a level-3 subheading).
  • Correct (Using CSS for style):
    • HTML: <p><span class="large-bold-text">Make this text big and bold!</span></p>
    • CSS: .large-bold-text { font-size: 1.5em; font-weight: bold; }

✅ Observation: No code changes. Review the headings in index.html. We are currently using them appropriately for structure (Page Title -> Section -> Subsection). The visual appearance (large/bold) comes from default browser styles for these semantic tags, which is fine, but we didn't choose them for that appearance.

✨ Key Takeaway: Use <h1>-<h6> strictly to define the logical outline and hierarchy of your content. Use CSS to control visual presentation like size, weight, and color. Separate structure (HTML) from style (CSS).
