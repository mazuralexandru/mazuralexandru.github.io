16. Pruning Code for Focus

Before moving on to new topics like Developer Tools, let's clean up our current example again. The <h1> element and its associated CSS rule served their purpose in demonstrating headings and external CSS. To keep the preview clean and focused for upcoming lessons, we'll remove them.

What's New?

  • HTML Removal: The <h1>Document Heading</h1> element has been removed from index.html. A comment <!-- Removed H1 element for focus --> marks the location.
  • HTML Addition: A simple <p>Preparing for next steps...</p> has been added to ensure the page isn't completely blank visually.
  • CSS Removal (Commented Out): The CSS rule targeting h1 in style.css has been commented out (/* ... */). We could have deleted it entirely, but commenting it out serves the same purpose for now (making it inactive) and shows another use of comments. A comment /* Removed H1 style rule for focus */ clarifies the intent.

Code Changes:

<!-- index.html snippet -->
<body>
  <!-- Removed H1 element for focus -->
  <p>Preparing for next steps...</p> <!-- Basic paragraph for context -->

  <!-- Commented out paragraph remains -->
  <!-- <p>This paragraph is commented out and won't appear.</p> -->

  <script src="script.js"></script>
</body>

/* style.css snippet */
  min-height: 150px; /* Ensure space for preview */
}

/* Removed H1 style rule for focus */
/* h1 {
  color: tomato;
  border-bottom: 2px solid #eee;
  padding-bottom: 5px;
} */

/* CSS files can also have comments! */

Code Pruning Rationale (Integrated Removal)

  • As before, we are removing code that is no longer directly relevant to the immediate learning objective. This keeps the index.html and style.css files concise and ensures the visual preview in the browser only shows elements pertinent to the current or upcoming concept.
  • Removing or commenting out the corresponding CSS rule (h1) prevents orphaned styles that don't apply to anything currently in the HTML.
  • This cleanup is integrated into the current step, preparing a clean slate before we discuss developer tools.
