311. Box Sizing: The box-sizing Property

Files Modified: None

We've mentioned box-sizing a few times. This property fundamentally changes how the browser calculates the total width and height of an element based on its width, height, padding, and border values.

  • Problem with Default (content-box): By default, width and height apply only to the content area. Padding and borders are added outside of this, increasing the element's final rendered size. This makes layout difficult. If you set two boxes to width: 50%; and then add padding or borders, they will no longer fit side-by-side in a 100% width container because their total rendered width will exceed 50% each.
  • Solution (border-box): Setting box-sizing: border-box; tells the browser to include the padding and border within the element's specified width and height. If you set width: 50%; and add padding/border, the content area will shrink to make room, but the element's total rendered width will remain exactly 50% of its container.

Understanding box-sizing is crucial for predictable sizing and layout.
