263. Attributes: Recap & Purpose

Before diving into global attributes, let's quickly recap what HTML attributes are and their general purpose.

What are Attributes?

  • Definition: Attributes are special keywords used inside the opening tag of an HTML element to provide additional information, configuration, or metadata about that specific element.
  • Purpose: They modify the element's default behavior, provide identifiers, associate it with external resources, define its relationship to other elements, or offer extra semantic context.
  • Analogy: Think of HTML elements as nouns (like "paragraph", "image", "link") and attributes as adjectives or configuration settings describing that specific noun (e.g., a blue paragraph, an image from this source, a link opening in a new tab).

Examples We've Seen:

We've already encountered several attributes specific to certain elements:

  • lang on <html> (language)
  • charset on <meta> (character encoding)
  • href on <a> (link destination)
  • src on <img> (image source)
  • alt on <img> (alternative text)
  • width / height on <img> (dimensions)
  • cite on <blockquote> / <q> (source URL)
  • target on <a> (link opening behavior)
  • rel on <a> / <link> (relationship)
  • type on <ol> / <a> (marker type / MIME type)
  • start / reversed on <ol> (list numbering)
  • value on <li> (list item number)
  • download on <a> (download hint/filename)

Global vs. Specific:

The key difference is that the attributes listed above generally apply only to specific elements or element types. Global attributes, which we'll focus on in this module, can be applied to almost any element.
