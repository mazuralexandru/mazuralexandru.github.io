368. Link State: :visited & Privacy Restrictions

Files Modified: style.css

The :visited pseudo-class allows you to style links that the user has previously visited.

  • :visited:

    • Selects: Links (<a> elements with an href attribute) whose target URL is present in the browser's history.
    • Purpose: Provides a visual cue that a link leads to a page the user has already seen.

  • Privacy Restrictions: To prevent websites from potentially sniffing a user's browsing history (by checking the computed style of many different links), browsers heavily restrict the CSS properties that can be effectively changed using :visited. You can generally only change:

    • color
    • background-color
    • border-color (and individual border sides)
    • SVG fill and stroke colors
    • Column rule colors
    • Transparency cannot be changed (e.g., you can't make visited links semi-transparent using rgba() if they weren't already).
    • You cannot change layout, font size, text-decoration, add background images, etc., based on the visited state.

✅ Observe style.css:

  • We've added a rule for a:visited.
  • We're setting a different color (mediumpurple) for visited links.
/* Style visited links differently */
a:visited { 
  color: mediumpurple; /* Different color for visited links */
}

✨ Observe the Preview:

  • Look at the links in index.html.
  • If the simulated browser environment tracks history, any links you might have "visited" (like potentially the placeholder # link if clicked previously, or maybe the DuckDuckGo link if the environment allows external navigation tracking) could now appear medium purple instead of cyan. Links definitely not visited should remain cyan.
  • This subtle color change is the primary way browsers allow styling for visited links due to privacy constraints.
