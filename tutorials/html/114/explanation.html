114. Preparing for li value: Resetting <ol> Attributes

We've seen how the type, start, and reversed attributes on the <ol> element itself can control the list's numbering scheme.

Now, we're going to look at an attribute that applies to the individual <li> element within an <ol>: the value attribute.

To demonstrate value clearly, let's first reset our <ol> element by removing the start and reversed attributes, reverting it back to a standard, ascending list starting from 1.

Why Reset?

  • Isolate Concept: To show the effect of the <li>'s value attribute without the interacting influence of start or reversed on the <ol>.
  • Clear Baseline: Starting from a standard list (1, 2, 3...) makes the jump caused by value more obvious.

Code Change:

We have removed the reversed and start attributes from the <ol> tag in index.html.

<p>Resetting list before showing li 'value' attribute.</p> <!-- Adjusted P -->

<ol> <!-- Removed reversed and start -->
  <li>Item 1</li> <!-- Text updated -->
  <li>Item 2</li>
  <li>Item 3</li>
</ol>

<p>Attributes removed.</p> <!-- Adjusted P -->

  2. Look at the browser preview. The list should be back to the standard numbering: 1., 2., 3..

✨ Key Takeaway: Resetting the <ol> attributes provides a clean baseline to demonstrate the effect of the value attribute on individual <li> elements next.
