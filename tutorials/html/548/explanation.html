548. Multimedia: Video (<video>) - preload Attribute

The preload attribute provides a hint to the browser about whether and how much of the video file should be downloaded before the user explicitly requests playback.

The preload Attribute:

  • Purpose: To optimize the loading experience by suggesting whether to prioritize bandwidth saving or faster playback startup.
  • Syntax: <video src="..." preload="value">
  • Values:
    • none: Hints that the browser should not preload any part of the video file. Download only starts when the user hits play. Use if playback is unlikely or conserving bandwidth is critical.
    • metadata: Hints that only the video's metadata (duration, dimensions, tracks) should be fetched initially. This allows displaying information like the video length in the controls without downloading actual video frames. Uses minimal bandwidth. Often a good default.
    • auto: Hints that the browser can preload as much of the video file as it deems necessary (potentially the whole file) to ensure smooth playback once started. Uses the most bandwidth upfront but can lead to faster start times.
  • Default Behavior: Varies between browsers! Some default to metadata, others might default to auto. Relying on the default is unpredictable; explicitly set preload if you have a preference.
  • Interaction: The preload attribute is only a hint. Browsers may ignore it based on network conditions, data saver modes, or other factors. If the autoplay attribute is present, it takes precedence, and the browser must start downloading the video regardless of the preload value.

Code Change:

We've removed the autoplay, muted, and loop attributes from the previous example and added preload="metadata". We've also reverted the src to the generic example path.

<!-- index.html snippet -->
  <!-- Removed autoplay/muted/loop, added preload="metadata" -->
  <video controls width="480" height="270"
         src="/videos/example.mp4" <!-- Hypothetical path -->
         preload="metadata">
    <p>Sorry, your browser doesn't support embedded videos.</p>
  </video>

✅ Observation:

  1. Locate the preload="metadata" attribute on the <video> tag.
  2. The browser preview still shows the video player with controls. There's no direct visual change from adding preload.
  3. (Conceptual): With preload="metadata", if the video file existed, the browser would fetch just enough information to potentially display the video duration in the controls, but wouldn't start downloading the main video content until the user clicked play. If it were preload="auto", the browser might start downloading the video data in the background immediately. If preload="none", it would wait entirely for the user action.

✨ Key Takeaway: Use the preload attribute (none, metadata, auto) to provide a hint to the browser about how aggressively to download video data before playback starts. metadata is often a good balance. autoplay overrides preload. Browser compliance varies.
