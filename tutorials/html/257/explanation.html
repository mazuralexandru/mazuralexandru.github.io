257. <figure> Example: Code Block with Caption

As mentioned earlier, <figure> isn't limited to images. It's also the semantically correct way to provide a caption for a block of code (<pre><code>).

Use Case: Displaying a code listing (like in documentation or a tutorial) with an associated title or description (e.g., "Listing 1: Basic Setup").

Code Change:

We've replaced the multi-image <figure> content with one containing a <pre><code> block (with escaped HTML) and a relevant <figcaption>. We've placed the <figcaption> first this time to show that variation.

<body>
  <h1>Figures and Captions</h1>
  <p>Figures can also group code blocks with captions:</p> <!-- Adjusted P -->

  <figure>
    <figcaption>Listing 1: Basic HTML document structure.</figcaption> <!-- Figcaption first -->
    <pre><code>&lt;!DOCTYPE html&gt;
&lt;html lang="en"&gt;
&lt;head&gt;
  &lt;title&gt;Example&lt;/title&gt;
&lt;/head&gt;
&lt;body&gt;
  &lt;p&gt;Hello!&lt;/p&gt;
&lt;/body&gt;
&lt;/html&gt;</code></pre> <!-- Code block inside figure -->
  </figure>

  <p>This provides semantic context for code examples.</p> <!-- Adjusted P -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

✅ Observation:

  1. Examine the <figure> in index.html. Note the <figcaption> comes before the <pre><code> block.
  2. Look at the browser preview. You should see:
    • The caption ("Listing 1: Basic HTML document structure.") displayed first.
    • The block of escaped HTML code displayed below the caption, rendered in monospace and preserving whitespace due to the <pre> tag.
    • The caption and code block are visually grouped together.

✨ Key Takeaway: Use <figure> with <figcaption> (placed first or last) to semantically associate a caption or title with a block of preformatted code (<pre><code>).
