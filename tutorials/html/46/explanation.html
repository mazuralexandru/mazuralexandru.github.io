46. Cascade Factor: Source Order

The final major factor in the cascade (after Origin/Importance and Specificity) is Source Order. If two rules have the same origin and the same specificity, the one that appears later in the stylesheet code wins.

What's New?

  • CSS Pruning: Removed the #unique-para ID selector rule.
  • CSS Addition: Added a second rule using the same class selector .special, placed after the first .special rule, and setting a different color.

Source Order Explained

  • When the browser evaluates styles, it reads the CSS files from top to bottom.
  • If it finds two rules that target the same element (or have rules targeting it via different selectors) and those rules have exactly the same specificity (and same origin/importance), the property value declared in the rule that comes last in the source code will be applied.

Applying the Cascade:

  • Our paragraph <p class="special"> is targeted by three rules for color:

    1. p { color: #ccc; } -> Specificity (0,0,0,1)
    2. .special { color: lightcoral; } -> Specificity (0,0,1,0) - Rule A
    3. .special { color: yellowgreen; } -> Specificity (0,0,1,0) - Rule B

  • Specificity Check: Both .special rules (A and B) have specificity (0,0,1,0), which is higher than the p rule's (0,0,0,1). So, the p rule's color is eliminated.

  • Source Order Check: Rules A and B have the same specificity (0,0,1,0). Now, source order applies. Rule B (color: yellowgreen;) appears later in the style.css file than Rule A (color: lightcoral;).

  • Result: Rule B wins due to source order. The paragraph's color will be yellowgreen.
