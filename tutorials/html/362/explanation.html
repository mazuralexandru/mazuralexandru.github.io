362. Reset: Preparing for <noscript> Element

We have explored the <script> element, focusing on linking external files and the importance of defer/async for performance.

The final element in the <head> section we'll cover is <noscript>, which provides fallback content for users whose browsers do not support JavaScript or have it disabled.

To focus on <noscript>, we will remove the <script> examples from index.html.

Why Remove Code?

  • Focus: Isolate the <noscript> element and its purpose.
  • Clean Slate: Remove the JavaScript-related tags before introducing the JavaScript-fallback tag.

Code Change:

  1. Removed the external <script> tag from the <head>.
  2. Removed the inline <script> tag from the <body>.
  3. Updated the <h1> and paragraph text.
<!-- index.html -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>HTML Metadata - NoScript Element</title> <!-- Updated title -->
  <link rel="stylesheet" href="style.css">

  <!-- Removed external script tag -->

</head>
<body>
  <h1>Metadata: The &lt;noscript&gt; Element</h1> <!-- Updated H1 -->
  <p>Preparing to discuss the 'noscript' element.</p> <!-- Adjusted P -->

  <!-- Removed inline script tag -->
  <!-- <script> console.log(...) </script> -->

  <p>It provides content when scripting is disabled.</p> <!-- Adjusted P -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>
</html>

✅ Observation:

  1. Verify the removal of both <script> tags from index.html.
  2. The browser preview shows the updated heading and paragraphs. Check the Developer Tools Console – the "Inline script executed!" message should no longer appear.

✨ Key Takeaway: Removing the <script> examples allows us to focus on the <noscript> element, which provides content specifically for environments where scripts don't run.
