Reason 3 Recap: Representing Invisible Characters

  • The most common use case here is the non-breaking space.

The Non-Breaking Space Entity (&nbsp;)

  • Purpose: Represents a space character ( ) that the browser is not allowed to use as a point for automatic line breaking (word wrapping).
  • Behavior: Ensures that the content immediately before and after the &nbsp; entity stays together on the same line, even if it occurs near the edge of a container.
  • Use Cases (Legitimate):
    • Keeping units with numbers (e.g., 10&nbsp;kg, £&nbsp;50).
    • Keeping titles with names (e.g., Dr.&nbsp;Smith, Section&nbsp;A).
    • Preventing breaks in multi-part names or specific phrases where a break would be confusing.
    • Occasionally used to ensure an element with no other content still occupies space (e.g., an empty table cell), though CSS min-height or padding is often better.
