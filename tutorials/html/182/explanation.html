182. Linking to Non-HTML Resources: Examples

Anchor elements aren't limited to linking just to other HTML web pages. You can use the href attribute to point directly to various other types of files.

Concept:

The href value simply needs to be the correct URL (absolute or relative) pointing to the desired file on a web server.

Common Examples:

  • PDF Documents: Linking to user manuals, reports, articles.
  • Image Files: Linking to view a larger version of an image, or for download.
  • Zip Archives / Other Downloads: Linking to software, data files, project archives.
  • Word Documents, Spreadsheets, Presentations: Linking to office documents.
  • Media Files: Linking directly to video or audio files (though embedding using <video>/<audio> is often preferred for playback).

Code Change:

We've added several examples of links pointing to hypothetical non-HTML files using relative URLs in index.html. We've used <br> tags simply to stack the links vertically for this demonstration.

<p>
  Example Links (will likely show 'Not Found' here):<br>
  <a href="/docs/user-manual.pdf">User Manual (PDF)</a><br> <!-- PDF Link -->
  <a href="/images/logo-large.png">Large Logo (PNG)</a><br> <!-- Image Link -->
  <a href="/downloads/archive.zip">Download Archive (ZIP)</a> <!-- ZIP Link -->
</p>

✅ Observation:

  1. Examine the href attributes in the new links. Note how they point to file extensions like .pdf, .png, and .zip.
  2. In the browser preview, you see the three links styled as hyperlinks.
  3. Behavior: Clicking these links in our current tutorial environment will almost certainly result in a "File Not Found" error because these files (user-manual.pdf, etc.) don't actually exist in our project. On a live website where these files were uploaded to the specified paths, clicking them would trigger the browser to handle the file (as discussed next).

✨ Key Takeaway: You can create links pointing directly to non-HTML files (PDFs, images, ZIPs, etc.) by setting the href attribute to the URL of that file.
