230. <img> Loading & Error Handling (Conceptual)

Let's briefly discuss what happens when the browser processes an <img> tag and handles potential errors.

Asynchronous Loading:

  • When the browser parses the HTML and encounters an <img> tag with a valid src attribute, it initiates a separate HTTP request to fetch the image file from the specified URL.
  • Crucially, this fetching process happens asynchronously. The browser doesn't usually wait for the image to download before continuing to parse and render the rest of the HTML and CSS below the <img> tag.
  • This prevents large images from blocking the rendering of the rest of the page content, improving perceived performance.
  • Once the image data is downloaded, the browser decodes it and paints it into the reserved space on the page. (This is why width and height attributes are important to reserve the space correctly beforehand - see Topic 51).

Error Handling: Broken Images

  • What happens if the src URL is incorrect, the server cannot find the file (404 error), the file is corrupted, or there's a network issue preventing the download?
  • The browser cannot display the intended image. Instead, it typically:
    1. Displays a "broken image" icon (the appearance varies between browsers).
    2. Crucially, it should display the content of the alt attribute as fallback text within the space reserved for the image. This makes the alt text vital not only for screen readers but also as a fallback for sighted users when images fail to load.
