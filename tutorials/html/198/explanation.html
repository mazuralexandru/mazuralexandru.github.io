198. Relative Unit: rem (Root Em)

Files Modified: None

The rem (rem) unit stands for "root em". It is relative to the font-size of the root element (<html>) only.

  • Meaning: 1rem equals the font size set on the <html> element. 2rem is twice the root font size, 0.5rem is half. The browser's default root font size is typically 16px, so 1rem often defaults to 16px unless overridden on the <html> tag.
  • Key Advantage: Unlike em, rem values do not compound based on parent element font sizes. 1.5rem will always mean 1.5 times the root font size, regardless of how deeply the element is nested or what font sizes its parents have.
  • Use Cases:
    • Highly recommended for setting font-size on elements throughout the page. This allows the entire page's text to scale proportionally if the user changes their browser's base font size (which typically affects the root <html> font size).
    • Often used for setting global spacing values (padding, margin) to maintain a consistent scale relative to the base font size.
  • Pros: Predictable scaling, avoids compounding issues of em, respects user font size preferences (good for accessibility).
  • Cons: Requires understanding the root font size (often 16px by default, but can be changed).

Example:

html { 
  font-size: 16px; /* Explicitly set root font size (or rely on browser default) */
} 
body {
  font-size: 1rem; /* Equivalent to 16px */
}
h1 {
  font-size: 2rem; /* Equivalent to 32px */
  margin-bottom: 1rem; /* Equivalent to 16px margin */
}
div {
  font-size: 1.2rem; /* Equivalent to 19.2px */
}
div p {
  font-size: 1rem; /* Still 16px, NOT 1.2 * 16px */
  padding: 1rem; /* 16px padding */
}

Recommendation: Use rem for most font sizes and often for margins/padding to create accessible, scalable interfaces. Use em when you specifically need sizing relative to the element's own font size (e.g., padding around text). Use px for fixed small values like borders.
