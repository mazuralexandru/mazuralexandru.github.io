476. Forms: Legends (<legend>) - Fundamentals

While <fieldset> groups controls, the <legend> element provides a visible caption or title for that group.

The <legend> Element:

  • Definition: Represents a caption for the content of its parent <fieldset> element.
  • Purpose:
    • Provides a visible label for the group of related controls, explaining their collective purpose (e.g., "Shipping Address", "Contact Details", "Choose Size").
    • Crucial for Accessibility: Screen readers announce the <legend> text when the user navigates into the <fieldset> or focuses on the first control within it. This provides vital context, especially for groups of radio buttons or checkboxes where individual labels might not fully explain the overall question being asked.
  • Element Category: Usually rendered somewhat like a block element visually embedded within the <fieldset> border.

Placement Rule (CRITICAL):

  • A <legend> element MUST be the very first child element placed immediately inside the opening <fieldset> tag.
  • Only one <legend> is permitted per <fieldset>.

Syntax:

  • Requires opening <legend> and closing </legend> tags.
  • The caption text goes between the tags.
    <fieldset>
  <legend>Your Caption Here</legend>
  <!-- Controls and labels -->
</fieldset>

Code Change:

We've added a <legend> element as the first child inside our <fieldset> in index.html.

<!-- index.html snippet -->
<form action="/submit-address" method="POST">
  <fieldset>
    <legend>Shipping Address</legend> <!-- Added legend -->

    <p>Street Address, City, Zip etc. would go here...</p> <!-- Placeholder -->

  </fieldset>
</form>

✅ Observation:

  1. Locate the <legend>Shipping Address</legend> placed immediately inside the <fieldset>.
  2. Look at the browser preview. You should see:
    • The border rendered by the <fieldset>.
    • The text "Shipping Address" displayed, typically visually embedded within the top border of the fieldset. This default rendering clearly associates the legend with the bordered group.

✨ Key Takeaway: Use the <legend> element as the first child inside a <fieldset> to provide a visible and accessible caption for the group of related form controls. This is essential for accessibility, especially for radio/checkbox groups.
