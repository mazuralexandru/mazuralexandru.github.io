570. Multimedia: Audio (<audio>) - Core Attributes & Comparison to <video>

The <audio> element shares most of its functional attributes with <video>.

Shared/Similar Attributes:

  • src: URL of the single audio file (omit if using <source>).
  • controls (Boolean): Shows default browser audio controls. Essential for basic user interaction.
  • autoplay (Boolean): Attempts automatic playback (subject to browser restrictions, often blocked unless muted or due to user interaction). Use very cautiously for audio!
  • muted (Boolean): Starts playback muted. Rarely useful for audio-only content unless used solely to enable autoplay for background sounds (generally discouraged).
  • loop (Boolean): Repeats the audio automatically upon ending.
  • preload (none|metadata|auto): Hints browser about preloading audio data. metadata is often sufficient.
  • crossorigin (anonymous|use-credentials): For CORS requests when loading audio or tracks from different domains (e.g., for Web Audio API access).
  • Global Attributes: id, class, style, title, etc.

Attributes NOT Applicable to <audio>:

  • width, height, poster: These relate to the visual display area of a video and have no meaning for the non-visual <audio> element. They are invalid on <audio>.
  • playsinline: Relates to iOS fullscreen video behavior, not applicable to audio.

Styling:

  • You can style the <audio> element itself with CSS (e.g., display, margin, width if you want to constrain the controls), but styling the internal parts of the default browser controls is extremely limited and inconsistent. Custom audio players require JavaScript and HTML/CSS for the UI.

✅ Observation: No code changes. Review the list of shared attributes (src, controls, autoplay, muted, loop, preload, crossorigin) and those specific to <video> (width, height, poster, playsinline). Look at the <audio> tag in index.html (lesson 569) and recognize which attributes could be added (loop, preload, etc.).

✨ Key Takeaway: <audio> shares most functional attributes (src, controls, autoplay, loop, preload, crossorigin) with <video>. Visual attributes like width, height, and poster do not apply to <audio>.
