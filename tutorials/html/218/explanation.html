218. Module 4 Recap: Links and Navigation

This module covered the cornerstone of web navigation: the anchor element (<a>) and its associated attributes and best practices.

Key Concepts Reviewed:

  • Anchor Element (<a>): Creates hyperlinks for navigation. Inline element by default.
  • href Attribute: Essential for functional links; specifies the destination URL or protocol.
    • Absolute URLs: Full addresses (https://...) for external links. Prefer https.
    • Relative URLs: Partial paths for internal links (highly recommended for portability).
      • Same Directory: page.html
      • Subdirectory: dir/page.html
      • Parent Directory: ../page.html
      • Root-Relative: /path/from/root/page.html
    • Fragment Identifiers: Link to specific sections within pages using #elementID (requires target element to have matching unique id).
  • target Attribute: Controls where the link opens.
    • _self (Default): Same tab/window.
    • _blank: New tab/window. Requires rel="noopener noreferrer" for security/privacy.
    • _parent / _top: For frame navigation.
  • Security (rel="noopener noreferrer"): Prevents tabnabbing and hides referrer when using target="_blank". Always use them together with _blank.
  • Special Protocols:
    • mailto:address?subject=...&body=...: Opens default email client. Requires URL encoding for parameters. Subject to spam risk/client dependency.
    • tel:+1555...: Initiates calls on capable devices. Use international format in href. Relies on device/software capability.
  • Non-HTML Resources: Links can point to PDFs, images, ZIPs, etc. Browser behavior depends on server headers (Content-Disposition) and browser settings.
    • download Attribute: Suggests download instead of inline display (mainly for same-origin). Can suggest filename (download="name.ext").
  • Link Content: The content between <a>...</a> (usually text or image).
    • Descriptive Text: CRITICAL for usability and accessibility. Must make sense out of context. Avoid "Click Here". Indicate file types.
    • Image Links: If image is sole content (<a><img></a>), the <img>'s alt text MUST describe the link's destination/purpose.
  • Other Attributes:
    • rel: Describes link relationship (e.g., nofollow, external, next/prev).
    • hreflang: Hints language of linked resource (SEO).
    • type: Hints MIME type of linked resource (rarely needed).
  • Links vs. Buttons: Use <a> for navigation (going somewhere). Use <button> for actions (doing something on the page). Choosing correctly is vital for semantics and accessibility. Avoid using <a href="#"> for button actions.
