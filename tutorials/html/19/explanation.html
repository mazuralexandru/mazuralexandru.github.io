19. Minimal HTML Document: Character Encoding (<meta charset>)

Inside the <head> section, one of the most crucial pieces of metadata is the character encoding declaration.

What is Character Encoding?

Computers store text as numbers (bytes). A character encoding defines the mapping between specific characters (letters, numbers, symbols) and the numbers used to represent them.

  • Using the wrong encoding can cause characters to display incorrectly (e.g., showing strange symbols like â€ or <?> instead of expected characters).
  • UTF-8 is the universal standard encoding for the web. It can represent virtually any character from any language, making it the ideal choice.

The <meta charset> Element:

  • Purpose: Tells the browser which character encoding the HTML file is saved in, so it can interpret the bytes correctly and display the text properly.
  • Syntax (HTML5): <meta charset="UTF-8">
    • <meta>: A generic tag for specifying metadata. It's a void element (no closing tag needed).
    • charset: An attribute specific to the <meta> tag when declaring the encoding.
    • "UTF-8": The value specifying the encoding. Always use UTF-8 for new web projects.

Placement:

  • Rule: The <meta charset="UTF-8"> declaration should be placed as early as possible within the <head> section.
  • Reason: The browser needs to know the encoding before it starts interpreting characters that might be affected by it, including potentially characters within the <title> tag itself. Ideally, it should be the very first element inside <head>.

Saving Files:

Remember (from Topic 8, which we'll revisit formally) that you also need to ensure your text editor actually saves the .html file using UTF-8 encoding. Declaring it in the HTML is only half the battle; the file itself must be saved correctly. Most modern editors default to UTF-8.

Code Change:

We've added the essential <meta charset="UTF-8"> tag as the first item inside the <head> section.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8"> <!-- Added character set declaration -->
  <!-- Metadata goes here -->
</head>
<body>
  <!-- Visible content goes here -->
</body>
</html>

✅ Observation:

  1. Check index.html to see the new <meta> tag inside <head>.
  2. The preview is still blank. This tag affects how text would be displayed if there were any, ensuring it displays correctly across different languages and symbols.

✨ Key Takeaway: Always include <meta charset="UTF-8"> as the first element inside your <head> section to ensure correct text display. Make sure your editor saves files as UTF-8.
