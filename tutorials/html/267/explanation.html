267. The id Attribute - Core Mechanics (Syntax & Uniqueness)

Now let's apply the id attribute and discuss its core mechanics.

Syntax:

  • The syntax is straightforward: id="unique-identifier"
  • It's placed within the opening tag of the element you want to identify.
  • The value (unique-identifier) is enclosed in quotes.

The Uniqueness Rule (CRITICAL):

  • The most important rule for id is that its value MUST be unique within the entire HTML document.
  • You cannot have two or more elements on the same page sharing the same id value.
  • Violating this rule leads to invalid HTML and unpredictable behavior (covered later).

Code Change:

We've added an id attribute to the <h1> element in index.html.

<!-- index.html snippet -->
<body>
  <h1 id="main-page-title">Global Attributes</h1> <!-- Added id attribute -->
  <p>Welcome to Module 6! Let's learn about attributes applicable to most HTML elements.</p>

  <p>These provide identification, styling hooks, and more.</p>

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

In this case, we've given the main heading the unique identifier main-page-title. No other element on this page can now use this exact ID.

✅ Observation:

  1. Locate the id="main-page-title" attribute added to the <h1> tag in index.html.
  2. Look at the browser preview. There is no visual change to the heading itself. The id attribute provides an identifier used internally by browsers, CSS, and JavaScript; it doesn't directly affect the default appearance.

✨ Key Takeaway: Add the id attribute to an element using id="unique_value". The value must be unique across the entire page. This attribute doesn't change the element's visual appearance directly.
