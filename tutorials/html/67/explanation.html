67. Text Styling: Handling Overflow (text-overflow)

What happens when inline text content is too long to fit within its container, especially when combined with white-space: nowrap or overflow: hidden? The text-overflow property provides a way to indicate this overflow visually, often with an ellipsis (...).

text-overflow Explained

  • Purpose: Specifies how hidden overflowed content should be signaled to the user. It only works on block containers where overflow is hidden.
  • Common Prerequisite Properties: For text-overflow: ellipsis; to have a visible effect on a single line, you typically need:
    1. white-space: nowrap; (To keep the text on one line)
    2. overflow: hidden; (To clip the content that doesn't fit)
  • Values:
    • clip: (Default) Simply cuts off the text at the container's edge. No visual indicator.
    • ellipsis: Displays an ellipsis ("...") to represent the clipped text. This is the most common use case.
    • <string> (Experimental): Allows specifying a custom string (e.g., " [...]") to indicate overflow.
    • fade (Non-standard, Firefox only): Fades out the text near the edge. Avoid for cross-browser compatibility.
  • Multi-line Ellipsis: Applying ellipsis to text that spans multiple lines is significantly more complex and often requires JavaScript or older, less standard CSS techniques (-webkit-line-clamp). text-overflow primarily targets single-line overflow.

In Our Example:

  • We set white-space: nowrap; and overflow: hidden; on .ellipsis-box to force the text onto one line and hide the excess.
  • We then apply text-overflow: ellipsis;.
  • We set a max-width to ensure the text actually overflows.
