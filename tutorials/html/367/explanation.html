367. Link States & Pseudo-classes: Purpose, :link

Files Modified: style.css

To make links interactive and provide visual feedback, CSS uses pseudo-classes. These are keywords added to selectors that specify a special state of the selected element(s). For links, the main states are unvisited, visited, hovered, active, and focused.

  • Purpose: Allow styling links differently depending on their state or how the user is interacting with them. This is crucial for usability and accessibility.

Let's start with the :link pseudo-class.

  • :link:
    • Selects: Links (<a> elements with an href attribute) that the user has not yet visited.
    • Usage: Used to define the default appearance of unvisited links.

✅ Observe style.css:

  • We've modified the a rule to specifically be a:link. This explicitly targets only unvisited links for the initial style.
  • We've set a specific color (cyan) for unvisited links, overriding the inherit value from the previous step.
/* Section 4: Styling Links Deep Dive */

/* Style unvisited links */
a:link { 
  color: cyan; /* Specific color for unvisited links */
  text-decoration: none; /* Remove default underline */
}
