255. Styling <hr>

Files Modified: style.css

While <br> isn't styled directly, the <hr> element can be styled quite extensively using CSS, allowing you to move beyond the default simple line.

Common Styling Techniques:

  • Controlling Thickness & Color: Use height and background-color, often after removing the default border.
  • Removing Default Border: Browsers often use a border to draw the line. You can remove this with border: none; or border-width: 0;.
  • Using Borders: Alternatively, you can remove the default background/height and use border-top (or border-bottom) to create the line, giving you control over style (solid, dotted, dashed) and color.
  • Margins: Adjust the default vertical margins using margin-top and margin-bottom.
  • Width: Control the width using width (e.g., width: 50%;) and potentially center it with margin-left: auto; margin-right: auto;.
  • Background Images: You can even use background images or gradients for highly customized dividers.

Semantic vs. Decorative: Remember that <hr> has semantic meaning (thematic break). If you just need a decorative line (e.g., under a heading), it's often better to use a CSS border-bottom on the heading element itself, rather than adding a semantically incorrect <hr>.

✅ Observe style.css:

  • We've added a rule to style all hr elements. It removes the default border, sets a height, adds a background color (a faint white line on our dark theme), and adjusts margins.
/* Styling Thematic Breaks */
hr {
  border: none; /* Remove default border */
  height: 1px; /* Define thickness */
  background-color: rgba(255, 255, 255, 0.2); /* Faint white line */
  margin: 2em 0; /* Add more vertical space */
}

✨ Observe the Preview: The horizontal lines separating the sections should now look different – they should be thinner, lighter gray (faint white), and have more vertical space around them compared to the browser's default <hr> style.
