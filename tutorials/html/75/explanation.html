75. Editor Settings: Saving as UTF-8

Files Modified: None

Declaring <meta charset="UTF-8"> in your HTML is only half the battle. You also need to ensure that your text editor actually saves the .html file using UTF-8 encoding.

  • Consistency is Key: The encoding declared in the <meta> tag must match the actual encoding used to save the file.
  • Modern Editors: Most modern code editors like VS Code default to saving files as UTF-8, so this is often handled automatically.
  • Checking/Changing Settings: You can usually check or change the file encoding in your editor's status bar or file saving options. In VS Code, the encoding is typically shown in the bottom-right status bar (clicking it often allows changing the encoding for saving).

If you declare UTF-8 in the HTML but save the file using a different encoding (like ISO-8859-1 or Windows-1252), you will still encounter garbled text issues (mojibake) because the browser's interpretation (based on the <meta> tag) won't match the file's actual byte structure.

Rule: Always ensure your editor is configured to save HTML files as UTF-8.
