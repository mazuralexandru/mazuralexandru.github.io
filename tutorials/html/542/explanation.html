542. Multimedia: Video (<video>) - Syntax & Fallback Content

Let's add the basic structure of the <video> element, including its fallback content.

Syntax Recap:

  • <video ...attributes...> Optional Source/Track Elements ... Fallback Content ... </video>
  • Requires opening and closing tags.

Fallback Content:

  • Purpose: Provides content to be displayed only by extremely old browsers that do not recognize or support the <video> element at all.
  • Placement: Any content placed directly between the <video> and </video> tags (after any <source> or <track> elements if they are used) acts as fallback content.
  • Content: Typically includes a message informing the user that their browser is outdated and often provides a direct link to download the video file or view it on a compatible platform.
  • Modern Relevance: Since all modern browsers support <video>, this fallback content is rarely seen by users today. However, it's still good practice to include a simple fallback message or link.

Code Change:

We've added <video> tags to index.html, including placeholder fallback content between them. We haven't added essential attributes like src or controls yet, so it won't function.

<!-- index.html snippet -->
  <video>
    <!-- Attributes (src, controls etc.) go in the opening tag -->
    <!-- Source/Track elements can go here -->
    <!-- Fallback content: -->
    <p>Sorry, your browser doesn't support embedded videos.</p>
    <p><a href="/videos/fallback_video.mp4">Download the video</a> instead.</p>
  </video> <!-- Added video tags with fallback -->

✅ Observation:

  1. Locate the <video>...</video> block in index.html. Note the fallback paragraph and link inside.
  2. Look at the browser preview. Because your modern browser does support <video>, it ignores the fallback content. Since we haven't provided a src or dimensions, you likely see nothing or perhaps a very small, empty placeholder box where the video element exists but has no source or size. The fallback text is not displayed.

✨ Key Takeaway: The <video> element requires opening and closing tags. Content placed between them serves as fallback for very old, non-supporting browsers and is ignored by modern browsers. Essential attributes (src, controls, dimensions) are still needed for functionality.
