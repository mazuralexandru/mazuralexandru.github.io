209. Other <a> Attributes: hreflang (Language Hint)

The hreflang attribute provides a hint about the language of the resource being linked to.

The hreflang Attribute:

  • Purpose: Specifies the human language of the document located at the link's destination URL.
  • Syntax: hreflang="language-code"
    • The language-code should be a valid BCP 47 language tag (e.g., en for English, es for Spanish, fr-CA for Canadian French, zh-Hans for Simplified Chinese).
  • Primary Use Case: SEO: It's mainly used by search engines (like Google) to understand the relationship between alternate language versions of the same page. If you have English, Spanish, and French versions of a page, you use hreflang (often in combination with rel="alternate" on <link> tags in the <head>, or directly on <a> tags) to tell search engines about these equivalent pages in different languages. This helps them serve the correct language version to users in search results.
  • Browser Behavior: Browsers generally don't do much with hreflang by default, though accessibility tools or browser extensions might potentially use it. Its main audience is search engine crawlers.

Code Change:

We've removed the previous rel examples and added a new link demonstrating the hreflang attribute in index.html. We also added conceptual <link rel="alternate"> tags in the <head> to show the typical context for multi-language SEO (though these <link> tags aren't the focus here).

<head>
  <meta charset="UTF-8">
  <title>HTML Links</title>
  <link rel="stylesheet" href="style.css">
  <!-- Example showing alternate language links -->
  <link rel="alternate" hreflang="es" href="https://example.com/es/page"> <!-- For SEO -->
  <link rel="alternate" hreflang="de" href="https://example.com/de/page"> <!-- For SEO -->
</head>
<body>

  <p>The 'hreflang' attribute specifies the language of the linked resource.</p>

  </p>
