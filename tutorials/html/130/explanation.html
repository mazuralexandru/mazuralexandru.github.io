130. Description Lists (<dl>): Multiple Groups

A description list (<dl>) typically contains multiple term/description groups. Each group follows the pattern of one or more <dt> elements followed by one or more <dd> elements.

Structuring Multiple Entries:

You simply add subsequent <dt>/<dd> groups directly within the <dl> container.

Code Change:

We've added a second term/description group (<dt>CSS</dt> followed by its <dd>) to the <dl> in index.html.

<dl>
   <dt>HTML</dt>
   <dd>HyperText Markup Language - used for structuring web content.</dd>

   <dt>CSS</dt> <!-- Added second dt -->
   <dd>Cascading Style Sheets - used for styling web content.</dd> <!-- Added second dd -->
</dl>

✅ Observation:

  1. Examine the structure in index.html. You now have two distinct dt/dd pairs within the single <dl>.
  2. Look at the browser preview:
    • The first term ("HTML") and its indented description appear.
    • Below that, the second term ("CSS") appears (unindented, possibly bold).
    • Below "CSS", its corresponding description ("Cascading Style Sheets...") appears, also indented.
    • The browser's default styling visually separates and formats each term/description pair.

✨ Key Takeaway: To include multiple entries in a description list, simply add more <dt>/<dd> groups sequentially within the <dl> container.
