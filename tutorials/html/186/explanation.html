186. The download Attribute: Boolean Syntax

The simplest way to use the download attribute is as a boolean attribute.

Boolean Syntax:

  • You simply add the download keyword attribute to the <a> tag, without assigning it a value.
    <a href="path/to/file.ext" download>Download File</a>
  • Behavior: The presence of the download attribute signals the download intention. When the user clicks the link:
    • The browser should initiate a download prompt.
    • The default filename offered in the "Save As..." dialog will typically be inferred from the last part of the URL path (file.ext in this case) or potentially from the Content-Disposition header sent by the server.

Code Change:

We've added a link to a hypothetical image file (/images/logo-large.png) and included the boolean download attribute in index.html.

<body>
  <h1>The 'download' Attribute</h1>
  <p>Boolean usage (suggests download, uses original filename):</p>

  <p>
    <a href="/images/logo-large.png" download>Download Logo (PNG)</a> <!-- Added boolean download -->
  </p>

  <p>Clicking should trigger a download prompt (browser/origin dependent).</p>

</body>

✅ Observation:

  1. Locate the download attribute (with no value) in the <a> tag in index.html.
  2. In the browser preview, you see the "Download Logo (PNG)" link.
  3. Behavior: Clicking this link should ideally trigger a download prompt instead of trying to display the image (even though browsers can usually display PNGs). However, this behavior strongly depends on browser support and same-origin policies. If the link points to a resource on a different domain (cross-origin), the download attribute might be ignored for security reasons unless specific server headers (CORS) are configured. In our local tutorial environment without the actual file, it will likely fail, but the HTML syntax is correct.

✨ Key Takeaway: Adding the download attribute without a value to an <a> tag suggests the linked resource should be downloaded, using the original filename by default. Its effectiveness depends on browser support and same-origin policies.
