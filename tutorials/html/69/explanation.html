69. Semantic HTML for Emphasis: <em> and <strong>

Moving back to HTML for a bit, let's revisit elements specifically designed for adding semantic meaning to text: <em> (emphasis) and <strong> (strong importance). While browsers style them by default (italic and bold), their primary purpose is meaning, not just appearance.

<em> and <strong> Explained

  • <em> (Emphasis):
    • Semantic Meaning: Represents stress emphasis on its contents. Think about which word you'd stress if reading the sentence aloud. Nested <em> tags increase the level of emphasis.
    • Default Styling: Typically rendered as italic text by browsers.
  • <strong> (Strong Importance):
    • Semantic Meaning: Indicates that its contents have strong importance, seriousness, or urgency. It's semantically stronger than <em>. Nested <strong> tags increase the level of importance.
    • Default Styling: Typically rendered as bold text by browsers.

Why Semantics Matter:

  • Accessibility: Screen readers can use these tags to change their voice inflection, conveying the intended emphasis or importance to visually impaired users. Using <em> and <strong> correctly improves the listening experience.
  • Search Engines & Browsers: While maybe less critical than headings, semantic tags can provide additional context to browsers and search engines about the structure and meaning of your content.
  • Maintainability: Using tags for their meaning makes your HTML more understandable. If you later decide all emphasized text should be blue instead of italic, you just change the CSS for em, rather than hunting down every place you used italics purely for presentation.

Styling Semantics:

  • You can (and often should) style these elements using CSS just like any other element.
  • In our example, we've kept the default font-style and font-weight but added custom colors (lightcoral for em, lightskyblue for strong) to make them stand out visually and show they can be styled.
