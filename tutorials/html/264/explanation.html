264. width Values: auto, Lengths, Percentages

Files Modified: style.css

Let's explore the common values for the width property:

  1. auto (Default for block elements):

    • The browser calculates the width.
    • For block-level elements, this typically means expanding to fill the full available horizontal space within its containing block, minus any horizontal margins, padding, or borders of the container.
    • For inline-block or replaced elements, auto usually means the width shrinks to fit the content.

  2. Length Units (px, em, rem, vw, etc.):

    • Sets a fixed or relative width based on the unit used.
    • width: 500px; sets a fixed width.
    • width: 20em; sets a width relative to the element's font size.
    • width: 50vw; sets the width to 50% of the viewport width.

  3. Percentage (%):

    • Sets the width as a percentage of the width of the containing block.
    • width: 50%; makes the element half the width of its parent container's content area. This is fundamental for creating fluid and responsive layouts.

✅ Observe style.css:

  • We've changed the width of .box from 300px to 75%.
.box {
  /* width: 300px; */ /* Changed from px */
  width: 75%; /* Set width relative to container */
  background-color: cornflowerblue; 
  padding: 10px; 
}

✨ Observe the Preview: The blue box should now take up 75% of the available width of the page body (its container). If you resize the preview window (if possible in your environment), the box's width should adjust fluidly, always maintaining that 75% proportion. This demonstrates percentage-based width.
