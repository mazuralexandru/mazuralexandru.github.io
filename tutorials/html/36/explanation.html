36. Text: Headings (<h1> to <h6>) - Hierarchy and Best Practices

Using headings correctly involves understanding their hierarchical nature and some important best practices.

1. Heading Hierarchy Rule:

  • Concept: Headings should form a logical, ordered hierarchy. You should start with the highest-level heading appropriate for the section (usually <h1> for the page) and step down sequentially without skipping levels.
  • Example:
    • Correct: <h1> -> <h2> -> <h3> -> <h2>
    • Incorrect: <h1> -> <h3> (Skipped <h2>)
    • Incorrect: <h2> -> <h4> (Skipped <h3>)
  • Why it Matters: Skipping levels can confuse users, especially those using screen readers who rely on the heading structure to navigate and understand the document's organization. It breaks the logical outline.

2. The <h1> Element:

  • Best Practice: Generally, use only one <h1> element per page. This <h1> should represent the main topic or title of that specific page. Think of it like the title of a book or a chapter.
  • HTML5 Nuance: While the HTML5 specification technically allows multiple <h1> elements if each is the primary heading of its own "sectioning content" element (like <article> or <section>), having a single, clear <h1> for the overall page is still widely considered the best practice for clarity and SEO. We will stick to the one-<h1>-per-page rule for this tutorial.

Code Change:

To illustrate the hierarchy, we've added an <h3> element logically nested under the <h2> within index.html.

<body>
  <h1>Main Page Title (h1)</h1>
  <p>This is the introductory paragraph following the main title.</p>

  <h2>A Major Section Title (h2)</h2>
  <p>This paragraph belongs to the first major section, introduced by the h2 heading above.</p>

  <h3>A Subsection Title (h3)</h3> <!-- Added h3 -->
  <p>This paragraph discusses a specific point within the major section, under the h3 subheading.</p>

  <p>Another paragraph within the same section (under the h2, but after the h3 subsection).</p>

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

Note how the last paragraph still logically falls under the <h2> because no new <h2> or <h1> has started.

✅ Observation:

  1. See the new <h3> in index.html.
  2. In the preview, observe the <h3> text. It should be bold (by default) but smaller than the <h2> and significantly smaller than the <h1>, visually reinforcing the hierarchy (H1 > H2 > H3).

✨ Key Takeaway: Use headings in a strict hierarchical order (h1 -> h2 -> h3...) without skipping levels. Aim for one main <h1> per page. This creates a logical and accessible document structure.
