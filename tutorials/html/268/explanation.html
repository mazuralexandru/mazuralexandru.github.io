268. height Values: auto, Lengths, Percentages

Files Modified: style.css

The common values for height are similar to width, but with a key difference for percentages:

  1. auto (Default):

    • The element's height adjusts automatically to fit its content. This is the most common behavior for elements like paragraphs or divs containing text.

  2. Length Units (px, em, rem, vh, etc.):

    • Sets a fixed or relative height based on the unit.
    • height: 200px; sets a fixed height.
    • height: 10em; sets height relative to the element's font size.
    • height: 50vh; sets the height to 50% of the viewport height.

  3. Percentage (%):

    • Sets the height as a percentage of the height of the containing block.
    • CRITICAL CAVEAT: For a percentage height to work, the containing block (parent element) must have an explicitly defined height (e.g., set in pixels, rems, vh, etc.). If the parent's height is auto (the default), then a percentage height on the child will typically be treated as auto as well. This often confuses beginners. Exceptions exist in specific layout contexts like Flexbox or Grid where percentage heights can sometimes work without an explicit parent height.

✅ Observe style.css:

  • We've changed the height of .box from 100px back to auto.
.box {
  width: 75%; 
  /* height: 100px; */ /* Changed from px */
  height: auto; /* Let height adjust to content (default) */
  background-color: cornflowerblue; 
  padding: 10px; 
}

✨ Observe the Preview: The blue box should now shrink or grow vertically to tightly contain its text content plus the 10px top/bottom padding. This demonstrates the default auto behavior. If we tried setting height: 50%;, it likely wouldn't work as expected because the parent (<body>) doesn't have an explicit height set.
