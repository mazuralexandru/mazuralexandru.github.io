37. Void / Empty Elements

Files Modified: None

Not all HTML elements follow the <tag>Content</tag> pattern. Some elements are void or empty.

  • Definition: These are elements that, by definition, cannot have any content (text or other elements nested inside them).
  • Syntax: They consist of a single tag, which is effectively the opening tag. They do not have a closing tag.
  • Examples:
    • <br> (Line break)
    • <hr> (Horizontal rule/thematic break)
    • <img> (Image)
    • <input> (Form input field)
    • <link> (Link to external resource, like CSS)
    • <meta> (Metadata)

You simply write <br> or <img>, not <br></br> or <img></img>.
