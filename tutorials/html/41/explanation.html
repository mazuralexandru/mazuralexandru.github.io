41. Inline Semantics: Emphasis (<em>)

Now let's start exploring inline semantic elements â€“ tags that wrap around specific words or phrases within a block-level element (like a paragraph) to add extra meaning. First up is <em>.

The Emphasis Element (<em>)

  • Purpose: Used to apply stress emphasis to its content. Think about how you might verbally emphasize a word in a sentence to change its nuance or meaning. That's the semantic role of <em>.
  • Semantics: It indicates that the enclosed text should be stressed relative to the surrounding text.
  • Type: It's an inline element. This means it doesn't start a new line and only takes up as much space as its content requires. It flows naturally within the line of text.
  • Syntax: Uses an opening <em> and a closing </em> tag.

Default Rendering:

Browsers typically render text within <em> tags in an italic font style by default. However, this is just a common visual convention. The primary meaning is emphasis, not italics. You could use CSS to make <em> text bold, red, or anything else, although sticking close to convention is usually less confusing for users.
