389. Grouping Content: <tfoot> - Fundamentals

The final row grouping element is <tfoot>, used for summary or footer rows.

The Table Foot Element (<tfoot>)

  • Definition: Represents the block of footer rows in an HTML table.
  • Purpose: To semantically group row(s) containing summary information about the table's columns, such as totals, averages, or footnotes that apply to the whole table.
  • Core Concept: It provides a distinct section for summary data, separate from the main data (<tbody>) and headers (<thead>).

Placement & Structure:

  • Must be a direct child of the <table> element.
  • Order in HTML Source: Can appear either before or after <tbody> elements, but must appear after any <caption> or <thead> elements. Historically, placing it before <tbody> was sometimes done to allow browsers to render the footer before receiving all body rows, but modern browser rendering makes this less relevant. Placing it logically at the end (after <tbody>) is often clearer.
  • Render Order: Regardless of source order (before or after <tbody>), browsers always render the <tfoot> content at the bottom of the table, after all <thead> and <tbody> rows.
  • A table can have at most one <tfoot> element.
  • Content Model: Must contain one or more <tr> elements. These rows often contain <td> cells, potentially using colspan to span across columns for totals.

Code Change:

We've added a <tfoot> element containing a summary row, placing it after the <tbody> in the source code of index.html.

<!-- index.html snippet -->
  <table>
    <thead>
      <tr>
        <th scope="col">Header 1</th>
        <th scope="col">Header 2</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>Data A1</td>
        <td>Data B1</td>
      </tr>
      <tr>
        <td>Data A2</td>
        <td>Data B2</td>
      </tr>
    </tbody>
    <tfoot> <!-- Added tfoot -->
      <tr>
        <td>Summary 1</td>
        <td>Summary 2</td>
      </tr>
    </tfoot> <!-- Closed tfoot -->
  </table>

✅ Observation:

  1. Locate the <tfoot> tags wrapping the final <tr> in index.html.
  2. Look at the browser preview. The "Summary 1" / "Summary 2" row appears at the bottom of the table, below the data rows, as expected. Again, there might be no distinct default styling difference just from adding <tfoot>, but it provides the semantic grouping and styling hook.

✨ Key Takeaway: Use the optional <tfoot> element to wrap summary row(s) (<tr>) for a table. It must appear after <caption> and <thead>, but can precede or follow <tbody> in the source (though it always renders last). A table can have only one <tfoot>.
