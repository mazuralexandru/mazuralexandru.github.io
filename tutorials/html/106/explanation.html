106. HTML Comments: Purpose Explained

Files Modified: None

Let's reiterate the main purposes of using HTML comments (<!-- ... -->):

  1. Explanation & Documentation: Add notes directly within the HTML to explain why certain code exists, what a particular section does, or leave reminders for future updates. This greatly improves code understanding, especially for complex structures or when returning to code after a long time.
    <!-- Navigation menu starts here -->
<nav>...</nav> 
<!-- End Navigation -->
  2. Debugging / Temporarily Disabling Code: If a part of your page isn't working correctly or you want to temporarily remove it without deleting the code, you can "comment it out". The browser will completely ignore the commented-out section. This is extremely useful during development and testing.
    <!-- Temporarily disabling the image gallery 
<div class="gallery">
  <img src="image1.jpg" alt="Image 1">
</div>
-->

Use comments judiciously to make your code clearer and easier to manage.
