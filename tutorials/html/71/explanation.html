71. Semantic HTML for Edits & Annotations

HTML provides specific tags to semantically mark up edits and annotations within text: <s>, <del>, <ins>, and <mark>.

Edit & Annotation Tags Explained

  • <s> (Strikethrough):
    • Semantic Meaning: Represents content that is no longer accurate or relevant. It's not for indicating document edits (use <del> for that). Think of a crossed-out price on a sale item.
    • Default Styling: Strikethrough text (text-decoration: line-through;).
  • <del> (Deleted Text):
    • Semantic Meaning: Represents a range of text that has been deleted from the document, often used for tracking changes or showing revisions.
    • Default Styling: Strikethrough text. Often styled further (like our reddish color) to distinguish it visually. Can optionally have cite (URL explaining change) and datetime attributes.
  • <ins> (Inserted Text):
    • Semantic Meaning: Represents a range of text that has been added to the document, often used alongside <del>.
    • Default Styling: Underlined text (text-decoration: underline;). Often styled further (like our greenish color). Can also have cite and datetime attributes.
  • <mark> (Mark Text / Highlight):
    • Semantic Meaning: Represents text marked or highlighted for reference purposes, due to its relevance in a particular context (e.g., highlighting search terms in results). It's not for general emphasis or importance.
    • Default Styling: Typically a yellow background (background-color: yellow;).

Styling Considerations:

While browsers provide default styles, you can customize the appearance of these elements heavily using CSS to fit your design or better convey the intended meaning (e.g., making deleted text less prominent).
