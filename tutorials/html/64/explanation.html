64. Preparing for Inline Code: Removing <pre> Block (Part 1)

We've covered the essentials of the <pre> element, including whitespace preservation, default font, overflow handling, and the crucial need for HTML escaping when displaying code.

Now, we'll shift our focus to the <code> element, which is used for marking up inline fragments of code. To provide a clear space for this, we'll remove the <pre> block containing the escaped HTML example.

Why Remove Code?

  • Focus: Transitioning from block-level preformatted text (<pre>) to inline code semantics (<code>). Removing the <pre> example prevents distraction.
  • Relevance: The specific escaped HTML example isn't needed to understand inline <code>.
  • Incremental Cleanup: Continuing our practice of clearing the stage after covering a specific element or concept.

Code Change:

We have removed the entire <pre>...</pre> block (which contained the escaped HTML) from index.html. The surrounding paragraphs have also been updated.

<body>
  <h1>Main Page Title (h1)</h1>
  <p>Preparing to discuss inline code.</p> <!-- Adjusted text -->

  <!-- Removed <pre> block with escaped code -->
  <!--
  <pre>
  ... escaped HTML content ...
  </pre>
  -->

  <p>This paragraph follows the preformatted text location.</p> <!-- Adjusted text -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

✅ Observation:

  1. Confirm that the <pre> block is gone from index.html.
  2. The browser preview should now only show the heading and the two plain paragraphs again, similar to the state before we introduced <pre>.

✨ Key Takeaway: Removing the block-level <pre> example clears the way for introducing the related but distinct concept of inline <code>.
