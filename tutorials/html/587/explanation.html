587. Graphics: Overview - Beyond <img>

Let's start by quickly revisiting standard images (<img>) and understanding why we need other graphics technologies.

<img> Recap:

  • Purpose: Displays pre-existing image files (JPEG, PNG, GIF, WebP, AVIF, SVG).
  • Nature: Primarily displays static content (though animated GIFs/WebP/AVIF exist).
  • Limitations:
    • Resolution Dependence (Bitmap): For formats like JPEG, PNG, GIF, WebP, AVIF (which are raster/bitmap), scaling them up significantly leads to pixelation and loss of quality. They have fixed pixel dimensions.
    • Limited Interactivity: Difficult to interact with individual parts of a raster image using CSS or JavaScript.
    • Dynamic Generation: Cannot easily generate complex graphics dynamically on the client-side based on data or user input using only <img>.

Need for Dynamic/Scalable Graphics:

Many web scenarios require more than static images:

  • Resolution Independence: Logos and icons that need to look crisp on any screen size or zoom level (including high-DPI/Retina displays).
  • Interactivity: Graphics where users can interact with specific parts (clicking shapes in a diagram, hovering over chart segments).
  • Dynamic Data Visualization: Charts and graphs generated based on changing data.
  • Animation: Complex visual effects or character animations beyond simple GIFs.
  • Games: Real-time rendering of game scenes.
  • Image Manipulation: Client-side photo editing or effects generation.

HTML5 Solutions: <canvas> and <svg>

These two technologies address the limitations of <img>:

  1. <canvas>: Provides a bitmap drawing surface controlled by JavaScript. Excellent for dynamic graphics, pixel manipulation, and complex scenes where performance is key. Resolution-dependent.
  2. <svg>: Defines vector graphics using XML markup. Scales infinitely without quality loss. Ideal for logos, icons, illustrations. Integrates with CSS and DOM/JavaScript.

We'll explore the fundamental differences next.

✅ Observation: No code changes. Understand the limitations of standard bitmap images (<img>) concerning scaling and interactivity, setting the stage for why <canvas> and <svg> are necessary for other types of graphics.

✨ Key Takeaway: While <img> is for displaying existing images, <canvas> (bitmap, JS-driven) and <svg> (vector, XML-based) provide native HTML mechanisms for creating dynamic, interactive, or resolution-independent graphics.
