334. Metadata: The <link> Element - Fundamentals

Let's formally introduce the <link> element, which we've already been using for our CSS.

Purpose:

  • The <link> element specifies relationships between the current HTML document and external resources.
  • It's used to link assets like stylesheets, icons, fonts, or provide hints about related documents (like alternate language versions or canonical URLs).

Element Category:

  • <link> is a void (or empty) element. It represents a relationship or connection, not content itself, so it has no closing tag.
  • It belongs in the <head> section of the HTML document.

Core Concept:

  • The <link> tag doesn't directly embed content like <img> or <script>. Instead, it defines a link to an external resource and, crucially, describes the type of that link or the relationship of that resource to the current document using the mandatory rel attribute.
  • The browser uses the rel attribute to understand how to use the resource specified in the href attribute.

✅ Observation: No code changes yet. Review the existing <link rel="stylesheet" href="style.css"> in the <head> of index.html. Understand that link connects to an external file, and rel="stylesheet" tells the browser how to use that file (apply it as CSS).

✨ Key Takeaway: The <link> element, placed in the <head>, is a void element used to define relationships between the current document and external resources like stylesheets, icons, or related pages. The rel attribute is key to defining this relationship.
