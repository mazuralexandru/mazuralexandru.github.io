166. Linking to Fragments: Links to Other Pages (href="page.html#id")

You can also link directly to a specific fragment (a section with an id) on a different HTML page.

Syntax (Different Page):

Combine the URL of the target page (which can be absolute or relative) with the fragment identifier.

  • href="target_page.html#elementID" (Relative URL example)
  • href="/path/to/target_page.html#elementID" (Root-relative URL example)
  • href="https://othersite.com/page.html#elementID" (Absolute URL example)

Behavior:

  1. The browser first navigates to the specified page (target_page.html or the external URL).
  2. Once the page loads, the browser then attempts to scroll to the element with the matching id (elementID).

Example (Conceptual):

If you had a separate page named faq.html with a section like <h2 id="billing">Billing Questions</h2>, you could link directly to it from your current page like this:

<!-- Conceptual Example -->
<a href="faq.html#billing">Jump to Billing Questions on FAQ page</a>

Code Change:

No changes are made to our index.html file in this step. We are explaining the concept of linking to fragments on other pages. Adding a functional example would require creating a second HTML file (faq.html) with appropriate ids, which is beyond the scope of simply modifying index.html.

✅ Observation: Understand conceptually that you can append #elementID to any valid URL (absolute or relative) in an href attribute to instruct the browser to load that page and then attempt to scroll to the specified id.

✨ Key Takeaway: To link to a specific section on a different page, use the format href="URL_of_page#elementID".
