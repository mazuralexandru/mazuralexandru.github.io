173. Combinators: Defining Relationships

Files Modified: index.html

So far, our selectors have targeted elements based on their type, class, ID, or attributes individually. Combinators allow us to create more complex selectors that target elements based on their relationship to other elements in the HTML document structure (their position in the DOM tree).

  • Purpose: To select elements based on whether they are descendants, children, or siblings of other elements.
  • Types: There are four main combinators:
    1. Descendant Combinator ( space)
    2. Child Combinator (>)
    3. Adjacent Sibling Combinator (+)
    4. General Sibling Combinator (~)

These allow for much more contextual and precise styling.
