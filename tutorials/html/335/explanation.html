335. Metadata: The <link> Element - Core Mechanics

Let's examine the core syntax and key attributes used with the <link> element.

Basic Syntax:

  • <link attribute1="value1" attribute2="value2" ...> (No closing tag)

Key Attributes:

  1. rel (Relationship) [Required]:

    • Defines the relationship between the current document and the linked resource specified by href.
    • This is the most crucial attribute, determining the link's purpose.
    • Common values include: stylesheet, icon, preconnect, preload, prefetch, dns-prefetch, alternate, canonical, author, license, help, next, prev, search, manifest. (We'll cover some of these).

  2. href (Hypertext Reference) [Usually Required]:

    • Specifies the URL (absolute or relative) of the external resource being linked.
    • Required for most rel types (like stylesheet, icon, preload). Might be optional for purely informational rel types in some contexts, but usually present.

  3. type (MIME Type Hint):

    • Provides a hint about the MIME type of the linked resource (e.g., type="text/css" for stylesheets, type="image/png" for icons).
    • Often optional for well-known types like CSS (rel="stylesheet" implies text/css) or when the server sends the correct Content-Type header. It's good practice for less common types or for clarity.

  4. media (Media Query):

    • Specifies the media conditions under which the linked resource (usually a stylesheet) should be applied.
    • Uses standard media types (e.g., screen, print) or media query syntax (e.g., (max-width: 600px)).

  5. hreflang (Href Language):

    • Used with rel="alternate" to indicate the language of an alternate version of the page.
    • Example: <link rel="alternate" hreflang="fr" href="french-page.html">

  6. as (Resource Type):

    • Required when using rel="preload" (and rel="modulepreload").
    • Specifies the type of content being preloaded (e.g., as="style", as="script", as="font", as="image", as="fetch"). Helps the browser prioritize the fetch correctly.

  7. crossorigin (Cross-Origin Settings):

    • Used when fetching resources (especially fonts, scripts, or via preload) from a different origin (domain) to configure CORS (Cross-Origin Resource Sharing) behavior (anonymous, use-credentials).

  8. sizes (Icon Sizes):

    • Used with rel="icon" (or apple-touch-icon) to specify the size(s) of the linked icon, helping the browser choose the best match.
    • Value can be any (for vector icons like SVG) or space-separated WidthxHeight pairs (e.g., sizes="16x16 32x32").

  9. title (Alternate Stylesheet Title):

    • Used with rel="stylesheet" or rel="alternate stylesheet" to define names for user-selectable stylesheets.

  10. disabled (Disable Stylesheet):

    • Boolean attribute to disable a linked stylesheet, usually manipulated via JavaScript.

Placement:

  • <link> elements must be placed within the <head> section.

✅ Observation: No code changes. Review the list of common attributes for <link>. Notice how rel and href are almost always present, while others (type, media, as, sizes, etc.) provide additional context or control depending on the rel value.

✨ Key Takeaway: The <link> element uses attributes like rel (required, defines relationship), href (usually required, specifies URL), and others (type, media, as, sizes, etc.) to connect external resources and provide context about them within the <head>.
