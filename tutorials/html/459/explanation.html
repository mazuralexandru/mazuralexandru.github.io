459. background Shorthand: Multiple Backgrounds

Files Modified: None

The ability to layer multiple backgrounds using comma separation within the background shorthand (or the individual properties like background-image, background-repeat, etc.) is very powerful.

  • Layering: background: <layer1>, <layer2>, ..., <last_layer_with_color>;
  • Order: The first layer listed is rendered on top, the last layer is rendered at the bottom (just above the optional background-color which can only be specified on this last layer).
  • Property Application: Each comma-separated group of values applies to the corresponding layer. If a property (like background-repeat) has fewer comma-separated values than the number of images, the list of values is repeated as needed.
  • Transparency: Upper layers need transparency for lower layers to be visible.

Example:

div {
  background: 
    /* Layer 1 (Top): Small icon, no repeat, top right */
    url(icon.png) no-repeat top right / 30px, 
    /* Layer 2: Gradient overlay */
    linear-gradient(rgba(0,0,255,0.5), rgba(0,0,255,0.5)), 
    /* Layer 3 (Bottom): Repeating pattern with background color */
    url(pattern.png) repeat scroll 0 0 / auto, lightgray; 
}

This allows for complex visual effects by combining images, gradients, repeating patterns, and solid colors within a single element.

We could make it more explicit:

/* More explicit version of our current shorthand */
.origin-box {
  /* ... other styles ... */
  background: 
    linear-gradient(to bottom, rgba(0,0,0,0.5), transparent) no-repeat scroll center top / auto padding-box content-box, /* Gradient layer */
    rgba(0, 100, 0, 0.2) url('images/placeholder-pattern.png') repeat scroll top left / 40px padding-box content-box; /* Image layer + Color */
}

(Note: This explicit version might render slightly differently if browser defaults were implicitly relied upon before, but illustrates applying properties per layer).
