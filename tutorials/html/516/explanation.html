516. Sectioning Content: <article> vs. <section> vs. <div>

Choosing between <article>, <section>, and <div> can sometimes be confusing. Let's clarify their distinct roles:

<article>:

  • Purpose: Self-contained, independently distributable/reusable content.
  • Test: Could this piece make sense on its own if syndicated (e.g., via RSS)? Could it stand alone as a complete unit?
  • Examples: Blog post, news article, forum post, individual comment, product description block.
  • Contains: Often its own header/footer, multiple sections. Can be nested.

<section>:

  • Purpose: Generic thematic grouping of content within a larger context (like an article or the main page content). Represents a distinct topic or functional area that belongs in the document outline.
  • Test: Does this group of content deal with a specific theme, and does it make sense to give it a heading in the document's outline?
  • Examples: Chapters within an article, different tab panels in a UI, introduction/features/conclusion sections on a marketing page.
  • Requires Heading: Should almost always have a heading (<h1>-<h6>) to identify its theme.
  • Not Self-Contained: Usually only makes full sense within the context of its parent document or article.

<div>:

  • Purpose: Generic grouping element with no semantic meaning.
  • Test: Is there any semantic element (<article>, <section>, <nav>, <aside>, <header>, <footer>, <p>, <ul>, etc.) that accurately describes this content or its role?
  • Use Case: Only use when no semantic element fits, primarily for:
    • Styling hooks (applying class or id for CSS).
    • Layout wrappers (e.g., creating containers for Flexbox/Grid items purely for positioning).
    • JavaScript hooks.
  • Avoid As Substitute: Do not use <div class="article"> instead of <article>, or <div class="sidebar"> instead of <aside>, etc.

Relationship Summary:

  • An <article> can contain multiple <section>s.
  • A <section> can contain multiple <article>s (e.g., a "Latest News" section containing several article summaries).
  • Both <article> and <section> can contain <div>s used for purely presentational grouping inside them.
  • <article> and <section> contribute to the document outline (especially with headings); <div> does not.

✨ Key Takeaway: Choose <article> for independent, self-contained content. Choose <section> for thematic groupings within a larger context (always with a heading). Use <div> only as a last resort for non-semantic grouping for styling/layout.
