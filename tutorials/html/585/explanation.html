585. Module 11 Recap: Looking Ahead to Module 12 (Graphics)

With Module 11, we've learned how to natively embed time-based media (<video>, <audio>), provide alternatives (<source>), ensure accessibility (<track>), and understand the context of legacy embedding (<embed>, <object>).

Connecting to the Next Module:

Our next module, Module 12: Graphics, will explore ways to create and embed graphical content beyond standard raster images (<img>). This module will focus primarily on:

  • Scalable Vector Graphics (SVG):
    • Revisiting SVG (.svg files), which we touched upon as an image format in Module 5.
    • Understanding SVG as an XML-based markup language for describing vector graphics.
    • Exploring how to embed SVG directly into HTML using the inline <svg> element.
    • Briefly mentioning the benefits (scalability, small size for non-complex graphics, CSS styling, JS manipulation).
    • Comparing inline <svg> with <img src="image.svg"> and <object data="image.svg">.
  • Canvas API (Introduction):
    • Introducing the <canvas> element, which provides a bitmap drawing surface.
    • Explaining that <canvas> itself is just a container; all drawing, animation, and interactivity are handled via JavaScript APIs (like the 2D Rendering Context or WebGL).
    • Highlighting its use cases (dynamic graphics, visualizations, games, image manipulation).
    • Note: This module will focus on the HTML aspect (<canvas> element, width/height, fallback content). Detailed Canvas drawing is a JavaScript topic beyond the scope of this HTML curriculum.

The semantic elements learned previously (<figure>, <figcaption>) can be used effectively to provide context and captions for SVG graphics or Canvas visualizations embedded in the page.
