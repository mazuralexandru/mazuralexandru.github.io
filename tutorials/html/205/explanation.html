205. font-family

Files Modified: style.css

The font-family property specifies a prioritized list (a "font stack") of one or more font family names for the selected element(s).

  • Purpose: To define the typeface used to render the text.
  • Font Stack: You provide a comma-separated list of font names. The browser starts with the first font in the list. If that font is available on the user's system (or loaded via @font-face), it's used. If not, the browser tries the next font in the list, and so on.
  • Generic Families (Fallbacks): It's crucial to end your font stack with a generic font family keyword. These keywords represent general categories of fonts, and the browser will pick a locally available font from that category if none of the specific fonts listed earlier are found. The main generic families are:
    • serif: Fonts with serifs (small strokes finishing off the main strokes, like Times New Roman).
    • sans-serif: Fonts without serifs (like Arial, Helvetica, Verdana). (Used in our base_styles.css)
    • monospace: Fonts where all characters have the same fixed width (like Courier New, Consolas). Used for code.
    • cursive: Fonts that emulate handwriting (like Comic Sans MS - use with caution!, Brush Script MT).
    • fantasy: Decorative or playful fonts (like Impact).
  • Quoting Names: Font names containing spaces must be enclosed in quotes (single or double). Single-word names don't strictly require quotes, but it's good practice. Generic family keywords should not be quoted.

✅ Observe style.css:

  • We've added a font-family rule to the #main-heading selector.
#main-heading {
    color: cornflowerblue; 
    text-decoration: underline; 
    font-family: "Georgia", Times, serif; /* Added font stack */
}

✨ Observe the Preview:

  • The "Main Heading" text should now be rendered in the Georgia font (if available on your system). If Georgia isn't available, it will try Times New Roman (or just "Times"). If neither is found, it will fall back to the browser's default serif font. Notice how it likely looks different from the paragraph text, which is still using the default sans-serif font inherited from the body style in base_styles.css.

Providing a good font stack ensures your text remains readable even if the user doesn't have your preferred font installed.
