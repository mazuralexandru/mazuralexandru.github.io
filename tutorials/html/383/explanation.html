383. Horizontal Nav (Inline Block): Spacing & Styling Links

Files Modified: style.css

With the list items arranged horizontally using display: inline-block;, we can now refine the spacing and link appearance.

  • Spacing Between Items: Since the <li> elements are now inline-block, we can use margin-left or margin-right on them to create space between the navigation items.
  • Link Styling: We style the <a> tags within the <li> elements. It's common practice to give the <a> tag itself display: block; (or inline-block) and apply padding to it. This makes the entire padded area of the link clickable, not just the text itself, improving usability.

✅ Observe style.css:

  • We've added margin-right to nav li to space out the items (except for the last one, using :not(:last-child) to avoid extra space at the end).
  • We've added display: block; to nav a to make the entire padded area clickable.
nav li {
    margin: 0; 
    display: inline-block; 
    margin-right: 0.5em; /* Add space between items */
}
/* Remove margin from the last item */
nav li:last-child {
    margin-right: 0;
}


/* Basic link styling within nav */
nav a {
    text-decoration: none;
    color: lightblue; 
    padding: 0.5em 1em; 
    display: block; /* Make the link fill the li for easier clicking */
    border-radius: 4px; /* Add some rounding */
}

nav a:hover, 
nav a:focus-visible {
    background-color: rgba(255, 255, 255, 0.1);
    color: white;
    outline: none;
}

(Note: :last-child and :not() are pseudo-classes we haven't formally covered but are used here for a common spacing pattern).

✨ Observe the Preview:

  • There should now be a small horizontal gap between each navigation item ("Home", "About", "Services", "Contact").
  • The clickable area for each link (indicated by the hover background effect) should extend to the full padded size defined for the <a> tag.
