259. Module 5 Recap: Images

This module focused on embedding images and related semantic structures.

Key Concepts Reviewed:

  • Image Element (<img>):
    • Embeds images into HTML documents.
    • Void element (no closing tag). Inline/Embedded content.
  • src Attribute:
    • Mandatory. Specifies the URL (path) of the image file.
    • Can use Absolute URLs (external images/CDNs) or Relative URLs (internal website images - preferred for portability).
  • alt Attribute:
    • Mandatory. Provides alternative text.
    • Crucial for: Accessibility (screen readers), fallback display (broken images), SEO.
    • Content: Describe the image's content or function accurately and concisely within context.
    • Decorative Images: Use alt="" (empty value) for purely decorative images that add no information, instructing assistive tech to ignore them. Never omit alt entirely.
    • Functional Images (Links/Buttons): alt text must describe the destination or action.
  • width & height Attributes:
    • Specify the image's intrinsic dimensions in pixels (unitless values).
    • Essential for Performance: Prevent Cumulative Layout Shift (CLS) by allowing the browser to reserve space before the image loads.
    • Should match the actual image dimensions to avoid distortion.
    • Work with CSS for responsive design (provide aspect ratio for layout reservation, while CSS controls final rendered size).
  • Image Formats:
    • JPEG: Best for photos (lossy compression). No transparency.
    • PNG: Best for graphics needing transparency or sharp lines (lossless compression, alpha transparency).
    • GIF: Best for simple animations (limited colors, basic transparency).
    • SVG: Best for logos, icons, illustrations (vector, scalable, styleable). Not for photos.
    • WebP: Modern format, often better compression than JPEG/PNG/GIF, supports transparency/animation. Good browser support.
    • AVIF: Newer format, potentially even better compression, supports HDR/transparency/animation. Support growing.
    • Optimization: Always optimize images (resize, compress) for web performance.
  • Figures & Captions:
    • <figure>: Semantically groups self-contained content (image, code, quote, etc.) often referenced from the main text.
    • <figcaption>: Provides a visible caption for its parent <figure>. Only one per figure (first or last child). Crucial for accessibility context. Complements, but does not replace, alt text for images within the figure.

✅ Observation: No code changes. This is a conceptual summary of Module 5.

✨ Looking Ahead: Having covered text, lists, links, and images, Module 6 will introduce Global Attributes – attributes like id, class, style, title, etc., that can be applied to almost any HTML element to provide identification, styling hooks, and additional information.
