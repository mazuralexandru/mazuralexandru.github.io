352. Reset: Preparing for <style> Element

We've explored the purpose, mechanics, and specific use cases (primarily SPAs) of the <base> element.

Next, we'll look at the <style> element, which allows embedding CSS rules directly within the HTML document (internal styles).

To focus on <style>, we will remove the <base> element and the example link from our index.html. We'll also restore the default CSS loading by removing the comment warning about the broken path.

Why Remove Code?

  • Focus: Isolate the <style> element and the concept of internal stylesheets.
  • Restore Styling: Removing <base href="..."> will allow our external style.css (containing the base body styles) to load correctly again via its relative path.
  • Clean Slate: Provide a basic structure to demonstrate adding internal styles.

Code Change:

  1. Removed the <base> element from the <head>.
  2. Removed the example link (<a href="doc.pdf">...</a>) from the <body>.
  3. Updated the <h1> and paragraph text.
<!-- index.html -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>HTML Metadata - Style Element</title> <!-- Updated title -->
  <!-- Removed base element -->
  <link rel="stylesheet" href="style.css"> <!-- CSS should load correctly now -->
</head>
<body>
  <h1>Metadata: The &lt;style&gt; Element</h1> <!-- Updated H1 -->
  <p>Preparing to discuss the 'style' element for embedding CSS.</p> <!-- Adjusted P -->

  <!-- Removed example link -->

  <p>This allows CSS rules directly within the HTML document.</p> <!-- Adjusted P -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>
</html>

✅ Observation:

  1. Verify the removal of the <base> element from the <head> and the link from the <body>.

  3. The page shows the updated heading and paragraphs, ready for the <style> element discussion.

✨ Key Takeaway: Removing the <base> element restores standard relative URL resolution and prepares the page for demonstrating internal CSS using the <style> element.
