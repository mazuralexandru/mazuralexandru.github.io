235. Decorative Images (alt="")

What if an image provides no information and is purely decorative? Examples include background flourishes, spacer images, or icons that visually duplicate adjacent text.

In these specific cases, providing descriptive alt text would actually be detrimental to accessibility, adding unnecessary "noise" for screen reader users.

The Empty alt Attribute (alt="")

  • Purpose: To explicitly signal that an image is decorative and should be ignored by assistive technologies.
  • Syntax: Use the alt attribute, but leave its value empty.
    <img src="images/decorative-swirl.png" alt="">
  • Crucial Distinction: This is different from omitting the alt attribute altogether. Omitting alt is invalid HTML and leads to unpredictable behavior. Providing an empty alt="" is valid and conveys the specific instruction: "This image is decorative, ignore it."

Behavior:

  • Screen readers typically skip right over images with alt="", providing a cleaner experience.
  • If a decorative image with alt="" fails to load, browsers usually display nothing in its place (no broken icon, no text).

Code Change:

We've added a second <img> tag to index.html, representing a purely decorative image, and given it an empty alt attribute.

<body>
  <h1>Embedding Images</h1>
  <p>Example of an image with appropriate alt text:</p>

  <img src="images/logo.png" alt="Example Corp Logo">

  <p>For purely decorative images, use an empty alt attribute:</p> <!-- Added P -->

  <img src="images/decorative-border.png" alt=""> <!-- Added decorative img with alt="" -->
