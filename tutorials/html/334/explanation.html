334. List Items (<li>, <dt>, <dd>)

Files Modified: None

Let's quickly recap the roles of the elements used within lists:

  • <li> (List Item):

    • Purpose: Represents a single item within an ordered (<ol>) or unordered (<ul>) list.
    • Placement: Must be a direct child of a <ul> or <ol> element.
    • Content: Can contain "flow content," meaning most other HTML elements like paragraphs, headings, images, links, and even nested lists.

  • <dt> (Description Term):

    • Purpose: Represents the term or name part of an item in a description list (<dl>).
    • Placement: Must be a direct child of a <dl> element (or a <div> wrapper within a <dl>). Must be followed by one or more <dd> elements.
    • Content: Can only contain "phrasing content," which generally means inline elements like <span>, <a>, <em>, <strong>, <code>, etc., but not block-level elements like <p> or <div>.

  • <dd> (Description Details):

    • Purpose: Represents the description, definition, or value associated with the preceding term(s) (<dt>) in a description list (<dl>).
    • Placement: Must be a direct child of a <dl> element (or a <div> wrapper within a <dl>). Must follow one or more <dt> elements.
    • Content: Can contain "flow content," similar to <li>.

Understanding these placement and content rules helps ensure you create valid and semantically correct lists.
