137. Preparing for Accessibility: Removing <div> Wrappers

We've seen the optional <div> wrapper pattern for <dl> groups and discussed its benefits, primarily for CSS styling and layout hooks.

For the upcoming discussion on the accessibility of description lists, the core semantic relationship exists between the <dt> and <dd> elements themselves, regardless of whether a <div> wrapper is present. To keep the focus on this core relationship and simplify the HTML structure for the accessibility discussion, we will remove the <div> wrappers we just added.

Why Remove Code?

  • Focus: Concentrate on the essential <dl>, <dt>, and <dd> elements and how assistive technologies interpret their relationship.
  • Simplicity: Revert to the most basic, valid <dl> structure.
  • Core Semantics: The fundamental accessibility features relate to the dt/dd pairing, not the optional wrapper.

Code Change:

We have removed the <div> wrappers from around the <dt>/<dd> groups in index.html.

<body>
  <h1>Description Lists (dl)</h1>
  <p>Reverting to simple dt/dd structure.</p> <!-- Adjusted P -->

  <dl>
     <dt>Term 1</dt> <!-- Removed div wrapper -->
     <dd>Description 1.</dd>

     <dt>Term 2</dt> <!-- Removed div wrapper -->
     <dd>Description 2.</dd>
  </dl>

  <p>Ready to discuss accessibility.</p> <!-- Adjusted P -->

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>
