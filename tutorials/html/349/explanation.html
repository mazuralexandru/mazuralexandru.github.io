349. Other <head> Elements: The <base> Element - Core Mechanics

Let's examine the syntax and attributes of the <base> element.

Syntax:

  • As a void element: <base attribute1="value1" ...>
  • Must be inside <head>.
  • Only one <base> element allowed.

Key Attributes:

  1. href (Base URL):

    • Purpose: Specifies the base URL to use for resolving all relative URLs in the document.
    • Value: Must be an absolute URL (including the scheme, like https://). It usually ends with a filename or, more commonly, a directory path (ending in /).
    • Effect: If <base href="https://example.com/docs/"> is set, a relative link <a href="setup.html"> will resolve to https://example.com/docs/setup.html, regardless of where the current HTML file is located. A root-relative link <a href="/images/logo.png"> will resolve to https://example.com/images/logo.png (the base URL's domain is used, but the path starts from the root). Absolute URLs in links are unaffected.

  2. target (Default Link Target):

    • Purpose: Specifies the default browsing context (like _self, _blank, _parent, _top, or a framename) for all links (<a> tags with href) and potentially form submissions in the document that do not have their own explicit target attribute.
    • Effect: If <base target="_blank"> is set, any link like <a href="page.html"> will open in a new tab/window, unless it has its own target attribute (e.g., <a href="page.html" target="_self">).
    • Caution: Setting a global target="_blank" can be annoying for users if all links suddenly open new tabs. Use judiciously.

Code Change:

We've added a <base> element to the <head> of index.html, specifying both an href and a target.

<!-- index.html -->
<head>
  <meta charset="UTF-8">
  <title>HTML Metadata - Base Element</title> <!-- Updated title -->
  <base href="https://www.example.com/files/" target="_blank"> <!-- Added base element -->
  <link rel="stylesheet" href="style.css">
  <!-- Note: The href for style.css is now relative to the base URL! -->
  <!-- This will likely break CSS loading in this example setup. -->
</head>
<body>
  <h1>The &lt;base&gt; Element</h1> <!-- Updated H1 -->
  <p>Added a base element to the head (check the HTML).</p> <!-- Adjusted P -->
  <p>It sets a base URL for relative links (like <a href="doc.pdf">doc.pdf</a>) and a default target (_blank).</p> <!-- Added example link -->

  <p>Beware: It affects ALL relative URLs after its declaration!</p>

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

✅ Observation:

  1. Locate the new <base href="https://www.example.com/files/" target="_blank"> tag in the <head>.
  2. CSS Loading Issue: Notice the comment. Because <base href="..."> affects all relative URLs after its declaration, our relative link <link rel="stylesheet" href="style.css"> will now try to load https://www.example.com/files/style.css, which doesn't exist! This will likely cause our base styling (dark background, light text) to disappear in the preview. This demonstrates the powerful and potentially disruptive effect of <base href>.
  3. Link Behavior: Hover over the "doc.pdf" link. The browser might show its target URL as https://www.example.com/files/doc.pdf. Clicking it would attempt to open that URL in a new tab because of target="_blank" set in the <base> tag. (It will fail as the URL is hypothetical).

✨ Key Takeaway: The <base> element sets a base URL (href) for relative paths and a default link target (target). It affects all relative URLs declared after it in the document, including stylesheets, scripts, and images, which requires careful consideration.
