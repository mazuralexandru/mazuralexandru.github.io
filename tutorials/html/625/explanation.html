625. Module 12 Recap: Graphics (<canvas>, <svg>)

This module explored the two primary native HTML technologies for creating graphics beyond standard images: <canvas> and <svg>.

Key Concepts Reviewed:

  • Bitmap (Raster) vs. Vector:
    • Bitmap (<canvas>, JPEG, PNG): Grid of pixels, resolution-dependent (blurs on scaling). Good for photos.
    • Vector (<svg>): Mathematical descriptions of shapes, resolution-independent (scales perfectly). Good for logos, icons, illustrations.
  • Immediate vs. Retained Mode:
    • Immediate (<canvas>): JS commands draw pixels directly; browser doesn't store object model. Requires manual redraws for changes/animation.
    • Retained (<svg>): Browser builds/retains a DOM tree from XML markup. Browser redraws automatically when DOM is changed via JS/CSS.
  • <canvas> Element:
    • HTML container (<canvas id="..." width="..." height="...">Fallback</canvas>).
    • width/height attributes set drawing buffer resolution (use pixels).
    • Requires JavaScript for all drawing via a rendering context.
    • getContext('2d'): Provides the 2D drawing API (fillRect, stroke, beginPath, lineTo, arc, fillText, drawImage, save/restore, transform, etc.).
    • getContext('webgl'/'webgl2'): For 3D graphics.
    • Performance: Good for large numbers of simple objects, pixel manipulation, complex real-time animations/games. Degrades with canvas pixel size.
    • Accessibility: Major challenge. Requires significant effort (fallback content, ARIA, parallel DOM, custom keyboard handling) as content is not inherently accessible.
    • Use Cases: Games, complex visualizations, image processing, high-framerate animations.
  • <svg> Element (Scalable Vector Graphics):
    • XML-based markup language for vector graphics.
    • Embedding: Inline (<svg>...</svg>), external (<img src=".svg"> - common), CSS background, <object>, <iframe>.
    • <svg> Root Element Attributes: width/height (viewport size), viewBox (internal coordinate system/scaling), preserveAspectRatio, xmlns.
    • Core Elements: Shapes (<rect>, <circle>, <ellipse>, <line>, <polyline>, <polygon>), complex shapes (<path> with d attribute), text (<text>, <tspan>, <textPath>), grouping (<g>), reuse (<defs>, <use>, <symbol>).
    • Styling: Prefer CSS (targeting type, class, ID) over presentation attributes or inline style. Use fill, stroke, transform, etc. Supports CSS Transitions/Animations.
    • Interactivity: Elements are part of DOM; use standard JS DOM manipulation and event listeners (addEventListener).
    • Performance: Good for fewer complex objects, large areas. Degrades with DOM complexity. Optimize with tools (SVGO) and techniques (<use>).
    • Accessibility: Good foundation. Text is real, structure is navigable. Use <title> (accessible name) and <desc> (description) elements inside <svg>/<g>, plus ARIA (role="img", aria-labelledby) for best results.
    • Use Cases: Logos, icons, illustrations, diagrams, infographics, moderate data visualizations, UI elements needing scalability.
  • Choosing <canvas> vs. <svg>: Based on requirements for scalability, performance profile (many simple vs. few complex objects), DOM interaction needs, text handling, accessibility priority, and development approach (imperative JS vs. declarative markup/CSS/DOM). SVG often preferred for static/moderately interactive graphics and icons due to scalability and accessibility.
