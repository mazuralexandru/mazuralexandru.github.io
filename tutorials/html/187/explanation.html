187. The download Attribute: With Value (Suggesting Filename)

You can also provide a value to the download attribute to suggest a specific filename for the downloaded file.

Syntax with Value:

  • Assign the desired filename as a string value to the download attribute.
    <a href="path/to/resource.abc" download="user-friendly-name.ext">Download Now</a>
  • Behavior:
    • Still triggers a download prompt (subject to same-origin policies).
    • The filename provided in the download attribute's value (user-friendly-name.ext) will be suggested as the default filename in the browser's "Save As..." dialog, potentially overriding the filename in the original URL (resource.abc).
    • The file extension in the download value doesn't have to match the original, but it's usually best if it does to avoid user confusion. Browsers might handle extension mismatches differently.
    • The browser might sanitize the suggested filename (e.g., replacing invalid characters like / or \).

Use Cases:

  • Providing more descriptive or user-friendly filenames than the original resource URL might have (e.g., changing doc_id_12345.pdf to Project-Proposal.pdf).
  • Ensuring a consistent filename format for downloads.
  • Downloading dynamically generated content (like data URIs) with a meaningful filename.
