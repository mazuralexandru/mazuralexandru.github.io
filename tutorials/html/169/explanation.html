169. Grouping Selectors

Files Modified: None

Often, you want to apply the exact same set of CSS declarations to several different elements or selectors. Instead of writing duplicate rulesets, you can group the selectors together.

  • Purpose: To apply a single declaration block to multiple selectors simultaneously, reducing repetition and making the CSS more concise and maintainable (following the DRY - Don't Repeat Yourself - principle).
  • Syntax: List the individual selectors separated by commas (,).
/* Without Grouping */
h1 {
  font-family: Georgia, serif;
  color: #444;
}
h2 {
  font-family: Georgia, serif;
  color: #444;
}
h3 {
  font-family: Georgia, serif;
  color: #444;
}

/* With Grouping */
h1, h2, h3 {
  font-family: Georgia, serif;
  color: #444;
}

Both achieve the same result, but the grouped version is much shorter and easier to update. You can group any combination of selector types (type, class, ID, attribute, etc.).
