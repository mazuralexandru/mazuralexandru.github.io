525. Other Semantics: <mark> - Fundamentals

The <mark> element is used to highlight text for reference or notation purposes, similar to using a highlighter pen.

The <mark> Element:

  • Definition: Represents a run of text marked or highlighted due to its relevance in a specific context, but not necessarily implying importance or emphasis in the original document context.
  • Purpose: To draw the reader's attention to a particular piece of text because of its relevance to the user's current activity (like a search query) or for notation.

Core Concept & Semantics:

  • It signifies relevance or notation, distinct from:
    • <strong> (importance)
    • <em> (stress emphasis)
    • <i> (alternate voice)
    • <b> (stylistic offset)

Syntax:

  • <mark> ... text to highlight ... </mark>
  • Requires opening and closing tags. Inline element.
  • Content: Phrasing content.

Default Rendering:

  • Browsers typically render the content of <mark> with a yellow background (or similar highlight color). This can be customized with CSS.

Use Cases:

  • Highlighting Search Results: When displaying search results within a page, the matching query terms are often wrapped in <mark> tags.
  • Quoted Text Reference: Highlighting a specific portion of quoted text (<blockquote>) that is being discussed or referred to in the main text.
  • User Annotations: Representing text that a user might have highlighted in a document viewer application.
  • Drawing Attention: Marking specific terms or phrases for attention in instructional text (though <strong> might sometimes be more appropriate if importance is implied).
