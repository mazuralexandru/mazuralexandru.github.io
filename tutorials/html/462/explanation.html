462. Forms: Text Areas (<textarea>) - Other Attributes & Behavior

Let's look at other common attributes applicable to <textarea>.

Common Attributes:

  • required (Boolean): Makes input mandatory. The textarea cannot be empty on submission if this is present.
  • minlength / maxlength: Specify minimum and maximum number of characters allowed. maxlength prevents further typing; minlength is validated on submission.
  • placeholder: Provides a short hint or example within the textarea that disappears when the user starts typing. Not a substitute for <label>.
  • readonly (Boolean): The content is displayed but cannot be edited by the user. The value is submitted.
  • disabled (Boolean): Disables the textarea entirely. It cannot be focused or edited, and its value is not submitted.
  • wrap: Controls how line breaks within the submitted value are handled.
    • soft (Default): Line breaks seen by the user due to wrapping within the textarea's width are not submitted as actual line breaks. Only literal line breaks entered by the user (pressing Enter) are submitted (usually as CRLF: %0D%0A).
    • hard: Line breaks induced by text wrapping are included in the submitted value along with user-entered line breaks. Requires the cols attribute to be set to define the wrap width. Rarely used.
    • off: Prevents wrapping altogether (text becomes horizontally scrollable). Very rarely used.
  • autocomplete: Controls browser autofill (on or off).

Behavior Notes:

  • Whitespace Preservation: Unlike most HTML elements, whitespace within the <textarea> tags (the initial value) and whitespace entered by the user is generally preserved, including multiple spaces and line breaks.
  • Submission: The text content entered by the user (or the initial content if unchanged) is sent as the value associated with the textarea's name attribute. Line breaks are typically encoded (e.g., as %0D%0A).

Code Change:

Let's add required, maxlength, and placeholder to our <textarea> example. We'll remove the initial content to see the placeholder.

<!-- index.html snippet -->
<form action="/submit-comment" method="POST">
  <p>
    <label for="user_comment">Your Comment:</label><br>
    <textarea id="user_comment" name="comment" rows="5" cols="40"
              required
              maxlength="200"
              placeholder="Max 200 characters..."></textarea> <!-- Added attributes, removed initial content -->
  </p>
  <p><button type="submit">Post Comment</button></p>
</form>

✅ Observation:

  1. Examine the updated attributes on the <textarea>.
  2. In the preview, the textarea should now be empty initially, but display the greyed-out text "Max 200 characters...".
  3. Try typing more than 200 characters – the browser should prevent you.
  4. (Conceptual): If you tried submitting the form with the textarea empty, native validation (because of required) would likely trigger an error.

✨ Key Takeaway: <textarea> supports attributes like required, minlength/maxlength, placeholder, readonly, disabled, and wrap to control its behavior and validation. CSS is preferred for sizing (rows/cols are hints).
