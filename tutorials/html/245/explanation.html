245. Escaping HTML within Code

Files Modified: style.css

It's critical to reiterate the need for escaping HTML characters (<, >) when displaying HTML code itself within a <code> or <pre><code> block.

  • Why? If you write literal < and > characters inside your code example, the browser will try to interpret them as actual HTML tags, potentially breaking your page layout or rendering the code incorrectly.
  • How? Replace literal < and > characters within your code examples with their corresponding HTML entities:
    • < becomes &lt; (less-than)
    • > becomes &gt; (greater-than)
  • Ampersands: If your code example itself contains an ampersand (&), you should also escape it as &amp; to avoid confusion with the start of an entity.
<!-- Correctly escaped HTML code example -->
<pre><code>&lt;div class="alert alert-danger"&gt;
  &lt;strong&gt;Error:&lt;/strong&gt; Please check your input &amp;amp; try again.
&lt;/div&gt;</code></pre>

Many code editors have functions to automatically escape selected text for HTML. Server-side languages and JavaScript frameworks also provide tools for safe HTML escaping when displaying dynamic code snippets.
