61. Specialized Text: Preformatted Text (<pre>)

Now let's explore elements designed for specific types of text content. We'll start with <pre>, used for preformatted text.

The Preformatted Text Element (<pre>)

  • Purpose: Defines a block of text where whitespace (spaces, tabs, line breaks) found in the HTML source code is preserved exactly as written. Standard HTML whitespace collapsing rules do not apply within <pre>.
  • Type: Block-level element.
  • Syntax: Uses an opening <pre> and closing </pre> tag.

Behavior:

  • Whitespace Preservation: This is its key feature. Multiple spaces won't collapse into one, and line breaks in the source code will render as line breaks on the page.
  • Default Font: Browsers typically display text inside <pre> using a monospace font (like Courier or Consolas), where all characters have the same width. This is ideal for aligning text vertically, such as in code examples or ASCII art.

Use Cases:

  • Displaying blocks of computer code.
  • Showing ASCII art.
  • Presenting formatted output from programs or command lines.
  • Any situation where the exact spacing and line breaks of the text are significant and need to be maintained.

Code Change:

We've added a <pre> block between the two paragraphs in index.html, containing text with deliberate extra spaces and line breaks.

✨ Key Takeaway: Use the <pre> element when you need to display text exactly as it is formatted in your HTML source, preserving all whitespace and typically rendering in a monospace font. It's essential for code blocks and ASCII art.
