11. Void / Empty Elements (Part 2: Examples & Syntax)

Let's look at the syntax of a couple of common void elements: <hr> and <br>.

Code Changes:

We've added two void elements and another paragraph to index.html:

<!-- index.html -->
<!-- Content will go here soon! -->
<p>This is a paragraph.</p>
<hr> <!-- Added this horizontal rule -->
<p>This is another paragraph.<br>This text appears on a new line.</p> <!-- Added this paragraph with a line break -->

Examples:

  1. <hr> (Thematic Break / Horizontal Rule):

    • Represents a thematic break between paragraph-level elements. It's often displayed visually as a horizontal line by default, but its semantic meaning is about separation.
    • Syntax: <hr> - Just the single tag.

  2. <br> (Line Break):

    • Represents a line break within a block of text. It forces the content that follows it onto a new line without starting a new paragraph (which would typically add more vertical space).
    • Syntax: <br> - Just the single tag. Notice how it's placed within the second <p> element's content.

Syntax Variations (Self-Closing Slash):

You might sometimes see void elements written with a trailing slash, like <hr /> or <br />.

  • HTML5 Syntax: The slash is optional and generally omitted. <hr> and <br> are preferred.
  • XHTML Syntax: In the older, stricter XHTML standard, this slash was required for void elements (called "self-closing tags").
  • Explanation: Modern HTML5 browsers understand both forms. The trailing slash is mostly ignored.
  • Best Practice: For HTML5, consistently use the simpler form without the slash (<hr>, <br>, <img>). Be aware that you might encounter the other style in older code or code aiming for XML compatibility.

✅ Observation:

  1. Examine index.html. Note the placement of <hr> between the paragraphs and <br> inside the second paragraph.
  2. Look at the preview pane. You should see:
    • The first paragraph.
    • A horizontal line below it (the <hr>).
    • The second paragraph, with the text after <br> appearing directly on the line below the preceding text within that same paragraph.

✨ Key Takeaway: Void elements like <hr> and <br> use a single tag. The trailing slash (/) is optional in HTML5 and usually omitted.
