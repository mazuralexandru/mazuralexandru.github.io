561. Multimedia: Multiple Sources (<source>) - media Attribute (Responsive Video)

Beyond format compatibility, the <source> element can also be used with the media attribute to implement responsive video, serving different video files based on device characteristics like viewport width.

The media Attribute on <source>:

  • Purpose: Specifies a media query that must be true for the browser to consider using this particular <source> element.
  • Syntax: <source src="..." type="..." media="media-query">
  • Value: A valid CSS media query string (e.g., (max-width: 600px), (min-width: 1024px), (orientation: portrait)).
  • Behavior: During source selection, the browser checks if the media query defined in the media attribute currently evaluates to true. If it doesn't match, the browser skips this <source> and moves to the next one, even if it supports the type.

Use Case: Saving Bandwidth:

  • Serve a lower-resolution, lower-bitrate (smaller file size) video version to devices with smaller viewports (e.g., mobile phones).
  • Serve a higher-quality version to larger screens (desktops, tablets).
  • This prevents mobile users from unnecessarily downloading large video files designed for desktop viewing, saving bandwidth and improving load times.

Code Change:

We've modified the <source> elements in index.html to demonstrate using the media attribute. We list a hypothetical "small" video source first, conditioned on a max-width of 600px, followed by a "large" version as the default fallback. (We've also added preload="metadata" back to the <video> tag as it's relevant here).

<!-- index.html snippet -->
  <video controls width="480" height="270" preload="metadata"> <!-- Added preload -->
    <!-- Serve smaller video on smaller screens -->
    <source src="/videos/example-small.mp4" type="video/mp4" media="(max-width: 600px)"> <!-- Added media query -->
    <!-- Default video for larger screens (or if small isn't supported) -->
    <source src="/videos/example-large.mp4" type="video/mp4">

    <p>Fallback content if no format supported.</p>
  </video>

✅ Observation:

  1. Examine the two <source> elements. Note the media="(max-width: 600px)" attribute on the first one.
  2. (Conceptual):
    • If the browser viewport is 600px wide or less, the browser will match the first <source>'s media query and attempt to load /videos/example-small.mp4.
    • If the viewport is wider than 600px, the first media query fails, so the browser skips it and moves to the second <source>, attempting to load /videos/example-large.mp4.
    • This allows tailored video delivery based on screen size. (Visual effect depends on having different video files and resizing the preview window).

✨ Key Takeaway: Use the media attribute on <source> elements with appropriate media queries to provide different video files optimized for different viewport sizes or other device characteristics, enabling responsive video delivery.
