203. CSS Color Values: currentColor Keyword

Files Modified: style.css

currentColor is a dynamic keyword value that resolves to the computed value of the element's own color property.

  • Purpose: Allows you to make other properties (like borders, backgrounds, SVG fills/strokes) automatically match the element's current text color, without having to repeat the color value. If the text color changes (e.g., on hover, or via inheritance), any property set to currentColor will update automatically.
  • Syntax: property: currentColor;

Use Cases:

  • Borders matching text: border: 1px solid currentColor;
  • Backgrounds matching text: (Less common, but possible) background-color: currentColor;
  • SVG Icons: Styling the fill or stroke of SVG icons to match the surrounding text color is a very common use case.
    .icon { 
  fill: currentColor; /* SVG icon color matches text color */
  width: 1em; 
  height: 1em; 
}
button { color: blue; }
button .icon { /* Icon inside blue button will be blue */ }
a { color: green; }
a .icon { /* Icon inside green link will be green */ }
  • Gradients or Shadows: Using the text color as part of a gradient or shadow.

✅ Observe style.css:

  • We've modified the p[title] rule. Instead of border: 1px dotted gray;, it now uses border: 1px dotted currentColor;.
/* Use currentColor for the border */
p[title] {
  border: 1px dotted currentColor; /* Border color now matches text color */
}

✨ Observe the Preview:

  • The first paragraph ("This is a standard paragraph.") still has a dotted border, but the border color should now be yellow, matching the paragraph's text color (which is set to yellow by the .container > p rule).
  • If we were to change the color of .container > p, the border color on the first paragraph would automatically update to match because it uses currentColor.

currentColor promotes consistency and reduces repetition when styles should be tied to the text color.
