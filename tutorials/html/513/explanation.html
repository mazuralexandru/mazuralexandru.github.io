513. Sectioning Content: <article> Element - Fundamentals

The <article> element is used to mark up self-contained compositions.

Definition & Purpose:

  • Represents a complete, self-contained piece of content that could, in principle, be distributed or reused independently (e.g., syndicated via RSS).
  • Answers the question: "Could this content make sense on its own, separate from the rest of the page?"

Examples:

  • A blog post.
  • A news article.
  • A forum post.
  • An individual user comment.
  • An interactive widget or gadget.
  • An individual product listing on a category page.

Core Mechanics:

  • Block-level sectioning content element.
  • Syntax: <article> ... </article>
  • Content: Flow content. Typically contains its own structure, often including:
    • A <header> with a heading (<h1>-<h6>) identifying the article.
    • Paragraphs (<p>), lists (<ul>/<ol>), images (<img>), etc., forming the main body.
    • Potentially a <footer> with metadata like publication date (<time>) or author info (<address>).
  • Nesting: Articles can be nested (e.g., comments within a blog post article).

Accessibility:

  • Maps to the ARIA article role, helping assistive technologies identify distinct, self-contained content units.

Code Change:

We've added an <article> element inside <main> in index.html, representing a blog post. It includes its own nested <header> with an <h2> and a <time> element. We also added basic CSS for the article heading.

<!-- index.html snippet -->
<head>
  ...
  <style>
    ...
    article header h2 { margin: 0 0 0.5em 0; } /* Basic article heading style */
  </style>
</head>
<body>
  ...
  <main>
    <h1>Main Content Title (Blog Page Example)</h1> <!-- Updated main title -->

    <!-- Added article element -->
    <article>
      <header> <!-- Article header -->
        <h2>Blog Post Title One</h2>
        <p>Published on <time datetime="2024-03-15">March 15, 2024</time></p>
      </header>
      <p>This is the first paragraph of the blog post. It stands alone as a self-contained piece of content...</p>
      <p>More content for the first blog post...</p>
    </article>

  </main>
  ...
</body>

✅ Observation:

  1. Locate the <article> element within <main>. Note its internal structure (<header>, <h2>, <time>, <p>).
  2. In the browser preview, the blog post content ("Blog Post Title One", publication date, paragraphs) appears within the main content area. The entire <article> block should have its own dashed border from our temporary styles, visually separating it as a unit within the main content.

✨ Key Takeaway: Use the <article> element to wrap complete, self-contained compositions like blog posts, news items, or forum comments that could potentially be distributed independently. It helps define the document structure for assistive technologies and content syndication.
