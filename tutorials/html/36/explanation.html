36. CSS Selectors: The Universal Selector

The Universal Selector (*) is unique because it matches every single element on the page. While powerful, it should be used cautiously.

Universal Selector (*)

  • Syntax: An asterisk (*).
  • Scope: Selects every single element in the HTML document, including <html>, <head>, <body>, <h1>, <p>, <div>, etc.
  • Specificity: The universal selector itself has zero specificity (0,0,0,0). It won't override rules with any specificity (like type, class, or ID selectors) unless combined with !important.
  • Use Cases:
    • CSS Resets: Often used in reset stylesheets to apply baseline styles globally, like setting box-sizing: border-box; or removing default margins/paddings. Example: * { margin: 0; padding: 0; box-sizing: border-box; } (Note: More targeted resets are often preferred over * for performance).
    • Debugging: Can be useful temporarily during development to visualize element boundaries (like adding a border to everything, as we've done here). Remove it afterward!
    • Targeting all descendants: #some-id * { ... } targets all elements inside #some-id.

Caution: Because it targets everything, applying styles with the universal selector can have unintended consequences and potentially impact performance if used carelessly or with complex properties.
