362. Link Attribute: rel

Files Modified: index.html

The rel attribute specifies the relationship between the current document and the linked document/resource.

  • Purpose: Provides metadata about the link type, primarily used by search engines, browsers, and assistive technologies. It does not typically have a direct visual effect.

  • Common Values:

    • noopener: (Security) When used with target="_blank", it prevents the newly opened page from having access to the originating page's window object via window.opener. This prevents potential phishing attacks ("tabnabbing"). It's highly recommended for all target="_blank" links.
    • noreferrer: (Privacy/Security) Prevents the browser from sending the Referer HTTP header to the destination site when the link is followed. This means the destination site won't know where the user came from. Often used together with noopener.
    • nofollow: Instructs search engines not to follow this link or pass ranking authority (PageRank) to the linked page. Used for untrusted content, paid links, or links you don't want to endorse.
    • external: Indicates that the linked document is not part of the current site.
    • author: Links to the author of the current document or article.
    • license: Links to the licensing information for the document.
    • prev / next: Indicate the previous/next document in a series.
    • stylesheet: (Used on <link> elements, not <a>) Indicates a CSS stylesheet.

  • Multiple Values: You can include multiple relationship values separated by spaces (e.g., rel="noopener noreferrer external").

✨ Observe the Preview: There is no visual change in the preview. The rel attribute provides non-visual metadata. However, adding noopener noreferrer significantly improves the security of opening links in a new tab.
