96. List Items (<li>): Core Mechanics (Syntax & Placement)

Let's add our first list item.

Syntax:

  • The list item uses an opening <li> tag and a closing </li> tag.
  • The content of the list item goes between these tags.
    <li>Content of the list item</li>
  • The closing tag </li> is mandatory in standard HTML5. (Older HTML versions had optional closing tags in some contexts, but this is bad practice and not allowed in XHTML or modern standards).

Placement Rule:

  • As established when discussing <ul>, an <li> element must always be a direct child of a parent list container element (<ul> or <ol>).
  • It cannot exist on its own directly within the <body>, a <p>, a <div>, etc. It must be inside a <ul> or <ol>.

Code Change:

We have added our first <li> element, containing simple text, inside the <ul> in index.html.

<ul>
  <li>First item</li> <!-- Added first li -->
  <!-- More list items will go here -->
</ul>

✅ Observation:

  1. Verify the presence of the <li>First item</li> line inside the <ul> in index.html.
  2. Look at the browser preview. You should now see:
    • The text "First item".
    • A bullet point (or similar marker, depending on your browser's default) appearing to the left of the text.
    • The item is likely indented from the left margin (due to default browser padding/margin on the <ul> or <li>).

✨ Key Takeaway: Create list items using <li>...</li> tags. Each <li> must be placed directly inside a <ul> or <ol> container. Adding an <li> makes content appear in the list, typically preceded by a default marker.
