629. Entities: What are HTML Entities? - Core Definition

Let's start with the basic definition.

Definition:

  • HTML Entities are special sequences of text (codes) used within HTML markup to represent specific characters. These characters might be:
    • Reserved Characters: Characters like <, >, and & that have structural meaning in HTML itself and need to be "escaped" to be displayed literally.
    • Invisible Characters: Characters like the non-breaking space.
    • Other Characters: Symbols or characters not easily typable on a standard keyboard (though direct UTF-8 input is often preferred for these now).

Structure:

HTML entities typically come in two main forms:

  1. Named Character References:

    • Start with an ampersand (&).
    • End with a semicolon (;).
    • Contain a predefined, case-sensitive entity name between the & and ;.
    • Example: &lt; represents the < character.

  2. Numeric Character References:

    • Start with an ampersand and a hash (&#).
    • End with a semicolon (;).
    • Contain a number representing the character's Unicode code point between the &# and ;.
    • Can be Decimal: &#<number>; (e.g., &#60; for <).
    • Or Hexadecimal: &#x<hex-number>; (e.g., &#x3C; for <).

Purpose Recap:

Entities allow us to reliably include characters in our HTML content that might otherwise be misinterpreted by the browser's parser or are difficult to type directly. The browser replaces the entity code with the actual character during rendering.

Analogy: Think of them as "escape codes" or placeholders that tell the browser, "Don't treat this sequence like normal markup; display the specific character it represents instead."

✅ Observation: No code changes yet. Focus on the definition and the two syntax forms (named like &name; and numeric like &#number; or &#xhex;).

✨ Key Takeaway: HTML entities are special codes (&name; or &#number;) used to represent reserved characters or other specific characters within HTML markup.
