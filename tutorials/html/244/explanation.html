244. <code> & <pre>: Default Styling

Files Modified: index.html

Browsers apply specific default styles to these elements:

  • <code>: Renders text in the browser's default monospace font (e.g., Courier, Consolas).
  • <pre>: Renders text in the default monospace font AND preserves whitespace and line breaks (white-space: pre).

✅ Observe index.html:

  • We've added a paragraph containing an inline <code> element.
  • We've added a <pre> element containing a <code> element for a block of code, including escaped HTML.
    <hr>
    <p>Use the <code>document.getElementById()</code> method.</p>

    <pre><code>&lt;!DOCTYPE html&gt;
&lt;html&gt;
  &lt;head&gt;
    &lt;title&gt;Example&lt;/title&gt;
  &lt;/head&gt;
  &lt;body&gt;
    Content here.
  &lt;/body&gt;
&lt;/html&gt;</code></pre>

✨ Observe the Preview:

  • In the first new paragraph, the text document.getElementById() should appear in a monospace font, distinct from the surrounding paragraph text.
  • Below that, the HTML code block should appear:
    • In a monospace font.
    • With all the indentation and line breaks preserved exactly as typed in the source code within the <pre> tag.
    • The < and > characters should be displayed literally, not interpreted as tags, because we used &lt; and &gt; in the source.
