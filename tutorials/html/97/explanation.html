97. List Items (<li>): Adding Multiple Items & Default Rendering

A list typically contains multiple related items. Let's add a couple more <li> elements to our unordered list.

Structure:

Each distinct item in the list gets its own <li>...</li> element, placed sequentially within the parent <ul> container.

Code Change:

We've added two more <li> elements to the <ul> in index.html.

<ul>
  <li>First item</li>
  <li>Second item</li> <!-- Added second li -->
  <li>Another item</li> <!-- Added third li -->
</ul>

✅ Observation:

  1. Examine the index.html file. You now see three <li> elements nested directly within the <ul>.
  2. Look at the browser preview. You should see:
    • Each list item ("First item", "Second item", "Another item") displayed on its own line.
    • Each item preceded by the same default bullet point marker.
    • All items sharing the same level of indentation.
    • This clearly presents the content as a distinct list, separate from the surrounding paragraphs.

✨ Key Takeaway: Add multiple <li> elements inside a <ul> (or <ol>) to represent multiple items in the list. Each <li> will typically render on a new line with the list's default marker style and indentation.
