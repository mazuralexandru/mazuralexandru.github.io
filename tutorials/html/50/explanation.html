50. CSS Units: em vs rem for Font Size

We've briefly mentioned different CSS units. Let's focus on two crucial relative units often used for font sizes and spacing: em and rem. Understanding their difference is key for creating scalable and accessible designs.

Relative Units: em vs rem

  • em Unit: Relative to the font-size of the parent element. If used for font-size itself, it's relative to the inherited font size.
    • Compounding Effect: This is the crucial part. If you set font-size: 1.2em on a parent, and then also set font-size: 1.2em on a child, the child's font size becomes 1.2 * 1.2 = 1.44 times the grandparent's font size. This compounding can make em units tricky to manage for font sizes in deeply nested structures.
    • Use Cases: Often useful for padding, margin, or width when you want the spacing or size to scale proportionally with the element's own text size.
  • rem Unit (Root EM): Relative to the font-size of the root element (the <html> element).
    • No Compounding: Since rem always refers back to the root element's font size (which is typically controlled by the browser's default setting, often 16px, unless you override it), setting font-size: 1.2rem on nested elements will result in all those elements having the same computed font size (1.2 times the root size). This makes rem much more predictable for managing font sizes across a whole application.
    • Use Cases: Generally recommended for font-size. Also excellent for global spacing (margins, paddings) to ensure consistent scaling if the user changes their base font size preference in the browser (improves accessibility).
