14. Adding CSS Comments

Just like in HTML, you can add comments to your CSS code to explain sections, leave notes, or temporarily disable rules.

CSS Comments (/* ... */)

  • Purpose: Same as HTML comments â€“ add explanations, notes for developers, or temporarily disable CSS code.
  • Syntax: CSS comments start with /* and end with */. Everything between these markers is ignored by the browser when parsing the CSS.
  • Placement: Can be placed almost anywhere in a CSS file, including on their own lines or at the end of a line after a declaration (though usually placed on separate lines for clarity).
  • Multi-line: CSS comments can span multiple lines.
  • No Nesting: Similar to HTML, nesting CSS comments (/* Comment /* Nested? */ End */) is generally not supported reliably.

Using Comments Effectively:

  • Explain why a complex style rule exists.
  • Label different sections of your stylesheet (e.g., /* --- Header Styles --- */, /* --- Button Styles --- */).
  • Temporarily disable a property or an entire ruleset during debugging:
    h1 {
  color: tomato;
  /* border-bottom: 2px solid #eee; */ /* Temporarily disabled border */
  padding-bottom: 5px;
}
