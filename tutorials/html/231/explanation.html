231. Presentation Elements (<i>, <b>, <u>)

Files Modified: None

HTML also includes elements that traditionally indicated presentation (italic, bold, underline) but have been given slightly more nuanced semantic meaning in HTML5. However, their primary association remains presentational.

  • <i> (Idiomatic Text):
    • Semantic Meaning (HTML5): Represents text in an "alternate voice or mood", or otherwise offset from the normal prose without conveying extra importance. Examples include technical terms, foreign words, thoughts, or ship names.
    • Default Styling: Typically rendered in italics.
    • When to Use: Use when there's a semantic reason for offsetting text (like a foreign phrase or technical term) that conventionally uses italics, but where <em> (stress emphasis) isn't appropriate.
  • <b> (Bring Attention To):
    • Semantic Meaning (HTML5): Represents text to which attention should be drawn for utilitarian purposes without conveying extra importance or alternate voice. Examples include keywords in a summary, product names in a review, or actionable words in software.
    • Default Styling: Typically rendered in bold.
    • When to Use: Use when you want to draw attention stylistically (bold) without implying the strong importance of <strong>.
  • <u> (Unarticulated Annotation):
    • Semantic Meaning (HTML5): Represents text with an unarticulated, non-textual annotation, such as labeling a proper name in Chinese text or indicating a misspelling. Its use is highly discouraged in most contexts.
    • Default Styling: Typically rendered with an underline.
    • Why Avoid: Underlines are strongly associated with hyperlinks on the web. Using <u> for other purposes can cause significant user confusion. Use CSS text-decoration: underline; (perhaps via a class) if you need an underline for purely stylistic reasons, but even then, consider alternatives to avoid confusion with links.

Key Takeaway: While <i> and <b> have specific HTML5 semantics, they are often used (sometimes incorrectly) simply for their default italic/bold appearance. If you only need the visual style without any specific semantic meaning, using a <span> with a CSS class (.italic, .bold) is often a clearer and more maintainable approach. Avoid <u>.
