164. Attribute Selectors: Use Cases

Files Modified: None

Let's summarize the common use cases for attribute selectors:

  1. Styling Form Elements:
    • Targeting inputs by type: input[type="text"], input[type="checkbox"], input[type="email"]
    • Styling required fields: [required]
    • Styling disabled elements: [disabled], button[disabled]
    • Styling read-only fields: [readonly]
  2. Styling Links:
    • External links: a[target="_blank"]
    • Links to specific file types: a[href$=".pdf"], a[href$=".zip"]
    • Secure/insecure links: a[href^="https://"], a[href^="http://"]
    • Links to specific domains: a[href*="example.com"]
    • Telephone links: a[href^="tel:"]
  3. Accessibility Styling:
    • Styling elements based on ARIA roles: [role="alert"]
    • Styling elements based on ARIA states: [aria-expanded="true"]
  4. Custom Data Attributes:
    • Adding tooltips or styling based on data-*: [data-tooltip], [data-item-id="123"]
  5. Presence Styling:
    • Applying styles only if an attribute exists: img[alt] (style images that have alt text), abbr[title] (style abbreviations that have an expansion).

Attribute selectors provide powerful hooks into the semantic information already present in your HTML attributes.
