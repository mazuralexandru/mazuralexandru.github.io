556. Reset: Preparing for <source> Element

We've reviewed the overall best practices for using the <video> element. A key part of that is providing multiple video formats for broader browser compatibility using the <source> element.

To focus specifically on the <source> element and its attributes, we will reset our <video> tag again, removing the src attribute (as it shouldn't be used when <source> is present) and any specific playback attributes.

Why Reset?

  • Focus: Isolate the <source> element and its role within <video> (and <audio>).
  • Clarity: Demonstrate how to add multiple <source> tags correctly.
  • Remove src: Explicitly show that the <video> element's src attribute is omitted when using child <source> elements.

Code Change:

  1. Removed the src attribute from the <video> tag in index.html.
  2. Ensured only controls, width, height, and fallback content remain for now.
  3. Updated the <h1> and paragraph text.
<!-- index.html snippet -->
<body>
  <h1>Multiple Video Sources (&lt;source&gt;)</h1> <!-- Updated H1 -->
  <p>Resetting video to demonstrate the 'source' element.</p> <!-- Adjusted P -->

  <video controls width="480" height="270"> <!-- Removed src -->
    <!-- Source elements go here -->
    <p>Fallback content</p>
  </video>

  <p>Use 'source' when providing multiple video formats.</p>

  <!-- <p>This paragraph is temporarily commented out.</p> -->
</body>

✅ Observation:

  1. Verify that the src attribute is gone from the <video> tag in index.html. The <video> tag itself is now just a container with controls and dimensions, waiting for sources.
  2. The browser preview shows the empty video player area with controls.

✨ Key Takeaway: Removing the src attribute from <video> prepares the element to receive its source information from child <source> elements, which is the standard way to provide multiple format options.
