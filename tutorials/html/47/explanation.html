47. Inline Semantics: Stylistic Offset (<b>)

Similar to <i>, the <b> tag also had its meaning redefined in HTML5 away from purely presentation.

The <b> Element (HTML5 Semantics):

  • Purpose: Represents text that should be stylistically offset from the normal prose, without conveying any extra importance or emphasis. It draws attention to the text for utilitarian purposes where standard conventions use bold text.
  • Specific Use Cases (Examples):
    • Keywords in an abstract or summary.
    • Product names in a review.
    • Lead sentences or opening phrases in an article.
    • Action verbs in a software manual.
  • Type: Inline element.
  • Syntax: <b>...</b>

Default Rendering:

Browsers render <b> text in a bold font weight by default, reflecting its historical use.

<b> vs. <strong>:

  • Use <strong> when you want to mark text as having strong importance, seriousness, or urgency.
  • Use <b> when you need to draw attention stylistically (like highlighting a keyword or product name) without implying that the text is more important than the surrounding content. If none of the specific <b> use cases fit, and you just want bold text for presentation, use CSS (font-weight: bold;) instead.
