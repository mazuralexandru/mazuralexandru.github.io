481. Forms: Validation & Assistance - The placeholder Attribute

The placeholder attribute provides a way to display a short hint or example text inside an input field before the user enters a value.

The placeholder Attribute:

  • Purpose: To provide a brief hint suggesting the expected data format or an example value for an input field.
  • Applies To: <input> (text, search, url, tel, email, password, number types) and <textarea>.
  • Syntax: placeholder="Short hint or example"
  • Behavior:
    • The placeholder text is displayed inside the input field, often in a lighter color, only when the field's value is empty.
    • It disappears as soon as the user starts typing in the field or when the field receives focus (behavior varies slightly across browsers).
    • The placeholder text itself is never submitted as the field's value.

Accessibility & Usability Warning:

  • NOT A LABEL REPLACEMENT: Placeholders are NOT a substitute for a proper <label> element.
    • Disappears: The hint vanishes once the user starts typing, removing context. Users might forget the required format.
    • Memory Load: Forces users to remember the hint if they need to refer back to it.
    • Contrast Issues: Default placeholder styling often has poor color contrast, making it hard to read for users with visual impairments.
    • Screen Reader Support: Not consistently announced as a label by all screen readers.
  • Best Use: Use placeholder only for supplementary hints or examples that are not essential for understanding the field's purpose (which the <label> should provide). Examples: "e.g., johndoe@example.com", "YYYY-MM-DD", "Search products...".

Code Change:

We've removed the previous readonly/disabled examples and added placeholder attributes to a search input and a textarea.

<!-- index.html snippet -->
<form action="/submit-data" method="POST">
  <!-- Removed readonly/disabled examples -->
  <p>
    <label for="search_term">Search:</label>
    <input type="search" id="search_term" name="q" placeholder="e.g., HTML forms"> <!-- Added placeholder -->
  </p>
  <p>
     <label for="message">Message:</label><br>
     <textarea id="message" name="message" rows="4" cols="40" placeholder="Enter your message here..."></textarea> <!-- Added placeholder -->
  </p>
  <p><button type="submit">Submit</button></p>
</form>

✅ Observation:

  1. Examine the placeholder attributes on the <input> and <textarea>.
  2. In the browser preview:
    • The search input should display the greyed-out text "e.g., HTML forms" inside it.
    • The textarea should display "Enter your message here..." inside it.
    • Click into either field and start typing – the placeholder text immediately disappears. Delete your text – the placeholder reappears.

✨ Key Takeaway: Use the placeholder attribute for non-essential hints or examples within text inputs and textareas. It disappears on focus/input and is never a replacement for a proper <label> element due to significant accessibility and usability limitations.
