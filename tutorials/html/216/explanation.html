216. line-height

Files Modified: None

The line-height property controls the amount of space between lines of text within an element. It essentially sets the height of a line box.

  • Purpose: To adjust the vertical spacing between lines for better readability. Sometimes called "leading" in typography.
  • Applies To: Elements containing text (inline and block).
  • Inherited: Yes.
  • Common Values:
    • normal: (Default) Lets the browser determine a reasonable line height based on the font (usually around 1.2).
    • Unitless Number: (Recommended) A number without a unit (e.g., 1.5). This number is multiplied by the element's own font-size to calculate the line height. This method is preferred because it scales proportionally if the font size changes or is inherited differently by child elements.
    • Length: A fixed length like 20px or 1.5em. Using em or % can lead to unexpected results on child elements with different font sizes, as the calculated pixel value is inherited, not the relative multiplier.
    • Percentage: 150% is calculated based on the element's own font size, similar to the unitless number but potentially less flexible with inheritance nuances.

Recommendation: Use unitless numbers (e.g., line-height: 1.6;) for line-height in most cases. This provides the best scalability and avoids inheritance issues. A value between 1.4 and 1.8 is often considered good for readability for body text.

Example:

body {
  line-height: 1.6; /* Set base line height for the page */
}
h1 {
  line-height: 1.2; /* Tighter line height for headings */
}
