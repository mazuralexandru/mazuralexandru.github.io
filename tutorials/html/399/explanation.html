399. Responsive Images Technique 1: max-width: 100%

Files Modified: style.css

A fundamental technique for making images responsive (adapt to different screen sizes) is using max-width: 100%;.

  • max-width: 100%;: Allows the image to shrink proportionally to fit its container if the container becomes narrower than the image's intrinsic width. However, it prevents the image from growing larger than its original (intrinsic) size if the container is wider.
  • height: auto;: Must be used in conjunction with max-width to ensure the height scales proportionally as the width changes.
  • display: block;: Often added to images when using this technique. By default, <img> is an inline-level element, which can sometimes have extra space below it due to line-height/baseline alignment. Setting display: block; removes this potential extra space and makes the image behave more predictably in block layout contexts.

✅ Observe style.css:

  • We've modified the .sample-image rule.
  • We removed the fixed width.
  • We added max-width: 100%; and display: block;.
.sample-image {
  /* width: 250px; <- Removed fixed width */
  max-width: 100%; /* Scale down to fit container, but don't exceed original size */
  height: auto; /* Maintain aspect ratio */
  display: block; /* Remove potential bottom space, treat as block */
  border: 2px solid yellow; 
  margin: 1em auto; /* Center the block image */
}
  • We also added margin: 1em auto; to center the block-level image horizontally.

✅ Switch to index.html to see the HTML structure.

✨ Observe the Preview:

  • The image area should now be centered.
  • Its width will be its intrinsic width (300px, as set in the HTML attributes) because the container (body) is likely wider than 300px.
  • If you could make the preview area narrower than 300px, the image would shrink proportionally to fit, never overflowing its container horizontally. This is the essence of this responsive technique.
