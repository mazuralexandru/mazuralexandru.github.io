404. Making Images Block-Level

Files Modified: index.html, style.css

Often, you want an image to behave like a distinct block, taking up its own line and allowing for easy horizontal centering using margin: auto;.

  • Technique: Set display: block; on the <img> element.
  • Effect:
    • The image now forces line breaks before and after itself, stacking vertically like a paragraph.
    • It respects width and height.
    • Crucially, if it has an explicit width less than its container, margin-left: auto; and margin-right: auto; will now work to center it horizontally.

✅ Observe style.css:

  • We've removed the previous inline styles.
  • We've added a rule for .block-image.
  • It sets display: block;, max-width: 100%; (for responsiveness), an explicit width, and margin: 1em auto; to center it and add vertical space.
/* Block Image Alignment */
.block-image {
    display: block; /* Treat image as a block */
    max-width: 100%; /* Responsive */
    height: auto; /* Maintain aspect ratio */
    width: 300px; /* Explicit width needed for auto margin centering */
    margin: 1em auto; /* Center horizontally, add vertical margin */
    border: 2px solid lightgreen; /* Visualize */
}

✅ Switch to index.html:

  • We've removed the previous inline image examples.
  • We've added a standalone <img> with the block-image class.
    <h1>Image Alignment</h1>

    <img 
        src="images/placeholder-landscape.jpg" 
        alt="Placeholder landscape image (300x200)" 
        width="300" 
        height="200"
        class="block-image">

    <p>Some text following the block image.</p>

✨ Observe the Preview:

  • The image (area) should now appear on its own line, centered horizontally within the page body.
  • The paragraph below it appears on a new line, respecting the vertical margin set on the image.
  • This is a standard way to handle standalone images that need centering or specific block-level layout treatment.
