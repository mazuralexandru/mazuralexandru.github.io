696. Accessibility (A11y): Table Accessibility Review

Data tables (<table>) present unique accessibility challenges if not marked up correctly. Let's review the essential HTML features for accessible tables.

1. Use Tables for Data Only:

  • Rule: Never use <table> for page layout. Use CSS. Layout tables destroy semantic structure and reading order for screen readers.

2. Provide a Caption (<caption>):

  • Requirement: Every data table must have a <caption> element as its first child.
  • Benefit: Provides the accessible name (title) for the table, announced by screen readers, allowing users to understand the table's purpose immediately.

3. Identify Headers (<th>):

  • Requirement: Use <th> (table header) cells for all cells that act as headers for columns or rows. Do not use <td> styled to look like headers.
  • Benefit: Semantically distinguishes header information from data.

4. Define Header Scope (scope):

  • Requirement: Add the scope attribute to every <th> element.
  • Values: Use scope="col" for column headers and scope="row" for row headers.
  • Benefit: Explicitly tells assistive technologies which data cells (<td>) are associated with which header (<th>). This allows screen readers to announce the correct row and column headers as a user navigates through data cells, providing crucial context. This is the most common and recommended method for associating headers in simple-to-moderate tables.

5. Use Structural Groups (<thead>, <tbody>, <tfoot>):

  • Purpose: Group header, body, and footer rows semantically.
  • Benefit: Provides additional structural context for AT, enables potential browser features (scrolling body, repeating headers/footers on print), and offers useful CSS hooks. Remember <tbody> is mandatory if <thead> or <tfoot> is used.

6. Handle Complex Tables (headers/id - Advanced):

  • For tables with irregular structures or multiple levels of headers where scope alone is insufficient, use the headers attribute on <td> elements.
  • Assign unique ids to all relevant <th> elements.
  • List the ids of all applicable headers (space-separated) in the headers attribute of each data cell.
  • This creates the most explicit associations but is more verbose and only needed for complex cases.

✅ Observation: No code changes. Recall the table examples from Module 8. Visualize a screen reader reading a cell like "15%" – without proper <th> and scope (or headers), the user wouldn't know this percentage relates to "Q4 Sales" (column header) for "Product B" (row header).

✨ Key Takeaway: Accessible data tables require a <caption>, proper use of <th> cells with explicitly defined scope attributes (col or row), and logical structure using <thead>/<tbody>/<tfoot>. Avoid layout tables entirely. Use headers/id for complex table structures if scope is insufficient.
