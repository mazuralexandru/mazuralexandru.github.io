80. Purpose of CSS

Files Modified: None

Why do we use CSS? What problems does it solve?

  1. Separation of Concerns: As mentioned, it cleanly separates document structure (HTML) from visual presentation. Imagine trying to set the color and font for every single paragraph using only HTML attributes â€“ it would be incredibly repetitive and hard to update.
  2. Consistency: Define a style once in a CSS file (like our base_styles.css) and link that file to multiple HTML pages. All pages will share the same base look. Changing the style in the CSS file updates it everywhere instantly.
  3. Maintainability: Need to change the brand color across your entire website? Modify one line in your CSS file instead of editing hundreds of HTML files.
  4. Sophisticated Styling: CSS offers a vast range of properties to control layout (positioning, flexbox, grid), typography, colors, backgrounds, borders, animations, responsiveness (adapting to different screen sizes), and much more â€“ far beyond what HTML attributes alone could ever achieve.
  5. Accessibility: Users can apply their own custom stylesheets to override website styles, which can be crucial for readability (e.g., high contrast modes, larger fonts). Keeping styles separate facilitates this.
  6. Bandwidth: Browsers can cache external CSS files. After the first download, the styles load instantly for subsequent pages, saving bandwidth and speeding up page loads compared to repeating styles directly in HTML.

CSS is fundamental to creating visually appealing, maintainable, and modern websites.
