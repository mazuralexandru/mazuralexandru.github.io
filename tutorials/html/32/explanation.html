32. Using Attribute Selectors: Exact Value

Often, you want to select elements where an attribute has a specific value. The exact value attribute selector allows this. Let's use it to style only the input field whose type attribute is exactly text.

What's New?

  • We added a new CSS rule using the exact value attribute selector [type="text"] to style.css.
/* style.css snippet */
[required] {
  border: 2px solid orange;
  background-color: #444; /* Darker background for input */
}

/* Added Exact Value Attribute Selector rule */
[type="text"] {
  border-color: lightgreen; /* Override border color only for type=text */
  font-family: monospace;
}

/* CSS files can also have comments! */

Exact Value Attribute Selector ([attribute="value"])

  • Syntax: The attribute name, an equals sign (=), and the desired value enclosed in quotes "" (single or double), all within square brackets [] (e.g., [type="text"], [href="https://example.com"], [lang="en-US"]).
  • Scope: Selects all elements where the specified attribute exists and its value exactly matches the value given in the quotes (case-sensitive by default, unless the attribute/document implies otherwise, or the i modifier is used - see later).
  • Use Cases: Styling form inputs based on their specific type (text, password, submit), styling links pointing to specific URLs, targeting elements with specific data attributes (data-*).

In our example, [type="text"] targets only elements with type="text". We're changing the border-color to lightgreen and setting the font-family to monospace.
