392. width & height Attributes: CLS & Aspect Ratio

Files Modified: None

While CSS is the primary way to control image dimensions for layout, the width and height attributes on the <img> tag itself play a crucial role, especially for performance and layout stability.

  • Purpose: These attributes provide hints to the browser about the image's intrinsic aspect ratio (the ratio of its width to its height) before the image file is fully downloaded.
  • Preventing CLS (Cumulative Layout Shift):
    • If you don't provide width and height, the browser initially doesn't know how much space to reserve for the image.
    • When the image finally loads, it suddenly pops into place, potentially pushing other content down the page. This jarring shift in layout is called CLS and is bad for user experience (and search engine rankings).
    • By providing the width and height attributes (matching the image's actual dimensions or at least its aspect ratio), the browser can calculate the aspect ratio early and reserve the correct amount of vertical space in the layout before the image downloads. This prevents the content jump when the image appears.
  • Syntax: <img src="..." alt="..." width="600" height="400">
    • The values should be unitless integers representing pixels.
    • Use the image's intrinsic (natural) dimensions or values that maintain the correct aspect ratio.
  • Interaction with CSS:
    • CSS width, height, and max-width properties will typically override the HTML width and height attributes for the final rendered size.
    • However, the browser still uses the HTML attributes initially to calculate the aspect ratio and reserve space, preventing CLS.
  • Best Practice: Always include width and height attributes on your <img> tags, reflecting the image's intrinsic dimensions or aspect ratio, even if you also control the size with CSS.
