171. target Attribute: _self (Default Behavior)

Let's look at the default behavior when the target attribute is not specified, which is equivalent to using target="_self".

target="_self" (or Omitting target)

  • Value: _self
  • Behavior: Opens the linked document in the same browsing context (the same tab, window, or frame) where the link was clicked. The current page is replaced by the linked page.
  • Default: This is the default behavior for all hyperlinks if the target attribute is completely omitted. You rarely need to explicitly write target="_self".

Use Case:

This is the standard behavior for most internal navigation links within a website, keeping the user within the same browser tab or window as they move from page to page.

Code Change:

We've added a simple link to an external site (developer.mozilla.org) in index.html, but we have not added a target attribute, so it will use the default _self behavior.

<p>
  Example Link (Default):
  <a href="https://developer.mozilla.org/">MDN Home</a> <!-- No target attribute -->
</p>

✅ Observation:

  1. Locate the <a> tag in index.html. Note the absence of the target attribute.
  2. In the preview, click the "MDN Home" link.
  3. Observe that the MDN homepage replaces the current content within the preview pane (or your current browser tab if you opened the file directly). It did not open in a new tab or window. This demonstrates the default (_self) behavior. (You'll need to navigate back in your browser/previewer to return to the lesson page).

✨ Key Takeaway: If the target attribute is omitted, or explicitly set to target="_self", the linked document opens in the same tab/window/frame, replacing the current page. This is the default behavior.
