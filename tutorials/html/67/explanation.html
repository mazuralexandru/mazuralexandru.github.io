67. Specialized Text: Code Blocks (<pre> + <code>)

We've seen <pre> for preserving whitespace in blocks and <code> for marking inline code fragments. How do we represent multi-line blocks of code correctly and semantically?

The Standard Pattern: <pre> + <code>

The standard and recommended best practice is to combine both elements: nest a <code> element directly inside a <pre> element.

  • Syntax:
    <pre><code>
... Your multi-line code ...
... with escaped HTML characters ...
</code></pre>
  • Structure:
    • The outer <pre> element handles the whitespace preservation and typically provides the block-level formatting and default monospace font.
    • The inner <code> element provides the semantic meaning, clearly indicating that the content is computer code.

Benefits of this Pattern:

  • Semantic Accuracy: Correctly identifies the block as both preformatted and code.
  • Styling Hooks: Provides clear selectors for CSS styling (e.g., pre, code, pre code).
  • Syntax Highlighting: This <pre><code>...</code></pre> structure is the standard pattern recognized by almost all JavaScript syntax highlighting libraries (like Prism.js, highlight.js). These libraries often look for <code> tags within <pre> tags to apply coloring.

HTML Escaping Still Required:

Remember that even when using this pattern, any special HTML characters (<, >, &) within your code block must still be escaped (&lt;, &gt;, &amp;) to be displayed literally. Notice the &amp;amp; used to display a literal ampersand in the example!
