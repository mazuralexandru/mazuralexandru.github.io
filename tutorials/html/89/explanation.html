89. Module 2 Recap: Structuring Text Content

This module focused on essential HTML elements used to structure and add semantic meaning to text content within the <body>. Let's recap the key elements and concepts:

Block-Level Text Structuring:

  • Paragraphs (<p>): Define distinct blocks of text, grouping related sentences. Crucial for readability and accessibility. Don't use for spacing or misuse <br> instead.
  • Headings (<h1> - <h6>): Establish the document's hierarchical outline. Use in logical order without skipping levels. <h1> is typically the main page title (one per page). Vital for navigation (esp. screen readers) and SEO. Use for structure, not just style.

Inline Text Semantics (Emphasis & Importance):

  • Emphasis (<em>): Apply stress emphasis, changing sentence meaning (like vocal stress). Defaults to italic.
  • Strong Importance (<strong>): Indicate strong importance, seriousness, or urgency. Defaults to bold.
  • Key Principle: Choose <em>/<strong> based on meaning, not just desired style. Use CSS for purely visual italics/bolding.

Inline Presentation vs. Semantics:

  • Italic (<i>): Alternate voice/mood (technical terms, thoughts, foreign words). Defaults to italic.
  • Bold (<b>): Stylistic offset without added importance (keywords, product names). Defaults to bold.
  • Underline (<u>): Non-textual annotation (e.g., misspelling). Discouraged due to confusion with links. Defaults to underline.
  • Preference: Prefer <em>/<strong> for semantic emphasis/importance. Use CSS for purely visual styling. Use <i>/<b>/<u> only for their specific, narrow semantic meanings if appropriate.

Structural Formatting Elements:

  • Line Break (<br>): Force a line break within a block (e.g., addresses, poetry). Void element. Do not use multiple <br>s for spacing.
  • Horizontal Rule (<hr>): Represents a thematic break between paragraph-level content (topic shift, scene change). Void, block-level element. Defaults to a horizontal line. Use for semantic breaks, not just visual lines (use CSS borders for that).

Specialized Text Containers:

  • Preformatted Text (<pre>): Preserves whitespace (spaces, line breaks) exactly as in the source. Block-level, defaults to monospace font. Requires HTML escaping (&lt;, &gt;, &amp;) for literal < > &. Use CSS for overflow control.
  • Inline Code (<code>): Semantically marks inline fragments of code (variable/function names, etc.). Defaults to monospace font.
  • Code Blocks (<pre><code>...</code></pre>): The standard pattern for multi-line code blocks, combining <pre>'s formatting with <code>'s semantics. Essential for syntax highlighting compatibility. Remember to escape HTML inside.

Quotation Elements:

  • Block Quote (<blockquote>): For long, block-level quotations from another source. Often indented by default. Use cite attribute for source URL (not displayed). Attribution goes outside the tag.
  • Inline Quote (<q>): For short, inline quotations. Browsers add language-appropriate quotes automatically (don't type them in). Can also use cite attribute.
  • Citation (<cite>): Marks up the title of a creative work (book, film, article, etc.). Inline, defaults to italic. Use only for titles.

Other Inline Semantics:

  • Small Print (<small>): For side comments, legal text, copyright. Defaults to smaller font. Use semantically, not just for size.
  • Subscript (<sub>): For chemical formulas, math indices. Lowers baseline. Crucial for accessibility.
  • Superscript (<sup>): For exponents, ordinals, footnote markers. Raises baseline. Crucial for accessibility.
  • Strikethrough (<s>): For text no longer accurate/relevant (e.g., old price). Defaults to strikethrough. Use semantically, not just for style.

Core Principle Revisited: Choose HTML elements based on their semantic meaning first and foremost. Use CSS to control visual presentation.
