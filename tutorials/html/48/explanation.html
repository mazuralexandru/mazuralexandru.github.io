48. Inheritance: Non-Inherited Properties

Let's explicitly see how non-inherited properties behave. We already saw that border and padding applied to the parent <div> were not inherited by the child <p>. Let's add a background-color to the parent and observe.

What's New?

  • HTML Modification: Added background-color: #444; to the inline style attribute of the parent <div>.
<!-- index.html snippet -->
<body>
  <h1>CSS Inheritance Demo</h1>

  <!-- Pruned previous cascade examples -->

  <div style="color: mediumpurple; border: 1px solid grey; padding: 10px; background-color: #444;"> <!-- Added background-color -->
    This text is directly inside the div (inherits color).
    <p>This paragraph is inside the div.</p> <!-- Child P -->
  </div>

  <!-- Commented out paragraph remains -->
  <!-- <p>This paragraph is commented out and won't appear.</p> -->

  <script src="script.js"></script>
</body>

Non-Inherited Properties

  • As mentioned, most properties related to an element's box model (size, spacing, borders, background) are not inherited by default.
  • If you want a child element to have the same background or border as its parent, you generally need to apply those styles directly to the child element as well (or use special CSS values like transparent or currentColor in specific scenarios).
