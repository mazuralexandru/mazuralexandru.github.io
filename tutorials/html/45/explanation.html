45. Inline Semantics: <em> & <strong> Accessibility

Let's revisit the accessibility implications of using <em> and <strong>.

How Screen Readers May Handle Them:

As mentioned before, screen readers and other assistive technologies can potentially use the semantic information provided by <em> and <strong> to enhance the user experience. This might involve:

  • Voice Changes: Adjusting the pitch, speed, volume, or synthesized voice characteristics to mimic vocal stress (<em>) or convey greater weight (<strong>).
  • User Settings: Some screen readers might allow users to customize how emphasis and importance are announced or indicated.

Important Caveats:

  • Variability: Support and specific behavior vary significantly between different screen readers (e.g., JAWS, NVDA, VoiceOver) and even between different versions of the same software.
  • User Configuration: The actual experience depends heavily on the user's settings and preferences.
  • No Guarantee: You cannot guarantee that a specific audible change will occur just because you used these tags.

Why Still Use Them? The Semantic Value:

Even if audible changes aren't consistently applied, using <em> and <strong> correctly remains crucial for accessibility because:

  1. Provides Intent: It clearly communicates the author's intent to emphasize or stress importance, even if the software doesn't act on it overtly. This structural information is valuable context.
  2. Potential Future Support: As assistive technologies evolve, semantic markup becomes increasingly important for enabling richer interactions and interpretations. Using semantic tags correctly future-proofs your content.
  3. Consistency: It establishes a consistent way to mark up these nuances, which is better than relying on ad-hoc methods or purely visual styling (which provides no semantic information to screen readers).

Contrast with <i>/<b>:

We'll discuss <i> (italic) and <b> (bold) next. Generally, screen readers do not announce these tags differently from regular text, as their semantic meaning (in HTML5) is weaker and more nuanced than <em>/<strong>. This further highlights the importance of choosing <em>/<strong> when true emphasis or importance is intended.
