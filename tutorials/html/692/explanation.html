692. Accessibility (A11y): Semantic HTML Review (<h1>-<h6>, Landmarks)

Let's specifically review how the semantic elements related to document structure contribute to accessibility.

1. Heading Structure (<h1>-<h6>) Recap:

  • Purpose: Define the hierarchical outline of the page content.
  • Accessibility Impact:
    • Screen readers use the heading structure (H key, 1-6 keys) as a primary navigation method. Users can quickly jump between sections.
    • Provides a mental map of the page organization.
    • Rule: Must be hierarchical (no skipped levels). Only one <h1> per page recommended. Use for structure, not styling.
  • Failure Example: Using styled <p> tags instead of headings, or skipping levels (H1 -> H3), breaks this navigation and understanding.

2. Landmark Elements Recap:

  • Elements: <header>, <nav>, <main>, <footer>, <aside>, <section> (if named), <form> (if named).
  • Purpose: Define the major functional regions of a page.
  • Accessibility Impact:
    • Mapped to ARIA landmark roles (banner, navigation, main, contentinfo, complementary, region, search, form).
    • Screen readers announce these landmarks and allow users to list them and jump directly to a specific region (e.g., "Jump to main content"). This bypasses repetitive header/navigation blocks.
  • Best Practices:
    • Use landmark elements appropriately for their intended purpose.
    • Ensure <main> wraps the primary content.
    • Provide accessible names for multiple instances of the same landmark (e.g., multiple <nav>s using aria-label).
    • Ensure <section> has a heading (or aria-label/labelledby) to potentially become a region landmark.
  • Failure Example: Using only <div> elements for page structure provides no landmark information, forcing users to navigate linearly.
