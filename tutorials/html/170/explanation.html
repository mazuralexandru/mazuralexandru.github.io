170. Grouping Selectors: Syntax

Files Modified: style.css

The syntax for grouping selectors involves listing each selector you want to target, separated by a comma. The declaration block that follows applies to all selectors in the group.

selector1,
selector2,
.some-class,
#some-id,
[data-attribute="value"] {
  /* Declarations apply to elements matching ANY of the selectors above */
  property: value;
}
  • Whitespace: Whitespace (including line breaks) around the commas is optional but commonly used for readability, especially with longer lists of selectors.
  • Validity: If any selector in a grouped list is invalid, the browser is supposed to ignore the entire ruleset. This means a typo in one selector can prevent the styles from applying to any of the other (valid) selectors in the group.

✅ Observe style.css:

  • We've added a new ruleset that groups the #main-heading ID selector and the .highlight-text class selector.
  • This grouped rule applies a font-style: italic; declaration to both elements.
/* Grouping Example */
#main-heading,
.highlight-text {
  font-style: italic; /* Apply italic to both */
}

✨ Observe the Preview: Both the main heading ("Main Heading") and the highlighted paragraph ("This is a standard paragraph.") should now appear in italics, in addition to their previous styles. This is because the grouped selector applies the font-style: italic; rule to both the element with id="main-heading" and the element with class="highlight-text".
