184. Linking to Non-HTML Resources: Best Practice (Inform User)

Since the browser's behavior when linking to non-HTML files can vary (inline display vs. download prompt), it's crucial to provide users with clear information within the link itself so they know what to expect before they click.

Best Practice: Indicate File Type and Size

  • Why:
    • Manages Expectations: Users know if they are about to view a document directly or initiate a potentially large download.
    • Informed Decisions: Users on limited bandwidth or mobile data can decide if they want to download a large file.
    • Usability: Prevents surprise downloads or unexpected content types opening.
  • How: Include the file type (e.g., PDF, ZIP, DOCX) and, if practical, the file size within or immediately adjacent to the link text. Parentheses are commonly used.

✅ Observation:

  1. Examine the updated link text in index.html. Notice the addition of "(PDF, 3.1MB)", "(PNG, 150KB)", and "(ZIP, 15MB)".
  2. Look at the browser preview. The links now clearly communicate the type and size of the file they point to. This is much more user-friendly than just having generic link text like "Manual" or "Logo".

✨ Key Takeaway: When linking to non-HTML resources, always include clear indicators of the file type (PDF, DOCX, ZIP, etc.) and ideally the file size within or near the link text to help users make informed decisions.
