333. Reset: Preparing for <link> Element Introduction

We have now covered the essential <meta> tags: charset, description, viewport, and discussed the historical context of keywords and the limited use of author.

Next, we will shift focus within the <head> section to the <link> element. While we've consistently used it to link our stylesheet (<link rel="stylesheet" href="style.css">), this element has many other important uses for defining relationships with external resources.

To prepare for a focused exploration of the <link> element and its various rel attribute values, we will simplify the <head> slightly by removing the <meta name="description"> and <meta name="viewport"> tags we just added. We will keep <meta charset="UTF-8"> and <title> as the minimal required/essential metadata alongside the CSS link for now.

Why Remove Code?

  • Focus: Isolate the <link> element and its attributes (rel, href, type, media, etc.).
  • Simplify <head>: Reduce the number of elements in the <head> temporarily to make the <link> examples clearer. We can re-add essential meta tags like viewport later.

Code Change:

  1. Removed the <meta name="description" ...> tag.
  2. Removed the <meta name="viewport" ...> tag.
  3. Updated the <h1> and paragraph text in the <body>.
