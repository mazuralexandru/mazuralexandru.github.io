511. Landmarks: Review (<header>, <footer>, <nav>, <main>)

We've now introduced the four primary HTML elements used to define the major landmark regions of a typical web page layout. Let's quickly review their specific roles:

  1. <header>:

    • Role: Introduces the page or section.
    • Content: Site title/logo, main navigation (<nav>), search form, taglines.
    • ARIA Landmark (Top-Level): banner.

  2. <nav>:

    • Role: Contains major navigation blocks.
    • Content: Lists (<ul>) of navigation links (<a>).
    • ARIA Landmark: navigation. (Use aria-label if multiple <nav>s).

  3. <main>:

    • Role: Contains the primary, unique content of the page.
    • Content: The core <article>s, <section>s, text, images, forms related to the page's main purpose.
    • ARIA Landmark: main. (Should be unique per page).
    • Placement: Not inside <header>, <footer>, <nav>, or <aside>.

  4. <footer>:

    • Role: Concludes the page or section.
    • Content: Copyright (<small>), contact info (<address>), related links (privacy policy, terms), site map links.
    • ARIA Landmark (Top-Level): contentinfo.

Why Use Landmarks?

  • Accessibility: Provides essential structure for screen reader users, allowing them to quickly understand the page layout and jump directly to relevant sections (banner, navigation, main, contentinfo, complementary for <aside>, region for named <section>s, form for <form> with label, search for search role elements). This dramatically improves navigation efficiency compared to navigating through non-semantic <div> structures.
  • SEO: Helps search engines understand the different parts of your page (e.g., distinguishing main content from boilerplate header/footer content).
  • Maintainability: Makes the HTML structure clear and self-documenting.

Code Change: No changes. This is a conceptual review of the four landmarks added so far.
