03. Introduction: What is HTML? (Part 3: History & Standards)

Let's briefly touch upon the history of HTML and the importance of standards.

A Brief History:

  • Origins: HTML was created by Sir Tim Berners-Lee in the early 1990s while working at CERN (the European Organization for Nuclear Research). His goal was to create a system for scientists to easily share documents and link between them.
  • Evolution: HTML has gone through several versions:
    • HTML (1), HTML 2.0, HTML 3.2, HTML 4.01
    • XHTML 1.0 / 1.1 (a stricter, XML-based version of HTML)
    • HTML5: The current major version, which introduced many new features for modern web applications (multimedia, graphics, semantic structure, offline capabilities). HTML is now considered a "Living Standard," meaning it evolves continuously rather than through large, discrete version releases.
  • Why the Evolution? To add new capabilities needed for richer web experiences, improve consistency, enhance accessibility, and adapt to new devices.

Standards Bodies:

Who decides what goes into HTML? Two main organizations collaborate on its development:

  • W3C (World Wide Web Consortium): An international community that develops open standards to ensure the long-term growth of the Web. Historically the primary body for HTML specifications.
  • WHATWG (Web Hypertext Application Technology Working Group): A group of browser vendors (Apple, Google, Mozilla, Microsoft) and other stakeholders focused on the practical evolution of HTML and related technologies. WHATWG now maintains the core HTML Living Standard.

Why Standards Matter:

Imagine if every web browser interpreted HTML slightly differently! Websites would look broken or behave unexpectedly depending on the browser used. Standards ensure:

  • Consistency: Web pages look and function similarly across different browsers and devices.
  • Predictability: Developers know what to expect when they write HTML code.
  • Interoperability: Different web technologies (HTML, CSS, JS) work together smoothly.
  • Future-Proofing: Adhering to standards helps ensure websites remain functional as browsers evolve.
