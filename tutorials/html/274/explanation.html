274. Minimum and Maximum Dimensions: Values

Files Modified: None

Let's quickly recap the values usable with min-width, max-width, min-height, and max-height:

  • Length Units: px, em, rem, vw, vh, etc. Define fixed or relative minimum/maximum sizes.
  • Percentages (%): Relative to the corresponding dimension (width for width properties, height for height properties) of the containing block. Remember the caveat about needing an explicit parent height for percentage heights to work reliably.
  • Keywords:
    • none: (Default for max-width/max-height) No limit.
    • auto: (Default for min-width/min-height) No minimum constraint beyond the content's intrinsic size.
    • max-content: Intrinsic preferred width/height. The element tries to be as wide/tall as needed for its content without wrapping if possible.
    • min-content: Intrinsic minimum width/height. The element tries to be as narrow/short as possible, wrapping aggressively.
    • fit-content(length-percentage): A more complex value that tries to fit the content but not exceed the specified argument, while also not being smaller than the intrinsic minimum size.

For most common use cases, you'll use length units or percentages. The content-based keywords are useful in specific layout scenarios, particularly with Grid and Flexbox.
