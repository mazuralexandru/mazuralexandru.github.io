456. Forms: Submit Buttons (type="submit") - <input> vs. <button>

The most crucial button type in a form is the one that triggers the submission process. This functionality is enabled by setting the type attribute to submit. There are two elements that can function as submit buttons:

1. <input type="submit">

  • Syntax: <input type="submit" name="optional_name" value="Button Label">
  • Appearance: Renders as a standard platform button.
  • Label: The text displayed on the button is determined solely by its value attribute. If value is omitted, it defaults to "Submit".
  • Content: Cannot contain other HTML elements (it's a void element).

2. <button type="submit">

  • Syntax: <button type="submit" name="optional_name" value="optional_submitted_value">Button Label Content</button>
  • Appearance: Also renders as a standard platform button.
  • Label: The content displayed on the button is everything placed between the <button> and </button> tags. This content can include text, inline formatting (<em>, <strong>), and even images (<img>).
  • value Attribute: Specifies the value sent to the server if this button is used to submit the form and if it has a name attribute. It does not set the visible label.
  • Flexibility: The ability to include HTML content makes <button> much more flexible for styling and creating visually rich buttons.

Why <button type="submit"> is Preferred:

  • Richer Content: Allows embedding icons, images, or specific text formatting directly within the button label.
  • Clearer Separation: Separates the button's visible label (content) from its submitted value (value attribute), which can be useful in complex forms.
  • Consistency: Matches the syntax of <button type="reset"> and <button type="button"> more closely.

Code Change:

We've added examples of both types of submit buttons to our form in index.html. We also added a simple text input so there's some data to potentially submit.

<!-- index.html snippet -->
<form action="/submit-data" method="POST">
  <p> <!-- Added basic input -->
    <label for="data">Data:</label>
    <input type="text" id="data" name="data_field" value="Test">
  </p>

  <!-- Method 1: Input Submit -->
  <p>
     <input type="submit" value="Submit via Input"> <!-- Input submit -->
  </p>

  <!-- Method 2: Button Submit -->
   <p>
     <button type="submit" name="action" value="save"> <!-- Button submit -->
       Submit via Button <em>(Recommended)</em> <!-- Rich content -->
     </button>
  </p>
</form>

✅ Observation:

  1. Examine both the <input type="submit"> and the <button type="submit"> elements. Note how the label is set differently (value attribute vs. inner content). Note the <em> tag inside the <button>.
  2. Look at the browser preview:
    • Both elements render as clickable buttons.
    • The first button's text is "Submit via Input" (from its value).
    • The second button's text is "Submit via Button (Recommended)", with the "(Recommended)" part likely italicized due to the nested <em>.
    • Clicking either button would attempt to submit the form data (including data_field=Test) via POST to /submit-data. If the second button was clicked, the pair action=save would also be submitted because it has both a name and a value.

✨ Key Takeaway: Both <input type="submit"> and <button type="submit"> trigger form submission. <button type="submit"> is generally preferred because its content model allows for richer labels (text, images, formatting) compared to the plain text value attribute of <input type="submit">.
