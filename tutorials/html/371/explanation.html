371. Link State: :focus

Files Modified: style.css

The :focus pseudo-class is critically important for accessibility. It styles an element when it receives focus, typically via keyboard navigation (using the Tab key) or sometimes by clicking.

  • :focus:
    • Selects: An element (links, buttons, form inputs, etc.) that currently has keyboard focus.
    • Purpose: Provides a clear visual indicator of which element is currently active for keyboard interaction. Without a visible focus style, keyboard users cannot tell where they are on the page.
    • Default Styling: Browsers typically provide a default focus outline (often a blue or dotted ring), but this is frequently overridden (and sometimes completely removed, which is bad practice) by CSS resets or custom styles.
    • Requirement: You must ensure that focusable elements have a clearly visible focus state. If you remove the browser's default outline, you must provide an alternative using properties like outline, border, background-color, etc.

✅ Observe style.css:

  • We've added a rule for a:focus.
  • We're applying a distinct outline style. The outline property draws a line outside the element's border without affecting layout space.
/* Style links when focused */
a:focus { 
  /* Ensure a clear visual indicator for keyboard users */
  outline: 2px dashed orange; 
  outline-offset: 2px; /* Add space between element and outline */
  /* Optionally change other styles too */
  /* background-color: rgba(255, 165, 0, 0.2); */
}

✨ Observe the Preview:

  • Try navigating through the links using the Tab key on your keyboard (you might need to click into the preview area first to give it focus).
  • As each link receives keyboard focus, you should see a dashed orange outline appear around it.
  • Pressing Shift+Tab will navigate backwards.
  • This outline clearly indicates which link is currently selected for keyboard activation (pressing Enter would follow the focused link).
