484. Forms: Validation - minlength & maxlength Attributes

For text-based inputs, you can constrain the allowed length of the user's input using minlength and maxlength.

Attributes:

  1. minlength:

    • Purpose: Specifies the minimum number of characters (in Unicode code points) the user must enter for the input to be considered valid.
    • Applies To: <input> (text, search, url, tel, email, password types) and <textarea>.
    • Syntax: minlength="number"
    • Behavior: This constraint is checked on form submission. If the entered text length is less than the minlength, native browser validation fails.

  2. maxlength:

    • Purpose: Specifies the maximum number of characters the user is allowed to enter into the field.
    • Applies To: Same types as minlength.
    • Syntax: maxlength="number"
    • Behavior: This constraint is typically enforced as the user types. The browser physically prevents the user from entering more characters than the specified maxlength.

Use Cases:

  • minlength: Ensuring comments or descriptions meet a minimum length requirement.
  • maxlength: Limiting usernames, passwords, tweet-like messages, or ensuring data fits database field limits.
  • Setting both minlength and maxlength to the same value enforces an exact length.

Code Change:

We've removed the previous examples and added a <textarea> with minlength="10" and maxlength="100", and a text <input> with both set to 6 (enforcing exact length).

<!-- index.html snippet -->
<form action="/submit-data" method="POST">
  <!-- Removed number/date examples -->
  <p>
     <label for="feedback">Feedback (10-100 chars):</label><br>
     <textarea id="feedback" name="feedback" rows="4" cols="40"
               minlength="10" maxlength="100" required></textarea> <!-- Added min/maxlength -->
  </p>
   <p>
     <label for="code">Code (exactly 6 chars):</label>
     <input type="text" id="code" name="code"
            minlength="6" maxlength="6" required> <!-- Added min/maxlength -->
  </p>
  <p><button type="submit">Submit</button></p>
</form>

✅ Observation:

  1. Examine the minlength and maxlength attributes on both the <textarea> and the <input>.
  2. In the preview:
    • Textarea: Try typing more than 100 characters; the browser should stop you. Now, type fewer than 10 characters (but more than 0) and click Submit; native validation should trigger an error because the minlength is not met.
    • Input: Try typing more than 6 characters; the browser should stop you. Try typing fewer than 6 characters and click Submit; validation should fail due to minlength. Only exactly 6 characters should pass validation.

✨ Key Takeaway: Use minlength and maxlength to constrain the number of characters in textual inputs (<input> types text, email, etc., and <textarea>). maxlength prevents further typing, while minlength is validated on submission.
