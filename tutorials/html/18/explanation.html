18. Applying CSS with Internal Stylesheets (<style>)

We've seen how to apply styles directly to elements using the style attribute (inline styles), but this mixes presentation with structure and isn't easily reusable. A step towards better organization is using an internal stylesheet.

The <style> Element

  • Purpose: Allows you to embed CSS rules directly within an HTML document, typically inside the <head>.
  • Syntax: The CSS rules are placed between the opening <style> and closing </style> tags. You write standard CSS syntax inside it, just like you would in a .css file.
  • Scope: Styles defined within the <style> tags apply only to the HTML document they are embedded in. They don't affect other pages.
  • Use Cases: Useful for single-page websites, quick demonstrations, HTML emails (where external CSS might be stripped), or sometimes for embedding critical "above-the-fold" styles for performance reasons (though external files are generally preferred).
  • Pros vs. Inline: Better organization than inline styles because rules are grouped together. Allows the use of CSS selectors (like targeting all <p> elements) which inline styles cannot do.
  • Cons vs. External: Styles are still tied to a single HTML page, making them harder to reuse across a multi-page site compared to external stylesheets.
