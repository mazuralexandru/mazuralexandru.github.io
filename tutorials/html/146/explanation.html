146. Type Selectors: Use Cases

Files Modified: None

Given their broad scope, type selectors are best suited for:

  1. Global Resets/Base Styles: Applying foundational styles to common HTML elements to create a consistent starting point across the site. For example, setting default margins, padding, font sizes, or line heights for headings, paragraphs, lists, etc. Many CSS reset/normalize libraries rely heavily on type selectors.
    /* Example Base Styles */
body {
  margin: 0;
  font-family: sans-serif;
  line-height: 1.6;
}

p {
  margin-bottom: 1em;
}

img {
  max-width: 100%;
  height: auto;
}
  2. Styling Very Common Elements Consistently: If all instances of a specific element should always look the same (e.g., perhaps all <hr> elements should be a thin gray line), a type selector is appropriate.

However, avoid using type selectors for styling specific UI components or sections where variations are needed. For instance, styling all <div> elements with a specific border is usually a bad idea, as <div> is used for many different structural purposes. Use classes for that.
