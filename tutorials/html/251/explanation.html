251. Figures (<figure>) - Fundamentals

When an image (or other content like a code block, chart, or quote) is self-contained and referenced from the main text, it's often semantically appropriate to wrap it in a <figure> element.

The Figure Element (<figure>)

  • Definition: Represents self-contained content, potentially with an optional caption (<figcaption>), that is referenced from the main flow of the document but whose position could potentially be changed (e.g., moved to an appendix or sidebar) without affecting the main flow's meaning.
  • Purpose: To semantically group content units (like an image and its caption) that belong together and function as a distinct "figure" within the document.
  • Type: Block-level element.

Core Concept:

  • Think of it like figures in a book or article – diagrams, photos, code listings that have numbers or captions and are referred to from the main text (e.g., "See Figure 1"). The <figure> element provides the container for such units.

✨ Key Takeaway: Use the block-level <figure> element to semantically wrap self-contained content like images, diagrams, or code blocks, especially when they are referenced from the main text and might have an associated caption.
