253. Figure Captions (<figcaption>) - Fundamentals

While a <figure> groups content, the <figcaption> element provides its visible caption or legend.

The Figure Caption Element (<figcaption>)

  • Definition: Represents a caption or legend describing the rest of the content of its parent <figure> element.
  • Purpose: To provide a visible title, description, or explanation directly associated with the figure's content (image, code, etc.).
  • Semantics: Clearly and programmatically associates the caption text with the figure. This is crucial for accessibility, as screen readers can announce the figure and its caption as a related unit.
  • Type: Typically renders as a block-level element, though its content model is flow content.

Placement:

  • Must be placed directly inside a <figure> element.
  • Only one <figcaption> is allowed per <figure>.
  • It can be the first or the last child within the <figure>. Both are valid, but placing it first is sometimes recommended for slightly better accessibility parsing in some older tools. Consistency within a project is key.
