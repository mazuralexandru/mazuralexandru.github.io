115. List Items (<li>): The value Attribute (in <ol>)

While attributes on <ol> control the overall list numbering, the value attribute can be added to an individual <li> element (specifically within an <ol>) to override its specific number in the sequence.

The value Attribute on <li>:

  • Context: Only meaningful when the <li> is a child of an <ol>. It is ignored inside <ul>.
  • Purpose: Sets a specific ordinal value (number) for that particular list item, overriding where it would normally fall in the sequence.
  • Syntax: <li value="integer">
  • Value: An integer representing the desired number for this item.
  • Behavior:
    • The <li> element with the value attribute gets that number as its marker.
    • Crucially: Subsequent <li> elements without their own value attribute will continue numbering sequentially from the overridden value.

Use Cases (Use Sparingly):

  • Representing steps or clauses that are intentionally non-sequential (e.g., referring back to a previous step number, skipping irrelevant steps).
  • Bridging gaps in numbering if items are dynamically added or removed.
  • Matching numbering to external documents or standards where gaps exist.

Caveat: Overusing the value attribute can make list numbering confusing and harder to follow. Often, if complex or non-standard numbering is needed purely for visual presentation, CSS Counters offer more robust control.
