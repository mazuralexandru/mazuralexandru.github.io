416. Forms: The method Attribute - Fundamentals

Along with where to send the data (action), we need to specify how to send it. This is done using the method attribute.

Definition & Purpose:

  • The method attribute specifies the HTTP method (or "verb") that the browser should use when submitting the form data to the server endpoint defined by the action attribute.
  • It determines the way the form data is packaged and transmitted within the HTTP request.

Core Concept:

  • The two primary HTTP methods used for HTML form submission are GET and POST. Each has distinct characteristics affecting how data is sent, data size limits, security, and idempotency (whether repeating the request has side effects).

Syntax:

  • <form action="..." method="http-method"> ... </form>
  • The value (http-method) is typically GET or POST. Method names are case-insensitive, but uppercase is conventional.

Default Value:

  • If the method attribute is omitted, the browser defaults to using the GET method.

Code Change:

We've added method="GET" to our <form> tag in index.html. Although GET is the default, we're adding it explicitly here to prepare for discussing it.

<!-- index.html snippet -->
<form action="/submit-form-data" method="GET"> <!-- Added method="GET" (default) -->
  <!-- Form controls will go here -->
</form>

✅ Observation:

  1. Locate the method="GET" attribute added to the opening <form> tag.
  2. Look at the browser preview. There is no visual change. Like action, the method attribute defines behavior (how data is sent on submission), not appearance.

✨ Key Takeaway: The method attribute specifies the HTTP method (GET or POST) used to submit form data. It defaults to GET if omitted.
