614. Graphics: SVG (<svg>) - Behavior & Interactions (DOM, CSS, JS)

Let's focus on how inline SVG integrates with the surrounding HTML document and interacts with CSS and JavaScript.

DOM Integration:

  • When SVG is embedded inline (<svg>...</svg> directly in HTML), its elements (<rect>, <circle>, <g>, <text>, etc.) become part of the main document's DOM tree.
  • This means you can directly target and manipulate these SVG elements using the same JavaScript DOM APIs you use for HTML elements:
    • document.getElementById('my-circle')
    • document.querySelector('svg .outline-shape')
    • element.setAttribute('fill', 'blue')
    • element.classList.add('animated')
    • element.addEventListener('click', () => { ... })

CSS Styling:

  • As seen previously (Lesson 612), standard CSS rules defined in the parent document (internal <style> or external <link>) can directly target and style elements within the inline SVG using type, class, ID, or attribute selectors.
  • This includes applying CSS Transitions and Animations to SVG elements, allowing for declarative, performant animations without complex JavaScript (e.g., changing fill, stroke, or transform smoothly over time).

JavaScript Interaction:

  • Because SVG elements are in the DOM, JavaScript can:
    • Modify Attributes: Change position (x, y, cx, cy), size (width, height, r), appearance (fill, stroke), path data (d), text content, etc., dynamically.
    • Modify Styles: Add/remove classes (classList) or directly set CSS properties (element.style).
    • Handle Events: Attach event listeners (click, mouseover, mousedown, etc.) directly to individual shapes or groups (<circle>, <rect>, <g>) to create interactive graphics. This is a major advantage over Canvas's single event listener and manual hit detection.
    • Create/Remove Elements: Dynamically generate new SVG elements (e.g., adding bars to a chart based on data) or remove existing ones.

SMIL (Synchronized Multimedia Integration Language):

  • SVG has its own native, declarative animation language defined within the SVG spec using elements like <animate>, <animateTransform>, <animateMotion>, <set>.
  • While powerful, SMIL adoption and browser implementation consistency have lagged behind CSS animations/transitions and JavaScript animation libraries.
  • Recommendation: For most web animations involving SVG, using CSS Transitions/Animations or well-supported JavaScript animation libraries (like GSAP, Anime.js) targeting the SVG DOM elements is generally the more common and cross-browser compatible approach today.
