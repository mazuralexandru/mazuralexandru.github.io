386. Horizontal Nav (Flexbox): Spacing with gap

Files Modified: style.css

One of the major advantages of Flexbox is the gap property (also usable in CSS Grid).

  • Purpose: Defines the size of the gutters (gaps) between flex items (or grid items). It controls the spacing between items without adding space at the start or end of the container.
  • Syntax: gap: value; (sets both row and column gap) or row-gap: value; / column-gap: value;. For a single row/column layout, just gap is sufficient.
  • Applies To: The flex container (in our case, the <ul>).
  • Benefit: Much simpler and cleaner than adding margins to individual flex items (<li>) and then removing the margin from the last item.

✅ Observe style.css:

  • We've added the gap property to the nav ul rule.
nav ul {
    display: flex; 
    list-style: none; 
    padding: 0;       
    margin: 0;        
    background-color: rgba(0, 0, 0, 0.2); 
    padding: 0.5em 0; 
    gap: 0.5em; /* Add space BETWEEN flex items (li) */
    /* Alignment properties will be added */
}

✨ Observe the Preview:

  • There should now be a consistent horizontal gap (0.5em) between each of the navigation items ("Home", "About", "Services", "Contact").
  • This spacing is achieved with a single property on the container, avoiding the need for extra selectors like :last-child to remove margins.
