320. Contrast display: none; with visibility: hidden;

Files Modified: None

It's important to distinguish display: none; from another way to hide elements: visibility: hidden;.

  • display: none;

    • Removes the element entirely from the document flow and rendering.
    • Takes up no space in the layout.
    • Affects layout (surrounding elements reflow to fill the space).
    • Removes the element from the accessibility tree.

  • visibility: hidden;

    • Makes the element invisible (fully transparent).
    • The element still takes up its original space in the layout. It's like an invisible placeholder.
    • Does not affect layout (surrounding elements do not reflow).
    • The element may still be part of the accessibility tree (behavior can vary). Descendants can be made visible again by setting visibility: visible; on them.

When to Use Which:

  • Use display: none; when you want to completely remove an element as if it wasn't there, often for conditional showing/hiding with JavaScript.
  • Use visibility: hidden; when you want to hide an element but preserve the space it occupies in the layout, perhaps for certain animation effects or to maintain layout stability when toggling visibility.
