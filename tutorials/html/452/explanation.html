452. Forms: The value Attribute - Checkboxes & Radio Buttons

The meaning of the value attribute is significantly different for checkboxes (type="checkbox") and radio buttons (type="radio").

value on Checkboxes & Radio Buttons:

  • Purpose: Specifies the value that is sent to the server if that specific checkbox or radio button is selected (checked) when the form is submitted.
  • Not Displayed: The value attribute's content is not directly visible to the user on the page. The visible description comes from the associated <label> element.
  • Importance: It's crucial to provide a meaningful value for each checkbox and radio button option. If you omit the value attribute:
    • Browsers often default to sending the value "on" if the control is checked.
    • Receiving just name=on on the server is usually unhelpful, as you don't know which specific checkbox/radio button was selected if multiple share the same name.
    • Therefore, always define a unique and meaningful value for each checkbox/radio button option that needs to be identified server-side.

Code Change:

We've removed the text inputs and added two fieldsets: one with checkboxes and one with radio buttons. Crucially, each <input> now has a specific value attribute.

<!-- index.html snippet -->
<form action="/submit-data" method="POST">
  <fieldset>
    <legend>Options (Checkboxes):</legend>
     <input type="checkbox" id="cb1" name="options" value="option1"> <!-- Checkbox value -->
     <label for="cb1">Option 1</label><br>
     <input type="checkbox" id="cb2" name="options" value="option2"> <!-- Checkbox value -->
     <label for="cb2">Option 2</label>
  </fieldset>

  <fieldset>
    <legend>Choice (Radio):</legend>
      <input type="radio" id="rb1" name="choice" value="A"> <!-- Radio value -->
      <label for="rb1">Choice A</label><br>
      <input type="radio" id="rb2" name="choice" value="B"> <!-- Radio value -->
      <label for="rb2">Choice B</label>
  </fieldset>
</form>

✅ Observation:

  1. Examine the value attributes on the checkboxes (value="option1", value="option2") and radio buttons (value="A", value="B").
  2. In the preview, select "Option 2" checkbox and "Choice A" radio button.
  3. Submission (Conceptual): If submitted, the data sent would include options=option2 and choice=A. If "Option 1" was also checked, the data would include options=option1&options=option2&choice=A. The value attributes provide the actual data transmitted for the selected controls.

✨ Key Takeaway: For checkboxes and radio buttons, the value attribute defines the data submitted for that option when selected. It is not displayed visually (the <label> is) but is essential for identifying the user's choice(s) on the server. Provide a unique and meaningful value for each option.
