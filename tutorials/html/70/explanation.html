70. Presentational HTML Elements: <i>, <b>, <u>

HTML5 redefined the meaning of some older presentational tags like <i>, <b>, and <u>. While they still produce italic, bold, and underlined text by default, they now have specific semantic nuances. They should be used only when those specific meanings apply, and not as general substitutes for <em>, <strong>, or CSS styling.

HTML5 Meanings:

  • <i> (Idiomatic Text): Represents text in an "alternate voice or mood", or otherwise offset from the normal prose. Examples: technical terms, foreign phrases, taxonomic designations, thoughts, ship names in Western text. It's not for emphasis.
  • <b> (Bring Attention To): Represents text to which attention is drawn for utilitarian purposes without conveying extra importance. Examples: keywords, product names, lead sentences. It's not for strong importance.
  • <u> (Unarticulated Annotation): Represents a span of text with an unarticulated, non-textual annotation. Examples: marking text as misspelled, labeling proper names in Chinese. Avoid using <u> for general underlining, as it looks like a hyperlink. Use CSS text-decoration: underline; instead for purely presentational underlines.

Key Takeaway:

If you just want text to look italic or bold, use CSS (font-style: italic; or font-weight: bold;), perhaps applied via a class (<span class="italic-style">). Use <em> or <strong> if there's genuine semantic emphasis or importance. Only use <i> or <b> if the specific HTML5 semantic nuance applies. Avoid <u> unless for its specific annotation meaning, due to hyperlink confusion.
