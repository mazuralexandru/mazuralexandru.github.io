300. The <head> Element Revisited

Let's quickly revisit the purpose and rules of the <head> element itself, which acts as the container for all the metadata we'll be discussing.

Purpose Recap:

  • The <head> element is the container for metadata (data about the HTML data).
  • Content within the <head> is not displayed directly on the main part of the web page in the browser window. Its content provides information and instructions to the browser and other web services (like search engines).

Placement & Structure Recap:

  • It must be the first element directly inside the root <html> element.
  • It is immediately followed by the <body> element.
  • There should only be one <head> element per document.
  • It requires an opening <head> and a closing </head> tag.

Required Child Element:

  • Every valid <head> section must contain at least one <title> element, defining the document's title. (We added this back in Module 1).

Common Child Elements (To be explored in this module):

  • <meta>: For various metadata types (charset, viewport, description, keywords, etc.).
  • <link>: For linking external resources (primarily CSS, but also icons, fonts, alternate versions, etc.).
  • <style>: For embedding internal CSS (less common for large projects).
  • <script>: For embedding JavaScript or linking external scripts (can also appear in <body>).
  • <base>: Specifies a base URL for relative paths.
  • <noscript>: Content displayed only if scripting is disabled.
