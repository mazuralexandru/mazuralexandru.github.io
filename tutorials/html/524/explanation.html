524. Other Semantics: <time> - Fundamentals

The <time> element provides a way to semantically mark up dates, times, or durations, providing both a human-readable version and a machine-readable equivalent.

The <time> Element:

  • Definition: Represents a specific period in time (a duration) or a specific point in time (a date, time, or datetime).
  • Purpose: To allow software agents (like browsers, calendar applications, search engines, assistive technologies) to parse and understand dates and times unambiguously, while still presenting a human-friendly format to the user.

Core Mechanics & Syntax:

  • Syntax: <time datetime="machine-readable-value">Human-readable representation</time>
  • Requires opening <time> and closing </time> tags. Inline element.
  • Content: The text between the tags is the version displayed to the user (e.g., "March 15th, 2024", "Tomorrow", "2 hours 30 mins").
  • datetime Attribute [Key]: This attribute holds the machine-readable version of the date, time, or duration. Its value must follow specific standard formats (based on ISO 8601):
    • Year: YYYY (e.g., 2024)
    • Month: YYYY-MM (e.g., 2024-03)
    • Date: YYYY-MM-DD (e.g., 2024-03-15)
    • Week: YYYY-Www (e.g., 2024-W11)
    • Time: HH:MM or HH:MM:SS or HH:MM:SS.sss (e.g., 10:30, 10:30:05.123)
    • Date and Time (Local): YYYY-MM-DDTHH:MM... (e.g., 2024-03-15T10:30)
    • Date and Time (Global with UTC offset): YYYY-MM-DDTHH:MM...[+/-]HH:MM or YYYY-MM-DDTHH:MM...Z (Z = UTC) (e.g., 2024-03-15T10:30:00+01:00, 2024-03-15T10:30:00Z)
    • Duration: PTnHnMnS (e.g., PT2H30M for 2 hours 30 minutes, P3D for 3 days).
    • If the content between the tags is already a valid machine-readable format, the datetime attribute can be omitted.

Benefits:

  • Accessibility: Screen readers can announce dates/times more intelligently.
  • SEO: Search engines can understand temporal information accurately for event indexing or timelines.
  • Interoperability: Calendar applications or browser extensions could potentially parse this data to offer "Add to Calendar" features.

Code Change:

We've removed the <address> example and added two paragraphs containing <time> elements: one for a specific datetime and one for a duration.

<!-- index.html snippet -->
<body>
  <h1>Other Semantic Elements</h1>
  <p>The &lt;time&gt; element represents dates/times.</p>

  <!-- Removed address example -->

  <p>
    Article published on <time datetime="2024-03-15T10:30:00Z">March 15th, 2024 at 10:30 UTC</time>. <!-- Added time element -->
  </p>
  <p>
    The event lasts for <time datetime="PT2H30M">2 hours and 30 minutes</time>. <!-- Added time element for duration -->
  </p>

  <p>The 'datetime' attribute provides the machine-readable value.</p>
  ...
</body>

✅ Observation:

  1. Locate the two <time> elements. Note the human-readable text between the tags and the corresponding machine-readable format in the datetime attribute.
  2. Look at the browser preview. The date/time and duration text ("March 15th...", "2 hours...") should appear as normal inline text. The <time> tag itself adds no default visual styling. Its value is purely semantic.

✨ Key Takeaway: Use the <time> element to mark up dates, times, or durations. Provide the machine-readable format in the datetime attribute (following standard formats) and the human-readable version as the element's content. This enhances accessibility and machine-parsability.
